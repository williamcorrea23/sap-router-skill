# SAP GUI Web Enrich — Auto-Fill Missing Navigation Data

When GUI fallback triggers but transaction navigation data is incomplete,
this skill auto-searches the web to fill the gaps.

## Philosophy (Karpathy: Think Before Acting)

Don't guess field IDs. Don't assume screen numbers.
If data missing: search, verify, then navigate.

```
GUI Fallback Triggered: "Create material via MM01"
    │
    ▼
Check: Do we have BDC sequence for MM01?
    │ NO → sap-gui-web-enrich (this skill)
    │       │
    │       ▼
    │   1. WebSearch: "SAP MM01 BDC recording field IDs"
    │   2. WebSearch: "SAP transaction MM01 screen sequence"
    │   3. WebFetch: SAP Help Portal MM01 documentation
    │   4. WebFetch: SAP Community blog with BDC example
    │   5. Build enriched BDC sequence
    │   6. Cache in MEMORY.md GUI_DATA section
    │       │
    │       ▼
    │   Return to sap-gui-scripting with complete data
    │ YES → Execute GUI navigation immediately
    ▼
Execute transaction + verify result
```

## Web Search Strategy (Karpathy: Define Success Criteria)

For each missing transaction:

1. **SAP Help Portal** — authoritative source for field definitions
   - URL pattern: `https://help.sap.com/docs/SAP_NETWEAVER/{version}/...`
   - Search: `"{tcode} SAP transaction screen fields" site:help.sap.com`

2. **SAP Community** — real-world BDC examples
   - Search: `"{tcode} BDC recording ABAP example" site:community.sap.com`

3. **GitHub** — open-source ABAP with BDC calls
   - Search: `"{tcode} CALL TRANSACTION OR BDC_OPEN_GROUP" language:abap`

4. **Stack Overflow** — common issues and fixes
   - Search: `"SAP {tcode} field not found OR screen not reached"`

5. **SAP Notes** — known issues with transaction
   - Use `mcp-sap-notes` to search for transaction-specific notes

## CLI Integration

```bash
# Enrich specific transaction
python scripts/sap_router.py gui-enrich --tcode MM01

# Enrich all transactions for a module
python scripts/sap_router.py gui-enrich --module MM

# Check enrichment status
python scripts/sap_router.py gui-enrich --status

# Force re-enrich (ignore cache)
python scripts/sap_router.py gui-enrich --tcode MM01 --force
```

## Fallback Chain for GUI Enrichment

```
1. LOCAL CACHE (MEMORY.md GUI_DATA) → instant
2. SAP HELP PORTAL (help.sap.com) → authoritative, ~2s
3. SAP COMMUNITY (community.sap.com) → practical examples, ~3s
4. GITHUB CODE SEARCH (github.com) → real ABAP code, ~5s
5. SAP NOTES (mcp-sap-notes) → known issues, ~2s
6. ASK USER TO RECORD MANUALLY → use SHDB transaction recorder in SAP
```

## Gotchas

- **Web search not always accurate**: Always verify 2+ independent sources for HIGH confidence
- **SAP version differences**: ECC field IDs differ from S/4HANA. Version-tag cached data.
- **Rate limiting**: WebSearch tool has rate limits. Batch enrichments staggered 2s apart.
- **Sensitive data**: Never search with real customer data. Use generic field descriptions.
- **Cache invalidation**: Re-enrich after SAP version upgrades. Force re-enrich if BDC fails.
