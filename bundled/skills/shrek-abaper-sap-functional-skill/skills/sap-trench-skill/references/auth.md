# SAP 权限检查知识库（Authorization）

---

## 一、AUTHORITY-CHECK 语法

### 基本语法
```abap
AUTHORITY-CHECK OBJECT '权限对象名'
  ID '字段1' FIELD '值1'
  ID '字段2' FIELD '值2'.

IF sy-subrc <> 0.
  MESSAGE '无权限执行此操作' TYPE 'E'.
ENDIF.
```

### 常用权限对象
| 权限对象 | 字段 | 说明 |
|---|---|---|
| `S_TCODE` | `TCD` | 事务码访问权限（SAP 自动检查）|
| `M_BEST_BSA` | `BSART`/`ACTVT` | 采购凭证类型/活动 |
| `M_BEST_EKG` | `EKGRP`/`ACTVT` | 采购组/活动 |
| `M_BEST_EKO` | `EKORG`/`ACTVT` | 采购组织/活动 |
| `M_MSEG_BWA` | `BWART`/`ACTVT` | 货物移动类型 |
| `F_BKPF_BUK` | `BUKRS`/`ACTVT` | 公司代码/活动 |
| `F_BKPF_BLA` | `BLART`/`ACTVT` | 凭证类型/活动 |
| `F_BKPF_KOA` | `KOART`/`ACTVT` | 科目类型/活动 |

### ACTVT 活动代码
| 值 | 含义 |
|---|---|
| `01` | 创建 |
| `02` | 修改 |
| `03` | 显示 |
| `06` | 删除 |
| `23` | 维护（包含增/改/删）|

## 二、权限排查三步法

### 1. SU53 — 查看上次失败的权限检查
执行事务码报权限错误后，**立即**在同一窗口执行 `SU53`：
- 显示系统记录的最后一次权限检查失败对象
- 显示用户实际拥有的权限值 vs 所需权限值

> ⚠️ 注意：SU53 只显示**最后一次**失败。如果错误发生后又做了其他操作，可能已被覆盖。

### 2. SU56 — 分析权限缓冲区
检查用户的权限缓冲区是否已刷新：
- 角色分配后，用户必须重新登录才能生效
- 通过 SU56 可以查看当前已加载的权限值

### 3. PFCG — 分配/修改角色
根据 SU53 结果，在 PFCG 找到对应角色，维护缺少的权限对象和字段值。

```
报错 → SU53（找缺少的权限对象和值）→ PFCG（维护角色）→ 用户重新登录（刷新缓冲区）→ 再次测试
```

## 三、关键事务码

### 权限排查类
| 事务码 | 用途 |
|---|---|
| `SU53` | ★★★ 查看最后一次权限检查失败详情（最常用）|
| `SU56` | 分析当前用户权限缓冲区 |
| `SU22` | 事务码权限检查提案（原始，SAP标准）|
| `SU24` | 事务码权限检查提案（可修改版本）|
| `SU25` | 检查并更新 SU24 数据 |
| `SUIM` | 用户信息系统（查"谁有哪个权限"/"哪个用户能执行哪个T码"）|

### 角色与用户维护类
| 事务码 | 用途 |
|---|---|
| `PFCG` | 角色维护（创建/修改/查看角色）|
| `SU01` | 用户主数据维护（创建/修改用户，分配角色）|
| `SU10` | 批量用户维护 |
| `SU21` | 维护权限对象（开发用）|
| `SUPC` | 批量生成角色/配置文件 |
| `ROLE_CMP` | 角色比较（对比两个角色的差异）|
| `SU02` | 直接创建配置文件（不经过角色）|

## 五、用户类型说明

| 类型 | 代码 | 说明 |
|---|---|---|
| 对话用户 | `dialog` | 常规业务用户，有登录屏幕，密码有有效期 |
| 通讯用户 | `communication` | 用于 RFC/BAPI 调用，无登录屏幕，密码永不过期 |
| 系统用户 | `system` | 内部系统调用，不能交互式登录 |
| 服务用户 | `service` | 类似通讯用户，允许多人共用（如 Web Service）|
| 参考用户 | `reference` | 仅用于继承权限，不能直接登录 |

> **实际开发注意**：接口程序（PI/RFC调用）使用 `communication` 或 `service` 类型用户，不要用 `dialog` 用户。

## 六、ABAP 权限检查最佳实践

### 自定义权限对象开发流程
1. `SU21` → 创建权限对象（指定所属对象类、字段列表）
2. `SU24` → 将权限对象关联到事务码（让 PFCG 能自动扫描到）
3. ABAP 代码中使用 `AUTHORITY-CHECK OBJECT '<新对象名>' ...`
4. `PFCG` → 在角色的权限页签，找到新对象，维护字段值

### 常见坑

**坑1：角色改了但不生效**
- 根本原因：权限数据被缓冲，用户必须重新登录才会刷新
- 解决：让用户重新登录，或在测试环境执行 `SU56` 手动刷新缓冲区

**坑2：SU53 显示无权限但 PFCG 里明明有**
- 检查是否分配角色后未重新登录
- 检查是否有多个角色冲突（某角色明确限制了该权限）
- 通过 `SUIM` → "用户拥有哪些权限" 查看有效权限值

**坑3：代码里 AUTHORITY-CHECK 通过，但功能仍报权限错误**
- SAP 框架层（如 OData、Proxy）有自己的权限检查，独立于 ABAP 代码
- 需要在相应框架侧配置权限（如 OData 服务的 ICF 节点权限）

## 九、SUIM 常用查询场景

`SUIM`（用户信息系统）是权限分析最强的报表工具：

| 查询需求 | SUIM 菜单路径 |
|---|---|
| 查某用户拥有哪些角色 | 用户 → 按角色 |
| 查哪些用户有某个事务码权限 | 用户 → 按事务码 |
| 查某角色包含哪些事务码 | 角色 → 按事务码 |
| 查哪些用户有某权限对象的特定值 | 用户 → 按权限对象 |
| 批量导出用户权限报表 | 用户 → 完整列表 |

> **权限排查黄金法则**：遇到权限报错，**立即** 在同一 SAP 窗口执行 `SU53`（不要做任何其他操作），它只记录最后一次失败。如果已经做了其他操作导致 SU53 被覆盖，让用户重新执行一次出错操作后立刻跑 SU53。
