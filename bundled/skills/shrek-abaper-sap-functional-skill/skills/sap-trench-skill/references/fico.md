## 三、主要 BAPI 速查

### BAPI_ACC_DOCUMENT_POST（FI凭证过账）
标准测试程序：`SE38 → ACC_BAPI_TEST_DOCUMENT`

**关键参数结构**：
- `DOCUMENTHEADER`（BAPIACHE09）：公司代码、凭证日期、过账日期、凭证类型、参考号
- `ACCOUNTGL`（BAPIACGL09）：总账行（科目、成本中心、利润中心）
- `ACCOUNTRECEIVABLE`（BAPIACAR09）：客户行
- `ACCOUNTPAYABLE`（BAPIACAP09）：供应商行
- `CURRENCYAMOUNT`（BAPIACCR09）：金额
- `EXTENSION2`（BAPIPAREX）：扩展字段（UMSKZ特殊GL、BSCHL过账码、SGTXT项目文本）

```abap
" 扩展字段写法（如特殊总账标识）
wa_extension2-structure  = 'BSEG'.
wa_extension2-valuepart1 = lw_document-buzei.  " 行号
wa_extension2-valuepart2 = lw_document-umskz.  " 特殊GL标识
APPEND wa_extension2 TO it_extension2.
```

### 物料标准成本相关 BAPI
- `BAPI_COSTESTIMATE_CREATE_SPLIT` — 创建成本估算
- `BAPI_COSTESTIMATE_MARKING` — 标记
- `BAPI_COSTESTIMATE_RELEASING` — 发布
- `BAPI_COSTESTIMATE_DELETE` — 删除
- `BAPI_COSTESTIMATE_UPDATE_PRICE` — 更新价格

### FI_ITEMS_MASS_CHANGE（FB02 批量修改）
修改凭证文本字段（SGTXT/ZLSPR），通过 `IT_FLDTAB` 指定字段，`IT_BUZTAB` 指定行范围

### FBL1N/FBL5N 到期日显示
**参数ID**：`FIT_DUE_DATE_SEL = X` → 显示"净付款到期日"字段

## 四、FI 凭证增强（9大控制点）

| 顺序 | 控制点 | 说明 |
|---|---|---|
| 1 | **前端模块控制**（MM/SD） | 物料凭证驱动FI凭证，用户出口在前端模块中 |
| 2 | **BAdI AC_DOCUMENT** | Logistics→FI过账时触发；F-xx事务码**不触发** |
| 3 | **替换（OBBH/OKC9）** | 简单字段派生，可调用用户出口 |
| 4 | **CO成本对象检查** | S/4用OB28；ECC早期CO凭证需单独实现 |
| 5 | **RWIN (TRWPR) 子过程** | 自定义表同步更新，含检查/过账/后处理 |
| 6 | **OB28 规则验证** | RWIN Project阶段之后调用 |
| 7 | **BTE 1025（检查）** | 凭证校验 |
| 7 | **BTE 1120（处理）** | ⚠️ 直接修改数据无二次检查，数据错误将直接入账 |
| 8 | **ES_FINS_ACDOCA（S/4）** | 复杂场景，控制ACDOCA（非BSEG），如外币估值/费用分摊 |
| 9 | **后处理（BTE 1050）** | 凭证过账后处理，最常用 |

**特殊增强点**：
- **SEGMENT 派生**：专用 BAdI

## 六、汇率确定

### 5种确定方式（优先级从低到高）
| 方式 | 说明 |
|---|---|
| OB08 自动 | 系统按记账日期取维护值（最低优先级）|
| MM采购订单 | ME21N 时带出，可手工修改 |
| SD销售订单 | VA01 时带出，可手工修改 |
| FI凭证抬头 | 直接输入，覆盖OB08 |
| 行项目手工 | `BSEG-HWMET='E'`，若第一行被修改可同步更新抬头汇率 |

### 直接汇率 vs 间接汇率（ONOT）
- **直接**：1外币 = XX本位币（如 1 USD = 6.5 CNY）
- **间接**：XX外币 = 1本位币（如 2 USD = 1 CNY，OB08中以"/"开头显示）
- 配置货币对采用哪种形式：`ONOT`

### 翻译日期（Translation Date）
- 默认取**过账日期**（`BKPF-BUDAT`），非凭证日期
- 需要用凭证日期作为翻译日期 → 在调用点1做替换，或使用 BAdI `FI_TRANS_DATE_DERIVE`（仅限MIRO）
- SAP Note：42615, 574583, 575249

## 七、凭证分割（Document Splitting）

### 目的
将应付/应收等科目行按对方行项目的利润中心/段自动拆分，实现多维度报表（资产负债表按利润中心出具）

### 6步配置
1. **给总账科目分组**（OBC4/分割特性配置）— 收入、费用、客户、供应商、银行等分类
2. **定义零余额清帐科目** — 借贷利润中心不同时的中间科目，避免死循环
3. **定义FI分割特性** — 指定哪些字段（如利润中心、段）作为分割维度，设为必输+零余额
4. **定义CO分割特性**（可选）
5. **定义默认特性值** — 未输入时默认值，防止报错
6. **激活凭证分割** — 客户端级别，激活后所有公司受影响；需在公司代码级别选择是否参与

> ⚠️ 已激活的公司代码不能随意关闭分割，否则产生数据不一致

**标准分割程序**：`0000000012`（SAP交付，覆盖大多数业务）

## 八、字段状态（Field Status）

### 核心原则：SDRO 优先级规则
| 状态缩写 | 含义 | 优先级 |
|---|---|---|
| `S` | Suppressed（隐藏）| 最高（覆盖其他）|
| `D` | Displayed（显示只读）| 次高 |
| `R` | Required（必输）| 次低 |
| `O` | Optional（可选）| 最低 |

**GL主数据字段状态 + 过账码字段状态 → 取限制更严格的那个**

### 配置路径
- `OBC4` — 字段状态变式/字段状态组（GL主数据）
- `OB41` — 过账码字段状态
- `OMJJ` — 物料移动字段状态

### SAP标准字段状态组
| 组 | 描述 |
|---|---|
| `G001` | 含文本与分配的一般（通用）|
| `G003` | 物料消耗科目 |
| `G004` | 成本科目 |

## 九、COPA 获利能力分析

### 特征派生（Characteristic Derivation）
**事务码**：`KEDR`（派生策略），`KEDB`（调试/测试）

**5种派生方法**：
| 方法 | 说明 |
|---|---|
| 派生规则 | 条件 → 赋值（最常用）|
| 表格查询 | 查自定义Z表取值 |
| 移动 | 从一个特征复制到另一个 |
| 清空 | 清除特征值 |
| 增强 | COPA0001 用户出口 |

### COPA0001 增强
```abap
" EXIT_SAPLKEAB_001 - 对所有COPA凭证触发
" 从销售凭证读取字段赋值给COPA特征
READ TABLE i_itemcode INTO ls_itemcode
  WITH KEY refnr = is_reqdocument-refnr.
IF sy-subrc = 0.
  ls_char_values-copa_char = 'ZCUSTOMFIELD'.
  ls_char_values-copa_value = ls_itemcode-value.
  APPEND ls_char_values TO e_char_values.
ENDIF.
```

### COPA必输字段注意
COPA特征组（KEPA/KE4G）的必输校验仅在**前台可见时**触发，后台BAPI过账**不触发**必输检查 → 后台过账需在增强中自行校验

## 十、自动付款（F110）

### 配置路径（FBZP）
```
FBZP → 付款程序全局配置
  ├─ 公司代码付款数据
  ├─ 付款方式（支票/电汇/直扣）
  ├─ 银行选择规则（可用额度、排名）
  └─ 付款方式/公司代码配置
```

### 付款运行步骤（F110）
```
1. 输入运行日期和标识
2. "参数" → 选择公司代码、付款方式、截止日期
3. "建议清单" → 系统生成建议，检查异常项
4. 修改建议（如有需要）
5. "付款运行" → 生成付款凭证
6. 打印输出（支票/汇款通知）
```

### 付款数据表
| 表名 | 内容 |
|---|---|
| `REGUH` | 付款运行结果抬头（银行账户、金额）|
| `REGUP` | 付款运行行项目（含供应商、发票号）|
