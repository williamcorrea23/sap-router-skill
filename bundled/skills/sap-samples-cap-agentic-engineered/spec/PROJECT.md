# Financial Risk Analyzer

## What This Is

A CAP + Fiori Elements application that visualizes financial anomaly predictions from an XGBoost model deployed on SAP AI Core. It reads pre-enriched GL transaction data from SAP Business Data Cloud and displays risk classifications with business-language explanations in a list report. Built in this directory using **MCP-first development** — every SAP code artifact is generated by querying the relevant SAP MCP server (CAP, Fiori, UI5) before writing code. The original `../prototype/` serves as a reference implementation.

## Core Value

Finance teams can see which GL transactions are risky — unusual amounts, backdated postings, weekend entries, rare patterns — in a single Fiori list report with one-click inference from a deployed ML model.

## Context

- **Source data:** `shared_analytics.finance_risk.gl_features_for_ml` — a BDC data product shared from SAP Databricks via BDC Connect. Contains GL line items with 25 pre-computed features for ML inference.
- **AI Core model:** XGBoost multi-class classifier deployed as a FastAPI container on SAP AI Core. Endpoint: `POST /v1/predict`. Already deployed and reachable.
- **BDC connection:** Not yet configured. The data product is in BDC via BDC Connect, but the BTP Destination to pull it from CAP doesn't exist yet. Build with mock data first, wire real BDC later.
- **Anomaly classes:** 11 classes from Normal to General_Anomaly. Each maps to a business-language display label, explanation, and Fiori criticality level.
- **Feature order is critical:** The 25 features must be extracted and sent to AI Core in exact column order or predictions will be wrong.
- **Repo location:** This directory (`prototype-mcp/`) within the agentic engineering reference architecture repo. Reference implementation in `../prototype/`.

## Constraints

- **Tech stack**: SAP CAP (Node.js) + Fiori Elements list report + OData V4
- **Data source**: BDC via BTP Destination (OData) — mock CSV/JSON until destination is configured
- **AI runtime**: SAP AI Core — custom Docker deployment, direct HTTP POST (not via `@sap-ai-sdk/foundation-models`)
- **MCP servers**: Use `@cap-js/mcp-server` for CAP, `@sap-ux/fiori-mcp-server` for Fiori, `@ui5/mcp-server` for UI5
- **MCP-first methodology**: Every SAP framework artifact MUST be generated after querying the relevant MCP server. Domain logic (feature columns, risk labels, AI Core client) can be reused from `../prototype/`.
- **Agent execution**: Single-agent GSD framework execution (sequential phases)
