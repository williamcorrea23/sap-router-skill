**Step 1: Update srv/risk-service.cds**

Update the existing service CDS to define the analyzeRisks action with the correct return type. Per user decision from CONTEXT.md, this is an unbound (service-level) action returning `array of GLTransactions`:

```cds
using { risk } from '../db/schema';

service RiskService {
  @readonly
  entity GLTransactions as projection on risk.GLTransactions;

  // Unbound action: called at service level
  // POST /odata/v4/risk/analyzeRisks()
  action analyzeRisks() returns array of GLTransactions;
}
```

Key decisions per CONTEXT.md:
- analyzeRisks is an unbound (service-level) action
- Returns `array of GLTransactions` -- full entities with virtual fields populated
- Entity remains @readonly (data from BDC, not editable)

**Step 2: Create srv/risk-service.js**

Adapt the handler from ../prototype/srv/risk-service.js with these specific behaviors per CONTEXT.md locked decisions:

1. **Conditional predictor loading:** `process.env.AI_CORE_MOCK === 'true'` uses mock-predictor, otherwise ai-core-client
2. **Fetch all transactions:** `SELECT.from(GLTransactions)` to get all GL data
3. **Extract features:** Call extractFeatures(transactions) -- returns validRows + skippedTransactions
4. **Handle amount/amount_feat mapping:** Before passing transactions to extractFeatures, map each transaction's `amount_feat` CDS field to `amount` property so FEATURE_COLUMNS[12] finds it: `tx.amount = tx.amount_feat` for each tx. This bridges the CDS naming (amount_feat) with the BDC/Python naming (amount) used by FEATURE_COLUMNS.
5. **Handle skipped rows:** Set riskClassification='Incomplete Data', riskExplanation='Transaction has missing feature values...', anomalyScore=null, criticality=0
6. **Call predictor:** predictions = predictor.predictAnomalies(validRows) -- handle both sync (mock) and async (real) via Promise check
7. **Map predictions:** Use mapPrediction to get display/explanation/criticality for each prediction
8. **Persist results:** UPDATE each transaction in DB with the enriched fields (riskClassification, riskExplanation, anomalyScore, criticality)
9. **Error handling per CONTEXT.md:**
   - TIMEOUT -> req.reject(408, 'AI Core service timed out. Please try again.')
   - AUTH_ERROR -> req.reject(401, 'AI Core authentication expired. Contact your administrator.')
   - SERVICE_UNAVAILABLE -> req.reject(503, 'AI Core model is temporarily unavailable. Please try again later.')
   - Unknown -> req.reject(500, 'Analysis failed. Please try again later.')
10. **Logging:** Use `cds.log('risk-service')` -- log action invoked (row count), prediction start/end, errors. Never log feature values or prediction values (sensitive data).
11. **Return:** Full transactions array with enriched virtual fields
