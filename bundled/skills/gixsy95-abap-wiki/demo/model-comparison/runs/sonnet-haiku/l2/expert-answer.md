# Expert answer - repository owner (gixsy95github@gmail.com) (2026-07-03)

## Gap: abapgit-standalone-demo-g002

The initiative behind this object is a public demo, not an IT adoption: the owner downloaded the abapGit standalone program from the official site (docs.abapgit.org) on 2026-07-02 to demonstrate and benchmark the abap_wiki L0/L1/L2 ingest pipeline on a large, public, MIT-licensed ABAP program that is relevant to the ABAP community. It is not part of any business process and is not productively installed in any system; the upstream file may change in the future, so the committed snapshot of 2026-07-02 (abapGit 1.133.0) is authoritative for this demo.

## Gap: abapgit-standalone-demo-g004

The background/batch execution path is not used in this context: no job is scheduled and the program is never executed here (it is a source snapshot used as pipeline input, not an installation). In a real installation the documented trigger applies: interactive launch via SE38 for the standalone version.

## Gap: abapgit-standalone-demo-g007

The ZABAPGIT package and the single-row TADIR are synthetic, generated by the demo harness (demo/model-comparison/prepare.py) purely to exercise the pipeline's package layout. The upstream recommendation of a local $ package applies to real installations; the deviation is an intentional demo fixture, not a transport-governance decision.
