# raw/system-library/

Place the export of the SAP system's ABAP sources here.

Recommended procedure:

1. Install and configure ABAP FS for VS Code following the documentation of the
   `marcellourbani/vscode_abap_remote_fs` project:
   <https://github.com/marcellourbani/vscode_abap_remote_fs>.
2. Configure the connection to your SAP system.
3. Open the remote ABAP file system in VS Code.
4. Navigate to the custom packages to document.
5. On the package shown in the remote file system use:

   ```text
   right-click -> download
   ```

6. Save or copy the downloaded content under this directory, preserving
   the structure generated by the tool.

The pipeline indexes this directory once during
`pipeline.py resolve-sources` and looks up sources by TADIR object. Keep the
files exactly as exported: `raw/** -text` in `.gitattributes` avoids automatic
line-ending conversions and preserves hashes and citations.

The files must follow the object-as-file naming convention born with
[abapGit](https://github.com/abapGit/abapGit) (`ZFOO.prog.abap`, `.clas.abap`,
`.fugr.abap`, ...): ABAP FS produces it automatically. If you extract sources
another way (abapGit, or a manual download from a system without ADT), name
the files accordingly, or `resolve-sources` will not bind them. Full
explanation: `core/docs/09-first-clone-and-sap-input-guide.md` section 4b.

This directory is a template scaffold: it contains no real data.

Full guide: `core/docs/09-first-clone-and-sap-input-guide.md`.
