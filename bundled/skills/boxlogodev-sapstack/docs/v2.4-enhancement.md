## Pillar A — "증명한다": 진단 품질 eval 하니스 (갭 G4)

> gstack qa/canary/devex-boomerang 대응 · ETHOS ①Ground-truth ②Evidence 측정 가능화

- `data/eval/gold-set.yaml` — **32 케이스, 18개 SAP 모듈 전체 커버**. 케이스는 `symptom-index` 를 `symptom_ref` 로 참조(단일출처) + 채점 정답(`primary_root_cause`/`must_tcodes`/`must_checks`). primary_root_cause 는 `typical_causes[0]` 파생(추측 금지).
- `schemas/eval-gold-set.schema.yaml` · `data/eval/README.md`
- `scripts/check-eval-goldset.sh` — **무료** 참조 무결성 게이트(순수 bash). `ci.yml` 편입.
- `scripts/eval-diagnosis.sh` + `scripts/eval/run.mjs` — LLM-judge 러너.
  - **provider 자동선택**: `ANTHROPIC_API_KEY` 있으면 `api`, 없고 `claude` CLI 있으면 `claude-cli`(구독·추가비용 0), 둘 다 없으면 dry-run.
  - 실제 에이전트 `.md` 본문을 system 으로 써 운영 동작 충실 재현.
  - **judge 다수결**(기본 3표): root_cause=최빈값(동률 시 보수적), recall/coverage=중앙값, ethos=과반 → 공식 재계산. `score_spread` 로 측정 모호 케이스 식별.
- `docs/eval/methodology.md` · `docs/eval/REPORT.md`

**baseline (3표 합의, sonnet)**: 평균 score **0.585** · tcode recall 0.738 · check coverage 0.619 · root cause full 0.286 · **avg judge spread 0.058**.

## Pillar C — "스스로 자란다": Learning Loop + Codify

> roadmap v2.4 · gstack `/skillify`·`/learn`·`decisions.jsonl` 대응

## 부수 발견 — "도구화가 결함을 드러낸다"

측정/도구를 만드는 과정에서 기존 시스템의 숨은 결함이 드러나, 우회 없이 근본 수정:

- **미등록 실존 T-code 9종** (eval gold-set 작성 중) — symptom-index 가 참조하나 tcodes.yaml 누락. 등록.
- **pii-scrubber 이메일/IP 마스킹 크래시** (codify 작성 중) — `scrubPII` 가 mask 에 캡처그룹 미전달. replace 콜백 인자 전달로 수정.
- **`.sapstack` 런타임/config 커밋 갭** (setup 작성 중) — `.gitignore` 로 차단(PII·실 회사코드).
- **ESM 은 NODE_PATH 무시** (eval CI 자동화 중) — 로컬 Windows 통과/Linux CI 실패. js-yaml 상대경로 import 로 근본 수정(run/aggregate/codify 3파일).
