# 금융업 SAP 운영 가이드

> **목적**: 금융기관(은행, 보험, 증권)의 SAP 시스템 운영자가 모듈 활성화, 규제준수, 한국 금융 환경을 이해하도록 작성.  
> **대상**: 금융 시스템 운영팀, 회계팀, 컴플라이언스팀, IT 보안팀

## 1. 금융업 SAP 모듈 활성화 매트릭스

### 핵심 모듈 (필수, 매일 운영)

| 모듈 | 역할 | 활용도 | 담당자 |
|------|------|--------|--------|

## 2. FI (재무) 심화 운영

### 2.1 일일 재무 거래 처리

금융기관의 SAP FI는 **은행 코어 시스템과는 별개**로, ERP 역할에 제한된다.

```
┌─────────────────────────────────────┐
│ 은행 코어 시스템 (Separate System)   │
│ 예: 하나은행 Core System, 우리은행  │
│     Finacle, IBK System           │
│ 역할: 예금/대출/환전 거래 처리      │
└──────────────┬──────────────────────┘
               │ (야간 배치)
               ↓ SAP로 데이터 전송 (FI)
┌─────────────────────────────────────┐
│ SAP FI (ERP 역할)                    │
│ ├─ 일일 외상 관리                    │
│ ├─ 원장 기록                         │
│ ├─ 부서별 손익 통계                  │
│ └─ K-IFRS 재무제표                   │
└─────────────────────────────────────┘
```

일반 제조업과 달리 금융기관은 **금융감독원 규정**에 따른 고정된 CoA 사용.

```
계정과목 (Account)

│ 15   고정자산                        │

┌─ 부채 (Liabilities) ─────────────────┐
│ 21   예금 (기업예금, 개인예금)      │
│ 22   차입금 (은행차입, 채권발행)    │
```

**실제 예시** (T코드: FSH1):
```
거래 1: 기업고객 "삼성전자" → 5억원 정기예금 1년

기록:
  Debit:  11 (현금및예치금)        500,000,000원
  Credit: 21 (예금-기업예금)               500,000,000원

1년 후 이자 지급 시:
  Debit:  80 (이자비용)              5,000,000원
  Credit: 11 (현금및예치금)                5,000,000원
```

### 2.3 원화/외화 병행 회계 (다통화)

한국 금융기관은 국내 및 해외 지점에서 원화/외화 거래를 동시 처리.

**SAP 다통화 설정** (T코드: OB22, TCURR):
```
거래 통화: USD 1,000,000 (달러 송금)

환율 (기준일자 2026-04-12):
  USD/KRW = 1,320원

기록:
  거래 통화: 1,000,000 USD
  회계 통화: 1,320,000,000 KRW
  환차이익: 환율 변동 시 발생

월 말 현재환율로 재평가:
  환율 상승 (1 USD = 1,350원): 
    → 환차이익 30,000,000 KRW 인식 (OCI 항목)
  환율 하락 (1 USD = 1,290원): 
    → 환차손실 30,000,000 KRW 인식
```

**SAP 설정**:
- T코드: OB08 (환율 등록)
- OB22 (통화별 소수점 자릿수)
- TCURR (통화별 기준 환율)
- 자동 재평가: 월 1회 (월말)

### 2.4 연결결산 (Consolidation)

지주사 구조 또는 다중 자회사 보유 시 필수.

**SAP EC-CS 또는 BCS** (T코드: CONS):
```
그룹 구조:
  모회사 (하나은행 본점)
    ├─ 해외지점 (뉴욕, 런던, 홍콩)
    ├─ 자회사 (하나투자증권, 하나금융투자)
    └─ 연결대상 회사 20개사

월 말 절차:
  1. 각 회사별 월말결산 완료 (FI 마감)
  2. 연결 데이터 수집 (자동 업로드 또는 수동)
  3. 상호 거래 제거 (Intercompany Elimination)
     예: 모회사-자회사 간 대출금 제거
  4. 소수주권 조정 (Minority Interest)
     예: 자회사 지분 60% 보유 → 40% 소수주권 기록
  5. 연결재무제표 생성 (Consolidated FS)
     - 자산: 그룹 전체 합계
     - 부채: 그룹 전체 합계
     - 자본: 모회사 귀속 지분만 기록
```

## 3. CO (통제) 심화 운영

### 3.1 금융기관 특화 손익구조

은행의 손익은 **전형적인 제조업과 전혀 다르다**.

```
연간 손익계산서 (예: 중견 규모 지방은행, 1조 자산)

수 익
─────────────────────────────────────
이자수익                      5,000억원
  └─ 대출금 이자: 대출금 × 평균금리 (약 3%)
     예: 100조 자산 중 70조 대출 × 3% = 2,100억원
  └─ 예금 이자: 예금 × 평균금리 (약 1%)
     예: 예금 50조 × 1% = 500억원
  └─ 순이자마진: 2,100 - 500 = 1,600억원

수수료수익                    1,500억원
  └─ 계좌관리비, 송금수수료, 신용카드 수수료

환헤지이익                      500억원
  └─ 환율 변동으로 인한 이익

기타수익                        500억원
  └─ 증권매매이익, 수익용부동산 임차료

총수익                        7,500억원

비 용
─────────────────────────────────────
이자비용                      1,000억원
  └─ 고객 예금에 지급할 이자

급여및복리후생                2,000억원
  └─ 직원 3,000명 × 연봉 5.5억원 + 상여금

대손충당금                    1,500억원
  └─ 부도 기업 대출금 손실 대비

감가상각                        500억원
  └─ 점포 건물, IT 시스템

기타비용 (부동산, IT 등)       1,000억원
  └─ 점포 임차료, 시스템 유지비

총비용                        6,000억원

영업이익                      1,500억원
  └─ 총수익 7,500 - 총비용 6,000

세전순이익                    1,200억원
  └─ 영업이익 - 법인세, 지방세

당기순이익                    1,000억원
```

### 3.2 부서별/상품별 손익 (Profitability Segment)

금융기관은 **상품별 손익**을 매일 모니터링한다.

**SAP CO-PA 설정** (T코드: KE5A):
```
Profitability Segment: 상품 + 채널

1. 대출 상품 (Loans)
   ├─ 가계 대출 (주택담보)
   │  ├─ 이자수익: 500억원 (평균금리 3%)
   │  ├─ 기본 비용: 50억원 (점포, 인력)
   │  ├─ 대손충당금: 50억원 (연체율 0.5%)
   │  └─ 순이익: 400억원 (손실률 0.4%)
   │
   ├─ 기업 대출 (신용대출, 시설투자)
   │  ├─ 이자수익: 800억원 (평균금리 4%)
   │  ├─ 기본 비용: 100억원
   │  ├─ 대손충당금: 200억원 (기업 신용 위험)
   │  └─ 순이익: 500억원 (손실률 1.0%)
   │
   └─ 자영업 대출
      ├─ 이자수익: 300억원 (평균금리 5.5%)
      ├─ 기본 비용: 60억원
      ├─ 대손충당금: 150억원 (높은 부도율 3%)
      └─ 순이익: -10억원 (손실!)

결론: 자영업 대출 상품 성과 부진 → 금리 인상 또는 상품 중단 고려
```

### 3.3 Activity-Based Costing (ABC)

간접비(은행 인력, 시스템)를 상품별로 정확히 배부.

```
연간 간접비:
  ├─ IT 시스템 운영비: 500억원 (서버, 보안, 개발)
  ├─ 점포 임차료: 300억원 (전국 200개 점포)
  ├─ 본부 인력비: 200억원 (기획, 감시, 법무)
  └─ 기타 운영비: 200억원

배부 기준:
  1. IT 비용: 거래량 기반
     - 대출 거래: 월 500M건 (70% 비중)
     - 예금 거래: 월 200M건 (20% 비중)
     - 증권 거래: 월 50M건 (10% 비중)
     → IT 비용 배부: 대출 350억원, 예금 100억원, 증권 50억원

  2. 점포 비용: 방문 고객수 기반
     - 대출 상담: 월 100,000명 (40%)
     - 예금/송금: 월 120,000명 (48%)
     - 기타 업무: 월 30,000명 (12%)
     → 점포 배부: 대출 120억원, 예금 144억원, 기타 36억원

결과:
  상품별 배부 간접비:
    - 대출: 470억원
    - 예금: 244억원
    - 증권: 86억원
```

## 4. TR (Treasury & Risk Management) 심화

### 4.1 자금관리 (Cash Management)

금융기관의 **일일 자금 수급**을 관리.

**실시간 현금 흐름** (T코드: TR01):
```
당일 오전 10시 현황:

예상 수입:
  ├─ 기업 고객 대출 상환: 200억원
  ├─ 예금 만기 도래: 500억원
  ├─ 이자 수익 입금: 100억원
  └─ 기타: 50억원
  → 총 수입 예상: 850억원

예상 지출:
  ├─ 고객 예금 회수: 600억원
  ├─ 신규 대출 실행: 400억원
  ├─ 채권 매입: 200억원
  ├─ 수수료 지급: 50억원
  └─ 기타: 50억원
  → 총 지출 예상: 1,300억원

자금 수급 불균형: -450억원 (부족)

해결책:
  1. 단기 채권 발행: 300억원 (3개월 CP)
  2. 타행 차입: 100억원 (Call 시장, 야간금리 1.5%)
  3. 유가증권 보유 매각: 50억원
```

### 4.2 금융상품 관리

금융기관이 보유한 다양한 자산의 가치평가 및 손익 추적.

**금융자산 분류** (K-IFRS 9):
```
상품 A: 정부채권 1,000억원

분류: FVOCI (Fair Value Through OCI)
  → 공정가치 변동을 기타포괄손익에 반영

회계처리:
  1. 취득: 1,000억원에 매입
     Debit:  금융자산(채권)      1,000억원
     Credit: 현금                         1,000억원

  2. 6개월 후 공정가치 평가: 1,050억원 상승
     Debit:  금융자산(채권)       50억원
     Credit: 기타포괄손익                 50억원
     
     → 손익계산서에 반영 안함
     → 대차대조표 자본에만 기록 (OCI)

  3. 채권 만기일 공정가치: 1,040억원
     Debit:  금융자산(채권)       -10억원 (감액)
     Credit: 기타포괄손익                 -10억원

  4. 채권 만기 (원금 회수)
     Debit:  현금                  1,040억원
     Credit: 금융자산(채권)              1,040억원
     
     → OCI 누적 40억원은 자본에서 유지
```

### 4.3 리스크 관리

**VaR (Value at Risk)** — 시장 변동성에 따른 손실 추정.

```
포트폴리오 구성:
  ├─ 주식 펀드: 500억원 (변동성 σ = 15%)
  ├─ 채권 펀드: 300억원 (변동성 σ = 5%)
  └─ 환률 파생상품: 200억원 (변동성 σ = 12%)
  → 총 포트폴리오: 1,000억원

VaR 계산 (신뢰도 95%, 기간 1일):
  VaR = Portofolio Value × Z-score × σ
      = 1,000억원 × 1.645 × √(0.5×(0.15)² + 0.3×(0.05)² + 0.2×(0.12)²)
      = 1,000억원 × 1.645 × 0.096
      = 158억원

해석:
  → 내일 하루 중 95% 확률로 손실은 158억원 이내
  → 만약 158억원 이상 손실 → 극도의 시장 변동(테일리스크)

리스크 관리:
  ① 헤지: 파생상품으로 손실 범위 제한
  ② 스트레스 테스트: 금리 1% 상승 시 손실액 계산
  ③ 규제 자본금: 리스크 자산의 일정 비율 보유
```

### 4.4 Hedge Accounting (헤지회계)

금융상품의 공정가치 변동을 일부 상쇄.

**예시: 금리 상승 리스크 헤지**

```
기본 자산: 고정금리채 500억원 (금리 3%, 채권가격 역상관)

위험요소:
  금리 상승 → 채권가격 하락 → 포트폴리오 손실

헤지수단:
  금리스왑 (Interest Rate Swap)
    ├─ 받기: 고정 3% (채권으로 이미 수익)
    ├─ 지기: 변동금리 (SOFR + 0.5%)
    └─ 효과: 금리 상승 시 변동금리 지급액 감소 → 손실 상쇄

회계처리 (Cash Flow Hedge):
  1. 채권 취득
     Debit: 금융자산(채권)    500억원
     Credit: 현금                      500억원

  2. 금리 상승 (3% → 4%)
     채권 손실: -50억원 (공정가치 하락)
     스왑 이익: +30억원 (변동금리 인하로 지급액 감소)
     
     순 손실: -20억원 (헤지 효과: 70% 성공)
     
  3. 기록:
     Debit: 손실(스왑 이익)         30억원
     Credit: 기타포괄손익                   30억원
     
     → 손익계산서 변동성 완화
```

```

  - 금액 기준: 50만원
  - 승인 단계: 2단계 (팀장, 부점장)
  - Approval Workflow: 자동화 (workflow_APPROVAL_50M)
  - 타임아웃: 3영업일 (미승인 시 회수)
```

### 7.4 K-IFRS 17호: 보험계약 회계기준

보험업계에 가장 큰 변화. 2025년 적용 예정.

**변경 사항**:
```
기존 (K-IFRS 4):
  보험료 수입 → 보험금 지급 차이 = 수익

신규 (K-IFRS 17):
  Contract Service Margin (CSM) 개념 도입
  → 현재가치 기준 이익 인식

예시:
  생명보험 계약 1건
  - 월 보험료: 100,000원 (10년, 총 1.2M원)
  - 보험금: 만기 10,000,000원
  
  기존 회계:
    Year 1: 보험료 1.2M원 - 운영비 500K원 = 수익 700K원
  
  신규 회계:
    Year 1: 예상 순현재가치의 일부만 수익 인식
    (할인율, 사망률 변동 포함)

SAP 영향:
  - FI: 보험계약 일일 평가 필요
  - CO: CSM 계산 알고리즘 구현
  - BI: 월별 CSM 변동 보고
```
