# 유통업 SAP 운영 가이드

> **목적**: 유통·소매 기업의 SAP 시스템 운영자가 모듈 활성화, 프로세스 설계, 한국 이커머스 환경을 이해하도록 작성.  
> **대상**: 판매/물류 담당자, 재고 관리자, 이커머스 담당자, 물류센터 운영팀

## 1. 유통업 SAP 모듈 활성화 매트릭스

### 핵심 모듈 (필수, 매일 운영)

| 모듈 | 역할 | 활용도 | 담당자 |
|------|------|--------|--------|
| **SD** (Sales & Distribution) | 판매오더 관리, 배송, 청구 | 100% | 판매/주문팀 |

## 2. SD (판매 & 유통) 심화 운영

### 2.1 판매오더 처리 프로세스

유통업의 핵심은 **수주부터 배송까지의 속도**다. SAP SD는 매일 수백~수천 건의 오더를 처리한다.

```
┌─────────────────────────────────────────┐
│ 주문 수신 (Order Input)                  │
│ - 온라인: SAP Commerce Cloud 자동 import│
│ - 도매: EDI/XML 수신                    │
│ - 직통: 전화/이메일 → 수동 입력         │
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│ 판매오더 생성 (SO: Sales Order)          │
│ - 오더 번호: 1000000001                 │
│ - 고객: CUST-0001 (ABC마트)             │
│ - 라인: 5개 (5개 상품)                  │
│ - 배송처: 서울역점, 부산중앙점 등      │
│ - 배송일: 2026-04-13                    │
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│ 재고 확보 (Stock Check)                  │
│ - 온라인 실시간: ATP(Available-to-Promise)│
│ - 배치 프로세스: 야간 2회               │
│ - 초과매수 방지: SO 금액 제한            │
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│ 픽킹 및 분류 (Picking & Sorting)        │
│ - WM에서 위치 확인(Bin)                 │
│ - 피킹 리스트 인쇄/모바일              │
│ - 바코드 스캔 확인                      │
│ - 배송처별 분류                         │
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│ 배송 (Shipment)                          │
│ - 송장 인쇄(배송사 연동)                 │
│ - 배송상태: 실시간 추적                 │
│ - 고객: 문자/이메일 알림                │
└────────────────────┬────────────────────┘
                     ↓
┌─────────────────────────────────────────┐
│ 배송 완료 및 청구 (Invoice)              │
│ - 도착 확인: GPS/사진 증거               │
│ - 청구건 생성(고객사 인보이스)           │
│ - 외상 기록(매출채권)                    │
└─────────────────────────────────────────┘
```

### 2.2 대량 주문 처리: EDI/IDoc 자동화

대형 도매사(GS, CJ, Costco 등)는 매일 수천 건 오더를 EDI로 전송한다.

**SAP EDI 설정** (T코드: BD10, IDOC):
```
파트너: GS SHOP (한국 최대 도매사)
  ├─ EDI 주소: partner.gs.co.kr/edi
  ├─ 오더 양식: UN/EDIFACT D96A ORDERS
  ├─ 전송 빈도: 매일 오전 9시 배치
  └─ 자동 SO 생성: 90% (유효성 통과)

수신된 IDOC: ORDERS 메시지
  ├─ 파트너사 PO: GS-20260412-001
  ├─ SKU 5개: PROD-001, PROD-002 등
  ├─ 수량: 각 100개, 200개 등
  ├─ 배송 주소: 서울역점, 부산중앙점
  └─ 요청배송일: 2026-04-15

자동 변환:
  GS PO (EDI) → SAP SO (1000000234 생성)
  자동 완료율: 99% (오류 1% → 수동 검수)
```

**EDI 오류 처리**:
- 재고 부족: 자동 백오더(BO) 생성, 고객 통지
- 오더 형식 오류: ARC(Advanced Returns Concept) 오류 로그 생성
- 중복 오더: 자동 감지(PO 번호 비교), 운영팀 승인 대기

### 2.3 프로모션 가격 관리: Condition Type 계층

유통은 **일일이상 변동하는 가격**으로 특징지어진다.

**SAP Pricing 구조** (T코드: V/06, TCODE):
```
기본가 (Base Price)
  + 고객그룹 할인 (Cond Type: K004, 5%)
  + 수량할인 (Cond Type: K005, 주문액 1M 이상 3%)
  + 프로모션 쿠폰 (Cond Type: ZKUP, 10%)
  + 신규고객 할인 (Cond Type: K007, 15%)
  ─────────────────
  = 최종 판매가 (Net Price)

예시: 상품 A 기본가 100,000원
  - 마트 중대형 고객 할인 (5%) → 95,000원
  - 100개 이상 주문 할인 (3%) → 92,150원
  - 봄시즌 프로모션 (10%) → 82,935원
```

**마스터 데이터 관리**:
- 기본가(Base): 원가 + 마진 (연 1~2회 변경)
- 고객별 할인: 계약 기반 (매년 재협상)
- 수량할인: 계층식 (0~99, 100~499, 500+)
- 프로모션: 임시 (주간/월간, 승인 후 활성화)

**SAP Condition 유효기간 관리**:
```
11.11 대행사 (블랙프라이데이)
  Condition: ZKUP (쿠폰) 10% 할인
  유효기간: 2026-11-11 ~ 2026-11-11 (1일만)
  적용대상: 모든 고객
  우선순위: 1순위 (다른 할인 제외)
  
결과:
  - 오전 0시 자동 활성화
  - 오후 11시 59분 자동 비활성화
  - 시간대별 트래픽 관리 (예: 쿠폰 한정 10,000장)
```

### 2.4 Returns & 교환 프로세스

대형 이커머스(쿠팡, 무신사 등)는 **반품률이 20~40%** 수준으로 높다.

**SAP 반품 관리** (T코드: VA02, VL06O):
```
원본 SO (1000000001): 100개 판매
  ↓ [고객 반품 요청]
반품 SO (1000000001RE): 20개 반품

SAP 처리 흐름:
  1. Returns 오더 유형(RA) 생성
  2. 반품 배송 (Goods Issue) → 재고 복구
  3. 반품 검사 (QA): 양호/손상/오염 판정
  4. Credit Memo (크레딧 메모) 발행
     - 양호: 전액 환불
     - 손상: 50% 환불 + 폐기 처리
     - 오염: 0% 환불 + 폐기 처리
  5. 고객 환불 (FI에서 외상 조정)
```

**한국 이커머스 반품 규칙**:
- **배송료**: 선불(판매자 부담) 또는 착불(고객 부담) — 정책 명확화
- **반품 기한**: 구매 후 7일 이내 (법정 요구)
- **손상 판정**: 위탁 물류센터의 전담팀이 검사
- **환불 속도**: 반품 완료 후 3영업일 이내 환불 (규제 요구)

**SAP에서 추적**:
- Return 오더 생성일 기록 (SO Header의 반품 가능 여부 확인)
- GI(반품 입고) 날짜 기록
- QA 검사 결과(양호/손상) 기록
- Credit Memo 발행일 기록
- 환불 완료일 기록 (FI 외상 차감)

## 3. WM/EWM (물류센터) 운영

### 3.1 물류센터 아키텍처

대형 이커머스 물류센터(쿠팡, 마켓컬리 등)의 구조.

```
┌─ 입고 게이트 (Inbound Gate) ─────┐
│                                   │
│ ① ASN 수신 (Advance Shipment Notice)
│    - 공급사/도매사에서 미리 통지
│    - T코드: IDOC (EDI), 또는 API
│    - 정보: SKU, QTY, 도착 시간
│                                   │
└───────────────┬───────────────────┘
                ↓
   ┌─ Receiving Dock ─┐
   │ (수입 검수 구역)  │
   └─────────┬───────┘
             ↓
   ② Goods Receipt (입고)
      - T코드: MIGO, 또는 WM 모바일
      - 바코드/QR 스캔
      - 수량 검증
      - 회차별 분류
             ↓
   ③ Putaway (자동 저장위치 배정)
      - 상품 종류별 존(Zone) 분류
      │  ├─ Fast-Moving Zone (회전율 높음, 앞쪽)
      │  ├─ Slow-Moving Zone (회전율 낮음, 뒤쪽)
      │  └─ Temperature Zone (냉동/신선)
      │
      - 바코드 부착 및 입고 완료
             ↓
   ④ Storage (저장)
      - 자동 랙 시스템(AS/RS) 또는 수동 선반
      - 실시간 재고 추적(WM Master Data)
             ↓
   ⑤ Order Picking (주문 피킹)
      - 일일 수천 건 SO에 대한 피킹
      - 피킹 방식:
        ├─ Pick-to-Light: 조명으로 피킹 위치 표시
        ├─ Put-to-Light: 분류 시 조명 표시
        └─ Voice Picking: 음성 명령 기반
      - 효율: 시간당 300~500건
             ↓
   ⑥ Sorting (분류)
      - 배송지역별, 배송업체별 분류
      - 컨베이어 벨트 자동화
             ↓
   ⑦ Packing (포장)
      - 에어캡/종이 완충재
      - 송장 인쇄/부착
      - QC: 상품 확인
             ↓
   ⑧ Shipping (배송)
      - 배송사로 인수
      - 송장 전송(배송사 API)
      - 배송 추적(실시간)
```

### 3.2 WM vs EWM 선택 기준

| 항목 | WM (기본) | EWM (고급) |
|------|---------|----------|
| 입고 처리 | 수동 기반 | 자동화 + AI 최적화 |
| 피킹 방식 | 픽 리스트 인쇄 | Pick-to-Light 지원 |
| 라우팅 최적화 | 간단한 규칙 | 복잡한 알고리즘 |
| 배송비 계산 | 정적 | 동적(시간/날씨 반영) |
| 다중 선적지 | 제한적 | 완전 자동 |
| 비용 | 낮음 | 높음 (연 수억원) |
| 추천 기업 | 중소 유통사 | 쿠팡/무신사 같은 대형 |

**한국 유통사 사례**:
- **쿠팡**: EWM 기반 (독자 개발 + SAP EWM 커스터마이징)
- **마켓컬리**: 자체 WMS (Coupang 인수 후 마이그레이션 중)
- **롯데쇼핑**: SAP EWM + 택배사 자동 연동
- **GS Retail**: SAP WM + MES 스타일의 간단한 연동

### 3.3 DAS (Digital Assorting System)

한국 유통의 독특한 시스템. 소팅 자동화의 표준.

```
Pick-Wave: 1000개 상품
         (100개 × 10 배송지역)
  ↓
DAS 컨베이어 입구
  - 각 상품에 배송지역 바코드 라벨 부착
  ↓
고속 스캔 & 라우팅
  - 상품 1초에 1개 스캔
  - 배송지역에 따라 자동 분류
  ↓
배송지역별 구분 (10개 라인)
  - 서울 동부 500개
  - 서울 서부 350개
  - 경기 북부 150개
  (등등 10개 지역)
```

**DAS 연동 SAP 설정** (T코드: VS01, VL06O):
1. Shipment 생성 시 배송지역 코드 자동 입력
2. 픽킹 리스트에 DAS 바코드 포함
3. 배송 완료 확인(VL02N) 시 자동 배송지역별 통계

**효과**:
- 피킹 오류율: 0.5% 이하 (수동 기반 3~5%)
- 처리 속도: 시간당 2,000~3,000건 (수동 300~500건)
- 인력 절감: 50~60%

## 4. MM (재고 관리) 심화

### 4.1 ABC 분석 기반 재고 정책

유통은 보유 SKU가 매우 많다 (쿠팡: 1M+ SKU).

```
A 상품 (상위 20%, 매출 80%)
  예: 인기 의류, 일용품
  ├─ 재고 정책: 충분한 재고 (안전재고 높음)
  ├─ 발주 빈도: 주 3~4회
  ├─ 자동 재주문: ON
  └─ 관심: 재고 부족 (기회손실) 방지

B 상품 (중간 30%, 매출 15%)
  예: 계절 상품, 준인기 카테고리
  ├─ 재고 정책: 적정 재고
  ├─ 발주 빈도: 주 1~2회
  ├─ 자동 재주문: 부분 (임계값 초과 시만)
  └─ 관심: 선회율과 비용 균형

C 상품 (하위 50%, 매출 5%)
  예: 니치 상품, 카테고리 완성용
  ├─ 재고 정책: 최소 재고
  ├─ 발주 빈도: 월 1회 또는 수동
  ├─ 자동 재주문: OFF
  └─ 관심: 폐기 방지, 자금 효율
```

**SAP에서 ABC 분석** (T코드: MM04, MATNR):
```
자재마스터 데이터:
  SKU-001 (셔츠)
  ├─ 연간 매출: 50억원 (A 상품)
  ├─ 안전재고: 1,000개
  ├─ MRP 타입: V (수요 기반 발주)
  ├─ 발주량: 500개 (정량식)
  └─ 자동재주문: 예

  SKU-5000 (니치 가구)
  ├─ 연간 매출: 10억원 (C 상품)
  ├─ 안전재고: 50개
  ├─ MRP 타입: P (계획식, 월 1회)
  ├─ 발주량: 100개
  └─ 자동재주문: 아니오
```

### 4.2 Safety Stock 자동 계산

발주점(Reorder Point) = 평균수요 × 납기 + 안전재고

```
상품 A (마스크):
  - 일일 평균수요: 1,000개
  - 공급사 납기: 5일
  - 공급사 신뢰도: 85% (가끔 지연)
  - 재고 회전율 목표: 월 3회

안전재고 계산:
  = Z-value (85% 신뢰도 = 1.04) 
    × 표준편차 (500개)
  = 520개

재주문점(ROP):
  = (1,000개/일 × 5일) + 520개 = 5,520개

실행:
  - 현재 재고 5,520개 도달 시 자동 PReq 생성
  - 발주량: 5,000개 (경제주문량, EOQ)
  - 도착 예정일: 2026-04-17 (5일 후)
```

### 4.3 Seasonal Demand 반영

한국 유통은 명절과 쇼핑 시즌에 수요가 급변한다.

```
월별 수요 계수 (설정):
```

## 5. 한국 유통업 특수 사항

### 5.1 대형 유통사 운영 패턴

**쿠팡** (온라인 마켓플레이스 + 로켓배송):
- **모델**: 3PL (Third Party Logistics) 중심
- **물류**: 독자 로봇 자동화 + 쿠팡 데이터 센터 (MES 스타일)
- **SAP 활용**: 판매 데이터 수집만 (결제는 별도)
- **배송 속도**: 당일배송/새벽배송 (고정 물류 투자)
- **반품**: 고객 친화적 (반품 비용 회사 부담)

**마켓컬리** (신선식품 이커머스):
- **모델**: 위탁 물류 (Coupang 인수 후 자체 시스템)
- **특성**: 식품 신선도 관리 필수
- **SAP 활용**: Lot/Batch 추적, 유통기한 관리
- **냉동/냉장 구분**: 3개 온도 존(Temperature Zone)
- **배송 속도**: 새벽배송 (24시간 운영 물류센터)

**롯데/이마트** (오프라인 + 온라인 통합):
- **모델**: 자체 물류센터 (여러 지역)
- **SAP 활용**: 가맹점주문 + 온라인 SO 통합
- **Omnichannel**: 점포 픽업(BOPIS) 지원
- **재고 공유**: 본점/지점/온라인 간 실시간 재고 공유

**GS Retail** (GS25 + GS Shop):
- **모델**: 가맹점 중심 + 도매 물류
- **SAP 활용**: EDI 기반 도매 주문 자동화
- **배송**: 일일 다회차 (이른 아침, 오후 2회)
- **반품**: 엄격한 정책 (가맹점 수익성 보호)

### 5.2 POS 연동 및 실시간 재고 수집

한국 대형 마트/편의점은 **POS → SAP 실시간 동기화**가 표준.

**POS 시스템** (예: 삼성SDS POSWARE):
```
점포 A (강남점) POS 터미널
  ├─ 판매 발생: 오후 2시 15분
  │  SKU-001 (셔츠) 2개 판매 @ 50,000원
  │
  ├─ 로컬 DB 기록
  │
  └─ 배치 전송 (실시간 또는 1시간 주기)
      ↓
SAP IDOC 수신 (T코드: WE05)
  ├─ 판매 수량 기록 (매출 통계)
  ├─ 재고 차감 (MM03)
  ├─ 매출 기록 (FI 수익)
  └─ 대시보드 업데이트 (BI 리포트)

결과:
  - 재고: SKU-001 100개 → 98개 (자동 차감)
  - 매출액: +100,000원 (FI에 기록)
  - 실시간 대시보드: 강남점 당일 매출 5억원
```

**SAP 설정**:
- T코드: BD64 (IDoc 전송 규칙)
- Segment: ORDERS/ORDERS05 (판매 데이터)
- 빈도: 매 시간 또는 실시간 (이벤트 기반)
- 오류 처리: 재전송 3회, 이후 운영팀 알림

### 5.3 대규모 프로모션 관리

**11.11 데이(싱글즈 데이) 또는 블랙프라이데이**는 연간 매출의 30~40%를 차지.

**준비 단계** (1개월 전):
```
10월 10일:
  - 프로모션 상품 선정 (500~1,000개 SKU)
  - Condition 설정 (할인율 10~50%)
  - 재고 확보 (정상 재고의 3~5배)
  - 배송사 추가 계약 (인력+차량)

10월 15일:
  - SAP에 Condition 등록 (T코드: V/06)
  - 유효기간: 2026-11-11 00:00 ~ 2026-11-11 23:59
  - 재고 시뮬레이션 (DP, "이 할인으로 예상 매출은?")
  - 서버 증설 완료 (온라인 트래픽 대비)

10월 20일:
  - 마케팅 캠페인 시작
  - 고객 예약 오더 수용 시작
  - 배송사와 최종 검증

11월 10일:
  - 모든 준비 완료
  - 최종 Condition 활성화 (00:00 시작)

11월 11일:
  - 판매 데이터 실시간 모니터링
  - 인기 상품: 재고 부족 → 백오더 전환
  - 배송: 물류센터 풀 가동 (24/7)

11월 12일:
  - Condition 자동 비활성화 (23:59 종료)
  - 프로모션 종료, 재고 정상화
```

**SAP 모니터링** (T코드: SDFV, ZSD_PROMO_MONITOR):
```
실시간 대시보드:
  - 판매량 (실시간): 300,000개/시간
  - 매출액: 1,500억원/시간
  - 재고: 초기 2M개 → 현재 1.5M개 (25% 소비)
  - 예상 완판 시각: 자정
  - 배송 대기: 50,000건 (당일 배송 불가)
```

### 5.4 식품 유통: HACCP & 유통기한 관리

마켓컬리, 쿠팡프레시 등 신선식품 유통의 특수성.

**HACCP 요구사항**:
- 원재료부터 완제품까지 전체 추적 (Traceability)
- 온도 기록 (냉장: 0~4℃, 냉동: -18℃ 이하)
- 미생물 검사 기록 (월 1회 이상)
- 보존료 사용 기준 준수

**SAP에서 Batch 관리** (T코드: MSC2):
```
상품: 유자차
Batch 번호: 20260401-001
  ├─ 제조일: 2026-04-01
  ├─ 유통기한: 2027-04-01 (365일)
  ├─ 온도존: Cold (냉장)
  ├─ 보관 조건: 0~4℃
  └─ 입고: 1,000개

입고 후 추적:
  - 입고 날짜: 2026-04-01
  - 저장 위치: Zone-COLD-A (냉장실)
  - 픽킹 순서: FIFO (먼저 들어온 것 먼저)
  - 배송: 2026-04-05 (4일 소비 시간 고려)
  - 예상 고객 수령: 2026-04-06
  - 고객 소비 기한: 2027-04-01 (365일)
```

**Shelf Life 관리** (T코드: V_MSTD):
- 자동 Batch Selection: 유효기한 순 자동 픽킹
- 경고: Expiry Date 30일 전 자동 경고 메시지
- 폐기: 유통기한 경과 → 자동 폐기 처리 (WM에서 Scrap)

### 5.5 전자상거래 통합

**쇼핑몰 유형별 SAP 연결**:

1. **자체 쇼핑몰** (쿠팡, 마켓컬리):
   - SAP Commerce Cloud 또는 Magento 연동
   - 주문(SO) 자동 동기화
   - 결제는 별도 Payment Gateway

2. **마켓플레이스** (11번가, 쿠팡, 네이버 스토어):

   - 재고 동기화 (실시간 또는 1시간주기)
   - 반품 관리 (마켓플레이스별 규칙 상이)

3. **소셜커머스** (라이브 스트리밍 판매):
   - 예측 불가한 주문 폭증
   - SAP 처리 능력 테스트 필수 (부하테스트)
   - 재고 즉시 사중복방지

**SAP Commerce Cloud 연동 예시**:
```
고객이 Naver 스토어에서 구매
  ↓
Naver 결제 게이트웨이 (PG)
  ↓

SAP ERP에 주문 수신

  - 재고 복구
```

## 6. CO (Controlling): 채널별/상품별 손익 분석

유통 기업의 경영진이 가장 관심 있는 분석.

### 6.1 Profitability Analysis (CO-PA)

```
판매채널별 손익:
  
1. 온라인 다이렉트 (자체 쇼핑몰)
   ├─ 매출: 500억원
   ├─ 원가: 250억원 (원가율 50%)
   ├─ 배송비: 50억원
   ├─ 광고비: 100억원
   ├─ 마진: 100억원 (마진율 20%)
   └─ 순이익: 100억원

2. 마켓플레이스 판매 (쿠팡, 네이버)
   ├─ 매출: 400억원
   ├─ 원가: 200억원 (원가율 50%)
   ├─ 플랫폼 수수료: 60억원 (15%)
   ├─ 배송비: 40억원
   ├─ 마진: 100억원 (마진율 25%)
   └─ 순이익: 100억원

3. 도매 판매 (GS, CJ 등)
   ├─ 매출: 300억원
   ├─ 원가: 240억원 (원가율 80%)
   ├─ 도매 할인: 20억원
   ├─ 배송비: 10억원
   ├─ 마진: 30억원 (마진율 10%)
   └─ 순이익: 30억원
```

**SAP CO-PA 설정** (T코드: KE3A, KE5A):
- Profitability Segment: 판매채널 + 상품카테고리
- 가치사슬 분석: 제조원가 → 유통마진 → 순이익

### 6.2 ABC (Activity-Based Costing)

배송비, 포장비, 반품처리비 등 간접비의 정확한 배부.

```
전체 배송비: 50억원/월
배송 구분:
  - 당일배송 (20억원): 고객만족도 높음, 비용 높음
  - 일반배송 (30억원): 표준 배송

ABC 계산:
  상품 A (고마진, 고가 제품)
    ├─ 판매량: 100개
    ├─ 평균 무게: 3kg
    ├─ 배송 비중: 5% (15kg)
    └─ 배송비 배부: 2.5억원

  상품 B (저마진, 저가 제품)
    ├─ 판매량: 1,000개
    ├─ 평균 무게: 0.2kg
    ├─ 배송 비중: 20% (200kg)
    └─ 배송비 배부: 10억원
```

**결과**:
- 상품 A: 순이익 높음 (배송비 적음)
- 상품 B: 순이익 낮음 (배송비 높음)
- 경영 의사결정: 상품 B 마진율 상향 또는 프로모션 중단

## 7. 체크리스트: SAP 구축 및 운영 준비

### 구축 단계
- [ ] 판매채널 분류 (온라인/오프라인/도매/직통)

- [ ] 프로모션 Condition 템플릿 설계

- [ ] POS 연동 아키텍처 확정

### 운영 단계
- [ ] 매주 MRP 실행 (DP 수요예측 업데이트)
- [ ] 일일 SO 모니터링 (재고 부족, 배송 지연 추적)
- [ ] 일일 배송비 리포트 (채널별, 지역별)
- [ ] 월 1회 CO-PA 분석 (채널별/상품별 손익)

- [ ] 월 1회 배송 성능 리뷰 (OTIF: On-Time In-Full)
- [ ] 월 1회 재고 정확성 점검 (Inventory Variance)
- [ ] 분기 1회 공급사 성과 평가 (납기 준수율, 품질)
- [ ] 연 1회 물류센터 자동화 투자 검토 (DAS, EWM 업그레이드)
