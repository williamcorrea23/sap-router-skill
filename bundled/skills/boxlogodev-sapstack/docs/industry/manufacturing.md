# 제조업 SAP 운영 가이드

> **목적**: 제조기업의 SAP 시스템 운영자가 모듈 활성화, 프로세스 설계, 한국 산업 특수성을 이해하도록 작성.  
> **대상**: ERP 운영 담당자, PP/MM/QM 코디네이터, MES 담당자

## 1. 제조업 SAP 모듈 활성화 매트릭스

| 모듈 | 역할 | 활용도 | 담당자 |
|------|------|--------|--------|
| **PP** (Production Planning) | 생산계획, 자재소요량 계획, 생산오더 관리 | 100% | 생산계획팀 |

| **HCM** (Human Capital Management) | 3교대 근무관리, 인력 배치 | 30~60% | 대규모 공장(500명 이상) |
| **MES** 연동 (Manufacturing Execution) | SAP ↔ MES 실시간 동기화 | 80~100% | 스마트팩토리 구축 기업 |

## 2. PP (생산계획) 심화 운영

### 2.1 MRP 전략 선택

제조 기업은 생산 방식에 따라 MRP 전략을 결정한다. 이 선택은 BOM 구조, 재고 정책, 납기 관리에 직접 영향을 미친다.

#### MTS (Make-To-Stock) — 재고 생산
- **특징**: 수요 예측 기반 사전 생산, 자재소요량계획(MRP) 자동 생성
- **적합 산업**: 음료·식품·전자부품·화학·의약품
- **SAP 설정**:
  - Demand Program (DP) 또는 Forecast에서 월별 수요 입력
  - PP 마스터 데이터에서 `Planning time fence` 설정 (일반적 2~4주)
  - MRP 실행 (T코드: MD01) → 자동 구매요청(PReq) + 생산오더 생성
  - 안전재고(Safety stock) 계산식: `(Max lead time - Normal lead time) × Average demand`

#### MTO (Make-To-Order) — 수주 생산
- **특징**: 고객 주문 후 생산 시작, 리드타임 긴편
- **적합 산업**: 중공업·조선·기계·반도체 장비·가구
- **SAP 설정**:
  - 판매오더(SO) 생성 → 자동 생산오더 예약(Production Reservation)
  - Make-to-order BOM 사용 (BOM 유형: 40)
  - MPS(Master Planning Schedule) 수준에서 SO demand 반영
  - 납기 예측: Rough-cut 용량 계획(RCCP, T코드: MD28) 사용

#### ETO (Engineer-To-Order) — 설계 생산
- **특징**: 고객 맞춤 설계 후 생산, 리드타임 매우 김
- **적합 산업**: 플랜트·용역기계·반도체 제조장비
- **SAP 설정**:
  - 고객 인자(Characteristics)를 BOM에 반영 (Variant BOM 또는 Configurable BOM)
  - 프로젝트 코드 기반 추적 (PS 모듈과 연계)
  - SO 라인 → 생산오더 1:1 매칭
  - 변경관리(Change Control) 프로세스 필수

### 2.2 BOM (Bill of Materials) 관리 체계

BOM은 제조업의 중추이다. 하나의 제품에는 여러 BOM이 동시 존재한다.

```
┌─────────────────────────────────────┐
│   완제품 (Finished Product)          │
├─────────────────────────────────────┤
│  ↓ Engineering BOM (설계팀)          │
│    (설계도면과 일치, 전체 부품 나열)  │
│                                     │
│  ↓ Production BOM (제조팀)           │
│    (생산 실행용, 공정별 분류)        │
│    - Module A (공정 10 투입)         │
│    - Module B (공정 20 투입)         │
│                                     │
│  ↓ Costing BOM (원가팀)             │
│    (원가계산용, 간접비 배부 포함)    │
└─────────────────────────────────────┘
```

**SAP에서 BOM 관리** (T코드: CS01/CS02/CS03):
- **BOM Header**: 제품코드, 플랜트, 버전, 유효기간
- **BOM Item**: 부품/반제품, 수량, 손실율(Scrap %), 투입점(Operation)
- **BOM 버전**: 동일 제품의 여러 BOM 병행 가능 (예: v1.0→v1.1로 점진 전환)
- **대량 변경**: MASS BOM Change (T코드: CS90)로 수천 개 BOM 일괄 수정

**한국 제조업 일반 관행**:
- 설계 BOM과 생산 BOM의 차이가 큼 (설계는 부품 중심, 생산은 공정 중심)
- 하청/외주 부품은 별도 BOM 버전 관리 (예: BOM v1.0=자체제조, v1.1=OEM 수입)
- 변경 이력 추적 필수 (감시 규정)

### 2.3 작업장(Work Center) & 라우팅(Routing) 설계

생산 공정의 흐름과 시간을 정의한다.

#### 작업장(Work Center) 정의 (T코드: CR01)
```
공장 1-2호선 조립실
├─ 작업장 코드: WC-ASM-001
├─ 용량: 5명 × 8시간 = 40시간/일
├─ 기계 비용: 50,000원/시간
├─ 인력 비용: 20,000원/시간
└─ 설치 시간(Setup): 30분 (지그 교환)
```

#### 라우팅(Routing) 정의 (T코드: CA01)
- **라우팅 헤더**: 제품코드, 생산 수량, 유효기간
- **작업 단계(Operation)**:
  - Op 10: 미링(Milling) — WC-MILL-001, 표준시간 2시간/100개
  - Op 20: 조립(Assembly) — WC-ASM-001, 표준시간 3시간/100개
  - Op 30: 검사(Inspection) — WC-INS-001, 표준시간 1시간/100개
  - Op 40: 포장(Packing) — WC-PKG-001, 표준시간 0.5시간/100개

**SAP 라우팅 시간 구성**:
- **Setup time**: 공정 시작 준비 시간 (지그 교환, 기계 설정)
- **Machine time**: 기계 처리 시간
- **Labor time**: 작업자 실제 작업 시간
- **Wait time**: 공정 간 대기 시간
- **Move time**: 공정 간 이동 시간

**생산 스케줄링 방식**:
- **Forward Scheduling**: 현재부터 시작하여 완료 일자 역산 (MTO 수주생산)
- **Backward Scheduling**: 요구 완료 일자부터 역산하여 시작 일자 결정 (MTS 재고생산)

### 2.4 생산 실적 입력 (CO11N)

생산 진행률을 SAP에 기록하는 핵심 프로세스.

#### 방식 1: 자동 입력 (MES → SAP RFC)
대형 공장(삼성, SK 등)이 주로 사용. MES에서 완료한 수량을 실시간 SAP로 전송.
```
MES: "공정 10 완료 100개" 
  ↓ RFC 호출
SAP BAPIGOODSMVT_CREATE (확인 기록) 자동 생성
```

**장점**: 실시간 추적, 오류 최소화, 원가계산 자동화  
**단점**: MES 투자 비용 큼, 유지보수 인력 필수

#### 방식 2: 수동 입력 (CO11N)
중소 제조업에서 일반적. 라인별 운영자가 매일 1회(또는 교대별 3회) 입력.

**CO11N 화면 구성**:
- 생산오더 번호: PP-000001234
- 작업(Operation): 10, 20, 30, 40
- 완료 수량(Confirmed Qty): 100개
- 폐기 수량(Scrap Qty): 2개 (2%)
- 실제 시간: Setup 0.5h, Machine 2.1h, Labor 2.0h
- 원가 자동 계산 (표준시간 vs 실제시간의 차이 = 원가 차이)

**입력 규칙**:
- 당일 입력 원칙 (원가 정확성, 재고 추적)
- 부분 입력 가능 (예: 10개 완료 → 50개 완료 → 100개 완료)
- 오입력 시 역입력(Reversal) 후 재입력

## 3. MM-PP 연계: 자재 소요량부터 생산까지

### 3.1 표준 흐름

```
[MRP 실행]
    ↓
[자동 구매요청(PReq) 생성]
    ↓
[구매 발주(PO) 생성 및 송장]
    ↓
[입고(GR: Goods Receipt)]
    ↓
[생산 투입(투입점 라우팅에 따라)]
    ↓
[생산 실적 입력(CO11N)]
    ↓
[제조원가 계산(CK11N)]
```

### 3.2 외주 생산(Subcontracting) 프로세스

하청사에서 중간 부품을 생산받는 경우.

**SAP 설정**:
1. 자재마스터 플래그: "Purchased/Subcontracted Part" 체크
2. 라우팅 작업 유형: `30 (Subcontracting)`으로 설정
3. PO 생성 시 "Delivery Completed Indicator" 체크

**흐름**:
```
원자재 출고 (Movement type 541)
  원자재 → 하청사 이전
    ↓
[하청사에서 처리]
    ↓
완성품 입고 (Movement type 101)
  원자재 소비 → 원가 계산
```

**실제 예시** (전자부품 조립):
- 생산오더: PP-001001
- Op 10: 내부 배선 (자사 공장)
- Op 20: 리플로우 솔더링 (하청사 A)
- Op 30: 내부 배선 (자사 공장)
- Op 40: 최종검사 (자사 공장)

## 4. QM-PP 연계: 품질 관리

### 4.1 검사 유형별 프로세스

| 검사 유형 | 코드 | 대상 | 시점 | 담당 |
|---------|------|------|------|------|
| 원자재 수입검사 | 01 | 구매한 원자재/부품 | PO 입고 직후 | 품질검사팀 |
| 공정 중 검사 | 03 | 반제품 | Op 완료 후 다음 Op 전 | 라인 운영자 또는 검사팀 |
| 완제품 출하검사 | 04 | 최종 제품 | 생산 완료 후 | 품질검사팀 |
| 확대검사 | F1 | 불량 발생 후 | 불량 발생 제품 전후 | 품질관리팀 |

### 4.2 검사 로트(Inspection Lot) 생성 (T코드: QA01)

PO 입고 시 자동 생성 또는 수동 생성.

```
검사 로트 번호: 30000123456
├─ 자재: STEEL-PLATE-001 (강판)
├─ 로트 크기: 500개
├─ 샘플 크기: 50개 (AQL 1%)
├─ 검사항목: 
│  ├─ 두께 (20mm ± 0.5mm)
│  ├─ 인장강도 (450 MPa 이상)
│  └─ 표면 결함 (육안)
└─ 합격 기준: AQL 1% → 50개 모두 합격
```

### 4.3 불량 관리 (QA08, QA24)

불량 발생 시 처리 방법 지정 (결정/Decision).

| 결정 코드 | 의미 | 원가 영향 | 프로세스 |
|---------|------|----------|---------|
| 01 | 합격(Accept) | 원가 반영 | 공정 계속 진행 |
| 02 | 부격(Reject) | 폐기비용 발생 | 폐기 처리, 원가 손실 |
| 03 | 재검사(Rework) | 수정 비용 발생 | 별도 공정 추가, 원가 추가 |
| 04 | 특별 승인(Use-as-is) | 원가 반영 | 특수 용도 또는 고객 승인 |

### 4.4 SPC (Statistical Process Control)

생산 공정의 안정성을 통계적으로 관리.

**SAP QM에서 SPC 관리** (T코드: QA35):
- **관리도(Control Chart)**: 표본 크기, 상한/하한 설정
- **공정 능력(Process Capability)**: Cp, Cpk 자동 계산
- **패턴 인식**: 공정 이상 신호 (예: 연속 5개 표본이 중심선 위쪽)
- **알림**: Cpk < 1.33 또는 패턴 이상 시 자동 경보

**한국 제조업 표준**:
- 반도체: Cpk ≥ 1.67
- 자동차 부품: Cpk ≥ 1.33 (IATF 16949)
- 일반 부품: Cpk ≥ 1.0

## 5. PM-PP 연계: 설비 보전

### 5.1 예방보전 vs 생산 스케줄 조율

생산 스케줄과 보전 일정의 충돌을 최소화하는 것이 핵심.

**SAP PM에서 설비 관리** (T코드: IA02, IA04, IA05):
```
설비 코드: MACH-MILL-001 (밀링머신)
├─ 설비 분류: 기계 (주요 설비)
├─ 설비 가격: 5억원
├─ 평균수명: 10년
├─ MTBF(평균고장간격): 720시간 (월 1회 고장 예상)
└─ 보전 계획:
   ├─ 월 1회 예방보전 (토요일 오후 4시간)
   ├─ 연 2회 정기보전 (1월, 7월 각 2일)
   └─ 수리 시 파트 교체 (부품 1백만원)
```

### 5.2 OEE (Overall Equipment Effectiveness) 추적

```
OEE = 가용성(Availability) × 성능(Performance) × 품질(Quality)

가용성 = (가동시간) / (계획운전시간) × 100%
        예: 8시간 계획 중 7시간 가동 = 87.5%

성능 = (계획생산량) / (실제생산량) × 100%
      예: 시간당 100개 계획, 85개 생산 = 85%

품질 = (합격품 수) / (생산품 수) × 100%
      예: 85개 중 82개 합격 = 96.5%

OEE = 0.875 × 0.85 × 0.965 = 0.72 (72%)
```

**한국 제조 목표**:
- 우수: OEE > 85%
- 평균: OEE 65~75%
- 미흡: OEE < 65%

## 6. CO (원가) 관점: 제품 원가 계산

### 6.1 원가 계산의 종류 (T코드: CK11N/CK24/CK40N)

| 계산 유형 | 목적 | 시점 | 담당자 |
|---------|------|------|--------|
| **표준원가** (CK11N) | 제품 표준가 설정 | 연 1회 (연초) | 원가팀 |
| **실제원가** (CK24) | 실제 소비 원가 추적 | 월 1회 (월말) | 원가팀 |
| **다단계** (CK40N) | 반제품 원가 포함 | 월 1회 | 원가팀 |

### 6.2 원가 차이 분석 (Variance Analysis)

생산 완료 후, 예상(표준) vs 실제의 차이를 분석한다.

```
제품 A 생산오더: PP-001001
  표준원가 합계: 100,000원
  ├─ 원자재비: 60,000원
  ├─ 직접노무비: 20,000원
  └─ 제조간접비: 20,000원

  실제원가 합계: 108,500원
  ├─ 원자재비: 61,500원 (공급가 변동)
  ├─ 직접노무비: 21,000원 (초과시간 발생)
  └─ 제조간접비: 26,000원 (높은 가동 비용)

  원가차이: 8,500원 (8.5% 초과)
  분석:
    ├─ Price Variance (가격차이): 1,500원 (원자재 공급가 상승)
    ├─ Quantity Variance (수량차이): 500원 (스크랩 증가)
    └─ Efficiency Variance (효율차이): 6,500원 (직접노무 초과)
```

### 6.3 원가 흐름 (Material Ledger)

SAP MM에서 입고 → PP에서 소비 → CO에서 원가 기록까지 전체 추적.

```
자재 STEEL-PLATE-001
├─ 1월 1일: 1,000개 입고 @ 100,000원 = 100M원
├─ 1월 5일: 500개 출고 @ 100,000원 = 50M원 (생산오더 PP-001001)
├─ 1월 10일: 200개 입고 @ 102,000원 = 20.4M원 (공급가 상승)
├─ 1월 15일: 300개 출고 @ 101,000원 = 30.3M원 (가중평균 산정)
│           (← 이 가격이 생산 원가로 반영됨)
└─ 1월 31일: 잔고 400개
```

## 7. 한국 제조업 특수 사항

### 7.1 대기업 제조사 운영 패턴

**삼성/LG/SK 그룹 공장** (매출규모 1조 이상):
- **MES 필수**: SAP과 MES 양계 시스템 운영
- **다중 플랜트**: 국내 3~5개 + 해외 10개 이상
- **글로벌 공급망**: 국제 유가/환율 변동에 즉시 반응
- **자동화율**: 80% 이상 (로봇 활용)
- **스마트팩토리**: 예측 유지보수, 자율 스케줄링

**SAP 운영 체계**:
```
글로벌 마스터 플랜(Singapore HQ)
        ↓
지역별 플랜(Korea, China, Vietnam)
        ↓
공장별 상세계획(울산, 온양, 기흥)
        ↓
라인별 일일 스케줄(100+ 라인)
```

### 7.2 중견 제조사 운영 패턴

**매출규모 1,000억~1조원**:
- **MES 선택**: 자동화된 라인만 MES, 수동라인은 CO11N
- **다중 플랜트**: 국내 1~2개
- **재고 관리**: 주 1회 재계획 (MD01)
- **원가 모니터링**: 월 1회 실제원가 계산
- **공급사 관리**: 1차, 2차 협력사 (평균 200~500개사)

### 7.3 중소 제조사 운영 패턴

**매출규모 100억 이하**:
- **MES 미사용**: 전량 수동 입력(CO11N)
- **단일 플랜트**: 대부분 1개 공장
- **재계획 빈도**: 월 2회 (월초, 월중)
- **원가 계산**: 월 1회 또는 분기 1회
- **공급사 관리**: 10~50개사 (밀접한 협력)
- **SAP 활용도 낮음**: FI/CO, SD 중심 (PP/MM은 보조)

### 7.4 산업안전보건법 (OSHA) 준수

한국은 중대재해처벌법(2022~) 시행으로 설비 관리 의무가 강화됨.

**SAP PM에서 기록 의무**:
1. **설비 점검 이력** (PM 오더): 월 1회 이상 기록
2. **불안전 상태 보고**: QM 불량처럼 기록 유지
3. **수리 이력**: 모든 보전 작업 기록 (부품, 시간, 비용)
4. **예방정비 계획**: 설비별 정기검사 일정 명확화

**SAP T코드**:
- `IA04`: 설비 이력 조회
- `PM01`: 보전오더 생성
- `PM02`: 보전오더 실행 및 기록

**감사 체계**:
- 분기별 설비 건전성 보고서 (Management Report)
- 근로감독관 요청 시 5년분 이력 제출 가능해야 함

### 7.5 K-IFRS 제조원가명세서

한국 회계기준(K-IFRS)에서 요구하는 재무제표 추가 정보.

**연간 재무제표에 포함되어야 할 내용**:
```
제조원가명세서 (OOO년 12월 31일)

원자재비               -----억원
+ 직접노무비           -----억원
+ 제조간접비           -----억원
─────────────────────
제조원가 합계          -----억원

완성품 기초재고        -----억원
+ 당기 제조원가        -----억원
- 완성품 기말재고      -----억원
─────────────────────
제조원가(매출원가)     -----억원
```

**SAP에서 자동 생성** (T코드: CK40 + J1IFRS):
- 월별 CO 모듈 데이터 → J1IFRS 통합 → 재무제표 자동 생성
- K-IFRS 16호(리스): 사용권자산, 리스부채 구분 기록

### 7.6 외주/하도급 관리 (하도급법)

한국은 「하도급거래 공정화에 관한 법률」로 하청사 보호 규정이 강화됨.

**SAP MM에서 추적**:
1. **하도급 계약 기록**: 구매오더에 "하도급" 플래그 설정
2. **대금 지급 이력**: PO → GR → 인보이스 → 지급 완료까지 기록
3. **부실 납품 처리**: 30일 이내 기한 명확화, 과다 검수 금지

**위반 시 과태료**:
- 기한 내 지급 미준수: 위약금 (미지급액의 1~3%)
- 부도덕한 검수: 과태료 (최대 3천만원)

**SAP 운영 팀의 확인 사항**:
- PO 생성 시 하도급사 구분 필수 (자재마스터 공급사 유형)
- 월별 미지급금 보고 (FI 채무 현황)
- 검수 이력과 GR 일자 자동 동기화 (QM 검사 로트와 MM 입고 매칭)
