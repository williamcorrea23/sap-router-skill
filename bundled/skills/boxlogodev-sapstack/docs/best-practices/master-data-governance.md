## 2. 마스터데이터 생성/변경 프로세스

### 4단계 프로세스
```
1. 요청 → 2. 승인 → 3. 생성/변경 → 4. 검증
```

### 역할 분리
| 역할 | 책임 | T-code |
|------|------|--------|
| **요청자** | 마스터 생성/변경 요청서 작성 | 워크플로/이메일 |
| **승인자** | 업무 적정성 검토 → 승인 | 워크플로 |
| **데이터 관리자** | SAP에서 실제 생성/변경 | XK01/MM01/FS00 등 |
| **검증자** | 생성 결과 확인 (4-Eye Principle) | XK03/MM03 등 |

### 변경 이력 추적
- **SCU3**: 테이블 변경 이력 조회 (변경 문서 활성화 필수)
- **S_ALR_87012089**: 마스터 변경 보고서 (FI)
- **ME2M**: 구매 마스터 변경 추적

## 3. 마스터데이터 품질 체크리스트

### 공급업체 마스터 필수 필드
- [ ] **회사코드 레벨** (LFB1): 통제계정(AKONT), 지급조건(ZTERM), 지급방법(ZWELS)
- [ ] **구매조직 레벨** (LFM1): 오더통화(WAERS), 인코텀즈(INCO1)
- [ ] **은행 정보** (LFBK): 은행키, 계좌번호 (지급 실행 필수)
- [ ] **세금 정보**: 사업자등록번호 (한국 필수)

### 고객 마스터 필수 필드
- [ ] **회사코드 레벨** (KNB1): 통제계정(AKONT), 지급조건(ZTERM)
- [ ] **판매영역 레벨** (KNVV): 가격그룹, 고객그룹, 배송조건
- [ ] **여신 정보** (KNKK/UKM): 여신한도, 리스크 카테고리

### 자재 마스터 필수 뷰
- [ ] **기본 데이터** (MARA): 자재유형, 단위, 자재그룹
- [ ] **구매 뷰** (MARC/EINE): 구매그룹, 리드타임
- [ ] **MRP 뷰** (MARC): MRP 유형, Lot sizing, 안전재고
- [ ] **회계 뷰** (MBEW): 가격결정, 가격제어(S/V), 평가클래스
- [ ] **영업 뷰** (MVKE): 판매조직, 유통채널, 세금 분류

## 4. 기간마감 전 마스터데이터 검증

마감 D-5일 전에 아래를 확인:

- [ ] **미완성 마스터** — 필수 뷰가 없는 자재 (MMBE → 뷰 누락 확인)
- [ ] **중복 마스터** — 동일 공급업체/고객 이중 등록 여부
- [ ] **비활성 마스터** — 12개월 이상 거래 없는 마스터 → 잠금 검토
- [ ] **은행 정보 유효성** — 지급 실행 전 은행 마스터 확인
- [ ] **가격 유효기간** — 구매 Info Record, 판매 Condition 유효기간 확인

## 5. MDG (Master Data Governance) — S/4HANA

S/4HANA에서는 MDG를 통한 중앙 관리 권장:
- **MDG-S**: 공급업체 중앙 관리
- **MDG-C**: 고객 중앙 관리
- **MDG-M**: 자재 중앙 관리
- **MDG-F**: 재무 마스터 중앙 관리

### MDG 워크플로
요청 → 검토 → 승인 → 활성화 → 배포 (멀티 시스템)

## 6. 한국 특수 사항

### 사업자등록번호
- 공급업체/고객 마스터에 **사업자등록번호 필수** (전자세금계산서 연동)
- 형식: 3-2-5 (예: 123-45-67890)
- 국세청 진위 확인 서비스 연동 권장

### 외주 업체 관리
- 하도급법 준수: 외주 단가 이력 보관 (3년)
- 대금 지급 기한: 60일 이내 (하도급법)
