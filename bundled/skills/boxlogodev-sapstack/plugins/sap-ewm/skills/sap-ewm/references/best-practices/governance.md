## 체크리스트

### RF(Radio Frequency) 장비 관리
- [ ] **RF 디바이스 및 무선 인프라 운영** — T-code: /SCWM/RES + 시스템 관리
  - 정책:
    ```
    RF는 EWM 작업 효율의 핵심 인프라
    → 안정적 관리, 보안, 교육 필수
    ```
  
  - 관리 항목:
    ```
    1. 디바이스 관리
       - RF 단말기 보유 수: 창고 인력당 0.8~1.0대
       - 배터리 관리: 일일 야간 충전 (충전시간 4~6시간)
       - 배터리 교체: 2년마다 또는 성능 저하 시
       - 고장 수리: 목표 MTTR 4시간
    
    2. 무선 인프라
       - WiFi Access Point(AP): 신호 강도 > -70dBm 유지
       - 신호 맵핑: 월 1회 모니터링 (RF Survey)
       - 신호 사각지역: 0개 목표 (또는 기술적으로 불가피한 경우만)
       - 대역폭: 최소 10Mbps (동시 100명 기준)
       - 보안: WPA2 암호화 이상
    
    3. 사용자 인증
       - 로그인 방식: ID/Password (물리 로그인)
       - 세션 관리: 자동 로그아웃(15분 미사용)
       - 감사 로그: 모든 RF 활동 기록
    ```
  
  - 비용 추정:
    ```
    RF 단말기: 1대당 ₩3M ~ ₩5M
    WiFi AP: 1대당 ₩800K ~ ₩1.5M
    연간 유지보수: 하드웨어 가격의 15~20%
    ```

### 접근 권한(Authorization) 관리
- [ ] **EWM 사용자 권한 체계** — T-code: /SCWM/* 등
  - 정책:
    ```
    최소 권한 원칙(Least Privilege) 적용
    → 업무에 필요한 최소 권한만 부여
    ```
  
  - 역할별 권한:
    ```
    1. EWM 작업자 (창고 현장)
       - WO 조회/실행: /SCWM/ORD (READ, UPDATE)
       - Bin 조회: /SCWM/INV (READ)
       - 데이터 입력: RF 스캔 (자동)
       - 제약: 삭제, 구성 변경 불가
    
    2. EWM 감독자 (팀 리더)
       - 모든 작업자 권한 + 모니터링
       - Wave 생성/릴리스: /SCWM/WAVE (CREATE, RELEASE)
       - 우선순위 변경: /SCWM/ORD (UPDATE)
       - 임시방편 조정: /SCWM/INV (ADJUST)
    
    3. EWM 관리자 (매니저)
       - 모든 감독자 권한 + 설정 변경
       - Master Data 유지: /SCWM/IMG (설정)
       - 보고서 생성: /SCWM/MON (커스텀)
       - 권한 관리: User 할당/해지
    
    4. System Administrator
       - 전체 권한 (제약 없음)
       - 시스템 설정, 백업, 재해복구
    ```
  
  - 검증:
    ```
    분기별 권한 감사:
    - 활성 사용자 목록 추출
    - 퇴사자 권한 제거 확인
    - 이중 권한(SoD) 검사: 승인권 + 실행권 분리
    ```

### 프로세스 유형 변경관리
- [ ] **EWM 설정 변경 승인 프로세스** — Change Control Board
  - 정책:
    ```
    EWM 설정은 운영에 즉시 영향
    → 모든 변경사항 문서화 및 승인 필수
    ```
  
  - 변경 유형별 절차:
    ```
    1. Bin(저장유형) 추가/삭제
       영향: Wave 경로, 픽 시간
       승인: EWM 담당자 + 경영진
       테스트: Sandbox 선행 테스트 필수 (1주)
       구현: 야간/주말(Low-traffic 시간)
       검증: 다음날 정상 작동 확인
    
    2. Wave 전략 변경 (Sort Key, Release Rule)
       영향: 배송 효율, 비용
       승인: EWM 담당자 + CFO
       테스트: 10건 샘플로 테스트
       구현: 즉시 또는 예약 (정책에 따라)
    
    3. Pick 경로 재최적화
       영향: 사이클 타임, 인력 효율
       승인: EWM 담당자
       테스트: 실제 데이터로 시뮬레이션
       효과: 도입 후 30일간 모니터링
    
    4. RF 프로세스/메뉴 변경
       영향: 사용자 경험, 교육 필요
       승인: EWM 담당자 + HR(교육)
       교육: 구현 전 전수 교육 (1일)
       전환: 병렬 운영 불가능 → 즉시 전환
    ```
  
  - 문서화:
    ```
    모든 변경사항 기록:
    - Change Request Form 작성
    - 영향도 평가 (사용자, 시스템, 비용)
    - 테스트 계획 및 결과
    - 승인 사인(전자 서명)
    - 구현 일시 및 결과
    - 검증 완료 기록
    ```

### 성과 관리 및 SLA
- [ ] **EWM 성과지표(KPI) 정의 및 모니터링** — T-code: /SCWM/MON
  - 주요 KPI:
    ```
    1. 처리량(Throughput)
       - 일일 주문 처리: ___건/일 (목표: ___)
       - 시간당 처리: ___건/시간
       - 인력당 생산성: ___건/인원일
       추적: 일간 / 분석: 월간
    
    2. 기한 준수(On-Time Performance)
       - Wave 마감 기한 준수: ___% (목표: 95%)
       - 주문 처리 기한: ___% (목표: 99%)
       - 배송 기한: ___% (목표: 98%)
       추적: 일간 / 분석: 월간
    
    3. 품질(Quality)
       - Pick Accuracy: __% (목표: 99.5%)
       - 반품율: __% (목표: 0.5%)
       - 손상율: __% (목표: 0.1%)
       추적: 주간 / 분석: 월간
    
    4. 효율성(Efficiency)
       - 평균 Wave 사이클 타임: ___시간 (목표: <6)
       - 평균 픽 시간/라인: ___분 (목표: <15)
       - Resource 활용률: __% (목표: 80~90%)
       추적: 일간 / 분석: 월간
    
    5. 안전/준수
       - 안전 사고: ___건 (목표: 0)
       - 감시 지적사항: ___건 (목표: 0)
       - 데이터 보안 위반: ___건 (목표: 0)
       추적: 월간 / 분석: 분기
    ```
  
  - SLA 설정:
    ```
    고객 약속(Service Level Agreement):
    - 주문 수령 → 출고 준비: 24시간 이내
    - 출고 준비 → 배송: 2시간 이내
    - 반품 처리: 48시간 이내
    
    미달 시 패널티:
    - SLA 미달 1회: 경고
    - SLA 미달 3회/분기: 개선 계획 수립
    - 지속 미달: 운영 구조 개편 또는 자동화 투자
    ```

### 자동화 투자 및 ROI 관리
- [ ] **자동화 기술 도입 및 성과 추적** — 투자 심의
  - 자동화 기술:
    ```
    1. 자동 컨베이어 시스템
       - 투자: ₩200M ~ ₩500M (규모)
       - 예상 효율: 픽 시간 40% 단축
       - ROI 기간: 3~5년
       - 유지비: 연간 투자의 10~15%
    
    2. AS/RS(Automated Storage & Retrieval System)
       - 투자: ₩500M ~ ₩1.5B
       - 예상 효율: 공간 50% 절감, 시간 60% 단축
       - ROI 기간: 4~7년
       - 유지비: 연간 투자의 15~20%
    
    3. 로봇(AGV/AMR)
       - 투자: ₩100M ~ ₩300M (대당)
       - 예상 효율: 운송 시간 70% 단축
       - ROI 기간: 3~5년
       - 유지비: 연간 투자의 10~12%
    
    4. AI 최적화
       - 투자: ₩50M ~ ₩200M (소프트웨어)
       - 예상 효율: 픽 경로 15~25% 개선, Wave 효율 10%
       - ROI 기간: 1~2년 (빠름)
    ```
  
  - ROI 추적:
    ```
    투자 전: 비즈니스 케이스 수립
    - 현 운영비: 년 ₩___ (인력, 유지비 포함)
    - 투자 규모: ₩___
    - 기대 절감: 년 ₩___ (효율성 X, 비용 Y)
    - 기대 증대: 년 ₩___ (매출 증가 또는 고객만족)
    - Break-even: ___년
    
    투자 후: 분기별 성과 검증
    - 실제 절감액 vs 계획: __% 달성
    - 추가 이득: ___
    - 부정적 영향: ___
    - 조정 계획: ___
    ```

### 데이터 보안 및 개인정보 보호
- [ ] **EWM 데이터 보안 정책** — IT 보안팀 연계
  - 정책:
    ```
    EWM 데이터는 고객, 재고, 운영 정보 포함
    → 보안 및 개인정보보호법 준수 필수
    ```
  
  - 관리 항목:
    ```
    1. 접근 제어
       - 인증: 이중 인증(2FA) 고려 (VPN 접근 시)
       - 암호화: 데이터 전송 시 TLS/SSL
       - 로그: 모든 접근 로그 기록 (1년 보관)
    
    2. 개인정보 보호
       - 고객 개인정보: 전담 관리 (GDPR, 개인정보보호법)
       - 최소 수집: 필요한 정보만 수집
       - 삭제: 사용 목적 종료 시 안전 삭제
    
    3. 감시
       - 월 1회: 불법 접근 여부 모니터링
       - 분기 1회: 데이터 보호 감사
       - 연 1회: 보안 침투 테스트(Penetration Test)
    ```

### 재해복구 및 비즈니스 연속성
- [ ] **EWM 백업 및 복구 계획** — T-code: ADMREP 또는 시스템 관리
  - 정책:
    ```
    EWM 다운타임 = 직접 손실(처리 불가) + 고객 만족도 저하
    → 최대 30분 다운타임 목표
    ```
  
  - 백업 전략:
    ```
    1. 일일 백업
       - 시간: 매일 22:00 (업무 시간 외)
       - 범위: 마스터 데이터(Bin, WO, Wave)
       - 보관: 온사이트 + 오프사이트(클라우드)
       - 검증: 월 1회 복구 테스트
    
    2. 재해복구 계획(DRP)
       - RTO(Recovery Time Objective): 2시간
       - RPO(Recovery Point Objective): 1시간
       - 대체 시스템: 클라우드 또는 백업 데이터센터
       - 테스트: 연 2회 DRP 훈련
    
    3. 장애 대응
       - 장애 발생 → 즉시 IT 부서 연락
       - 1시간 이내: 임시 수동 처리 전환
       - 2시간 이내: 시스템 복구 또는 대체 시스템 활성화
    ```

### 기술 로드맵 및 혁신
- [ ] **EWM 기술 혁신 계획** — 연간 1회 수립
  - 계획 항목:
    ```
    1년차: 현 상태 최적화
    - WM → EWM 안정화
    - Pick Accuracy 99.5% 달성
    - 자동화 투자 효과 검증
    
    2년차: 고급 기능 도입
    - AI 기반 최적화 (시범)
    - 예측적 분석(Predictive Analytics) 도입
    - 고객 통합 (API)
    
    3년차: 자동화 확대
    - 로봇(AMR) 도입 검토
    - 클라우드 EWM 마이그레이션 검토
    - 공급망 연계 강화
    ```
  
  - 투자 우선순위:
    ```
    1순위: Pick Accuracy & Quality
           (고객 만족도 직결)
    
    2순위: 자동화 및 효율성
           (비용 절감)
    
    3순위: 고객/공급업체 통합
           (경쟁 우위)
    ```
