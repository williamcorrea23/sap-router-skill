# EWM 기간마감 Best Practice

## 적용 범위
- S/4HANA: ✓

## 개요
월 마감 프로세스에서 EWM 담당자가 수행해야 할 정산, 검증 작업 목록.

---

## 기간 마감 프로세스

### Phase 1: Pre-flight 검사 (T-5일)

#### 1. 미처리 Warehouse Order 정리
- [ ] **미결 WO 현황 검토** — T-code: /SCWM/ORD
  - 필터: Status ≠ 'Completed' AND Created ≤ 당월
  - 조치:
    ```
    - 진행 중(Started): 월말 전 완료 추진
    - 미시작(Created): 우선순위 재검토 또는 취소 결정
    - 지연(Overdue): 원인 파악 → 해결 방안 결정
    ```

#### 2. Physical Inventory(실사) 계획 확인
- [ ] **/SCWM/PI Physical Inventory 준비** — T-code: /SCWM/PI
  - 정책:
    ```
    EWM은 Annual Count(연 1회 전수) 또는 Cycle Count(월 순환) 지원
    
    선택:
    - Annual: 대규모 창고 (1회에 몇 일 소요 가능)
    - Cycle: 중소 창고 (월별 순환 권장)
    ```
  - 준비:
    ```
    PI 준비:
    1. 범위: 대상 저장유형 또는 전체 선택
    2. 인력 배정: 실사 담당자 확보
    3. 교육: 실사 프로세스, RF 조작법
    4. 일정: 예상 소요 시간 계획
    ```

#### 3. EWM-ERP 재고 대사 사전 확인
- [ ] **EWM vs MM 재고 비교** — T-code: /SCWM/INV + MMBE
  - 확인:
    ```
    EWM 재고(상세: 빈별 LPN)
    vs
    MM 재고(총수량)
    
    차이 발견:
    - 미확인 WO (자동 조정 대기)
    - 시스템 오류 (수정 필요)
    - 자연감모 (폐기 처리 필요)
    ```

### Phase 2: 월 마감 실행 (T-day)

#### 4. Physical Inventory 실행
- [ ] **/SCWM/PI 물리적 실사** — T-code: /SCWM/PI
  - 프로세스:
    ```
    1. PI 문서 생성 (/SCWM/PI)
       - 범위: 대상 저장유형 선택
       - 실사 방법: Annual 또는 Cycle
       - 실사 담당자 배정
    
    2. 현장 실사 (RF 또는 수동)
       - LPN(License Plate): 바코드 스캔 또는 입력
       - 수량 확인: 물리적 수량 입력
       - 위치 검증: 현재 위치 확인
    
    3. 시스템 비교
       - 시스템 재고(LPN) vs 물리 수량 비교
       - 차이(Variance) 자동 계산
    
    4. 차이 분석
       - 차이 = 물리 - 시스템
       - 양수: 시스템 미기재(과소계산)
       - 음수: 분실 또는 오류
    ```

#### 5. EWM-ERP 재고 동기화
- [ ] **Goods Issue(GI) 최종 정산** — T-code: /SCWM/MON
  - 조치:
    ```
    PI 차이 반영:
    1. EWM에서 조정 이동 생성
    2. 차이가 ERP(MM)로 자동 동기화
    3. 원가 영향: 차이 금액 계상
    
    예시:
    부품 A: 시스템 100개 vs 물리 98개 (-2개)
    → 원가 손실 ₩50,000 (@ ₩25,000/개)
    → FI 계정에 자동 포스팅
    ```

#### 6. Wave 및 WO 월별 정산
- [ ] **월별 처리량 및 효율성 정산** — T-code: /SCWM/MON
  - 통계:
    ```
    월간 Wave 통계:
    - 생성된 Wave: ___ 건
    - 완료된 Wave: ___ 건 (완료율: __%)
    - 평균 Wave 사이클 타임: __시간
    - 평균 주문당 사이클 타임: __분
    
    월간 WO 통계:
    - 생성된 WO: ___ 건
    - 완료된 WO: ___ 건
    - Pick Accuracy: __% (목표: 99.5%)
    - 품질 오류(재작업): __ 건 (원인분석)
    
    월간 자원 효율성:
    - 평균 인력 활용률: __% (목표: 80~90%)
    - 자동화 설비 가동률: __% (목표: 90~98%)
    - 시간당 처리량: ___주문/시간
    ```

#### 7. Rearrangement 영향도 분석
- [ ] **월별 Rearrangement 효과 측정** — T-code: /SCWM/REAR
  - 분석:
    ```
    지난달 Rearrangement 후:
    - Pick 시간 단축: __% (목표: 15~25%)
    - Pick Accuracy 개선: __% (오류 감소)
    - 자원 활용률: __% (개선도)
    
    차기월 Rearrangement 계획:
    - 대상 저장유형: ___
    - 예상 효과: 시간 ___% 단축
    ```

#### 8. 설비/자원 성능 평가
- [ ] **월별 Resource 성능** — T-code: /SCWM/RES
  - 평가:
    ```
    RF 터미널:
    - 가동률: __% (목표: 95%+)
    - 고장 건수: ___ (MTBF 추적)
    - 배터리 성능: 정상/교체 필요
    
    자동 컨베이어:
    - 가동률: __% (목표: 98%+)
    - 처리량: 시간당 ___상자 (설계값 대비)
    - 예방보전 실행: 예정된 유지보수 완료 여부
    
    로봇(도입 시):
    - 가동률: __% (목표: 90%+)
    - 처리 오류 발생: ___건 (원인분석)
    ```

### Phase 3: 검증 (T+3일)

#### 9. PI 차이 대사 및 원가 반영
- [ ] **물리 실사 차이와 FI 대사** — T-code: /SCWM/INV + FB01
  - 대사:
    ```
    EWM 차이 금액:
    - 부품별 재고 차이 × 단가 = 차이 금액
    
    FI 기록:
    - 재고 계정 변동 확인 (자동 포스팅)
    - 손실/증가 계정 기록
    
    일치 여부: 100% 일치 목표
    ```

#### 10. 운영 성과 리뷰
- [ ] **월별 EWM 운영 성과** — 경영진 보고
  - 핵심지표:
    ```
    SLA(Service Level Agreement) 달성:
    - 주문 처리 기한: __% (목표: 95%+)
    - Pick Accuracy: __% (목표: 99.5%)
    - 배송 지연: __ 건 (근본 원인 분석)
    
    비용 효율:
    - 월 운영 비용: ₩___ (전월 대비: +/- __)
    - 인력당 처리량: ___주문/인원일
    - 자동화 ROI: 추진 중 (또는 달성도)
    
    개선 사항:
    - 문제점: ___
    - 해결책: ___
    - 기대 효과: ___
    ```

### Phase 4: 마감 확인 (T+5일)

#### 11. EWM 마감 Lock
- [ ] **/SCWM/ARC 아카이빙 (선택사항)**
  - 조치:
    ```
    월별 마감 보고서:
    - PI 결과 저장
    - Wave/WO 통계 보관
    - 이상 사항 기록
    
    선금(Post-closing) 조정:
    - 한계값: ±₩500K 이내만 허용
    - 예: 새로운 재고 오류 발견 → 조정 이동 생성
    - 기한: 차기월 중순까지
    ```

#### 12. 차기월 계획 수립
- [ ] **다음달 EWM 운영 계획**
  - 계획:
    ```
    - 예상 물량: ___주문 (전월 대비 +/- %)
    - 필요 자원: ___명 (일일 평균)
    - 특수 작업: Rearrangement 계획
    - 설비 정비: 예방보전 일정
    - 교육/개선: 사항 있으면 기재
    ```

## 특수 상황 처리

### 대량 Pick 오류 발생
```
상황: 특정 주문에서 10% Pick 오류(잘못된 부품 출고)
원인: RF 스캔 오류 또는 작업자 부주의
대응:
1. 영향 고객 파악 → 즉시 통보
2. 반품 및 교환 처리
3. 근본 원인 분석:
   - RF 바코드 손상? → 라벨 재인쇄
   - 작업자 교육 부족? → 재교육
4. 임시 조치: 더블 체크 프로세스 추가
5. 월말 비용 기록: 반품 처리 비용 계상

예방: Pick 정확도 동향 모니터링 (99.5% 유지)
```

### 설비 고장으로 인한 지연
```
상황: 자동 컨베이어 고장 → 일일 처리량 50% 감소
영향: Wave 일정 1일 지연, 고객 배송 지연
대응:
1. 긴급 부분 처리: 수동 픽으로 임시 대응
2. 설비 수리: 당일 복구 또는 대체 장비 준비
3. 고객 연락: 배송 지연 사유 설명
4. 비용: 비상 인력 초과근무 비용 발생 (월 보고)

예방: 예방보전 강화, 백업 장비 보유
```

## 운영 KPI 대시보드 예시

```
EWM 운영 현황 (2026년 4월)
=========================================

Wave 성과:
- 월간 Wave: 125건 생성 / 120건 완료 (96%)
- 평균 사이클 타임: 5.2시간 (목표: <6시간) ✓
- 지연 Wave: 5건 (분석: 인력 부족 2건, 설비 고장 3건)

Pick 성과:
- 월간 주문: 3,500건
- Pick Accuracy: 99.6% (목표: 99.5%) ✓
- 오류 건수: 14건 (분석: RF 오류 8건, 작업자 실수 6건)
- 재작업 비용: ₩1,050,000

자원 효율:
- 평균 일일 인력: 15명 (목표: 14명)
- Resource Utilization: 87% (목표: 80~90%) ✓
- RF 터미널 가동률: 97% (목표: 95%) ✓
- 자동 컨베이어: 94% (목표: 90%) ✓

Rearrangement 효과:
- 실행: 3월 실시
- 효과: Pick 시간 22% 단축 ✓
- 차기: 5월 저장유형 03 (완성품) 대상

Physical Inventory:
- 실사 범위: 저장유형 01, 02 (순환)
- 소요 시간: 6시간
- 재고 차이: 3개 (복구율: 99.95%)
- 차이 금액: ₩72,500 (손실: 분실 의심)
- 조정 완료: ✓

성과 평가:
- SLA 달성율: 96% (목표: 95%) ✓
- 운영 비용: ₩12,500,000 (전월 대비: -3%)
- 처리 건수: 3,500주문 (전월 대비: +8%)
- 인력당 생산성: 233주문/인원일 (전월: 218)

주요 이슈:
1. Pick 오류 증가 추세 (0.2%) → RF 재교육 추진 중
2. 자동 컨베이어 간헐적 고장 → PM과 협의 중
3. 인력 부족 발생 → 4월 신규 채용 2명 예정

개선 계획:
- RF 사용자 교육 강화 (5월)
- Pick Accuracy 목표 99.7% 상향 (6월)
- Rearrangement 2차 실시 (5월)
```
