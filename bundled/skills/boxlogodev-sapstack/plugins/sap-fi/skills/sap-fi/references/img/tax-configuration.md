# 세금(Tax) 구성 IMG 가이드

## SPRO 경로
```
SPRO → Financial Accounting → Financial Accounting Global Settings
    → Tax on Sales/Purchases Accounting → Configuration
```

## 필수 선행 구성

- [ ] OX02: 회사코드 정의 (국가 코드 필수)
- [ ] OB13: GL 계정 차트 정의
- [ ] OBC4: 필드상태그룹 정의
- [ ] 국가별 세금 기본설정 (세법 확인)

## 세금 구성의 핵심 T-codes

### 1. FTXP — 세금코드 정의(Tax Code Definition)

**SPRO 경로**: `SPRO → Financial Accounting → Financial Accounting Global Settings → Tax on Sales/Purchases → Tax Codes → Define Tax Codes`

**목적**: 
- 매입/매출 세금코드 정의
- 세금율 설정
- 세금 계산 메커니즘 정의

**한국 세금코드 예시**:

| 세금코드 | 설명 | 세율 | 세금 유형 | 사용 용도 |
|---------|------|------|---------|---------|
| 1K | 매출 과세(10%) | 10% | Output VAT | 일반 판매 |
| 2K | 매입 과세(10%) | 10% | Input VAT | 일반 구매 |
| 3K | 매출 면세 | 0% | Output (Zero) | 수출/면세 |
| 4K | 매입 면세 | 0% | Input (Zero) | 면세 구매 |
| 5K | 매출 영세율 | 0% | Output (Zero) | 영세율 판매 |
| 6K | 수입세금 | 10% | Input VAT | 수입 물품 |
| 7K | 보정세금 | 0% | Adjustment | 세금 조정 |

**구성 단계** (T-code: FTXP):

1. **기본 세금코드 생성**
   ```
   세금코드: 1K
   설명: "매출 과세 (10%)"
   국가: "KR" (Korea)
   세금 유형: "MWST" (VAT - Mehrwertsteuer)
   계산 메커니즘: "A" (Amount - 금액 기반)
   ```

2. **세율 정의**
   ```
   세금코드: 1K
   과세기준: 100 (기본값)
   세율: 10 (10%)
   계산 결과: 110 (세금 포함 총액)
   
   공식: Tax = Net Amount × (Tax Rate / 100)
   예: 100,000 원 × 10% = 10,000 원 세금
   ```

3. **여러 세율 지원** (변형세율)
   ```
   기본세율: 10%
   감면세율: 5% (특정 거래처/자재)
   영세율: 0% (수출)
   면세: 0% (의료, 금융)
   ```

**필드 설명**:

| 필드 | 설명 | 값 | 주의사항 |
|------|------|-----|---------|
| Tax Code | 세금코드 | 1K, 2K 등 | 국가별 2자리 구분 |
| Tax Type | 세금 유형 | MWST (VAT) | 원천세는 다른 코드 |
| Calculation Type | 계산 방식 | A (Amount) | B (%), C (Net/Gross) |
| Tax Rate | 세율 | 10 | FTXP 테이블에서 직접 입력 |
| Valid From | 유효 시작일 | 01.01.2024 | 법인세율 변경 시 신규 생성 |

### 2. OBCL — 세금 절차 지정(Tax Procedure Assignment)

**SPRO 경로**: `SPRO → Financial Accounting → Financial Accounting Global Settings → Tax on Sales/Purchases → Assign Tax Procedure to Company Code`

**목적**: 회사코드별로 적용할 세금 절차 결정

**구성 단계**:

1. **세금 절차 선택**
   ```
   회사코드: 1000
   국가: "KR" (한국)
   세금 절차: "TSK" 또는 "KR" (한국식 부가가치세)
   ```

2. **절차별 특성**
   ```
   절차 코드 | 설명 | 세금 신고 기간 | 특징
   TSK      | Standard Korea | 분기별 | 표준 부가세
   KR_EXP   | Export VAT | 월별 | 수출 환급
   KR_DRAW  | Drawback | 분기별 | 원재료비 환급
   KR_SPECIAL| 특시 | 필요시 | 특별 신고
   ```

3. **회사코드에 할당**
   ```
   회사코드: 1000 → TSK (표준)
   회사코드: 2000 → KR_EXP (수출)
   회사코드: 3000 → KR_DRAW (특별)
   ```

**구성 검증** (T-code: OBCL):
```
회사코드: 1000
세금 절차: "TSK"
확인: "O" (유효)
```

---

### 3. OB40 — 세금 계정결정(Tax Account Determination)

**SPRO 경로**: `SPRO → Financial Accounting → Financial Accounting Global Settings → Tax on Sales/Purchases → Account Assignment → Assign Tax Accounts`

**목적**: 세금 거래의 GL 계정 자동 결정

**구성 단계**:

1. **세금 계정결정 규칙 정의**
   ```
   T-code: OB40 (또는 OB41 — Posting Keys 동시 확인)
   ```

2. **세금코드별 계정 할당**
   ```
   세금코드 | 세금 유형 | GL 계정 | 설명
   1K       | Output   | 210000  | 매출세금채무
   2K       | Input    | 110000  | 매입세금채권
   3K       | Zero     | 210100  | 면세 매출 (기록용)
   ```

3. **세금 계정 구조 (한국)**
   ```
   자산:
   - 110000: 부가가치세 대기납부금 (매입세 귀속)
   
   부채:
   - 210000: 부가가치세 납부액 (매출세 귀속)
   - 210100: 세금 환급액 (수출환급)
   
   비용:
   - 510000: 미환급 세금 (비용 처리)
   ```

**구성 예시** (한국식):

```
[매출 거래 — 세금코드 1K (10%)]
발행 금액: 100,000 원
세금: 10,000 원

전기:
Debit  110000 (유통채권) — 소비자로부터 받을 세금
Credit 400000 (매출) — 100,000 원
Credit 210000 (세금채무) — 10,000 원

[매입 거래 — 세금코드 2K (10%)]
구매 금액: 100,000 원
세금: 10,000 원

전기:
Debit  530000 (자재비) — 100,000 원
Debit  110000 (세금채권) — 10,000 원
Credit 200000 (채무) — 110,000 원
```

## 원천세(Extended Withholding Tax, EWT) 설정

### T-code: J1IJ — 원천세 코드 정의

**용도**: 정산금, 용역비, 로열티 등에 대한 자동 원천세 공제

**구성 단계**:

1. **원천세 유형 정의** (T-code: J1IJ)
   ```
   EWT Code | Description       | Rate  | Account
   001      | 정산금 원천세      | 3.3%  | 220000
   002      | 용역비 원천세      | 3.3%  | 220000
   003      | 배당금 원천세      | 14%   | 220100
   004      | 이자 원천세        | 14%   | 220100
   ```

2. **거래처별 원천세 신고번호 할당** (T-code: XK02)
   ```
   거래처마스터 → 세금 데이터 섹션
   신고번호: 123-45-67890 (사업자등록번호)
   주민번호: 123456-1234567 (개인)
   원천세 의무자 플래그: X (체크)
   ```

3. **송장/청구에 자동 적용** (T-code: F-43 또는 MIRO)
   ```
   거래처: ACME Inc. (원천세 의무자)
   금액: 1,000,000 원
   자동 계산: 원천세 = 1,000,000 × 3.3% = 33,000 원
   
   실제 지급: 1,000,000 - 33,000 = 967,000 원
   ```

## 세금 신고(Tax Reporting)

### T-code: J1IB — 세금 신고 리포트

**용도**: 분기별 부가가치세 신고

**신고 절차**:

1. **기간 선택**
   ```
   신고 기간: 2024년 1분기 (1월~3월)
   회사코드: 1000
   ```

2. **신고 데이터 수집**
   ```
   매출세금 (공급가액):
   - 세금코드 1K: 100,000,000 원 (세금 10,000,000 원)
   
   매입세금 (매입가액):
   - 세금코드 2K: 50,000,000 원 (세금 5,000,000 원)
   
   납부 세금 = 10,000,000 - 5,000,000 = 5,000,000 원
   ```

3. **신고 파일 생성**
   ```
   T-code: FTVD (또는 J1I5)
   → Excel 형식 또는 정부 제출 포맷
   ```

## ECC vs S/4HANA 차이점

| 항목 | ECC | S/4 |
|------|-----|-----|
| **세금코드** | FTXP (동일) | FTXP (동일) |
| **세금 계정결정** | OB40 (별도) | OB40 (GL 통합) |
| **세금 분류** | MWST / EUST | 동일 |
| **신고 프로세스** | J1IB 수동 | DBTAX (자동) |
| **원천세 추적** | 제한적 | ACDOCA에 자동 기록 |

## 주의사항

1. **세금율 변경 시**
   - FTXP에서 신규 세금코드 생성 (기존 수정 금지)
   - Valid From 날짜 명확히 설정
   - 기간 관리: OB52와 연계 (월별 적용 시점 명시)

2. **면세/영세율 구분**
   - 면세 (Tax Code 0): 매입세 환급 가능
   - 영세율 (Tax Code 0): 매입세 환급 불가
   - 혼동 시 세금 신고 오류 발생

3. **국가별 세금 코드**
   - 한국: 부가가치세(10%), 특례 (0%, 5% 등)
   - 해외: 각국 법인세/부가세 준용
   - 글로벌 기업: OBCL에서 회사코드별 절차 구분

4. **세금 신고 기한**
   - 부가세: 분기별 (1월, 4월, 7월, 10월 25일)
   - 소득세: 연 1회 (5월 31일)
   - 원천세: 월별 (익월 10일)

5. **세금 오류 정정**
   - 전년도 오류: 수정신고 또는 경정청구
   - 당년도 오류: 역분개 후 재전기 (기간 013 또는 014 사용)
