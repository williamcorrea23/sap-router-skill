## 체크리스트

### 🔐 K-SOX 준수 사항

#### 일반원장(GL) 거래 이중 승인
- [ ] **T-code SPRO → FI → General Ledger → Posting → Transaction Variants**
  - 목표: 특정 계정(예: 4000~4999 비용) 이상 금액 거래 2인 승인
  - 설정 위치: IMG → Financial Accounting → General Ledger → Posting → Define Posting Validation Rules
  - 담당: SAP BASIS / 재무시스템관리
  - 확인사항:
    - 승인 권한 사용자 등록 (SM01 또는 SU01Z)
    - 금액 임계값(예: 5천만원) 정의
    - 예외 계정 화이트리스트 관리

#### 수동 전표 감사 추적
- [ ] **T-code FB09 — 전표 변경 이력 추적**
  - 월간 표본: 전월 수동 전표(FB50, FB60) 중 20% 이상 감사
  - 변경 항목: 금액, 적요, 상대방 계정 등
  - 담당: 내부감사/재무감시
  - 점검항목:
    - 승인일과 생성일 차이 > 48시간 조사
    - 최종 승인자와 생성자 동일 여부 확인
    - 변경 이력 있는 전표 중 승인 기록 유무

#### 특별 기간 접근 제한 (Period Lock)
- [ ] **T-code OB52 — 회계 기간 설정**
  - 폐기된 기간(Closed Period): FB50, FB60 등 신규 전표 입력 불가
  - 수정 가능 기간(Modification Period): 설정 범위 내 변경 허용 (감시 대상)
  - 담당: 마스터데이터 관리자
  - 정책:
    - 재무보고 공시 후 수정 기간 14일 이내로 제한
    - 기간 변경 시 감사 부서에 공지
    - 긴급 조정 필요 시 CTO(Chief Financial Officer) 승인 필수

### 📊 전표 변경 관리

#### 전표 변경 권한 분리
- [ ] **T-code SU01Z, SU24 — 사용자 역할 및 권한 정의**
  - 역할 분리:
    - **입력**: FB50(자산), FB60(채권채무) 생성 권한만
    - **검토**: FB09 조회 권한만 (수정 불가)
    - **승인**: FBL1N/FBL5N 승인(F-04, F-32) 권한
    - **감시**: Authority 변경(SU24) 권한 없음 (BASIS 담당)
  - 담당: 인사/보안/SAP시스템관리
  - 검증빈도: 분기(3개월)

#### 삭제된 전표 추적
- [ ] **T-code BAPI_ACC_DOCUMENT_DELETE 또는 T-code SE01 (Change Log)**
  - 삭제된 전표 조회: Table BSAK/BSIK (Status = D)
  - 담당: 재무회계
  - 정책:
    - 월간 삭제 전표 현황 리포트 생성
    - 삭제 사유 문서화(예: 오입력, 중복)
    - 삭제 건수 월 3건 초과 시 상위보고

### 🔄 연간 정기 작업

#### 사용자 역할 재인증 (Annual Review)
- [ ] **T-code SU01 및 SU24 — 권한 정기 재검증**
  - 시기: 회계연도 말(12월) 또는 외감 예정 2개월 전
  - 절차:
    1. 현재 FI 권한 보유 사용자 목록 추출 (SE16 → USR03 테이블)
    2. 실제 업무 필요 권한과 대비
    3. 불필요 권한 회수, 신규 필요 권한 신청
    4. 각 사용자/팀장 전자서명 확인
  - 담당: 인사/보안팀
  - 문서: 권한 재인증 결과 보고서

#### 계정과목 정기 정비
- [ ] **T-code SM30 → SKA1, SKB1 — 계정과목 마스터 정비**
  - 시기: 연 1회 (회계연도 시작 전)
  - 확인사항:
    - 미사용 계정 비활성화 처리
    - 신규 사업 필요 계정 신청/생성
    - 계정 단계 체계(예: 4000~4999는 판매비, 5000~5999는 일반관리비) 정합성
  - 담당: 재정담당자
  - 검증: 각 계정별 YTD(Year-To-Date) 거래액 확인

#### 비용센터 정기 재검토
- [ ] **T-code SM30 → CSKS — 비용센터 마스터 재검토**
  - 시기: 연 1회 (조직 개편 후 또는 연초)
  - 항목:
    - 구조 변경(신규/폐기 센터)
    - 담당자 변경(예: 이직/보직변경)
    - 센터별 예산 기초 설정
  - 담당: 인사/재정담당자
  - 출력: 비용센터 정기보고서 (경영진 승인)

#### 감사보고 및 법인세 신고
- [ ] **T-code S_ALR_87012284 — 재무제표 감사용 보고서**
  - 시기: 분기말 및 년도말 (외감 일정 15~30일 전)
  - 산출물:
    - 재무상태표 (조정 전/후)
    - 손익계산서 (세부 라인)
    - 현금흐름표 (선택사항)
    - 계정별 상세 내역
  - 담당: 재정담당자 → 외부 감사인
  - 제출처: 공시시스템(예: DART, SEDARLING)

### 🛡️ 특별 기간 관리 (Fiscal Year-End)

#### 폐기 기간 설정 및 통제
- [ ] **T-code OB52 — 폐기 기간(Closed Period) 관리**
  - 연도 전환 시(예: 2025년 12월 말):
    1. 2024년 기간 → "Completely Closed" (모든 거래 불가)
    2. 2025년 1~11월 → "Modification Period" (14일, 감시)
    3. 2025년 12월 → "Open Period" (신규 거래 가능)
  - 담당: 마스터데이터 관리자
  - 정책: 연도 변경 후 역년도 기간 수정 불가 원칙

#### 년도별 기간 독립 관리
- [ ] **T-code OB52 설정의 "Year-Dependent" 체크**
  - S/4에서는 기간 설정이 회계연도별로 독립됨
  - 연도별 폐기 정책 상이한 경우 명확히 기재
  - 담당: SAP 시스템관리자

#### 특별 조정(Special Period) 필요 시
- [ ] **T-code OKP1 또는 OB52 — 특별 기간 활성화**
  - 연도말 후 발생 오류(예: 미청구 입고) 조정용
  - 선택적 활성화: 기간 "00" (또는 별도 특별기간)
  - 승인: CFO급 이상 + 외감인 사전 협의 필수
  - 담당: 재정담당자 + 감시팀

## 위험 시나리오 & 대응

| 시나리오 | 원인 | 예방 액션 | 심각도 |
|---------|------|---------|--------|
| 폐기기간에 신규 전표 입력 | 사용자 오인식/권한 오류 | OB52 설정 재확인 + 교육 | 🔴 Critical |
| 미승인 고액 전표 포함 | 승인 권한 설정 미흡 | 년 2회 권한 점검(SU01Z) | 🔴 Critical |
| 무단 전표 삭제 | 삭제 권한 소유자 부당 사용 | 월간 BSAK 삭제 현황 보고 | 🔴 Critical |
| 외감 중 기간 수정 | 폐기 기간 해제 과정 오류 | 외감 시작 2주 전 기간 잠금 | 🔴 Critical |
| 권한 보유 퇴직자 미회수 | 인사 부서 미통보 | 월간 SU01 신규/삭제 사용자 확인 | 🟠 High |

## K-SOX / 외감 체크사항

- **보고서 진정성 선언**: 최종 재무제표 공시 전 CFO/회계책임자 서명
- **내부통제 평가**: COO/CFO는 분기별 내부통제 효율성 평가 + 개선 계획 수립
- **감사인 협력**: 외감 시작 전 감사계획(Audit Plan) 합의, 재무보고 위험 영역 사전 공지
- **기록 보관**: FB09, SU01 변경 이력, 권한 재인증 문서 최소 7년 보관

## 참고 및 관련 규정

- **K-SOX 대상 기업**: 상장사 + 대규모비상장사 (매출액 2조 이상)
- **재무보고 내부통제평가**: COSO-ERM(Committee of Sponsoring Organizations) 준수
- **개인정보보호**: 사용자 권한 정보는 최소 필요한 인원만 열람
