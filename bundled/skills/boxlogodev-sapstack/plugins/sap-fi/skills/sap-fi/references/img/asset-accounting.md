# 자산회계(Asset Accounting, AA) IMG 구성 가이드

## SPRO 경로
```
SPRO → Financial Accounting → Fixed Assets
    → Valuation and Posting
```

## 필수 선행 구성

- [ ] OX02: 회사코드 정의 (통화 설정)
- [ ] OB13: GL 계정 차트 정의
- [ ] OB29: 회계연도 변형 정의
- [ ] OBC4: 필드상태그룹 정의

## 자산회계의 핵심 T-codes

### 1. OADB — 감가상각 영역 정의(Depreciation Areas)

**SPRO 경로**: `SPRO → Financial Accounting → Fixed Assets → Valuation → Depreciation Areas → Define Depreciation Areas`

**목적**:
- 자산의 다중 평가(Book, Tax, IFRS 등)
- 각 영역별 독립적인 감가상각 계산
- 국가별 세무/회계 기준 대응

**표준 감가상각 영역** (한국):

| 영역 | 영역명 | 목적 | 통화 | 연계 계정 |
|------|--------|------|------|---------|
| 01 | Book Depreciation | 재무보고서 | KRW | 100000 (자산) |
| 02 | Tax Depreciation | 세무신고 (법인세) | KRW | 100100 (세무자산) |
| 03 | IFRS | 국제회계기준 | KRW | 100200 (IFRS) |
| 04 | Group Accounting | 연결재무제표 | KRW | 100300 (그룹용) |
| 15 | Subledger | 보조원장 | KRW | 100400 (보조) |

**구성 단계** (T-code: OADB):

1. **감가상각 영역 생성**
   ```
   영역 코드: 01
   영역 설명: "Book Depreciation"
   자산 계정: 100000
   누적감가상각 계정: 160000
   감가상각비 계정: 510000
   ```

2. **영역별 특성 설정**
   ```
   영역 01 (Book):
   - 평가 수준: 개별 자산 (Individual)
   - 증감식 입력: X (매월 누적)
   
   영역 02 (Tax):
   - 평가 수준: 자산그룹 (Group)
   - 법인세 감가상각 기준 적용
   
   영역 03 (IFRS):
   - 평가 수준: 개별 자산
   - IFRS16 리스 자산 포함
   ```

3. **통화 및 환율 설정**
   ```
   영역별 통화: KRW (한국 원)
   다국적 기업: USD, EUR 등 별도 영역
   ```

**필드 설명**:

| 필드 | 설명 | 값 | 주의사항 |
|------|------|-----|---------|
| Area | 영역 코드 | 01, 02, 03 | 1~99 범위, 중복 금지 |
| Description | 영역 설명 | Book, Tax | 관리 편의성 |
| Asset Account | 자산 계정 | 100000 | GL 계정과 일치 필수 |
| Accumulated Depr. | 누적감가상각 | 160000 | Contra 계정 |
| Depreciation Expense | 감가상각비 | 510000 | 비용 계정 |

### 2. OAYZ — 자산 클래스 정의(Asset Classes)

**SPRO 경로**: `SPRO → Financial Accounting → Fixed Assets → Master Records → Maintain Asset Classes`

**목적**: 자산의 범주화 및 감가상각 정책 할당

**자산클래스 구조** (한국식):

```
자산클래스    | 설명              | 감가상각기간 | GL 계정
1000         | 건물 및 구축물     | 20~40년      | 100000
1100         | 건설중자산         | 해당없음     | 100100
1200         | 임차인개량        | 5~10년      | 100200
2000         | 기계장비          | 5~10년      | 110000
2100         | 연구개발장비      | 3~5년       | 110100
3000         | 차량             | 5년         | 120000
3100         | 당사용차량        | 4년         | 120100
4000         | 컴퓨터/IT        | 3~4년       | 130000
4100         | 소프트웨어        | 3~5년       | 130100
5000         | 비품             | 5년         | 140000
6000         | 임차자산(리스)    | 리스기간    | 150000
7000         | 무형자산          | 5~10년      | 170000
8000         | 건설중자산(CIP)  | 해당없음     | 180000
```

**구성 단계** (T-code: OAYZ):

1. **자산클래스 생성**
   ```
   클래스: 1000
   설명: "건물 및 구축물"
   감가상각키: "K001" (20년)
   GL 계정: 100000
   ```

2. **클래스별 감가상각 정책**
   ```
   자산클래스 | 감가상각키 | 방법 | 연수
   1000      | K001      | 정액법 | 20
   2000      | K002      | 정액법 | 10
   3000      | K003      | 정액법 | 5
   4000      | K004      | 정액법 | 3
   ```

3. **자산그룹 설정** (선택)
   ```
   자산그룹 코드: AG-001
   설명: "건물"
   자산클래스: 1000, 1100, 1200 포함
   ```

**필드 설명**:

| 필드 | 설명 | 값 | 비고 |
|------|------|-----|------|
| Asset Class | 클래스 코드 | 1000~9999 | 고유성 필수 |
| Description | 설명 | 건물 | 관리 편의성 |
| Depreciation Key | 감가상각키 | K001, K002 | AFAMS와 연계 |
| G/L Account | GL 계정 | 100000 | 취득가액 계정 |
| Useful Life | 감가상각기간 | 20, 10, 5 | 연 단위 |

### 3. AFAMS — 감가상각키 정의(Depreciation Keys)

**SPRO 경로**: `SPRO → Financial Accounting → Fixed Assets → Valuation → Depreciation → Depreciation Keys`

**목적**: 감가상각 계산 방식 및 기간 정의

**감가상각 방법** (한국식):

| 방법 | 방법코드 | 계산식 | 사용 시기 |
|------|---------|--------|---------|
| 정액법(Straight-Line) | 10 | Year 취득가 / 경제수명 | 표준 |
| 정율법(Declining-Balance) | 20 | Book Value × 정율 | 초기 감소 필요시 |
| 가속감가상각 | 30 | 구간별 다른 율 적용 | 세무 사유 |
| 한번에(Immediate) | 40 | 소액자산 100% 즉시 | <500,000원 |

**구성 단계** (T-code: AFAMS):

1. **감가상각키 생성**
   ```
   키: K001
   설명: "건물 정액법 (20년)"
   감가상각 방법: 10 (정액법)
   유효기간: 01.01.2024 ~ 31.12.2099
   ```

2. **방법별 파라미터 입력**
   ```
   방법 10 (정액법):
   - 경제수명: 20 (년)
   - 매월 감가상각: 1/20 = 5% (년 1/20)
   
   방법 20 (정율법):
   - 정율: 150% (1/연수 × 배수)
   - 예: 10년 자산 → 10% × 150% = 15%
   ```

3. **구간별 감가상각** (선택사항)
   ```
   기간 1 (1~5년): 30% (고감가상각)
   기간 2 (6~10년): 20% (중감가상각)
   기간 3 (11년~): 5% (저감가상각)
   ```

**필드 설명**:

| 필드 | 설명 | 값 | 용도 |
|------|------|-----|------|
| Key | 감가상각키 | K001 | 고유 |
| Description | 설명 | 건물 정액법 | 관리 |
| Depreciation Method | 방법 | 10 (정액) | 계산식 |
| Useful Life | 경제수명 | 20 | 기간 |
| Start Month | 시작월 | 01 | 연중 추가시 고려 |

### 4. AO90 — 자산 계정결정(Asset Account Determination)

**SPRO 경로**: `SPRO → Financial Accounting → Fixed Assets → Valuation → Account Determination → Maintain Asset GL Accounts`

**목적**: 자산클래스 및 거래유형별 GL 계정 자동 결정

**거래유형별 계정 할당**:

| 자산클래스 | 거래유형 | GL 계정 | 설명 |
|-----------|---------|---------|------|
| 1000 | 100 (취득) | 100000 | 건물 |
| 1000 | 110 (증가) | 100000 | 건물 증설 |
| 1000 | 120 (폐기) | 100000 | 자산 폐기 |
| 1000 | 200 (감가상각) | 160000 | 누적감가상각 |
| 1000 | 300 (폐기시 손실) | 515000 | 자산폐기손실 |
| 1000 | 310 (폐기시 이익) | 450000 | 자산폐기이익 |
| 2000 | 100 (취득) | 110000 | 기계장비 |
| 3000 | 100 (취득) | 120000 | 차량 |

**구성 단계** (T-code: AO90):

1. **자산클래스별 계정 설정**
   ```
   자산클래스: 1000
   거래유형 100: GL 계정 100000 (자산)
   거래유형 200: GL 계정 160000 (누적감가상각)
   거래유형 300: GL 계정 515000 (폐기손실)
   ```

2. **감가상각 영역별 계정 연계**
   ```
   자산클래스: 1000
   감가상각 영역: 01 (Book)
     - 취득: 100000
     - 감가상각: 160000, 510000
   
   감가상각 영역: 02 (Tax)
     - 취득: 100100
     - 감가상각: 160100, 510100
   ```

3. **저가자산 정책** (선택)
   ```
   자산클래스: 5000 (비품)
   취득가액 < 500,000원: 즉시 비용 처리
   거래유형: 150 (즉시 비용)
   GL 계정: 530000 (소품비 또는 유사)
   ```

**구성 검증** (T-code: AS01 — 자산 생성):
```
새 자산 생성:
- 자산클래스: 1000
- 취득가액: 100,000,000 원
- 취득일: 01.01.2024

결과:
- Debit 100000 (자산) 100,000,000
- Credit 200000 (채무) 100,000,000
```

### 자산마스터 주요 필드:

| 필드 | 설명 | 입력값 | ECC vs S/4 |
|------|------|--------|-----------|
| Asset Class | 자산클래스 | 1000 (건물) | 동일 |
| Acquisition Date | 취득일 | 01.01.2024 | 동일 |
| Acquisition Value | 취득가액 | 100,000,000 | 동일 |
| Company Code | 회사코드 | 1000 | 동일 |

| Profit Center | 이익센터 | PC-001 | S/4: 권장 |
| Useful Life | 감가상각기간 | 20 (년) | S/4: 자동 |

### 월별 감가상각 자동 계산:

```
자산: 건물 (클래스 1000)
취득가액: 100,000,000 원
취득일: 2024.01.15
감가상각기간: 20년 = 240개월
감가상각방법: 정액법

월별 감가상각비 = 100,000,000 / 240 = 416,667 원

2024년 1월: 416,667 원 (15일부터 월말)
2024년 2월: 416,667 원
...
2024년 12월: 416,667 원
2025년 1월: 416,667 원 (정상 월)

자동 전기 (월말 배치 AFAMA 또는 AFJK 실행):
Debit 510000 (감가상각비) 416,667 원
Credit 160000 (누적감가상각) 416,667 원
```

## S/4HANA 신자산회계(New AA) 전환

**S/4 필수 설정** (ECC와의 주요 차이):

1. **신자산회계 활성화** (T-code: S_BCE_68001658)
   ```
   Activate New Asset Accounting: "X" (체크)
   ```

2. **자산 GL 계정 통합**
   ```
   ECC: AA (독립 모듈) + GL 계정
   S/4: ACDOCA (통합) → GL 계정 직접 기록
   ```

3. **감가상각 영역 자동 생성**
   ```
   신GL 활성화 시:
   - 영역 01, 02, 03 자동 생성
   - 사용자 정의 영역은 선택사항
   ```

4. **리스 자산 (IFRS 16) 통합**
   ```
   S/4HANA: 리스 자산 AA에 자동 포함
   ECC: 별도 처리 필요
   ```

기간: 202401 (2024년 1월)

```

### 2. 감가상각 결과 검증

**T-code: AFVV (Asset Depreciation History)**
```

기간: 202401

- 자산: 100,000,000
- 누적감가상각: 416,667

## 자산 폐기(Retirement)

### T-code: ABGA 또는 AS04

**폐기 절차**:
```
1. AS04 (자산 폐기) 실행
2. 자산 선택
3. 폐기 방법 선택:
   - 전체 폐기: 100%
   - 부분 폐기: 일부만 (예: 50%)
4. 폐기일: 자산 소거 날짜
5. 폐기가격: 잔존가격 (있으면)
```

**자동 전기** (폐기 시):
```
Debit 160000 (누적감가상각) [누적치]
Debit 515000 (자산폐기손실) [손실액] (있으면)
Credit 100000 (자산) [취득가액]
Credit 450000 (자산폐기이익) [이익액] (있으면)
```

## 주의사항

1. **감가상각 영역 일관성**
   - OADB, OAYZ, AFAMS 세 T-code 연계 필수
   - 하나 빠지면 감가상각 계산 실패

2. **취득일 설정**
   - 월말 이전 취득 자산: 당월부터 감가상각
   - 월말 이후 취득 자산: 익월부터 감가상각
   - AFAMA 실행 전 확인 필수

3. **저가자산 정책**
   - 500,000원 이상: 자산 처리 (AA 마스터)
   - 500,000원 미만: 경비 처리 (즉시 비용)
   - 기준액은 회사 정책에 따라 조정

4. **S/4HANA 마이그레이션**
   - ECC AA → S/4 신AA 직접 통합 불가
   - 마이그레이션 전문가 필요
