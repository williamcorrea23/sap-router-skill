# QM 일간 운영 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
품질관리의 일일 점검 체크리스트. 검사 로트 관리, 품질통보, 공급업체 품질 모니터링을 포함합니다.

---

## 체크리스트

### 미결 검사 로트 확인
- [ ] **QA33 미결 검사 로트 조회** — T-code: QA33
  - 왜: 차단 재고 신속 처리, 생산 일정 유지
  - 빈도: 매일 08:00, 14:00 (2회)
  - 세부:
    ```
    QA33 필터:
    - Status: '계획'(계획됨), '진행중'
    - 검사 마감일: 오늘 또는 과기한(Overdue)
    - 우선순위: 1(상) 필터링
    ```
  - 조치:
    - Overdue 로트: 즉시 검사 또는 사용결정
    - 당일 마감: 검사 실행 일정 재확인

### 사용결정 대기 로트 처리
- [ ] **QA32 사용결정 대기 조회** — T-code: QA32
  - 왜: 품질 판정 → 재고 해제 또는 폐기 (WIP 정확성)
  - 빈도: 매일 15:00
  - 세부:
    ```
    QA32 필터:
    - Status: '검사완료'(결과입력됨, 사용결정 대기중)
    - 작업 소유자: 자신 또는 팀
    ```
  - 조치:
    ```
    각 로트별 사용결정:
    - ACCEPT(승인): 재고 자동 해제 → PP/MM 반영
    - REJECT(반품): 공급업체로 반품 처리
    - SCRAP(폐기): 원가 감소 → 폐기비용 발생
    - REWORK(재작업): 공정으로 재투입 (부분 합격)
    ```

### 품질통보 진행 현황
- [ ] **QM10 품질통보 진행 상황** — T-code: QM10
  - 왜: 고객 품질 이슈 대응 추적, 원인 파악
  - 빈도: 주간 월요일 09:00 리뷰
  - 세부:
    ```
    QM10 필터:
    - Status: 'OPEN'(미해결)
    - 중요도: 1(긴급), 2(높음)
    ```
  - 추적:
    ```
    각 통보별:
    - 신고 기한 초과 여부 → 보고 필요
    - 원인 분석 완료 여부 → 8D Report
    - 시정 조치 수행 여부 → 효과성 검증
    ```

### 공급업체 품질 점수 모니터링
- [ ] **공급업체 품질 KPI** — T-code: QE3 / QQR
  - 왜: 공급업체 선정, 단가 협상, 부품 신뢰성 평가
  - 빈도: 월 1회(월말)
  - 세부:
    ```
    공급업체별 품질지수:
    - 합격률: ACCEPT / (ACCEPT+REJECT+SCRAP) × 100%
      목표: 98% 이상
    - 반품률: REJECT / (수입량) × 100%
      목표: 0.5% 이하
    - 불량 원인별 집계 → 파레토 분석
    
    예시:
    공급업체 A: 합격률 96% (부족)
    - 원인: 형상 공차 초과 (60%)
    - 대책: 공급업체 협의 → 공정 능력 개선 요청
    ```

### 검사계획 유효성 검증
- [ ] **QP01 검사계획 유효성** — T-code: QP01
  - 왜: 불필요한 과잉 검사 제거 (비용 절감), 필요한 검사 보장
  - 빈도: 월 1회(목요일)
  - 세부:
    ```
    평가항목:
    - 고장률: 최근 6개월 부품 고장 = 0건?
      → 검사 샘플링 축소 검토
    - 새 부품: 초기 공정능력 확인 완료?
      → 특별 검사 기준 설정
    ```

### MIC(Measurable Inspection Characteristics) 적정성
- [ ] **검사특성 및 샘플링 검토** — T-code: QP02 / QP03
  - 왜: 측정 오류 최소화, 객관적 품질 판정
  - 빈도: 월 1회
  - 세부:
    ```
    평가항목:
    - 치수 측정: 측정 장비 보정(Calibration) 여부 확인
    - 특성 샘플 크기: AQL(Acceptable Quality Level) 적절성
      예: 부품 500개 입고 → AQL 0.65%, n=32 샘플 검사
    ```

### SPC(Statistical Process Control) 관리도 추이
- [ ] **SPC 관리도 모니터링** — T-code: QCC / QSU
  - 왜: 공정 변동성 조기 감지, 예방 조치
  - 빈도: 주 1회
  - 세부:
    ```
    관리도 판독:
    - 평균 추세: 점이 상한/하한 벗어남?
      → 공정 조정 필요
    - 변동성 증가: 표준편차 증가?
      → 공정능력 악화 신호
    - 제어 상태: 8개 연속 점이 중앙선 한쪽?
      → 공정 이상 신호
    ```

    ```
    - 사내 부적합: ___ 건

    ```

## 연계 프로세스

- **MM**: QA32 사용결정 → MIGO(입고) 또는 MB1C(폐기) 자동 연계
- **PP**: 부적합 로트 → MB1C 폐기 → 재생산오더 필요 여부 검토
- **SD**: 완성품 검사 → FI 출고 인정(Billing)
