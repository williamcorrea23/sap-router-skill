## 정책 및 절차

### 1. 급여 권한 분리 (Segregation of Duty)

#### 권한 매트릭스
- [ ] **역할 분리 검증 (분기별)**
  - 원칙: **인사마스터 변경 ≠ 급여 실행**
  - 적용:
    ```
    1. 인사마스터 변경(PA30) 권한: HRM 담당자
       - 신입/퇴직/전배/승진
       - 급여 담당자와 권한 분리 필수
    
    2. 급여 계산(PC00_M99_CALC) 권한: 급여 담당자
       - HRM 담당자와 권한 분리 필수
       - CFO 승인 필요(월간)
    
    3. 급여 지급(PA03) 권한: 재무/현금 담당자
       - HRM, 급여 담당자와 권한 분리 필수
    
    4. 근태 입력(PT60/CATS) 권한: 부서 관리자 또는 근태담당
       - 급여 담당자와 권한 분리 필수
    ```
  - 감시: 월간 권한 로그(T-code SU50) 표본 검사

#### 감사 추적
- [ ] **T-code PA30 / PC00_M99_CALC — 월간 거래 감시**
  - 대상:
    - PA30 인사변경 모두 추적 (특히 급여 변경)
    - PC00_M99_CALC 수동 보정 항목
    - PC00_M99_CEDT 수동 조정 항목
  - 수행자: 내부감사팀
  - 기록: 감사 로그 월간 보관
  - 예시:
    ```
    - 누가 언제 PA30으로 OOO 직원 급여를 ₩200,000 인상했는가?
    - 누가 언제 PC00_M99_CALC 결과를 수정했는가?
    - 월별 급여비 증감 사유 기록
    ```

### 2. 인사정보 보안 및 접근 제어

#### 인사마스터 접근 권한 세분화
- [ ] **T-code PA30 / INFOTYPE — 인포타입별 권한 설정**
  - 실행: 분기별 권한 재검토
  - 인포타입별 접근 제어:
    ```
    인포타입 0002 (개인정보: 주민번호, 은행계좌)
    - 접근권: HR팀장, 급여담당(최소화)
    - 금지: 일반 영업직원, 감시용 직원
    - 마스킹: 다른 직원 조회 시 주민번호 ****-****
    
    인포타입 0008 (급여)
    - 접근권: 급여담당, CFO
    - 금지: 직원 본인도 타인 급여 조회 불가
    
    인포타입 0001 (기본 정보: 직급, 부서)
    - 접근권: 전체 직원 (매니저는 부하직원만)
    - 공개: HR 포탈에서 조회 가능
    ```
  - 도구: T-code PFCG (권한 프로필 관리)

#### 개인정보보호법 준수
- [ ] **한국 개인정보보호법 준수**
  - 실행: 월간 점검
  - 항목:
    ```
    1. 주민번호 관리
       - 저장: PA30 인포타입 0002에만 저장
       - 조회: 권한자만 가능
       - 마스킹: 내부 보고서는 주민번호 끝 6자리 표시(예: 880101-*****)
       - 파기: 퇴직 후 3년 경과 시 시스템에서 삭제
    
    2. 이메일/전화 정보
       - 개인용과 회사용 명확히 분리
       - 개인용은 비활성화 또는 마스킹
    
    3. 건강정보(건강검진 결과)
       - 저장: PA30 특정 필드만 사용
       - 조회: 본인 + 보건담당자만
       - 타인 공개: 본인 동의 필수
    ```
  - 감시: HR시스템 접근 로그 분기별 감사

### 3. 근로기준법 준수

#### 주당 52시간 모니터링
- [ ] **T-code PT60 / CATS — 초과근무 모니터링 (주간)**
  - 실행: 주 1회(목요일)
  - 확인사항:
    ```
    법규:
    - 주당 최대 근무시간: 40시간(기본) + 12시간(초과) = 52시간
    - 일당 최대 초과근무: 4시간
    - 휴일 초과근무: 추가 제약 없음(단, 보상휴가 제공)
    
    모니터링:
    - 개별 직원별 주간 초과근무: 12시간 초과 여부
    - 누적 초과근무(월): 200시간 이상 경보
    - 위험군: 높은 초과근무 직원 사전 상담
    
    조치:
    - 12시간 초과: 경고 + 다음주 휴무 강제
    - 연속 높은 초과근무(3주 이상): 건강 상담 + 휴무 권고
    - 반복 위반: 인사 조치 또는 법정 휴가 강제
    ```
  - 기록: 주간 초과근무 리포트 (노무사 검토 후 보관)

#### 휴가 관리
- [ ] **T-code PT60 — 연차/휴가 관리 (분기별)**
  - 실행: 분기 말
  - 확인사항:
    ```
    1. 연차휴가(법정)
       - 발생: 근로자 1년 시간 기준 15일
       - 소진: 분기별 평균 소진율 50% 이상 목표
       - 미사용: 차년도 월 5일 이월 + 보상금 지급(대체휴가 미사용시)
    
    2. 조직 휴가 정책
       - 하계휴가(예: 8월): 전직원 5일 이상
       - 명절휴가(설/추석): 연차 + 특별휴가
       - 보상휴가(초과근무 보상): 유급 휴가
    
    3. 특별휴가
       - 경조휴가: 결혼(5일), 부모상(5일), 자식상(3일)
       - 병가: 의료증명 후 최대 90일(무급)
       - 공익휴가: 헌혈(1일), 투표(특별 휴가)
    ```
  - 목표: 분기 미사용 휴가 비율 < 20%
  - 기록: 휴가 현황 분기별 리포트

### 4. 연간 조직 관리

#### 조직 구조 정기 검토
- [ ] **T-code PPOME — 조직 구조 검토 (연간)**
  - 실행: 연초(1월) 또는 반기별(6월)
  - 확인사항:
    ```
    1. 조직 단위(Org Unit)
       - 미사용 부서(인원 0명) 비활성화
       - 신설 부서 추가
       - 부서명 변경 적용
    
    2. 직책(Position)
       - 공석 위치 정보 업데이트
       - 신규 직책 생성(필요 시)
       - 우선순위(hierarchy) 재정렬
    
    3. 직무(Job)
       - 직무 설명서 갱신
       - 필수 자격(qualification) 재정의
    ```
  - 승인: CEO 또는 인사이사
  - 문서: 조직 구조도 최신화 (연간)

#### 직위/직급 정책 검토
- [ ] **연간 직급 체계 검토**
  - 실행: 연초(1월)
  - 항목:
    ```
    1. 직급 체계(Grade) 정의
       - 이사, 부장, 과장, 대리, 사원, 인턴
       - 각 직급별 급여 표준(Min/Mid/Max)
       - 직급 간 급여 역전 제거
    
    2. 급여 인상 정책
       - 연간 기본 인상율(보통 3-5%)
       - 성과급(Bonus): 평가 결과별 차등
       - 직급 승진 시 급여 증액율(예: 12-15%)
    
    3. 성과평가 연계
       - 상위 10%: 인상율 8%
       - 중상위(25%): 인상율 5%
       - 중간(40%): 인상율 3%
       - 하위(25%): 인상율 1% 또는 동결
    ```
  - 문서: 직급 및 급여 정책 문서 (법무 검토)

### 5. K-SOX 준수 사항

#### 일반 준수
- [ ] **FRAMEWORK**

  - 감시 추적: T-code SU50로 월간 권한 변경 이력 확인

#### 한국 규정 준수
- [ ] **근로기준법 / 개인정보보호법**
  - 52시간 준수: PT60 모니터링 (주간)
  - 휴가 관리: 분기별 점검
  - 개인정보: 주민번호 마스킹, 접근 제한
  - 문서: 모든 정책은 노무사 검토 후 보관
