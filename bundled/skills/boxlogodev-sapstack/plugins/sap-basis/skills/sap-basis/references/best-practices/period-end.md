## 위험·롤백 계획

### 마감 잡 실패 시
- F110 fail → 즉시 트랜잭션 정지·롤백 (F110에서 reset proposal/run)
- F.05 fail → 임시 retry 가능, but 결과 회계기간 영향 사전 확인
- F.16 fail → **롤백 절대 금지** (이미 이월된 트랜잭션은 backout 불가)
  - Fix만: 누락 분 별도 ZZ_REVERSE_CARRY 절차

### 시스템 다운 시
- 우선순위: DB → SAP NW → 응용 잡
- 한국 시간대 기준 6시간 이내 복구 의무 (SLA 시)
- 외부 stakeholder 통보 절차 (재무 임원, IR, 감사인)
