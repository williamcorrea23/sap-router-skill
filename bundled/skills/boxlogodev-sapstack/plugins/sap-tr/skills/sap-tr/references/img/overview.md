# 송금관리(Treasury, TR) IMG 구성 가이드 — 개요

## SPRO 경로
```
SPRO → Financial Accounting → Treasury → Cash Management
```

## TR IMG 주요 영역

### 1. 기본 설정 (Cash Management)
- **하우스뱅크 정의** (T-code: FI12)
- **은행 계좌 설정** (T-code: FI12)
- **은행 연결 통신** (T-code: FI13)
- **지급 프로그램 설정** (T-code: FBZP)

### 2. 은행명세서 처리
- **은행명세서 포맷** (T-code: FF_5)
- **MT940 정의** (T-code: FF_5, SWIFT)
- **자동 거래 대사** (T-code: FF.5)
- **외부 거래 유형** (T-code: FFVT)

### 3. 지급 처리
- **지급 메서드** (T-code: FBZP)
- **지급 파일 포맷** (T-code: DMEE)
- **지급 조건 정의** (T-code: OBB8)
- **상계 규칙** (T-code: OBXP)

### 4. 외환 관리
- **환율 유형** (T-code: OB61)
- **환율 정의** (T-code: OB08)
- **실현 손익** (T-code: OBA1, OBA2)
- **통화 환전** (T-code: OBMA)

### 5. 현금관리 분석
- **현금 흐름 분석** (T-code: FF.3)
- **은행 조정** (T-code: FF.2)
- **포지션 분석** (T-code: TCURR)

## 구성 순서 (권장)

```
1단계 기본설정
  ├─ FI12: 하우스뱅크 정의
  ├─ FI12: 은행 계좌 ID 정의
  └─ FI13: 은행 연결 설정

2단계 지급 관리
  ├─ FBZP: 지급 프로그램 정의
  ├─ OBB8: 지급 조건 정의
  └─ DMEE: 지급 파일 포맷 설정

3단계 은행명세서
  ├─ FF_5: 명세서 포맷 정의
  ├─ FFVT: 외부 거래 유형
  └─ 자동 대사 규칙 설정

4단계 외환관리
  ├─ OB61: 환율 유형 정의
  ├─ OB08: 환율 마스터
  └─ OBA1/OBA2: 실현손익 계정

5단계 현금분석
  └─ FF.3: 현금 흐름 모니터링
```

## TR 모듈의 핵심 개념

### 현금 관리 사이클 (Cash Cycle)

```
1. 지급 계획
   ├─ 채무 조회 (F.13)
   └─ 지급 우선순위 결정

2. 지급 실행
   ├─ F110: 지급 제안
   ├─ F110S: 지급 실행
   └─ DMEE: 은행 파일 생성

3. 은행 송금
   ├─ SWIFT/SFTP 전송
   └─ 은행 처리

4. 은행명세서 수령
   ├─ 자동 다운로드 (MT940)
   └─ 현금 수량 확인

5. 조정 및 대사
   ├─ FF.2: 은행 조정
   ├─ FF.3: 현금 흐름 분석
   └─ 차이 조사
```

### 하우스뱅크 구조

```
Company Code 1000
  └─ Controlling Area KOR
      └─ House Bank HB-001 (우리은행)
          ├─ Account ID 1001 (당좌 계좌)
          │   └─ Currency KRW
          ├─ Account ID 1002 (저축 계좌)
          │   └─ Currency KRW
          └─ Account ID 1003 (외화 계좌)
              └─ Currency USD
```

## ECC vs S/4HANA

| 항목 | ECC | S/4 |
|------|-----|-----|
| **하우스뱅크** | FI12 (동일) | FI12 (동일) |
| **지급 관리** | F110/F110S | F110/F110S (동일) |
| **은행명세서** | FF_5 (수동) | TFFM (자동 가능) |
| **환율 관리** | OB08 (수동) | OB08 (자동 연계) |
| **현금분석** | FF.3 (배치) | FF.3 (실시간) |
| **DMEE** | 포맷 정의만 | 자동 생성 |

## 공통 용어

| 용어 | 설명 |
|------|------|
| **House Bank** | 거래 은행 |

| **DMEE** | 지급 파일 생성 엔진 |

| **Payment Run** | 지급 배치 |
| **Cash Position** | 현금 포지션 |

## 월별 TR 운영

### 월중
```
1. 현금 포지션 모니터링
   - FF.3: 일일 현금 흐름
   - 부족분 사전 예측
   
2. 채무 추적
   - F.13: 만기 채무 현황
   - 지급 스케줄 확인
```

### 월말
```
1. 지급 제안 생성
   - F110: 지급 대상 선정
   - 지급 우선순위 결정
   
2. 지급 실행
   - F110S: 최종 승인
   - DMEE: 은행 파일 생성
   - SWIFT 전송
   
3. 은행명세서 수령
   - MT940 다운로드
   - FF_5: 자동 대사
   
4. 조정
   - FF.2: 은행 조정
   - OBA1: 환율 손익 계산
   - GL 잔액 검증
```
