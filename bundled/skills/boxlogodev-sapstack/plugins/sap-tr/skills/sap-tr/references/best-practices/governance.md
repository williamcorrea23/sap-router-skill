- **ECC**: ✓

## 체크리스트

### 🏦 은행 마스터 거버넌스

#### 하우스뱅크 설정 정기 검토
- [ ] **T-code FF_1 — House Bank Configuration**
  - 시기: 연 2회 (상반기/하반기)
  - 검토 항목:
    - 기존 하우스뱅크 거래량/조건 평가
    - 신규 은행 추가 필요 여부 (지역, 상품 다양화)
    - 은행별 거래 수수료 협상
  - 담당: 재정담당자 / 은행담당자
  - 설정 내용:
    - Bank Code, Bank Name, Account Number
    - 우대 조건(금리, 수수료)
    - 담당 연락처

#### 은행 마스터 데이터 오류 방지
- [ ] **T-code SM30 → 은행 테이블 — Bank Master Validation**
  - 월간 체크:
    - Account ID 중복 여부
    - 폐지된 계좌(예: 지점 통폐합) 정보 업데이트
    - 통화 설정 일관성
  - 담당: SAP 시스템관리자
  - 액션: 오류 항목 즉시 정정 또는 비활성화

#### 은행 거래 권한 분리
- [ ] **T-code SU01Z — Bank Account 접근 권한 관리**
  - 역할 정의:
    - **입력**: 지급 청구서(F110) 작성 권한만
    - **승인**: FF50(지급 승인) 권한
    - **실행**: FTVD(자금 실행) 권한
    - **감시**: FF_5(Reconciliation 조회) 권한만
  - 주의: 같은 인원이 지급 생성+승인+실행을 하지 않도록 제어
  - 담당: 보안팀 / 인사팀
  - 검증빈도: 분기(3개월)

### 💰 지급 통제 (Payment Control)

#### 지급 권한 분리 (Segregation of Duties)
- [ ] **T-code SPRO → TR → Payments → Define Payment Program(F110 설정)**
  - 설정:
    - 지급 한도별 승인자 정의 (예: 500만원 이하 = 팀장, 초과 = CFO)
    - 지급 방식(수표, 계좌이체, 어음) 별 승인자
    - 특별 지급(일시금, 보조금) 최고 승인자 정의
  - 담당: SAP 구성담당자 / CFO
  - 정책: 400만원 이상 = 2인 승인 원칙

#### 지급 예약 추적
- [ ] **T-code F110 — Payment Program Monitoring**
  - 월간 점검:
    - 예약된 지급 건수 vs 실제 실행 건수 대비
    - 취소/지연된 지급 사유 조사
    - 월간 지급액 예산 대비율
  - 담당: 지급담당자
  - 빈도: 월 1회 (월초, 지난달 데이터)
  - 문서: 월간 지급 현황 보고

#### 부정 지급 방지
- [ ] **T-code FF_5 + 수동 감시 — Fraud Prevention**
  - 감시 항목:
    - 비정상적 지급처 (신규 공급업체, 해외 송금)
    - 과다 지급 (청구액 대비 125% 이상)
    - 이중 지급 (동일 금액, 동일 공급업체 중복)
  - 담당: 감시팀 / 내부감사
  - 액션: 의심 지급은 승인 전 조사
  - 빈도: 월 1회 (표본 검사)

### 🏦 은행 거래 감시

#### 은행 수수료 모니터링
- [ ] **T-code FF_5 — Bank Fee Monitoring**
  - 월간 은행 수수료 현황:
    - 계좌유지비
    - 이체/송금 수수료
    - 여신 이자료
  - 비정상 수수료 조사:
    - 전월 대비 30% 이상 증가 시
    - 정책상 수수료 면제 대상인데 부과된 경우
  - 담당: 은행담당자 / 재정담당자
  - 액션: 수수료 오류 시 은행에 이의 제기

#### 외화 거래 모니터링
- [ ] **T-code FCM5 — Foreign Exchange Transaction Monitor**
  - 월간 외화 거래 현황:
    - 거래 통화별 건수 및 금액
    - 환율 적용 적정성 (공시율 대비 ±3% 이내)
    - 환차손익 누적액
  - 담당: 재정담당자
  - 빈도: 월 1회
  - 문서: 외화 거래 리포트

#### 차입금/예치금 연계 검증
- [ ] **T-code FF_8 — Loan & Investment Verification**
  - 월간 확인:
    - 은행 거래 명세서 vs SAP 기록 대사
    - 이자 누적액 정확성
    - 만기일 추적 (3개월 이내 만기 사전 알림)
  - 담당: 재정담당자
  - 빈도: 월 1회

### 📊 자금 계획 및 예산

#### 연간 차입 한도 책정
- [ ] **T-code SPRO 또는 수동 — Annual Borrowing Limit**
  - 시기: 연 1회 (회계연도 시작 전, 11월)
  - 절차:
    1. 연간 자금 수요 예측 (Operating CF + CapEx)
    2. 내부 자금(Operating CF) 차감
    3. 필요 차입액 = 총수요 - 내부자금
    4. 차입 한도 = 필요액 × 1.2 (여유율 20%)
  - 담당: 재정담당자 / CFO
  - 승인: Board 또는 경영진 회의
  - 문서: 차입 한도 정책서

#### 월간 현금 예산 관리
- [ ] **T-code FF7B — Monthly Cash Budget**
  - 월간 현금 예산 vs 실적:
    - Operating 현금 입금
    - Operating 현금 지급
    - 투자 활동 현금
    - 재무 활동 현금 (차입/상환)
  - 담당: 자금담당자
  - 빈도: 월 1회 (월초, 지난달 데이터)
  - 목표: 예산 대비 실적 편차 < 5%

#### 자금 효율성 평가
- [ ] **T-code FF7A + 경영 분석 — Cash Efficiency KPI**
  - KPI 항목:
    - **Average Daily Cash Position**: 월 평균 현금 유지 수준
    - **Cash Turnover**: 일간 평균 거래액
    - **Interest Cost Ratio**: 이자료 / 총 자금 비율
  - 담당: 재정담당자 / CFO
  - 빈도: 월 1회
  - 목표: 현금 효율성 향상 및 이자 비용 절감

### 🔐 금융감독 컴플라이언스

#### 차입금 공시 준비
- [ ] **T-code FF_8 + 감시 — Loan Disclosure Preparation**
  - 분기별 공시 준비:
    - 차입금 잔액 및 이자율 현황
    - 만기 현황 (1년 이내/초과)
    - 저당/담보 현황
  - 담당: 재정담당자
  - 빈도: 분기 1회 (공시 1개월 전)
  - 문서: 차입금 공시 자료

#### 대외 공시 시스템 제출
- [ ] **DART/SEDARLING/금감원 시스템 — Disclosure Submission**
  - 대상:
    - 상장사: 분기별 재무제표 공시 (TR 내용 포함)
    - 대규모비상장사: 연간 외감 보고서
  - 담당: 재정담당자 + 홍보/법무팀
  - 빈도: 공시 일정에 따름
  - 검토: CFO 최종 서명

### 🔍 정기 감시

#### 월간 TR 거버넌스 체크
- [ ] **T-code FF_5, FF_8, F110 — Monthly TR Governance Check**
  - 점검 항목:
    - 은행 Reconciliation 완료 여부
    - 지급 한도별 승인 프로세스 준수
    - 외화 거래 모니터링
    - 부정 지급 의심 건 조사
  - 담당: 감시팀 / 재정담당자
  - 문서: 월간 거버넌스 점검표

#### 분기 감사 (Quarterly Review)
- [ ] **분기 거버넌스 평가**
  - 항목:
    - 지급 승인 프로세스 표본 감사 (10% 이상)
    - 은행 Reconciliation 품질 평가
    - 부정 지급 감지 현황
    - 자금 효율성 KPI 평가
  - 담당: 내부감사 / CFO
  - 문서: 분기 감사 보고서

#### 연간 K-SOX 평가
- [ ] **연간 TR 내부통제 평가**
  - 평가 항목:
    - 은행 거래 권한 분리 준수 현황
    - 월간 Reconciliation 완료율
    - 지급 한도별 승인 준수율
    - 부정 지급 적발 사례 및 개선
  - 담당: CFO / 감사위원회
  - 결과: 내부통제 평가 보고서 (최고 경영진 서명)

## K-SOX / 금감위 준수 체크사항

- **자금 관리 정책**: 차입 한도, 지급 승인 기준 문서화 및 주기 검토
- **내부통제 운영**: 월간 Reconciliation, 지급 승인 추적, 부정 감지
- **기록 보관**: 은행명세서, 지급 승인 건, Reconciliation 보고서 최소 7년
- **공시 신뢰성**: TR 정보(차입금, 현금) 정시 공시, 정확성 검증

## 참고 및 관련 규정

- **금융감독위 자금관리 가이드**: 차입 한도, 공시 기준
- **은행감시**: 부당 거래 보고(금융감시위원회)
- **외감**: 분기/연간 차입금, 현금 잔액 검증
