# PM 기간마감 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
월 마감 프로세스에서 보전 모듈 담당자가 수행해야 할 정산, 검증 작업 목록.

---

## 기간 마감 프로세스

### Phase 1: Pre-flight 검사 (T-5일)

#### 1. 미결 보전오더 확인
- [ ] **IW38 미결 오더 현황** — T-code: IW38
  - 필터: Status ≠ 'TECO'(기술완료) AND Status ≠ 'CLSD'(종료)
  - 조치:
    - 진행 중(REL): 마감 전 완료 추정 확인
    - 계획(PLAN): 차기월 이월 결정, 기한 변경 기록
    - 취소 대상: 주관자 승인 후 ITCO 처리

#### 2. 미정산 오더 확인
- [ ] **IW38 → KO88 정산 현황** — T-code: IW38 + KO88
  - 조건: Status = 'TECO' AND KOSTN(원가센터) = 대상
  - 조치: 미정산 건은 KO88로 월말 정산 처리
  - 확인: 기술완료(TECO) → 원가정산(KO88) 연계 자동화 점검

#### 3. 임시 비용 확인
- [ ] **KO88 임시 비용 항목** — T-code: KO88
  - 예시:
    ```
    - 외주비 선입금 (Advance payment)
    - 예상 재료비 (Estimated material cost)
    - 대기 청구서 (Pending invoice)
    ```
  - 조치: 확정 비용 또는 다음월 이월 명확화

### Phase 2: 월 마감 실행 (T-day)

#### 4. 보전오더 기간 마감
- [ ] **IW38 월별 정산** — T-code: IW38
  - 조치:
    ```
    Step 1: IW38 필터 (기간: 1/1~31, Status: TECO)
    Step 2: 각 오더 세부 비용 확인
            - 인력(Hours) + 급여
            - 자재(Parts) + 원가
            - 외주비(Subcontracting)
    Step 3: 오더별 수익성 분석 (실제 > 예상 예산?)
    Step 4: 차이 분석 → 사유 기록
    ```

#### 5. 비용 정산 (KO88)
- [ ] **KO88 보전오더 정산** — T-code: KO88
  - 목적: 보전오더의 모든 비용을 원가센터에 집계
  - 세부:
    ```
    KO88 실행:
    1. 범위: 월 단위 선택 (예: 2026-04)
    2. 오더 선택: 상태 = TECO 그룹만
    3. 정산 방법: 자동(Auto Settle)
       - 직접비: 오더→원가센터 이전
       - 간접비: 배부율로 배분
    4. 정산 후: 원가센터별 비용 총액 산출
    ```
  - 결과: KOSTN(원가센터)별 누적 비용 계산

#### 6. 보전비용 보고
- [ ] **원가센터별 보전비용 추출** — T-code: KE24 + KE71
  - 용도:
    ```
    예: 생산라인 #1 (원가센터 401)
    월간 보전비용: ₩12,340,000
    - 인력비: ₩8,500,000
    - 자재비: ₩2,100,000
    - 외주비: ₩1,740,000
    ```
  - 보고: CFO에 제출
  - 분석: 
    - 목표 대비: 예산 ₩10,000,000 vs 실제 ₩12,340,000 (Unfavorable)
    - 원인: 긴급 보전 3건 추가 (₩2,000,000)

#### 7. 설비 내용년수 추적
- [ ] **IM05 설비 감가상각** (FI 연계) — T-code: IM05
  - 주의: 
    ```
    보전오더 원가 ≠ 설비 감가상각
    설비 감가상각: FI-AA(자산관리) 담당
    보전 비용: PP 원가센터에서만 처리
    ```
  - 확인: 높은 보전비 설비 → 신규 설비 도입 검토 신호

### Phase 3: 검증 (T+3일)

#### 8. 재공품 잔액 대사
- [ ] **IW38 미결 오더 금액 vs FI** — T-code: IW38 + FB01
  - 대사:
    ```
    PM 측: IW38 Status ≠ TECO의 WIP 금액
    FI 측: T-Account "재공품"(설비 보전) 잔액
    목표: 100% 일치
    ```
  - 불일치 원인 분석:
    - 비용 중복 계산
    - 선입금 미반영
    - 환율 차이(수입 외주)

#### 9. 원가센터별 보전비용 검증
- [ ] **KE24 보전 원가센터 검증** — T-code: KE24
  - 확인:
    ```
    각 생산라인별 원가센터:
    - 예산 대비 실제: ±10% 범위 내?
    - 라인별 MTBF: 목표치 달성?
    - 보전비율(% of Production Cost): 정상?
    ```
  - 지표:
    ```
    보전비율 = 월간 보전비 / 월간 생산비 × 100%
    목표: 5~8% (산업 평균)
    > 10%: 설비 노후화 신호
    ```

#### 10. 청구서 정산
- [ ] **외주 보전 청구서 정산** — T-code: FB03 (FI)
  - 확인: 외주비 청구서 도착 → KO88로 이미 예상 비용 계상?
  - 조치: 실제 청구액 vs 예상액 차이 → 원가공사(CO72) 처리

### Phase 4: 마감 확인 (T+5일)

#### 11. FI와 최종 대사
- [ ] **FBL5N 미결제 대사** — T-code: FBL5N (FI)
  - 범위: 보전 관련 미수금/미지급금
  - 확인: 예상 비용 vs 실제 청구액 차이 0원

#### 12. 월 마감 Lock
- [ ] **KO-PER 기간 마감** — 보전 모듈 기간 마감
  - 효과: 월 데이터 수정 불가
  - 선금(후차월): 최소 3일 이내만 허가

## 특수 상황 처리

### 고장으로 인한 외주비 증액
```
상황: Press 긴급 수리 → 외주비 증가 예상 50%
처리:
1. IW38에서 임시 비용 설정 (추정치)
2. KO88 정산 시 임시 비용 포함
3. 청구서 도착 후 차액 조정 (CO72)
기한: 차기월 중순까지
```

### 연 1회 설비 대수선(Overhaul)
```
상황: Injection Molding Machine #1 대수선 (1주)
처리:
1. IP30에서 예방보전 기한 연장 (수동)
2. 별도 프로젝트 오더로 관리 (PM에서 PN01)
3. 월 마감에 포함: 별도 비용 센터 사용
결과: 통상 보전비와 분리 추적 가능
```
