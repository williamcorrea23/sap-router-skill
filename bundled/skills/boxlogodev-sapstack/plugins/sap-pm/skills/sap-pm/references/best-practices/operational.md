# PM 일간 운영 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
설비 보전 업무의 일일 점검 체크리스트. 장비 상태 모니터링, 보전 통보 처리, 예방보전 관리를 포함합니다.

---

## 체크리스트

### 미결 보전통보 확인
- [ ] **IW28 미결 통보 조회** — T-code: IW28
  - 왜: 긴급 보전 요청 신속 대응, 생산 라인 가동률 유지
  - 빈도: 매일 08:00, 12:00, 16:00 (3회)
  - 세부:
    ```
    IW28 필터 조건:
    - Status: '계획' 또는 '릴리스' (미확인 상태만)
    - 우선순위: 1(긴급) 필터링 우선 처리
    - 기한: 오늘 또는 과기한(Overdue)
    ```
  - 조치: 
    - Priority 1(24시간 이내): 즉시 현장 출동
    - Priority 2(48시간 이내): 당일 일정 확보
    - Priority 3(5일): 주중 스케줄 편성

### 긴급 보전오더 모니터링
- [ ] **IW38 보전오더 조회** — T-code: IW38
  - 왜: 진행 중인 보전 현황 파악, 원가 추적
  - 빈도: 매일 09:00, 15:00 (2회)
  - 세부:
    ```
    IW38 필터:
    - Status: 'REL' (릴리스, 진행중)
    - 계획마감일: 오늘
    - 우선순위: 1(상)
    ```
  - 모니터링:
    - 투입 인력 시간: 계획 대비 실제 추적
    - 사용 부품: MB1C 발고 확인
    - 잔여 작업: 예상 마감 시간 재산정

### 생산 계획 협의
- [ ] **보전과 생산 Daily 협의**
  - 시간: 매일 08:30 (15분 회의)
  - 참석: PP 차장, PM 담당자
  - 의제:
    - 예정된 보전이 생산 일정에 미치는 영향
    - 긴급 보전 요청 → 생산 일정 변경 검토
    - 예방보전 주간 스케줄 최종 확인
  - 기록: 일일 회의록 저장

### 기한 모니터링 (IP30)
- [ ] **IP30 예방보전 기한 추적** — T-code: IP30
  - 왜: 예방보전 누락 방지, 설비 신뢰성 유지
  - 빈도: 매주 월요일 08:00
  - 세부:
    ```
    IP30 필터:
    - 기한: 주간(Mon~Fri) 범위
    - Status: '계획' 상태 (미실행)
    ```
  - 조치:
    - 주중 스케줄: 생산 차질 최소 시점에 배정
    - 금주 미실행 항목: 다음주로 이월 → IP01에 변경 기록

### 카운터/미터(Meter) 읽기
- [ ] **IW65 카운터 읽기 입력** — T-code: IW65
  - 왜: 설비 사용 현황 추적, 예방보전 트리거 기준 제공
  - 빈도: 일일 1회(오후 16:00) 또는 주말 엔지니어링 (주1회)
  - 세부:
    ```
    카운터 종류:
    - 운영 시간(Operating Hours): 중장비
    - 운영 사이클(Cycles): 프레스, 사출 성형기
    - 생산량(Quantity): 펌프, 압축기
    ```
  - 예시:
    ```
    설비: Injection Molding Press #1
    카운터: 1,234,567 cycles (어제 대비 +12,345)
    임계값: 1,500,000 cycles (다음 대유지 시점)
    예상: 약 20일 후 예방보전 필요 알림
    ```

### 예비부품 재고 확인
- [ ] **MMBE 예비부품 재고 현황** — T-code: MMBE
  - 왜: 긴급 부품 부족으로 인한 보전 지연 방지
  - 빈도: 매주 목요일
  - 대상: Critical parts (설비 가동에 필수인 부품)
  - 조치:
    - 재고 < 안전재고: 긴급 구매발주 (ME41)
    - 폐기 부품: MM04로 비활성화 처리

### 보전 기술 이력 확인
- [ ] **IW29 설비 이력 조회** — T-code: IW29
  - 왜: 반복 고장 패턴 인식, 근본 원인 파악
  - 빈도: 월 1회 또는 고장 발생 즉시
  - 세부:
    ```
    예: Pump #5 잦은 축 파손
    IW29 조회:
    - 최근 1년 고장 내역: 5회
    - 기간: 평균 60일 주기
    - 원인분석: 회전수 과다? 윤활유 품질? 축정렬?
    ```
  - 의사결정: 근본원인 조사 → 라우팅 변경 또는 부품 업그레이드

### 하루마감 보고
- [ ] **보전 일일 보고서 작성**
  - 시간: 매일 17:00
  - 내용:
    ```
    - 본일 완료 보전: ___ 건
    - 미완료(연기): ___ 건, 사유
    - 긴급 요청: ___ 건
    - 고장 발생: ___ 건, 심각도
    - 예상 내일 작업: ___ 건
    ```
  - 배포: 경영진, PP(생산) 공유

|----------|---------|---------|------|

## 특수 상황 대응

### 설비 고장 발생 시
1. **IW28 긴급 통보 생성** (Priority 1)
2. **IW37 보전오더 생성** (자동 또는 수동)
3. **고장 분류**: 
   - 전기/기계 고장: 전문 기술자 호출
   - 설정 오류: 현장 작업자 복구
4. **원인 분석**: IW29에 결과 기록

### 예방보전 연기
- IP30에서 기한 변경: IP01로 계획 수정
- 사유 기록 필수 (감사 추적)
- 연기 최대 기간: 원 기한으로부터 2주

## KPI / 모니터링

| 지표 | 목표 | 측정 방법 |
|-----|------|----------|
| 설비 가동률 | 95% 이상 | (총시간 - 고장시간) / 총시간 |
| MTBF (Mean Time Between Failures) | 720시간 | IW29 고장 간격 추적 |
| MTTR (Mean Time To Repair) | 8시간 | (복구시간 - 신고시간) 평균 |
| 긴급 보전 비율 | 20% 이하 | 긴급 건수 / 전체 건수 |
