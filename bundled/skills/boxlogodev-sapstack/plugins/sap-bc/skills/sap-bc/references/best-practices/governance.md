# 한국 BC 거버넌스 (Tier 3) Best Practice

> K-SOX·전자세금계산서·망분리·개인정보보호법(PIPA)에 특화된 한국 거버넌스.

## 1. K-SOX (Korean Sarbanes-Oxley)

### 1.1 ITGC 5대 통제
| 통제 | BC 역할 |
|---|---|
| 변경 관리 | 트포 로그·승인 워크플로우 |
| 접근 통제 | PFCG·SU01 변경 이력, 분기 재인증 |
| 운영·모니터링 | Solman Korea 대시보드 |
| SoD | 위반 보유자 0 정책 |
| 데이터 무결성 | DB 백업·audit log 보존 7년 |

### 1.2 평가 주기
- 분기 자체 평가 (Internal Audit)
- 연 외부 감사 (Big 4 또는 지정 회계법인)
- 감사 결과 → 시정 조치 → 사후 검증

## 2. 전자세금계산서 (e-Tax Invoice)

### 2.1 국세청 연동
- 인증서 만료 30일 전 갱신 절차 시작 (STRUST)
- 인증서 갱신: 한국전자인증·코스콤 등 인정 기관
- 일일 전송 모니터링: 성공률 > 99%

### 2.2 부정 발급 방지
- 전송 실패 30분 내 재시도
- 발급 후 1일 내 국세청 confirm 확인
- 부정 발급 시 24시간 내 취소 + 재발급

## 3. 망분리 (Network Separation)

### 3.1 외부 통신 통제
- RFC: 모든 외부 통신 보안 게이트웨이 경유
- HTTPS: TLS 1.2+ 강제
- API: WAF 경유 + IP allowlist

### 3.2 모니터링
- SM21/SM19 — 침입 시도 탐지
- SECUI/APIPark — 외부 통신 로그
- 분기 1회 침투 테스트

## 4. 개인정보보호법 (PIPA)

### 4.1 PII 처리 원칙
- 수집 최소화: 필요 항목만
- 동의 의무: 명시적 사전 동의
- 보유 기간: 목적 달성 후 즉시 폐기
- 위·수탁: 외부 위탁 시 계약·관리

### 4.2 PII 시스템 통제
- 주민번호·계좌 등 PII 필드 마스킹
- 접근 로깅 (SLG1 또는 별도 감사 시스템)
- 동의 철회 → 데이터 삭제 절차

### 4.3 PIPA 위반 대응
- 사고 발생 24시간 내 신고 (개인정보보호위원회)
- 영향 받은 사용자 48시간 내 통지
- 시정 조치·재발 방지

## 5. 사회보험·근로기준

### 5.1 4대보험 통제
- 국민연금/건강보험/고용보험/산재보험
- 페이롤 → 4대보험 EDI → 정확 신고
- 이연·과부족 0 정책

### 5.2 근로기준법
- 페이롤 최저임금 자동 검증
- 시간외 수당 계산 정확성
- 연차 잔여 시스템 정확성

## 6. 데이터 거버넌스

### 6.1 데이터 분류
- 일반 (공개) / 내부 (회사) / 기밀 (제한) / 1급 비밀 (CFO+)
- PII는 항상 "기밀" 이상

### 6.2 보존·폐기
- 회계 데이터: 10년 (상법) 
- 세무 데이터: 5년 (국세기본법)
- HR 데이터: 직원 퇴사 후 3년
- 일반 거래: 5년

## 7. 거버넌스 대시보드

| 지표 | 임계치 | 측정 |
|---|---|---|
| 전자세금계산서 전송 성공률 | > 99% | 일별 |
| 인증서 갱신 사전 알림 | 30일 전 | 일별 |
| 4대보험 신고 성공률 | 100% | 월별 |
| K-SOX 감사 시정율 | > 95% | 분기별 |
| PIPA 위반 건수 | 0 | 분기별 |
| 망분리 통신 위반 | 0 | 일별 |
