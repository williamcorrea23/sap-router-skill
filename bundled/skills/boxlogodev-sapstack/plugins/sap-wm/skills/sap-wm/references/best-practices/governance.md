# WM 거버넌스 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: EWM 권장

## 개요
창고 관리 부문의 정책, 기술 전략, 준법성 항목. EWM 마이그레이션 로드맵을 포함합니다.

---

## 체크리스트

### EWM(Enhanced Warehouse Management) 전환 로드맵
- [ ] **EWM 마이그레이션 계획** — S/4HANA 업그레이드 연계
  - 정책:
    ```
    ECC WM은 SAP에서 더 이상 지원 중단(2030년 목표)
    → S/4HANA로 마이그레이션 필수
    
    EWM 개선 사항:
    1. 실시간 재고 추적 (IoT 센서 연계)
    2. RF/Handheld 터미널 고도화 (GPS 기반 위치)
    3. 자동화(Robotics, Conveyor) 지원
    4. 클라우드 연계 가능성
    ```
  
  - 로드맵:
    ```
    Phase 1 (준비 단계, 3~6개월):
    - 현 WM 설정 분석 (저장유형, 전략, 커스터마이징)
    - EWM 기능 비교 검토
    - 마이그레이션 영향도 평가 (비용, 시간, 인력)
    - 교육 계획 수립
    
    Phase 2 (설계 및 구축, 6~9개월):
    - EWM Sandbox 환경 구축 (별도 시스템)
    - EWM 설정 (저장유형, 전략, 프로세스)
    - 마스터 데이터 마이그레이션 (부품, 저장유형, BOM)
    - 통합 테스트
    
    Phase 3 (검증 및 Go-live, 3개월):
    - User Acceptance Testing (UAT): 실사용자 테스트
    - WM → EWM 병렬 운영 (2주): 양쪽 다 운영하면서 검증
    - 문제점 기록 및 수정
    - Go-live 실행 (커트오버)
    ```
  
  - 예상 비용:
    ```
    - SAP 라이선스: ₩500M ~ ₩1B (규모에 따라)
    - 구현비(SI): ₩1B ~ ₩3B
    - 교육/변경관리: ₩200M ~ ₩500M
    - 총 예상: ₩1.7B ~ ₩4.5B (소규모~중규모)
    ```
  
  - 기한: 
    ```
    S/4HANA 업그레이드 시 함께 추진 권장
    최대 연기: 2027년까지 (그 이후 지원 약화)
    ```

### 저장유형 및 빈 관리
- [ ] **저장유형 설정 및 효율성** — T-code: LX01 / LX04
  - 정책:
    ```
    저장유형은 창고 공간을 논리적으로 구분하는 핵심
    → 적절한 설정으로 효율성 극대화
    ```
  
  - 설계 기준:
    ```
    저장유형별 역할:
    01 입고 준비: Inbound staging (수령 후 검사 대기)
    02 원재료: Raw material storage (장기 보관)
    03 완성품: Finished goods (출고 준비)
    04 WIP: Work in progress (생산 중 임시)
    05 반품: Returns (고객 반품 격리)
    06 불량: Defective (품질 부적합)
    
    각 저장유형의:
    - 위치(Zone): 물리적 창고 구역
    - 빈 수용량: 최대 저장 수량
    - 적재방식: 높이 제한, 무게 제한
    - 입출 전략: FIFO(먼저 들어온 것 먼저), LIFO 등
    ```
  
  - 효율성 평가:
    ```
    월 1회 분석(LX04):
    - 저장유형별 점유율 추적
    - 목표: 70~85%
    - 개선안: 점유율 <50% 또는 >95% 저장유형 정리
    ```

### RF(Radio Frequency) 디바이스 관리
- [ ] **RF 단말기 및 무선 인프라** — T-code: LUI1, LUIXG
  - 정책:
    ```
    RF는 WM 작업 효율의 핵심 (수동 입력 오류 감소)
    → 안정적 관리 및 보안 필수
    ```
  
  - 관리 항목:
    ```
    1. 디바이스 관리
       - RF 단말기 보유 수: 창고 면적당 1대/500m²
       - 배터리 관리: 일일 충전, 월 1회 성능 테스트
       - 고장 수리: 24시간 이내 복구 목표
    
    2. 무선 인프라
       - WiFi AP(Access Point): 신호 강도 > -70dBm 유지
       - 신호 강도 맵: 월 1회 모니터링 (신호 사각지역 없는지)
       - 대역폭: 최소 10Mbps (동시 사용자 기준)
    
    3. 보안
       - RF 암호화: WPA2 이상
       - 사용자 인증: ID/Password (물리 로그인)
       - 감사: RF 데이터 접근 로그 월 1회 검토
    ```
  
  - 예상 비용:
    ```
    RF 단말기: 1대당 ₩2M ~ ₩4M
    WiFi AP: 1대당 ₩500K ~ ₩1M
    연간 유지보수: 하드웨어 가격의 10~15%
    ```

### 저장유형 간 이동(Transfer) 전략
- [ ] **효율적인 TO(Transfer Order) 프로세스** — T-code: LT01, LT06
  - 정책:
    ```
    TO는 창고 내 부품 이동을 추적
    → 명확한 정책과 우선순위로 효율성 확보
    ```
  
  - 전략:
    ```
    1. 자동 vs 수동 TO 생성
       - 자동: FIFO 기반 입고(MIGO) → 자동 TO 생성
         장점: 오류 감소, 일관성
         단점: 유연성 부족
       - 수동: 특정 요청(출고) 기반 TO 생성
         장점: 유연한 우선순위 지정
         단점: 수동 오류 가능성
    
    2. 우선순위 지정
       - Priority 1: 고객 긴급 주문 (24시간)
       - Priority 2: 정기 출고 (48시간)
       - Priority 3: 내부 이동 (1주)
    
    3. 배치 처리
       - 오전 08:00 / 정오 12:00 / 오후 16:00 (3회)
       - 배치별 우선순위 정렬 후 작업 배정
    ```

### 재고정확성 목표 및 감시
- [ ] **월별 물리적 실사 계획** — T-code: LI01N
  - 정책:
    ```
    목표 재고 정확성: 99.5% 이상
    방법: 월 1회 순환실사(Cycle Count) 또는 연 1회 전수실사
    ```
  
  - 계획:
    ```
    순환실사 방식 (권장):
    - 월별 25%씩 4개월 로테이션
    - 각 실사: 저장유형별로 나누어 진행
    - 시간: 각 배치당 4시간 (10명 기준)
    
    예시 연간 일정:
    1월: 저장유형 01 (입고)
    2월: 저장유형 02 (원재료)
    3월: 저장유형 03 (완성품)
    4월: 저장유형 04 (WIP)
    + 반복
    
    정확성 미달 저장유형: 추가 실사 (분기별)
    ```
  
  - 불일치 임계값:
    ```
    - 차이 > 5개 또는 > ₩100K: 근본 원인 분석
    - 분실 우려: 보안 강화 조치
    ```

### 변경관리 및 프로세스 개선
- [ ] **WM 프로세스 변경 승인** — Change Advisory Board (CAB)
  - 정책:
    ```
    WM 프로세스 변경은 시스템 전체에 영향
    → 모든 변경사항 문서화 및 승인 필수
    ```
  
  - 변경 유형:
    ```
    1. 저장유형 추가/삭제
       - 영향: TO 라우팅, 출고 시간
       - 승인: WM 매니저 + MM 매니저 + CFO
       - 테스트: Sandbox에서 선행 테스트 필수
    
    2. 이동 전략 변경 (FIFO → LIFO)
       - 영향: 재고 가치평가, 출고 품질(선도 관리)
       - 승인: CFO (원가 영향)
       - 효과: 1개월 선행 검증
    
    3. RF 프로세스 개선
       - 영향: 사용자 인터페이스, 작업 시간
       - 승인: WM 매니저 + 사용자 대표
       - 교육: 구현 전 전수 교육
    ```

### 감사 및 준법성
- [ ] **WM 정기 감사** — 반기 1회
  - 감시 항목:
    ```
    1. TO 프로세스 정확성
       - 샘플: 무작위 선택 10건 TO
       - 확인: 실제 물품 이동 vs 시스템 기록 일치
    
    2. 재고 정확성
       - 월별 정확성 추이: > 99.5% 지속 확인
    
    3. RF 데이터 무결성
       - 기록 손실 여부 확인
       - 사용자 접근 로그: 비인가 접근 0건
    
    4. 변경관리 준수
       - 미승인 변경사항: 0건
       - 문서화 완전성: 100%
    ```

- **ISO 9001**: 공정 설명서 문서화 필수
