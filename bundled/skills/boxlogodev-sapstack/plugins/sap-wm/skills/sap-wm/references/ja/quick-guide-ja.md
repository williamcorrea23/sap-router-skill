# sap-wm クイックガイド (日本語)

## 🔑 環境ヒアリング

SAP WM (Warehouse Management) 作業前に確認:

### 1. SAP リリースとサポート
- **ECC 6.0**: WM 完全サポート
- **S/4HANA**: WM は **非推奨** — EWM へ強制移行

### 2. WM 構成モード
- **Standard WM**: 単純保管, 小規模 DC
- **Lean WM**: 高度機能, 大規模 DC

### 3. バーコード/RF 構成
- RF 端末使用
- バーコードプリンタ・ラベル方針
- モバイルネットワーク依存

### 4. 物流環境
- **EC 配送**: B2C 日次処理量大
- **B2B 流通**: 一括入出庫
- **返品センター**: 返品 + 品質検査

## 📚 主要 T-code と役割

### 入庫 (Inbound)
| T-code | 機能 |
|--------|------|
| **LT01** | 入庫 (Goods Receipt) |
| **LT04** | 入庫ステータス |
| **LT03** | 入庫取消/訂正 |

フロー: PO 参照 → 数量入力/スキャン → bin 提案 → 確認 (即時在庫更新)。

### 保管 (Storage)
| T-code | 機能 |
|--------|------|
| **LS01N** | 在庫概要 |
| **LS01** | bin 別在庫照会 |

LS01N: bin 別リアルタイム在庫, 不良在庫追跡, 滞留警告。

### 出庫 (Outbound)
| T-code | 機能 |
|--------|------|
| **LB01** | ピッキングリスト作成 |
| **LI01N** | 出庫 (Goods Issue) |
| **LI04** | 出庫ステータス |

フロー: SO 参照 → LB01 ピッキング指示 (バーコード) → ピッキング + スキャン確認 → LI01N 最終出庫 → FI 在庫減少。

## 🇯🇵 日本ローカル

### オンライン DC 日常
- 日次処理量: 大規模フルフィルメントセンター
- 午前 入庫 (LT01) + ピッキング準備 (LB01) → 午後 ピッキング + 出庫 (LI01N) → 夕方 配送業者集荷 + 締め

### バーコード/RF 運用
- 入庫時バーコード自動印刷 → bin に貼付
- RF 端末: 倉庫チームがリアルタイムピッキング/出庫スキャン
- ネットワーク信頼性: WiFi 弱エリアはオフラインモード必要な場合

### 返品処理
- プロセス: 顧客返品 → 入庫 → 品質検査 → 再出庫 or 廃棄
- 返品ゾーン専用 bin, LS01N で追跡
- EC 返品率高 → 効率的 WM 処理必須

## ⚠️ WM vs EWM と移行

| 項目 | WM (ECC) | EWM (S/4HANA) |
|------|---------|--------------|
| サポート | ✓ (ECC EOL ~2027) | ✓ (推奨) |
| 拡張性 | 中 | 高 (多倉庫) |
| バーコード/RF | 基本 | 高度 (DAS, モバイル) |
| モバイルアプリ | 制限 | フル |
| 統合 | MM/SD | MM/SD/TM フル |

移行: ECC+WM → S/4HANA+EWM 推奨 (6-12 ヶ月プロジェクト, チーム再教育)。

## よくある問題

| 症状 | 原因 | 診断 | 解決 |
|-----|------|------|-----|
| 入庫遅延 | bin 異常/割当 | LT01 ログ | 保管戦略再設定 |
| 出庫漏れ | LI01N 未確認 | LI04 | 一括確認 |
| バーコードエラー | RF バッテリ/網 | 端末ログ | WiFi 確認, 充電 |
| 在庫不一致 | 未確認入出庫 | LS01N vs 実物 | 循環棚卸 |

## 📊 KPI
- 入庫スループット (LT01 日次件数)
- 出庫精度 (LI01N エラー率, 目標 < 1%)
- 在庫精度 (LS01N vs 実物, 目標 < 0.5%)

## プロセスフロー詳細 (Process Flows)

入庫 (LT01):
```
1. 伝票入力 (PO 参照)
2. 入庫数量入力/スキャン
3. bin 自動提案または手動
4. 確認 → 在庫マスタ即時更新
```

出庫 (LB01 → LI01N):
```
1. SO 参照または手動作成
2. LB01: ピッキング指示 (バーコード)
3. 倉庫: ピッキング + バーコードスキャン確認
4. LI01N: 最終出庫 → FI 在庫減少
```

日常運用:
```
午前: 入庫 (LT01) + ピッキング準備 (LB01)
午後: ピッキング実行 + 出庫 (LI01N)
夕方: 配送業者集荷 + 締め
```

S/4HANA 移行:
```
現在: ECC 6.0 + WM
選択 1: ECC 維持 (~2027 EOL まで) — リスク: 新技術なし
選択 2: S/4HANA + EWM (推奨) — 6-12 ヶ月プロジェクト
```
