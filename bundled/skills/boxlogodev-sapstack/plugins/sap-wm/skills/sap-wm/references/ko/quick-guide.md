# sap-wm 한국어 퀵가이드

## 🔑 환경 인테이크

SAP WM(Warehouse Management) 구현 전, 다음을 반드시 확인하세요:

### 1. SAP 릴리스 및 지원 여부
- **ECC 6.0**: WM 완전 지원
- **S/4HANA**: WM은 **Deprecated** — EWM(Enhanced Warehouse Management)으로 **강제 전환**

### 2. WM 구성 모드 선택
- **Standard WM**: 단순 보관, 소규모 물류센터 (한국 전형적)
- **Lean WM**: 고급 기능 활성화, 대규모 물류센터 (쿠팡/CJ 규모)

### 3. 바코드/RF 장비 구성
- RF 터미널(Radio Frequency) 사용 여부
- 바코드 프린터 및 라벨 정책
- 이동통신 네트워크(LTE/5G) 의존도

### 4. 한국 물류 환경
- **온라인 이커머스 배송**: B2C 일일 1000건 이상 처리
- **B2B 유통**: 일괄 입고/출고
- **반품 센터**: 반품 처리, 품질 검사 필요

## 📚 핵심 T-code 및 역할

### 입고(Inbound) 프로세스
| T-code | 기능 | 담당자 |
|--------|------|--------|
| **LT01** | 입고 등록 (Goods Receipt) | 입고팀 |
| **LT04** | 입고 상태 조회 | 입고팀/감시 |
| **LT03** | 입고 취소/정정 | 입고팀 |

**LT01 상세 흐름**:
```
1. 전표 번호 입력 (구매 발주 PO 참조)
2. 입고 수량 입력 또는 스캔
3. 보관 위치(Bin) 자동 제안 또는 수동 입력
4. 확정(Confirm): 재고 마스터 즉시 반영
```

### 보관(Storage) 프로세스
| T-code | 기능 | 담당자 |
|--------|------|--------|
| **LS01N** | 재고 현황 (Stock Overview) | 창고/감시 |
| **LS01** | 위치별 재고 조회 | 창고팀 |
| **LSMW** | 재고 이동(Transfer) | 창고팀 |

**LS01N 상세**:
```
- 보관 위치별 재고 수량 실시간 조회
- 불량 재고(Damaged) 별도 추적
- 회전율 낮은 상품 경보(Slow Moving)
```

### 출고(Outbound) 프로세스
| T-code | 기능 | 담당자 |
|--------|------|--------|
| **LB01** | 피킹(Picking) 목록 생성 | 영업/배송팀 |
| **LI01N** | 출고 확정 (Goods Issue) | 출고팀 |
| **LI04** | 출고 상태 조회 | 감시/배송팀 |

**LB01 → LI01N 흐름**:
```
1. 배송 오더(SO) 참조 또는 수동 생성
2. LB01: 피킹 작업 지시서 출력 (바코드 포함)
3. 창고팀: 상품 피킹 + 바코드 스캔으로 확정
4. LI01N: 최종 출고 확정 → FI 재고 감소 전기
```

## 🇰🇷 한국 현장 운영 지침

### 온라인 물류센터 일상
- **일 처리량**: 1,000~5,000건 (쿠팡/CJ 규모)
- **작업 프로세스**:
  1. 오전: 입고(LT01) + 피킹 준비(LB01)
  2. 오후: 피킹 실행 + 출고(LI01N)
  3. 저녁: 배송업체 픽업 + 마감 정산
- **WM 시스템 부하**: 물류센터 규모 > 중소기업 일일 처리량

### 바코드/RF 운영
- **바코드 레이블**: 입고 시 자동 인쇄(LT01) → 보관 위치에 부착
- **RF 터미널**: 창고팀이 휴대 — 피킹/출고 실시간 스캔 확정
- **네트워크 신뢰도**: WiFi 신호 약한 지역 → 오프라인 모드 필요 (일부 WM 버전)

### 반품 처리
- **프로세스**: 고객 반품 → 입고(LT01) → 품질 검사 → 재출고(LI01N) or 폐기
- **위치 관리**: 반품 구간 별도 Bin 설정 → LS01N에서 추적
- **한국 이커머스**: 반품율 높음(5-10%) → WM에서 효율적 관리 필수

## ⚠️ WM vs EWM 비교 및 전환 계획

### ECC의 WM은 **S/4HANA에서 사용 불가**

| 항목 | WM (ECC) | EWM (S/4HANA) |
|-----|---------|--------------|
| 지원 | ✓ (ECC 6.0 종료 기한 2027년) | ✓ (권장) |
| 확장성 | 중간 (단일 창고) | 높음 (다중 창고, 풀필먼트센터) |
| 바코드/RF | 기본 지원 | 고급(DAS, 모바일) |
| 모바일 앱 | 제한 | 풀 지원 (SAP Warehouse Management App) |
| 통합 | MM/SD 긴밀 | MM/SD/TM 완전 통합 |

### 한국 물류센터의 S/4HANA 전환 시나리오
```
현재: ECC 6.0 + WM
│
├─ 선택1: ECC 유지 (비용 최소, 2027년 종료까지)
│  └─ 위험: 신기술 미지원, 보안 패치 제한
│
└─ 선택2: S/4HANA + EWM 전환 (권장)
   ├─ 이점: 클라우드 마이그레이션 가능, 고급 AI 예측 (수요 예측)
   ├─ 비용: 전환 프로젝트 6-12개월, 팀 재교육
   └─ 한국 참조: 쿠팡(2021 전환 완료), CJ(RISE 도입 중)
```

## 흔한 이슈 및 해결

| 증상 | 원인 | T-code 진단 | 해결 방법 |
|-----|------|-----------|----------|
| 입고 시간 지연 | Bin 이상 또는 할당 오류 | LT01 로그 | IMG 보관 전략 재설정 (WMIMG_ZONE) |
| 출고 건 누락 | LI01N 미확정 | LI04 현황 | 미확정 건 일괄 확정 |
| 바코드 인식 오류 | RF 터미널 배터리/네트워크 | 터미널 로그 | WiFi 신호 확인, 배터리 충전 |
| 재고 불일치 | 입고/출고 미확인 | LS01N vs 실물 | 재고 조사(Cycle Count) 실행 |

## 📊 모니터링 및 리포팅

### 일간 KPI
- **입고 처리량**: LT01 일일 기록 건수 (목표: 500건/팀)
- **출고 정확도**: LI01N 오류율 (목표: < 1%)
- **재고 정확도**: LS01N vs 실물 차이율 (목표: < 0.5%)

### 주간 보고
- **WM 현황**: 처리량, 에러 건, 미결 항목
- **시스템 헬스**: RF 터미널 가용도, 네트워크 상태
