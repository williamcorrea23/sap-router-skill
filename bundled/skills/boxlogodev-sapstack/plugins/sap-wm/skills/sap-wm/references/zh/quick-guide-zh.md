# sap-wm 快速指南 (简体中文)

## 🔑 环境信息收集

SAP WM (Warehouse Management) 工作前确认:

### 1. SAP 版本与支持
- **ECC 6.0**: WM 完全支持
- **S/4HANA**: WM **已弃用** — 强制转向 EWM

### 2. WM 配置模式
- **Standard WM**: 简单存储, 小型物流中心
- **Lean WM**: 高级功能, 大型物流中心

### 3. 条码/RF 配置
- RF 终端使用
- 条码打印机与标签策略
- 移动网络依赖

### 4. 物流环境
- **电商配送**: B2C 日处理量大
- **B2B 分销**: 批量收发货
- **退货中心**: 退货 + 质检

## 📚 关键 T-code 与角色

### 入库 (Inbound)
| T-code | 功能 |
|--------|------|
| **LT01** | 收货 (Goods Receipt) |
| **LT04** | 收货状态 |
| **LT03** | 收货取消/更正 |

流程: PO 参照 → 数量录入/扫描 → bin 建议 → 确认 (即时库存更新)。

### 存储 (Storage)
| T-code | 功能 |
|--------|------|
| **LS01N** | 库存总览 |
| **LS01** | 按 bin 查库存 |

LS01N: 各 bin 实时库存, 不良品追踪, 滞销预警。

### 出库 (Outbound)
| T-code | 功能 |
|--------|------|
| **LB01** | 创建拣货清单 |
| **LI01N** | 发货 (Goods Issue) |
| **LI04** | 发货状态 |

流程: SO 参照 → LB01 拣货指示 (含条码) → 拣货 + 扫描确认 → LI01N 最终发货 → FI 库存减少。

## 🇨🇳 中国本地化

### 在线物流中心日常
- 日处理量: 大型履约中心
- 上午收货 (LT01) + 拣货准备 (LB01) → 下午拣货 + 发货 (LI01N) → 傍晚承运商揽收 + 收尾

### 条码/RF 运营
- 收货时自动打印条码 → 贴到 bin
- RF 终端: 仓库团队实时拣货/发货扫描
- 网络可靠性: WiFi 弱区可能需离线模式

### 退货处理
- 流程: 客户退货 → 收货 → 质检 → 再发货 or 报废
- 退货区单独 bin, LS01N 追踪
- 电商退货率高 (618/双11 大促后) → 高效 WM 处理必要

## ⚠️ WM vs EWM 与转换

| 项 | WM (ECC) | EWM (S/4HANA) |
|----|---------|--------------|
| 支持 | ✓ (ECC EOL ~2027) | ✓ (推荐) |
| 扩展性 | 中 | 高 (多仓) |
| 条码/RF | 基础 | 高级 (DAS, 移动) |
| 移动应用 | 受限 | 完整 |
| 集成 | MM/SD | MM/SD/TM 完整 |

转换: ECC+WM → S/4HANA+EWM 推荐 (6-12 月项目, 团队再培训)。

## 常见问题

| 症状 | 原因 | 诊断 | 解决 |
|-----|------|------|-----|
| 收货延迟 | bin 异常/分配 | LT01 日志 | 重配存储策略 |
| 漏发货 | LI01N 未确认 | LI04 | 批量确认 |
| 条码错误 | RF 电池/网络 | 终端日志 | 检查 WiFi, 充电 |
| 库存不一致 | 未确认收发货 | LS01N vs 实物 | 循环盘点 |

## 📊 KPI
- 收货吞吐量 (LT01 日计数)
- 发货准确率 (LI01N 错误率, 目标 < 1%)
- 库存准确率 (LS01N vs 实物, 目标 < 0.5%)

## 流程详解 (Process Flows)

入库 (LT01):
```
1. 输入凭证 (PO 参照)
2. 输入/扫描收货数量
3. bin 自动建议或手动
4. 确认 → 库存主数据即时更新
```

出库 (LB01 → LI01N):
```
1. SO 参照或手动创建
2. LB01: 拣货指示 (条码)
3. 仓库: 拣货 + 条码扫描确认
4. LI01N: 最终发货 → FI 库存减少
```

日常运营:
```
上午: 收货 (LT01) + 拣货准备 (LB01)
下午: 拣货执行 + 发货 (LI01N)
傍晚: 承运商揽收 + 收尾
```

S/4HANA 转换:
```
现在: ECC 6.0 + WM
选项 1: 保留 ECC (至 ~2027 EOL) — 风险: 无新技术
选项 2: S/4HANA + EWM (推荐) — 6-12 月项目
```
