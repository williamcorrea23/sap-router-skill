# WM 기간마감 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: EWM 권장

## 개요
월 마감 프로세스에서 창고 관리 담당자가 수행해야 할 정산, 검증 작업 목록.

**주의**: WM은 독립적인 기간 마감이 없고, MM(Materials Management) 기간마감(MMPV)에 종속됩니다.

---

## 기간 마감 프로세스

### Phase 1: Pre-flight 검사 (T-5일)

#### 1. 미처리 Transfer Order 정리
- [ ] **LT06 미결 TO 현황** — T-code: LT06
  - 필터: Status ≠ 'CO'(완료) AND Creation Date ≤ 당월
  - 조치:
    ```
    - 진행 중 TO: 월말 전 완료 추진
    - 미사용 TO(>7일): LT05로 취소 처리
    - 기술적 오류: 담당자 보고 → 해결
    ```

#### 2. WM vs MM 재고 최종 대사
- [ ] **MMBE(MM) vs LX01(WM) 대사** — T-code: MMBE + LX01
  - 필터: 전체 부품 (또는 특정 저장유형)
  - 대사:
    ```
    MM 재고(MMBE):
    - 무제한 저장유형: 1,000개
    
    WM 재고(LX01, 저장유형 합계):
    - 저장유형 01: 150개
    - 저장유형 02: 450개
    - 저장유형 03: 350개
    - 저장유형 04: 50개
    합계: 1,000개 ✓ (일치)
    
    차이 분석:
    - 차이 > 5개: 원인 조사
      * 미확인 TO (LT06)
      * 이동 오류 (LT05)
      * 분실 (MB1C 조정)
    ```

#### 3. 임시 저장 정리
- [ ] **제외 저장유형(Holding area) 정리** — T-code: LX04
  - 목표: 월말에 임시 보관 부품 0개
  - 조치:
    ```
    각 임시 저장유형별:
    - 보관 사유 확인 (결함품? 반품 대기?)
    - 처리 방안 결정 (승인? 폐기? 반품?)
    - 명확한 의사결정 후 이동 또는 처리
    ```

### Phase 2: 월 마감 실행 (T-day)

#### 4. 물리적 재고실사 (Cycle Count)
- [ ] **LI01N / LI01 물리적 재고확인** — T-code: LI01N (S/4) / LI01 (ECC)
  - 범위:
    ```
    옵션 1: 전수실사
    - 모든 저장유형 확인
    - 시간: 1~2일
    - 빈도: 연 1회(월말)
    
    옵션 2: 순환실사(Cycle Count)
    - 월별로 저장유형 로테이션
    - 각 달 20% 정도 확인
    - 빈도: 월 1회 (더 나은 방식)
    ```
  - 프로세스:
    ```
    1. LI01로 실사 문서 생성
    2. 현장 담당자: 물리적 수량 입력
       (RF 단말기 또는 종이)
    3. 시스템: 시스템 수량 vs 물리 수량 비교
    4. 차이 분석: 
       - 재고차이(Inventory Difference) 계산
       - 원인 파악 (분실, 오류, 자연감모)
    5. 조정: MB1A로 차이 이동전표 생성 (양수 또는 음수)
    ```

#### 5. 재고 조정 및 정산
- [ ] **MB1A 재고 조정 이동전표** — T-code: MB1A
  - 조치:
    ```
    실사 차이를 시스템에 반영:
    - 수량 초과(+): 미계상 부품 입고 → 자산 증가
    - 수량 부족(-): 분실 부품 → 원가 손실
    
    예시:
    부품 X: 시스템 100개 vs 물리 95개
    → MB1A로 -5개 이동
    → 원가 계정: 재고손실 계정에 차감
    
    기한: 월말 전 모든 조정 완료
    ```

#### 6. WM 기간 마감 (간접적)
- [ ] **MM 기간 마감(MMPV)과 동시** — T-code: MMPV
  - 조치:
    ```
    WM은 독립적 기간 마감 없음 → MM 마감에 종속
    
    MM MMPV 실행:
    1. 범위: 마감 월 선택 (예: 2026-04)
    2. 포함: 모든 재고 움직임 → Lock
    3. WM 영향:
       - TO 생성 불가 (새로운 이동 금지)
       - LT06 검사/확인 가능 (이미 생성된 것만)
    ```

### Phase 3: 검증 (T+3일)

#### 7. 재고 불일치 최종 검증
- [ ] **MMBE vs 실제 확인** — T-code: MMBE + 현장
  - 확인:
    ```
    월말 실사 이후:
    - 추가 불일치 발생 여부
    - MB1A 조정 완료 여부
    - 시스템 재고 = 물리 재고 (100% 일치 목표)
    ```

#### 8. FI와 재고 대사
- [ ] **FBL5N 미결제 재고 계정** — T-code: FBL5N (FI)
  - 대사:
    ```
    WM 측: 재고 금액 (MMBE × 단가)
    FI 측: T-Account "재고"(예: 121000) 잔액
    
    일치 여부 확인
    차이 있을 경우: 원인 파악 → 조정 입력 (CO72)
    ```

### Phase 4: 마감 확인 (T+5일)

#### 9. WM 데이터 Lock (간접)
- [ ] **MMPV MM 기간 마감 확정**
  - 효과:
    ```
    MM 마감 → WM도 자동으로 Lock됨
    - 월 데이터 수정 불가
    - 선금(후차월): 최소 3일 이내만 허가
    ```

## 특수 상황 처리

### 대량 재고 불일치
```
상황: 실사 결과 부품 A: 시스템 1,000개 vs 물리 950개 (-50개)
원인 분석:
1. LT06 미확인 TO: 40개 (시스템에 반영 안 됨)
2. 분실(자연감모, 도난): 10개

처리:
1. LT06 미확인 TO 40개 정리 → MIGO 확인
2. MB1A로 -10개 이동전표 생성
   → 원가 손실 계정 기록
3. 보안 강화: 고가 부품 출입 통제 강화

월말 결과: 부품 A 잔액 정확성 100% 회복
```

### 실사 담당자 오류
```
상황: 실사 결과 부품 100개 초과 기록 → MB1A +100
원인: 담당자 실수 (이중 계산 또는 품목 혼동)

발견: 차기월 실사에서 이상 감지
대응:
1. 담당자 재교육
2. 이전달 MB1A 역입력 (정정)
3. 재 실사 수행

예방: RF 단말기 사용 → 수동 입력 오류 감소
```

```
- 확인된 원인: 7개 (시스템 오류 등)

- 총 재고 원가: ₩1,234,560,000

```
