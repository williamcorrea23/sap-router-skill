# PP 일간 운영 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
생산 계획 및 실행 단계의 일일 점검 체크리스트. MRP 계획 실행, 생산오더 관리, 실적 입력의 주요 프로세스를 포함합니다.

---

## 체크리스트

### 일간 재고 및 소요량 확인
- [ ] **MD04 재고·소요량 분석** — T-code: MD04
  - 왜: 부품 부족으로 인한 생산 차질 예방, 외주 급발주 조기 포착
  - 빈도: 매일 08:00 경영진 보고 전
  - 세부: 예외 그룹(Exception Group) 필터링 → Red Flag(음수) 부품 우선 처리
  - S/4 TIP: IP(In Progress) 상태 필터로 MRP 실행 결과만 추출

### MRP 실행 및 예외 메시지
- [ ] **MD01N / MD01 MRP 일일 실행** — T-code: MD01N (S/4), MD01 (ECC)
  - 왜: 계획오더 자동 생성, 공급망 안정성 확보
  - 빈도: 06:00-07:00 야간 배치 또는 수동 실행
  - 세부: 예외 메시지 검토(MD04 → 예외 그룹 버튼) → 수급 불일치 근원 파악
  - 주의: ECC는 MD01, S/4는 /SAPAPO/RRP 또는 MD01N

### 생산오더 상태 점검
- [ ] **CO11N 생산 실적 입력** — T-code: CO11N
  - 왜: 실시간 생산 진도 파악, WIP 정확성 유지
  - 빈도: 생산 종료 후 즉시 (보통 매시간)
  - 세부: 작업 완료 여부, 원재료 투입량, 부산물 입력
  - S/4 TIP: Fiori 앱 "확정 및 보고" 사용 권장

### 생산오더 확인 검사
- [ ] **CO13 생산오더 확인(TECO)** — T-code: CO13
  - 왜: 생산 완료 로트 잠금, 원가 정산 준비
  - 빈도: 일일 1회(오후 15:00경)
  - 세부: 미확인 생산오더만 필터링 → TECO 실행
  - 주의: 미확인 상태로 월말 월초 경계 넘지 않도록 주의

### 자동 재고 이동(COGI) 오류 처리
- [ ] **MB1C / MB1B 불량품 처리** — T-code: MB1C
  - 왜: 불량 발생 즉시 시스템 반영, 재고 오염 방지
  - 빈도: 품질 판정 직후
  - 세부: 불량 수량 → 폐기 저장유형 또는 반품 처리
  - ECC: 자동 COGI 오류는 MB90으로 조정, S/4는 오류 자동 포스팅

### 하루마감 전 점검
- [ ] **CO01 생산오더 조회** — T-code: CO01
  - 왜: 미완료 오더, 미확인 오더 파악 → 다음날 우선순위 결정
  - 빈도: 매일 17:00 퇴근 전
  - 세부: 상태별 필터(CRTD=생성, REL=릴리스, PROC=진행중, CNF=확인)
  - 보고: 미완료 목록 생산 주재자에게 회신

## 특수 상황 대응

### 긴급 부품 발주 프로세스
1. MD04에서 음수(적자) 부품 확인
2. 공급업체 선택: MM01 → 구매 데이터탭
3. 긴급오더: ME41(구매요청) 또는 ME23(구매오더)
4. 입고: MIGO 후 MM/PP 협의

### 생산 차질 발생 시
1. CO01로 현황 파악
2. MD04로 원인 부품 확인(예: 외주 지연, 재고 부족)
3. 대체품 검토: STPO(BOM 대체품 조회)
4. 예외 승인 후 임시 계획

## 연계 프로세스

- **MM (재무관리)**: MD04 재고 부족 → 긴급 발주 (ME23)
- **QM (품질)**: CO11N 불량입력 → QA32로 사용결정 대기
- **CO (원가)**: CO11N 실적 → KKAO로 WIP 계산
