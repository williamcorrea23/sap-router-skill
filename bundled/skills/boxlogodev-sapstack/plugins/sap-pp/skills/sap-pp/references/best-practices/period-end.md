# PP 기간마감 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
월 마감 프로세스에서 생산 모듈 담당자가 수행해야 할 정산, 검증 작업 목록.

---

## 기간 마감 프로세스

### Phase 1: Pre-flight 검사 (마감일 T-5일)

#### 1. 미완료 생산오더 점검
- [ ] **CO01 미완료 오더 확인** — T-code: CO01
  - 필터: Status ≠ 'TECO' AND Plant = 현재 공장
  - 조치: 미완료 오더 목록 생산팀장 보고
  - 방침: 월말 전일까지 모든 오더 TECO 처리 또는 차기월 이월 결정

#### 2. COGI 오류 점검
- [ ] **MB90 자동이동 오류** — T-code: MB90
  - 검색: 스탠다드값 'X' = COGI 오류
  - 조치: 불일치 수량 원인 파악 → 수정 이동전표(MB1A) 처리
  - 주의: 월말 마감 전 오류 0건 달성 필수

#### 3. 외주비 선입금 확인
- [ ] **ME2M 외주구매 조회** — T-code: ME2M
  - 범위: 선입금 외주오더(선급금 발생)
  - 확인: 선급금 정산 여부, 미정산 건 마감 포함 결정

### Phase 2: 월 마감 실행 (T-day)

#### 4. 생산오더 기간 마감
- [ ] **KKAO WIP 계산** — T-code: KKAO
  - 목적: 월말 재공품 원가 계산
  - 범위: CO-PA(Controlling-Profitability) 마감과 연계
  - 세부:
    ```
    KKAO 실행 → 표준원가 vs 실제 비용 대사
    차이 분석 → 배치별 조정 항목 식별
    ```
  - 주의: KKAO 전 CO11N 모든 실적 입력 완료 필수

#### 5. 차이 분석
- [ ] **KKS1 차이 분석 보고서** — T-code: KKS1
  - 용도: 생산 비용 편차(가격, 수량, 구성) 분석
  - 보고: 
    - 표준원가 vs 실제원가
    - 원재료 가격 편차
    - 노무비 편차 (시간당 임금 상승)
    - 간접비 배부율 편차
  - 의사결정: 편차 > 임계값(보통 5%)인 경우 사유 기록

#### 6. 계획오더 정산
- [ ] **MD04 → CO41 계획오더 전환** — T-code: MD04 + CO41
  - 조치: 미전환 계획오더 → 생산오더 전환
  - 확인: 계획오더 잔액 = 0 또는 차기월 이월 수량 명확히 기록
  - ECC: CO41 일괄, S/4: /SAPAPO/RRP에서 거래 전환

#### 7. 분할 교활동(Scrapping) 처리
- [ ] **MB1C 불량품 최종 폐기** — T-code: MB1C
  - 범위: 미처리 스크랩 수량 모두 발고 처리
  - 원가: MB1C 시스템에서 자동 원가 감소
  - 보고: 스크랩율 보고서 FI 부서에 제출

### Phase 3: 검증 (T+3일)

#### 8. 재공품 잔액 대사
- [ ] **KKAO 결과 vs FI 미결제** — T-code: KKAO + FB01 (FI)
  - 대사:
    ```
    PP 측: KKAO → 재공품 금액 추출
    FI 측: T-Account "재공품" (예: 122000) 잔액
    차이 분석 → 근원 파악 → 조정 입력
    ```
  - 주의: 월 마감 전 차이 0 달성

#### 9. 원가센터별 생산비 검증
- [ ] **KE24 원가센터 실적** — T-code: KE24
  - 확인: 각 생산라인별 원가센터 비용 집계
  - 용도: 생산 효율성 분석, 라인별 원가 추적
  - 조치: 이상 수치(> 평년 대비 +10%) 원인 파악

#### 10. 표준원가 vs 실제원가
- [ ] **KKAK 표준원가 확인** — T-code: KKAK
  - 검토:
    - 자재 표준원가 유효성
    - 노무 표준시간 적정성
    - 간접비 배율 적절성
  - 차이 해석: 
    - Favorable (실제 < 표준): 효율성 증가
    - Unfavorable (실제 > 표준): 낭비 또는 구조 변화

### Phase 4: 마감 확인 (T+5일)

#### 11. FI와 최종 대사
- [ ] **FBL5N 미결제 대사** — T-code: FBL5N (FI)
  - 범위: 미수금/미지급금 중 PP 관련 항목
  - 확인: 외주비, 간접비 선입금 정산 여부
  - 사인: CFO 최종 승인

#### 12. 월 마감 Lock
- [ ] **MMPV 월 마감 설정** — T-code: MMPV
  - 조치: 현재 월 마감, 차기월 오픈
  - 효과: 현월 데이터 수정 불가, 감사 추적 활성화
  - 예외: 선금(후차월) 허가 시만 수정 가능


## 특수 상황 처리

### 긴급 외주 정산
- ME2M으로 외주비 선입금 조회
- 청구서 미수신 건: 예상 비용 추정 → 월 마감에 포함
- 다음월 정산: 실제 청구액 도착 시 조정

### 부정기 수정 (Post-closing)
- 월 마감 후 오류 발견 시: MM-PP 협의 → 원가공사 (CO72)
- 원가공사 최대 기간: 다음월 중순까지 (감사 기한)
