# PP 거버넌스 Best Practice

## 적용 범위
- ECC: ✓ | S/4HANA: ✓

## 개요
생산 계획 부문의 정책, 승인 체계, 컴플라이언스 항목. 주로 연간 또는 분기별 검토 대상.

---

## 체크리스트

### BOM(Bill of Materials) 변경관리
- [ ] **CS02 ECO(Engineering Change Order) 승인 프로세스** — T-code: CS02
  - 정책: BOM 변경 요청 → 엔지니어링+생산+품질 협의 → 승인 → CS01로 이행
  - 관련 문서: 설계 변경표(Design Change Form)
  - 빈도: 분기별 ECO 진행 현황 보고
  - 세부:
    ```
    예: 부품 사양 변경 (A→B) 필요성
    1. CS02 ECO 생성 → 변경 BOM 검토
    2. 품질+생산 Sign-off → 월 MRP 반영 시점 결정
    3. CS01 ECO 이행 → BOM 자동 갱신
    ```
  - 주의: ECO 이행 날짜 < 생산 스케줄 날짜 (충분한 리드타임 확보)

### 라우팅(Routing) 변경관리
- [ ] **CA02 라우팅 변경 승인** — T-code: CA02
  - 정책: 표준시간 변경 → 생산+원가회계 협의 → 승인
  - 영향: 노무비 변경 → 표준원가 재계산 필요
  - 빈도: 월 1회 라우팅 변경사항 검토
  - 세부:
    ```
    예: 공정 시간 단축 (10시간 → 8시간)
    1. CA02 변경 → 표준시간 수정
    2. KKAK로 표준원가 재계산
    3. BOM 유효기간 설정 (Old~New BOM 동시 존재 기간 최소화)
    ```

### MRP 파라미터 연간 검토
- [ ] **MRP1/MRP2 (MM01 Material Master)** — T-code: MM01
  - 검토 항목:
    - **Lot size**: EOQ(경제주문량) 재계산 필수 (1년 1회)
      ```
      EOQ = √(2×D×S / H)
      D: 년간 수요, S: 주문비용, H: 보관비율
      ```
    - **Lead time (Procurement)**: 공급업체 실제 납기 vs 시스템 설정값 비교
    - **Safety stock**: 수요 변동성 상승 감지 → 증가 검토
    - **Reorder point**: EOQ + 안전재고 합계 검증
  
  - 빈도: 연간 1회(1월 초), 분기별 검토(Q-end)
  - 도구: MM01N/MRP2 탭

### 외주 단가 관리
- [ ] **외주비 연간 협상 및 등록** — T-code: MM04 / PB00
  - 정책: 
    - 주요 외주 부품: 연 1회 단가 협상
    - 변경 이력 추적: PB00 (외주비 Master)에 유효기간 설정
  - 법규 준수: **하도급법** 준수 (공정거래위원회)
    - 외주비 구성 요소 명확화 (원재료+노무+간접비)
    - 단가 인상/인하 사유 서면 기록
    - 부당 수탈 방지 (최소 이익률 보장)
  
  - 빈도: 월 1회 외주 실적비 검증 (ME2M)

### 생산 라인 표준화
- [ ] **BOM/Routing 라인별 표준화 정책** 
  - 목표: 동일 제품 다중 라인 생산 시 표준화 수준 유지
  - 검토:
    - BOM 버전 관리 (Active vs Inactive)
    - 라우팅 결합(Routing combination) 활용 (자동 선택)
    - 라인별 선호 라우팅 설정 (TOR 유효기간)
  - 빈도: 반기 1회

### 역산(Reverse Engineering) 방지
- [ ] **BOM 접근 권한 제한** — Authorization
  - 정책: BOM 수정 권한 → PP 담당 선임자만 (경영진 승인 필수)
  - 시스템: CS00/CS01 T-Code 사용 로그 추적
  - 감사: 분기별 BOM 미인가 변경 확인

### SAP 승인 워크플로우 설정
- [ ] **CS01/CS02 (ECO) 자동화 승인** — BADI/Enhancement 
  - ECC: BADI_CS_CREATE, BADI_CS_RELEASE로 커스텀 승인 로직
  - S/4: Fiori 앱 "Engineering Change Order" 승인 프로세스
  - 예:
    ```
    ECO 생성 → 품질팀 자동 Notify → 품질팀 승인 → 생산팀 Notify
              → 생산팀 승인 → 재무팀 최종 Sign-off
    ```

## 제조 관련 준법성 (Compliance)

### 제조물책임(PL)법
- [ ] BOM 추적 가능성: 부품 Serial Number 기록 (MM)
- [ ] 품질 인증서 보관: 구매 증빙 5년 이상 (MM-PUR)

### 표준원가 공정성
- [ ] 표준원가 > 시장가: 감가상각 위험 검토
- [ ] 표준원가 < 시장가: 저평가 위험 (세무 감시)

## 문제해결 시나리오

### Scenario 1: BOM 오류로 인한 생산 차질
```
증상: CO11N 투입 시 "부품 부족" 오류
원인: BOM 소요량 과대 기록 (예: 1 대신 10으로 입력)
해결: 
  1. MM01로 BOM 검증
  2. CS02로 ECO 생성 → 수정
  3. 영향받은 생산오더(CO01) 재계산
  4. 재료 회수: MB1A (반품)
조치: BOM 변경 프로세스 재교육
```

### Scenario 2: 표준원가 vs 실제원가 큰 차이
```
증상: KKS1 편차 > 20% (Unfavorable)
원인: 
  - 외주비 단가 인상 미반영
  - 노무비 상승 미반영
  - 원재료 가격 급등
해결:
  1. KKAK로 표준원가 재계산
  2. 단가 인상/인하 원인 분석
  3. 재무 보고: 가격 손실 기록 (P&L에 영향)
조치: 분기별 표준원가 갱신 정책 수립
```

## 감사 체크리스트 (Annual Audit)

```
[ ] BOM 샘플 추출(20건) → 생산 실적 비교
    - 소요량 정확성: 100%
    - 부품 유효성: 폐지 부품 0건

[ ] Routing 샘플 추출(10건) → 시간 기록(HR) 비교
    - 표준시간 vs 실제시간 편차: ±10%

[ ] 표준원가 5건 검증
    - 부품 원가: 최근 구매가 기준
    - 노무비: 최근 시간급 기준

[ ] ECO 진행 현황 5건 추적
    - 생산 반영 선행성: 100%

[ ] 외주비 협상 결과 5건
    - 단가 인상 정당성 문서화: 100%

[ ] Authorization 검토
    - BOM/Routing 수정 로그: 승인자 서명 확인

[ ] 컴플라이언스 사항 확인
    - PL법: 부품 추적 5년
    - 표준원가: 공정성 검토
```
