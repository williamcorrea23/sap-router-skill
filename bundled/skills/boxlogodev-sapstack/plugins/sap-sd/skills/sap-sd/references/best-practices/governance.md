## 정책 및 절차

### 1. 가격결정 절차 (연간 검토)

#### 가격정책 수립
- [ ] **T-code V/86 — 가격 조건 정책 검토 (연 1회)**
  - 실행 시점: 연초(1월) 또는 상황 변화 시
  - 참석자: CFO, 영업이사, 가격결정 담당자
  - 검토 항목:
    ```
    1. 고객군별 가격 차등화
       - A등급(대형 거래처): 기본가 × 85-90%
       - B등급(중형 거래처): 기본가 × 92-97%
       - C등급(소형 거래처): 기본가 × 100%
       - 경쟁사 대비 위치
    
    2. 상품별 마진율 검토
       - 주력 상품: 30-40% 마진율 목표
       - 신제품: 초기 5-10% 마진율, 6개월 후 재검토
       - 저마진 상품(loss leader): 연간 1-2개 제한
    
    3. 채널별 할인 정책
       - 직거래: 기본가 기준
       - 대리점: 5-15% 추가 할인
       - 수출: 시장별 맞춤 가격
    
    4. 계절/프로모션 가격
       - 연말 세일: 10-15% 할인
       - 신제품 출시: 런칭 가격 5-10% 프리미엄
    ```
  - 승인: CFO 최종 사인
  - 문서화: 정책 문서 아카이빙 (compliance 증거)

#### 가격조건 설정 및 유지
- [ ] **T-code V/87 (가격조건 설정) — 월간 모니터링**
  - 실행: 매월 초
  - 확인사항:
    - 차월 신규 조건 활성화 완료
    - 만료 조건 사전 연장 또는 폐기
    - 고객별 특별조건(OSA 등) 최신 반영
  - 산출물: 월간 가격 조건 변경사항 보고

### 2. 여신정책 정비 (연간 검토)

#### 여신한도 정책 검토
- [ ] **T-code FD32 / VKM1 — 여신정책 수립 (연 1회)**
  - 실행 시점: 연초 또는 분기 말 (CFO 보고)
  - 참석자: CFO, 신용관리자, 판매이사
  - 검토 항목:
    ```
    1. 고객군별 여신한도 기준
       - AAA등급(우량): 월 매출액 × 3배
       - AA등급(양호): 월 매출액 × 2배
       - A등급(보통): 월 매출액 × 1.5배
       - B등급(주의): 월 매출액 × 1배
       - C등급(제한): 현금 선금제
    
    2. 신용 모니터링 기준
       - 연체율(지난이 비율) > 5%: 경고
       - 연체율 > 15%: 한도 축소 또는 현금 선금제 전환
       - 지속적 연체: 거래 중단
    
    3. 현금 선금제 고객
       - 신용도 미확립: 초기 3개월
       - 대량 주문: 전체 금액의 10% 선금
    
    4. 신용 차단/제한 프로세스
       - 한도 초과: 신용팀 즉시 차단 (VKM1)
       - 차단 해제: CFO 승인
    ```
  - 승인: CFO 최종 사인
  - 문서화: 여신정책 문서 (quarterly 재검토)

#### 신규 거래처 신용 심사
- [ ] **T-code FD01 — 신규 고객 신용등급 결정**
  - 실행: 신규 거래처 등록 시
  - 심사 절차:
    1. 신원 조사: 사업자 등록증 확인, 통신료 조회
    2. 재무 조사: 최근 재무제표(또는 신용 보고서)
    3. 거래 조사: 동종업체 2-3곳 ref 조회
    4. 신용등급 결정: A/B/C 배치
    5. 여신한도 설정: 등급별 기준 적용
  - 승인: 신용관리자 사인
  - 기록: 신용평가서 보관

### 3. SD 계정결정 검토 (분기별)

#### 오더-배송-빌링 계정 정합성
- [ ] **T-code VKOA — SD 계정결정 정기 검토**
  - 실행: 분기 말 (3월, 6월, 9월, 12월)
  - 확인사항:
    ```
    매출(Revenue) 계정:
    - 상품 카테고리별 계정 분류 명확한가?
    - 환율차이/반품 계정이 명시적으로 분리된가?
    - 예: 상품 A 매출 = GL 4001, 반품 = GL 4101
    
    미확인수익(Unbilled Revenue) 계정:
    - PGI 완료 but 빌링 미완료: GL 1401 (자산)
    - 월간 영업일 기준 정확한가?
    
    수익인식 정책:
    - 배송 기준(FOB Shipping Point) 적용되는가?
    - 선급금(Advance) 처리 계정이 별도인가?
    ```
  - 결과: 계정결정 정합성 리포트 (CFO 검토)

### 4. 권한 분리 (Segregation of Duty, SoD)

#### 역할 및 권한 분리
- [ ] **권한 매트릭스 검증 (분기별)**
  - 원칙: **오더 작성 ≠ 배송 확인 ≠ 빌링 발행**
  - 적용:
    ```
    1. 오더 작성(VA01) 권한: 영업팀
       - 매니저 승인 필요(금액 기준: 예 1,000만원 이상)
    
    2. 배송 확인(VL01) 권한: 물류/창고팀
       - 영업팀과 권한 분리 필수
       - 품질 확인 후 PGI 버튼
    
    3. 빌링 발행(VF01) 권한: 청구팀
       - 영업팀, 물류팀과 권한 분리 필수
       - CFO 또는 재무책임자 승인(일정액 이상)
    
    4. 신용 차단(VKM1) 권한: 신용관리팀
       - 영업팀과 권한 분리 필수
       - CFO 해제 승인
    ```
  - 감시: 월간 권한 로그 (T-code SU50) 표본 검사

#### 감사 추적
- [ ] **T-code VV09 / FBL5N — 월간 거래 감시**
  - 대상:
    - VA01 수동 오더 모두 추적
    - VL04 크레딧 메모(반품) 사유 확인
    - VF01 수동 빌링 사유 확인
  - 수행자: 내부감사팀
  - 기록: 감사 로그 유지

### 5. 한국 특화 거버넌스

#### 대리점/특약점 가격정책 관리
- [ ] **거래처별 가격 정책 문서화 (연간)**
  - 실행: 연초 및 정책 변경 시
  - 내용:
    ```
    1. 대리점 할인율 정책
       - Class A (월 5,000만원 이상): 15% 할인
       - Class B (월 2,000만원~): 10% 할인
       - Class C (월 500만원~): 5% 할인
    
    2. 인센티브 정책
       - 분기 목표 달성 시 추가 2% 할인
       - 연간 누적 보너스(차년도 선금)
    
    3. 특약점(Exclusive) 정책
       - 지역 독점권 조건
       - 월 최소 구매액 정의
       - 미달 시 재협상 또는 계약 해지
    ```
  - 문서: 대리점 거래약정서 (법무팀 검토)

#### 가격 인상 정책 (수익 보호)
- [ ] **연간 가격 인상 계획 수립**
  - 실행: 분기별 또는 반기별
  - 절차:
    1. 원가 상승률 분석 (원자재, 인건비)
    2. 시장 경쟁 분석 (경쟁사 가격 추이)
    3. 인상 폭 결정 (보통 3-8%)
    4. 고객군별 사전 공지 (60일 전)
    5. 신규 가격 V/86 설정 및 활성화
  - 승인: CFO + 영업이사

## K-SOX 준수 사항

- **거래 증거 보전**: VA01 수동 오더는 월간 표본 100건 검사, 문서화
- **권한 접근 제어**: SD 권한(VA01, VL01, VF01, VKM1)은 분기별 재검토
- **감시 추적**: T-code SU50로 월간 권한 변경 이력 확인
- **정책 문서화**: 가격정책, 여신정책, SoD 정책은 매년 갱신 및 서명

## 연계 및 외부 규정

- **FI 연계**: 매출 계정(VKOA) 설정은 재무팀과 공동 검토
- **HR 연계**: 영업사원 실적 평가 기준(매출액, 수거율 등)
- **한국 법규**:
  - 부가세법: 전자세금계산서 발행 (익월 10일까지, 필수)
  - 공정거래법: 과도한 반품 강요 금지, 납기 일정 공정성
  - 상법: 거래약정서 작성 및 보관
