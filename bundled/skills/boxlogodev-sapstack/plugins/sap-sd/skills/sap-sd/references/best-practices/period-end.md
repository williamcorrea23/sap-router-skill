# SD 기간말 마감 Best Practice

## 적용 범위
- **ECC**: ✓
- **S/4 HANA**: ✓
- **한국 특화**: ✓ (전자세금계산서 마감, 반품 정책)

## 마감 프로세스

### Phase 1: Pre-Flight (마감 5일 전)

#### 미배송/미빌링 확인
- [ ] **T-code VL06G — 미배송 배송 건 정리**
  - 월말 3일 전부터 실행
  - 확인사항:
    - PGI 예정일이 금월 내인 배송 건 모두 확인
    - 물류팀과 협의해 가능한 건 금월 배송 완료
    - 불가능한 건(고객 요청 연기 등) 명시적으로 차월 이월 기록
  - 산출물: VL06G 현황 보고서 (배송팀 서명)

- [ ] **T-code VF04 — 미발행 청구 건 정리**
  - 월말 3일 전부터 실행
  - 확인사항:
    - PGI 완료(배송 확정) 후 미빌링 건 모두 확인
    - 신용 차단 고객의 미빌링: CFO 승인 후 조건부 빌링
    - 클레임 예정: 명시적으로 차월 연기 결정
  - 산출물: VF04 현황 보고서 (청구팀 서명)

#### 반품 처리 완료 확인
- [ ] **T-code VL04 — 월간 반품 클로징**
  - 월말 2일 전부터 실행
  - 확인사항:
    - 당월 접수 반품 모두 입고(MIGO) 완료 여부
    - 반품 양품 재고 반영 완료
    - 반품 불량품: 폐기/판매 처리 결정
  - 결과: 반품 현황 보고서 (수량, 금액)

#### 가격 조건 유효성 검토
- [ ] **T-code V/86 — 가격조건 유효기간 확인**
  - 월말 1일 전부터 실행
  - 확인사항:
    - 금월 말 만료 조건 목록화
    - 월초 신규 조건 사전 생성/활성화 계획
    - 차월 신규 거래 조건 최종 확인
  - 승인: 가격결정 담당자 사인

### Phase 2: 실행 (마감 당월 말일 또는 다음월 초 2일)

#### VF04 일괄 빌링 실행
- [ ] **T-code VF04 — 월간 빌링 실행**
  - 실행 시점: 월말 오후 또는 차월 초 오전
  - 프로세스:
    1. 빌링 범위 설정: 기간(예: 2026-04-01 ~ 2026-04-30)
    2. 빌링 유형: 정상 청구 (제외: 크레딧 메모 별도)
    3. 일괄 빌링 실행: Report RAAVIS01 또는 수동 VA01
    4. 빌링 문서 생성 확인: VBRK/VBRP 테이블 확인
  - 검증:
    - 생성된 청구 건수 VF04 예상치와 일치
    - 청구금액 합계 매출 예상치와 ±5% 이내
  - 산출물: 일괄 빌링 로그 (RAAVIS01 결과)

#### 배송 완료 (PGI) 확인
- [ ] **T-code VL06G — PGI 완료 여부 최종 확인**
  - 빌링 전 한 번 더 확인
  - 확인사항:
    - 미배송 건이 금월 내 완료되었는가?
    - PGI 날짜가 빌링일 이전인가?
  - 결과: "PGI 완료" 보고서

#### 매출 전표 검증
- [ ] **T-code VV09 / VBRK — 빌링 후 재무 연계 확인**
  - 빌링 실행 후 2시간 이내
  - 확인사항:
    - 청구 건수 = VF04 + 크레딧 메모
    - 청구금액 합계 = 예상 매출액
    - FI 자동 전기(VF31 매출전표): 동시 또는 다음날
  - 미등재 건: 수동 전기(FB01 또는 F-02)

### Phase 3: 검증 (마감 후 2~3일)

#### 매출 대사
- [ ] **T-code FBL5N / VBRK — 매출 대사 (고객별)**
  - 마감 후 1일 이내
  - 프로세스:
    1. 고객별 빌링액 추출 (FBL5N)
    2. 매출 예상치 대비 비교
    3. 오차율 ±3% 이내 확인
  - 큰 차이 시 조사:
    - 환율 변동 (외화 매출)
    - 신용 메모 (반품/클레임)
    - 수익 인식 정책 (선급금 등)

#### 채권 잔액 확인
- [ ] **T-code FD10N — 고객별 채권 현황**
  - 마감 후 1일 이내
  - 확인사항:
    - 금월 신규 채권: 빌링액 합계
    - 회수 채권: 입금액
    - 미결 채권: 차월 이월액
  - 결과: 채권 대사표 (월간)

#### 여신 이용률 검토
- [ ] **T-code VKM1 — 여신 이용률 월간 현황**
  - 마감 후 1일 이내
  - 확인사항:
    - 고객별 미결 채권: 여신 한도 대비 비율
    - 한도 초과 고객 (emergency list)
    - 월간 평균 여신 이용률 (목표: 50~70%)
  - 조치: 한도 초과 고객 신용 축소 또는 한도 상향
  - 산출물: 여신 리포트 (경영진 보고)

### Phase 4: 한국 특화 마감 (월초)

#### 전자세금계산서 발행
- [ ] **T-code J1EX — 전자세금계산서 발행**
  - 발행 기한: 금월 익월 10일 (필수)
  - 프로세스:
    1. 빌링(VBRK) 확정 후 실행
    2. 청구서 정보 자동 매핑
    3. 국세청 EDI 연계 자동 송신
  - 검증:
    - 공급가액 + 세액 = 청구금액 일치
    - 고객 사업자번호 정확성
    - 세율(10% / 0% 면세) 올바름
  - 미발행 건 추적: J1EX 로그

#### 부가세 신고 준비
- [ ] **T-code J1E3 / VV09 — 부가세 과세표준 확인**
  - 월초 3일 이내
  - 확인사항:
    - 당월 공급가액(세제외) 합계
    - 매출세 = 공급가액 × 10%
    - 면세 구간 분류 (수출, 금융 등)
  - 산출물: 부가세 신고용 자료 (세무팀 제출)

## 주요 위험요소

| 위험 | 증상 | 대응 |
|-----|------|------|
| 미배송 배송이 금월 빌링됨 | VL06G 실제 배송 < VF04 빌링 | 빌링 역분개(크레딧 메모) 또는 차월 이월 명시 |
| 청구서 세율 오류 | J1EX 공급가액 ≠ 실제 청구 | 정정세금계산서 재발행 (한국 법정 기한 내) |
| 미결 채권 불확실성 | FD10N 차월 이월 예상과 실제 불일치 | 반품/클레임 즉시 처리, 명시적 기록 |

## 연계 모듈

- **MM (자재관리)**: 반품 입고(MIGO) → 재고 반영
- **FI (재무)**: 빌링(VF04) → 매출 전기(VF31) → 자동 전표 생성
- **CO (원가)**: 고객별 수익성 최종 분석 (월간)
