## CO 기간마감 절차 (CO Period-End Close)

### 📋 Pre-Close 체크 (T-5 ~ T-1일)

#### 미정산 거래 확인
- [ ] **T-code KO02 — 내부오더 조회**
  - 완료되었으나 미정산 오더 목록 추출
  - 정산 조건 확인: 프로젝트 100% 완료, 청구 완료 등
  - 담당: 원가관리
  - 액션: KO88 정산 우선순위 결정

#### 통계수치 최종 검증
- [ ] **T-code KEAT — 통계수치 입력 현황**
  - 월간 필수 통계수치(생산량, 작업시간, 기계시간) 모두 입력 여부 확인
  - 이전 월과 비교 비정상 수치 조사
  - 담당: 통계담당자
  - 액션: 미입력 항목 즉시 입력, 비정상 값 검증

#### 선급금 및 차용금 정리
- [ ] **T-code KO01 — 선급금/차용금 오더 확인**
  - 선급금 오더(KO용 특별오더) 상환 여부 확인
  - 사내 간접비 선급 회수
  - 담당: 재무회계

#### 미배부 거래 조회
- [ ] **T-code 99KO_RECEIVER 또는 SM30(COSC) — 배부 규칙 최신화**
  - 월간 신규 발생 원가센터/내부오더 배부 규칙 등록 여부
  - 배부 규칙 누락 시 배부 전 신규 등록
  - 담당: 시스템관리자

### 🔄 Close 실행 단계 (T일)

#### Step 1: 배부 시뮬레이션
- [ ] **T-code KSU5 — 배부 시뮬레이션**
  - 목표: 실제 배부 전 영향도 최종 검증
  - 실행:
    1. 월간 누적 거래 모두 포함 상태에서 실행
    2. 배부 예상액 vs 예산 대비율 검증
    3. 비정상 항목(음수, 과다 배부) 확인
  - 담당: 원가관리
  - 승인: 원가팀장

#### Step 2: 배부 실행
- [ ] **T-code KSV5 — 배부 실행(Distribution)**
  - 실행 전: Step 1 검증 완료 확인
  - 실행: KSU5 결과와 동일 매개변수로 배부 실행
  - 배부 후:
    - 배부 로그(Log) 확인 (에러/경고 없음)
    - 배부 처리된 거래 수 검증
  - 담당: 원가관리
  - 주의: 배부는 수동 실행 후 배치(BATCH) 등록하여 자동화 가능

#### Step 3: 내부오더 정산
- [ ] **T-code KO88 — 내부오더 정산**
  - 대상: 월내 완료/종료된 오더
  - 정산 규칙: 
    - **세부 정산(Line Item)**: 비용센터/계정별 상세 정산 → 자산 기록
    - **요약 정산(Summary)**: 오더별 합계만 정산 → 비용 처리
  - 실행 절차:
    1. KO02에서 정산 대상 오더 필터링
    2. KO88 정산 시뮬레이션 실행 (T-code KO88, 매개변수 "Simulate" = 1)
    3. 정산 전표 미리보기 검증
    4. 실제 정산 실행 (Simulate = 0 또는 확인 버튼)
  - 담당: 원가관리
  - 결과: 자산/비용 전표가 FI GL에 자동 생성

#### Step 4: CO-PA 배부
- [ ] **T-code KEU5 — CO-PA 배부(Profitability Distribution)**
  - 목표: 판매/마케팅/관리 간접비를 제품/고객/판매채널에 배분
  - 배부 기준: 사전 정의 규칙(예: 판매액 비례)
  - 실행:
    1. KEU5 배부 규칙 선택(예: "Sales Revenue Distribution")
    2. 시뮬레이션 모드 확인
    3. 배부 실행
  - 담당: 경영분석/원가관리
  - 결과: 제품별/고객별 수익률(Margin) 산출

#### Step 5: 원가 기간 마감
- [ ] **T-code OKP1 — 원가 회계 기간 폐기**
  - 상태 변경: "Open" → "Locked" → "Closed"
  - 잠금(Locked): 신규 거래 입력 불가, 정산/배부만 가능
  - 폐기(Closed): 모든 거래 입력/수정 불가
  - 담당: 원가관리자
  - FI와 동기화: OB52(FI)와 동일 시점에 실행

### ✅ Post-Close 검증 (T+1 ~ T+3일)

#### CO 보고서 검증
- [ ] **T-code S_ALR_87013611 — 원가센터 보고서 최종**
  - YTD 누적 비용 vs 예산 대비율 확인
  - 월간(MTD) 신규 비용 항목별 분석
  - 예산 초과 원가센터 원인 분석
  - 담당: 재무담당자

#### CO-PA 보고서 검증
- [ ] **T-code KEA0 또는 KEPM — 수익성 분석 보고서**
  - 제품군별 기여이익(Contribution Margin)
  - 고객별 수익률
  - 판매채널별 profitability
  - 담당: 경영분석

#### FI 대사 (Reconciliation)
- [ ] **T-code KMVV 또는 S_ALR_87012979 — FI/CO 대사**
  - 목표: CO 배부 결과가 FI 손익계산서와 일치 여부 확인
  - 확인항목:
    - CO 원가배부액 = FI 비용 계정 합계
    - CO 매출 = FI 매출 계정 합계
  - 불일치 원인: 시차 거래(예: 미청구입고) 또는 매개변수 오류
  - 담당: 재무회계/원가관리

#### 이월 거래 확인
- [ ] **T-code KMVV — CO 이월 거래 조회**
  - 차기 월로 이월되는 거래(미배부, 미정산) 현황
  - 이월액이 과다하지 않은지 확인(기간마감 품질 지표)
  - 담당: 원가관리

## 주요 위험 시나리오

| 시나리오 | 원인 | 대응 액션 | 심각도 |
|---------|------|---------|--------|
| 배부 금액 과다/과소 | 통계수치 오입력 또는 규칙 오류 | KSU5 재시뮬→규칙 정정→재배부 | 🔴 Critical |
| 정산 후 음수 비용 발생 | 미청구 입고 미정산 | 선급금 추가 정산 | 🟠 High |
| CO-PA 매출 vs FI 차이 | 시차 거래(미청구) | KMVV로 차이 항목 조사 | 🟠 High |
| 기간마감 후 신규 거래 입력 | 기간 상태 설정 오류 | OKP1 기간 상태 재확인 | 🔴 Critical |
| 배부 규칙 미적용 | 신규 원가센터/오더 규칙 누락 | SM30에서 규칙 신규 등록 | 🟠 High |

## S/4 HANA 특수 고려사항

- **New GL (New General Ledger)**: CO 배부가 자동으로 FI 전표 생성 (실시간)
  - 대사 복잡도 증가 가능 → KMVV 이용으로 자동 대사
  
- **기간마감 일정**: OB52(FI) + OKP1(CO) 반드시 동시 실행
  - OB52 먼저 폐기 시 CO 배부 불가 (순서 중요)
