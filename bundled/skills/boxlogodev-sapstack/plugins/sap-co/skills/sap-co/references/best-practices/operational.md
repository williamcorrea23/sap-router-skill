## 체크리스트

### 📊 일간 (Daily) 운영

#### 내부오더 예산 현황 확인
- [ ] **T-code KO01 — 내부오더(Internal Order) 예산 조회**
  - 왜: 오더별 예산 소진율 실시간 추적, 초과 오더 조기 감지
  - 빈도: 일간 (영업 영역별, 오전)
  - 담당: 프로젝트관리/부서장
  - 확인항목:
    - YTD(누적) 실적 vs 예산 대비 소진율
    - 월간(MTD) 신규 발생액
    - 재정자산화(Capitalization) 대상 여부

#### 실시간 CO-PA 모니터링
- [ ] **T-code KEA0 또는 KEPM — 수익성 분석(CO-PA)**
  - 왜: 제품군/고객별 수익성 실시간 추적
  - 빈도: 일간 (영업 종료 후 저녁)
  - 담당: 경영분석/영업관리
  - 확인항목:
    - 변동 수익(매출 - 변동비) 현황
    - 신규 주문 수익률 기대치
    - 손실 프로젝트/고객 조기 식별

#### 원가센터 일일 지출 추적
- [ ] **T-code GCAO 또는 S_ALR_87013611 — 원가센터 보고서**
  - 왜: 이상 거래(예: 월말 집중) 조기 인지
  - 빈도: 일간 (주요 원가센터, 오후)
  - 담당: 재무담당자
  - 확인항목:
    - 어제 대비 금일 누적액 비정상 변화
    - 예산 초과 신호(조기 경고: 90% 도달)

### 📈 주간 (Weekly) 운영

#### 배부 프로세스 시뮬레이션
- [ ] **T-code KSU5 — 배부 시뮬레이션(Simulation)**
  - 왜: 실제 배부 전 영향도 사전 검증
  - 빈도: 주 1회 (목요일, 월 배부 전)
  - 담당: 원가관리/시스템관리자
  - 절차:
    1. 현재 주차 거래 내용 최종 입력 확인
    2. KSU5로 배부 run 시뮬레이션
    3. 예상 원가배부 금액 검증
    4. 이상 금액 발견 시 소스 거래 재확인

#### 배부 주기 실행 및 검증
- [ ] **T-code KSV5 — 배부 실행(Distribution)**
  - 왜: 원가 누적 및 정산의 정확성 보증
  - 빈도: 주 1회 (금요일, 월 누적 예상 시점)
  - 담당: 원가관리
  - 실행후:
    - 배부 로그 확인 (에러/경고 없음)
    - 배부 후 CO-PA 변동 재확인

#### 원가센터 주간 보고서
- [ ] **T-code S_ALR_87013611 — 원가센터 보고서(Cost Center Report)**
  - 왜: 주간 비용 동향 파악, 부서 리더 의사결정 지원
  - 빈도: 주 1회 (월요일 오전, 지난주 데이터)
  - 담당: 재무담당자 / 각 부서장
  - 보고 대상:
    - 비용센터별 YTD/MTD 비용 실적
    - 예산 대비 초과/절감 현황
    - 주간 신규 비용 발생 항목별 분석

#### 미정산 내부오더 목록
- [ ] **T-code KO02 — 내부오더 조회**
  - 왜: 장기 미결 오더 조기 정산, 재정자산화 타이밍 최적화
  - 빈도: 주 1회 (금요일)
  - 담당: 프로젝트관리
  - 확인항목:
    - 90일 이상 미정산 오더 현황
    - 종료 예정 오더 vs 실제 종료 여부 확인

### 📅 월간 (Monthly) 운영

#### 통계 주요수치(Key Figures) 전기
- [ ] **T-code KEAT 또는 KES2 — 통계수치(Statistical Key Figures) 입력**
  - 왜: CO-PA, 원가배부의 기초가 되는 통계수치 실시간 최신화
  - 빈도: 월 1회 (월초 1-3영업일)
  - 담당: 통계담당자(예: 공정/생산부서)
  - 입력 대상:
    - 생산량(LOT 수, 투입량)
    - 작업시간(DH: Direct Hour)
    - 기계 가동시간(MH: Machine Hour)
  - 예시: 신발 제조사 → 월별 shoe 단위 수 입력

#### 월간 원가 정산 검토
- [ ] **T-code KO88 — 내부오더 정산(Settlement)**
  - 왜: 완료된 프로젝트/오더의 비용 최종 배부
  - 빈도: 월 1회 (월말 전 2~3일)
  - 담당: 원가관리
  - 절차:
    1. 종료 조건 만족 오더 확인(100% 완료, 청구 완료 등)
    2. KO88 정산 규칙 검증(규칙: 세부내역/요약 정산)
    3. 정산 시뮬레이션 → 검증 → 실행
  - 결과: 비용 → 재무제표(자산, 비용) 반영

#### CO-PA 배부 주기
- [ ] **T-code KEU5 — CO-PA 배부(Profitability Analysis Distribution)**
  - 왜: 판매/마케팅 비용 제품/고객에 배분
  - 빈도: 월 1회 (월말)
  - 담당: 원가관리/경영분석
  - 정책:
    - 배부 기준: 판매액, 판매수량, 또는 사전 정의된 Key
    - 배부 후 영업이익(EBITDA) 산출 재확인

#### 월말 CO 기간 마감 준비
- [ ] **T-code OKP1 — 원가 회계 기간 상태 조회**
  - 왜: FI 기간마감과 동기화, 이중마감 방지
  - 빈도: 월 1회 (월말)
  - 담당: 원가관리자
  - 확인:
    - 현재 개방 기간 확인
    - 미정산 거래 조회(KO88 미실행 건)
    - FI 마감 일정과 일치 확인

## 주요 위험요소 (Risk Alert)

| 위험요소 | 감지 T-code | 대응 액션 | 심각도 |
|---------|-----------|---------|--------|
| 예산 초과 오더 | KO01 | 부서장 승인 + 추가예산신청 | 🟠 High |
| 미정산 오더 120일 이상 | KO02 | 프로젝트 종료/정산 강행 | 🟠 High |

## 관련 모듈 연계

- **FI (재무회계)**: 월말 원가 배부 → 재무제표 반영 (동기화 필수)
- **MM (자재관리)**: 표준원가 사용 시 가격 차이 분석 (선행조건)
- **SD (판매/배송)**: CO-PA 고객/제품 수익성 분석 (의사결정 지원)
- **PP (생산계획)**: 생산 통계수치(생산량, 작업시간) 공급
