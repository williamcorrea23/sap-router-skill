## 체크리스트

### 🔐 원가배부 내부통제

#### 배부 규칙 정기 검토
- [ ] **T-code SM30 (COSC, CSKS) — 배부 규칙 마스터 정기 재검토**
  - 시기: 연 2회 (상반기 말/하반기 말)
  - 절차:
    1. 현재 배부 규칙(Sender/Receiver) 목록 추출
    2. 조직 개편으로 인한 신규/폐기 센터 반영
    3. 배부 기준(예: 매출액, 근무시간) 적정성 재검토
    4. 규칙 변경 시 효력 날짜 명시(예: 2026-01-01부터)
  - 담당: SAP 시스템관리자 / 원가관리
  - 문서: 배부 규칙 변경 사항 보고서

#### 배부 프로세스 감사
- [ ] **T-code SE38(ZREPORT) 또는 T-code KSU5 로그 — 배부 실행 이력 추적**
  - 월간 배부 실행 기록(KSU5, KSV5) 점검
  - 확인항목:
    - 배부 일시, 실행자, 배부된 거래 건수
    - 에러/경고 발생 여부 및 처리 결과
  - 담당: 감시팀 / 내부감사
  - 빈도: 월 1회
  - 기록: 배부 감사 보고서 보관

#### 정산 권한 분리 (Segregation of Duties)
- [ ] **T-code SU01Z — 내부오더 정산(KO88) 권한 분리**
  - 역할 정의:
    - **입력**: 오더 생성 + 거래 입력 (사용자)
    - **정산 승인**: KO88 정산 실행 권한 (원가팀장)
    - **감시**: KO88 결과 검증 (독립적 감시팀)
  - 주의: 동일 사용자가 생성+정산을 수행하지 않도록 통제
  - 담당: 보안팀
  - 검증빈도: 분기(3개월)

#### 정산 이중 승인
- [ ] **T-code SPRO → IMG → CO → Internal Orders → Settle Interdepartmental Orders**
  - 설정: 일정액 이상(예: 5천만원) 정산은 2인 승인
  - 사례: 자본화(Capitalization) 오더 정산 시 CFO 승인 필수
  - 담당: SAP 구성담당자
  - 검증: SM30에서 정산 규칙 설정 상태 확인

### 💰 CO-PA 거버넌스

#### 수익성 모델 정기 재검토
- [ ] **T-code KEA0 또는 KEPM — CO-PA 설정 정기 점검**
  - 시기: 연 1회 (회계연도 시작 전)
  - 검토 항목:
    - 수익성 차원(예: 제품, 고객, 판매채널) 적정성
    - 배부 기준(예: 판매액 vs 판매수량) 최신화
    - 신규 제품/고객 추가 필요 여부
  - 담당: 경영분석 / CFO
  - 결과: CO-PA 설정 변경 통지서

#### CO-PA 데이터 품질 감시
- [ ] **T-code KEPM — 수익성 분석 보고서 정기 감시**
  - 월간 CO-PA 보고서 검증:
    - 음수 수익률 제품/고객 조사
    - 매출 "0"인 행 조사 (계정 오류 가능성)
    - 매출액이 과도하게 높은 고객 확인
  - 담당: 경영분석
  - 빈도: 월 1회
  - 기록: CO-PA 데이터 품질 점검표

#### 이연 수익 정리
- [ ] **T-code KESR 또는 수동 — CO-PA 이연 정리**
  - 미청구 입고로 인한 이연된 수익 정기 정리
  - 분석: 고객별/제품별 이연 원인 분석
  - 담당: 원가관리 / 재무회계
  - 빈도: 월간 기간마감 시

### 📊 원가센터 관리

#### 원가센터 계층 정기 유지
- [ ] **T-code SM30 → CSKS — 원가센터 마스터 정기 정비**
  - 시기: 연 2회 (조직 개편 후 + 회계연도 초)
  - 항목:
    - 신규/폐기 센터 등록/삭제
    - 센터별 담당자 업데이트(인사부서 연계)
    - 센터 설명/목적 명확화
  - 담당: 인사 / 원가관리
  - 문서: 원가센터 정기보고서

#### 예산 설정 및 모니터링
- [ ] **T-code KP06 또는 KPAB — 예산 편성**
  - 시기: 연 1회 (전년도 11월) + 수정 (분기별)
  - 절차:
    1. 사전 년도 실적 + 인플레율 고려 예산 편성
    2. 경영진 검토 및 승인
    3. SAP에 입력(KP06)
  - 담당: 재무/예산담당자
  - 모니터링: 월간 예산 vs 실적 대비율 10% 미만 목표

#### 예산 초과 승인 프로세스
- [ ] **T-code KO01 — 예산 초과 오더 추적**
  - 정책: 예산 초과 오더는 부서장 재승인 필수
  - 절차:
    1. 시스템 경고: KO01에서 "Over Budget" 플래그
    2. 부서장 승인 증거(이메일/서명) 확보
    3. CFO 최종 승인(액수 기준)
  - 담당: 원가관리 / 부서장
  - 기록: 예산 초과 승인 현황 보고

### 🔍 주기적 감시 (Monitoring)

#### 월간 CO 거버넌스 체크
- [ ] **T-code S_ALR_87013611 — 원가센터 보고서 서명**
  - 월별 원가센터 보고서 최종 발행
  - 부서장별 서명 + CFO 최종 승인
  - 보고서 보관(종이 또는 PDF)
  - 담당: 원가관리 / CFO
  - 빈도: 월 1회 (기간 마감 후 5일)

#### 분기 감사 (Quarterly Audit)
- [ ] **분기말 거버넌스 검토**
  - 항목:
    - 배부 규칙 실행 현황
    - 정산 건수 및 금액 검증
    - 예산 초과 처리 현황
    - CO-PA 데이터 품질
  - 담당: 내부감사 / CFO
  - 문서: 분기 거버넌스 보고서

#### 연간 K-SOX 평가
- [ ] **T-code KMVV + 수동 검증 — 연간 내부통제 평가**
  - 평가 항목:
    - FI/CO 대사 기록(KMVV 결과)
    - 배부 규칙 변경 이력 및 승인 증거
    - 정산 이중 승인 기록
    - 예산 초과 승인 건수 및 사유
  - 담당: CFO / 감사위원회
  - 결과: 내부통제 평가 보고서 (최고 경영진 서명)

## K-SOX 준수 체크사항

- **내부통제 설계**: 배부 규칙 정의 + 정산 승인 구조 문서화
- **운영 효율성**: 월간 배부 로그 검증, 분기 감사 기록
- **재무보고 신뢰성**: FI/CO 월간 대사, TB 불일치 조사
- **기록 보관**: 배부 규칙 변경, 정산 승인 건 최소 7년 보관

## 참고 및 관련 규정

- **배부 기준 공시**: 연간 재무제표 주석에 배부 정책 명시
- **감사인 협력**: 분기/연간 감사 시 배부 규칙 및 기록 제출
- **예산관리 정책**: 예산 편성, 수정, 초과 승인 프로세스 정책 문서화
