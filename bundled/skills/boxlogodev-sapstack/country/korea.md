# CVI KR: 한국 (Korea) 로컬라이제이션

## 개요

| 항목 | 값 |
|------|-----|
| **CVI 코드** | KR |
| **SAP 활성화** | T코드: SFW5 → SWITCH → KR활성화 |
| **회계기준** | K-IFRS (한국채택국제회계기준) 또는 기업회계기준서 |
| **회계연도** | 1월 1일 ~ 12월 31일 (표준, 변경 가능) |
| **기본 통화** | KRW (원, 소수점 없음) |
| **법인세 기준 폐쇄** | 매년 3월 31일까지 신고 |
| **부가세 신고** | 분기별(1월 25일, 4월 25일, 7월 25일, 10월 25일) |
| **감시 규정** | K-SOX, 전자세금계산서(에티콜), 망분리(금융/공공/방산) |
| **급여 지급** | 월 1회 이상 (관례상 말일) |

## 1. 회계 기준 & 폐쇄

### 1.1 회계연도 설정
```
SAP 메뉴 → 재무 관리 (FI) → 기본 데이터 → 회기 설정
T코드: OKP1 (회기 생성) → 회기 변형 정의
```

**한국 표준**:
- 회계연도: 1월 1일 ~ 12월 31일
- 폐쇄 시점: 매년 12월 31일
- 법인세 신고: 매년 3월 31일까지

**분기 설정** (선택사항):
```
회기 1: 1월~3월
회기 2: 4월~6월  
회기 3: 7월~9월
회기 4: 10월~12월
```

### 1.2 K-IFRS vs 기업회계기준서

| 항목 | K-IFRS | 기업회계기준서 |
|------|--------|---------------|
| **대상** | 상장사·금융기관·특수목적기업 | 비상장 대기업·중소기업 |
| **재고평가** | 평균법·선입선출법 (후입선출법 불가) | 평균법·선입선출법·후입선출법 가능 |
| **감가상각** | 정액법 강제 | 정액법·정률법 선택 |
| **감액손실** | 매년 감사 필수 | 필요시만 |
| **SAP 설정** | IMG → FI → Valuation → K-IFRS Rules | 기본 설정 |

**SAP 구성**:
```
T코드: OBD1 (평가 방법)
T코드: OBD4 (감가상각)
T코드: OBALIT (자산 생명주기)
```

## 2. 세무 규정

### 2.1 부가세 (VAT)

**세율 및 분류**:
| 세율 | 코드 | 해당 거래 |
|------|------|---------|
| **10%** | V1 | 일반 재화/용역 (표준세율) |
| **8%** | V2 | 식품(가공), 농산물, 도서, 신문 (경감세율) |
| **0%** | V0 | 수출, 면세 거래 (영세율) |
| **면세** | X0 | 금융·보험·의료·교육·부동산 임대료 |

**SAP 구성**:
```
T코드: FTXP (세금 코드 정의)
  - 세금 코드 1: 부가세 10% (수입세액)
  - 세금 코드 2: 부가세 10% (산출세액)
  - 세금 코드 3: 부가세 0% (영세율 수출)
  - 세금 코드 4: 면세 (금융거래)

T코드: OBCL (세금 계산 절차)
```

**부가세 신고**:
- **신고 기한**: 분기별 마지막 달 25일까지
  - 1분기: 4월 25일
  - 2분기: 7월 25일
  - 3분기: 10월 25일
  - 4분기: 1월 25일 (다음해)
  
- **신고 방법**: 
  - 국세청 홈텍스 (홈페이지)
  - 세무서 방문 (서명 원본)
  - 세무대리인 위탁

**SAP 보고서**:
```
T코드: VOFI (부가세 신고)
또는
T코드: FB51 (부가세 조회)
```

### 2.2 원천세

**주요 원천세 종류**:

| 소득 종류 | 세율 | 신고 기한 | 비고 |
|----------|------|---------|------|
| **사업소득** (용역, 계약금 등) | 3.3% | 지급 후 10일 이내 | 급여 제외 |
| **기타소득** (강의료, 저작권료) | 8.8% | 지급 후 10일 이내 | - |
| **배당소득** | 15.4% | 결산 후 2개월 이내 | 법인세 신고 시 | 
| **이자소득** | 15.4% | 결산 후 2개월 이내 | - |
| **외국인 근로자** | 3.3% | 월 급여 지급 시 | - |

**SAP 구성**:
```
T코드: T001W (원천세 설정)
T코드: FB1B (원천세 코드)
T코드: F110 (원천세 지급 자동화)
```

**원천세 신고 절차**:
1. 월별 원천세 계산 (급여 지급 시)
2. 분기별 원천세 신고 (4월 10일, 7월 10일, 10월 10일, 1월 10일)
3. 연간 "**근로소득 지급명세서**" 제출 (3월 10일)

### 2.3 법인세

**주요 감면**:

- 중소기업 고용 증대 세액공제
- 환경 투자 세액공제
- 지역 산업 육성 세액공제

**SAP 구성**:
```
T코드: OBP6 (법인세 공제 정의)

```

## 3. 전자세금계산서 (에티콜)

### 3.1 개요
- **의무 대상**: 매출 1억 원 이상 사업자
- **발행 기한**: 익월 10일까지 발행 완료 (예: 1월 거래 → 2월 10일까지)
- **보관 기간**: 5년
- **발행 시스템**: 국세청 전자세금계산서 시스템 (홈택스)

### 3.2 SAP 통합

**SAP의 전자세금계산서 인터페이스**:
```
T코드: FTXP → 세금 코드 정의 시
  - 거래일 vs 발행일 구분
  - 순발행/역발행 플래그 설정

T코드: OBA0 → 영수증/청구서 숫자 관리
  - 순번호 자동 생성

Module: FI (재무회계) → 전자세금계산서
  → 자동 생성 또는 수동 검수 후 전송
```

**역발행** (구매자가 발행):
```
거래 유형: 역발행 마크 설정 (건설·물품 매입 등)
SAP 설정: T코드: FTXP → 역발행 코드 (예: R0)
```

**SAP Note**: **3099213** (한국 전자세금계산서 가이드)

## 4. 4대 보험 & 급여

### 4.1 4대 보험 종류

| 보험 | 보험료율 (본인) | 보험료율 (사업자) | 월 납기 |
|------|-----------------|------------------|--------|
| **국민연금** | 4.5% | 4.5% | 10일까지 |
| **건강보험** | ~7% | ~7% | 10일까지 |
| **고용보험** | 0.8% | 1.7~2.2% | 10일까지 |
| **산재보험** | 0% | 0.4~3.5% (업종별) | 10일까지 |

### 4.2 SAP HCM (Human Capital Management) 급여 모듈 설정

```
T코드: SPRO → IMG → Human Resources → Payroll → [국가 KR]
```

**필수 설정 항목**:

1. **회사 코드별 보험 설정** (T코드: PE03)
   - 4대 보험 가입 여부
   - 보험 비율 (월 단위로 변경 반영)

2. **급여 규칙** (T코드: PE04)
   - 기본급, 식사비, 교통비, 상여금
   - 연차 수당 (15일 ~ 25일)
   - 휴가비 충당금

3. **세금 공제** (T코드: PE05)
   - 인적공제: 기본 150만원
   - 배우자공제: 50만원
   - 부양가족공제: 1인당 150만원
   - 신용카드 사용액 공제: 15% (한도)

4. **상여금 설정** (T코드: PE06)
   - 정기상여금 (2회: 1월, 7월)
   - 성과급 (연 1회)
   - 특별상여금

**급여 처리 프로세스**:
```
1. 근태 입력 (T코드: PSAB)
   - 결근, 연차, 반차, 월차 입력

2. 급여 계산 (T코드: HRPC)
   - 기본급 + 수당 - 공제 = 순급

3. 4대 보험 계산 (자동)
   - 국민연금, 건강보험, 고용보험, 산재보험

4. 세금 계산 (자동)
   - 소득세 (근로소득세)
   - 지방소득세 (10%)

5. 급여 지급 (T코드: HR_PAY_REGISTER)
   - 은행 이체 또는 현금 지급

6. 4대 보험료 납부 (T코드: F110)
   - 자동화된 이체 처리

7. 급여명세서 출력 (T코드: HCMT)
   - 근로자별 명세서 제공
```

### 4.3 연차/휴가 관리

**연차 발생**:
- **기본 연차**: 15일 (1년 근속)
- **적립 연차**: 1개월마다 1.25일 (총 15일)
- **추가 연차**: 8년마다 1일 (최대 25일)

**휴가 종류**:
| 휴가 | 일수 | 급여 지급 |
|-----|------|---------|
| 연차 | 15~25일 | O |
| 병가 | 자유 | 회사 규정 |
| 경조사 | 3~7일 | O |
| 보훈 | ~10일 | O (정부 규정) |

**SAP 설정**:
```
T코드: TCODE (휴가 유형)
T코드: VTYPE (휴가 계획)
```

## 5. K-SOX (내부회계관리제도)

### 5.1 개요
- **대상**: 상장사, 감시대상 회사
- **규정**: 자본시장과 금융투자업법 제161조
- **핵심**: 회계 거래의 **완전성·정확성·적시성** 보증 필수

### 5.2 SAP 내 K-SOX 대응

**1. 접근 권한 관리** (SAP GRC)
```
T코드: SUIM (사용자 권한 분석)
  - 필수 분리: 
    - 증빙 생성 vs 승인 권한 분리
    - 입력 vs 수정 권한 분리
    - 지급 vs 감시 권한 분리
```

**2. 감시 추적 (Audit Trail)**
```
T코드: ZLOG (변경 내역 조회)
  - 장부 수정: 원본 기록 유지
  - 삭제 금지: 마킹(삭제 표시)만 허용
  - 타임스탬프 기록: 모든 거래마다

SAP 기본 설정: 
  → 데이터 변경 내역 자동 기록
  → 변경일시/작업자ID/변경내용 저장
```

**3. 원본 증빙**
```
T코드: CV01N (전자 문서 보관)
  - 세금계산서 원본 (PDF)
  - 송장, 영수증, 계약서
  - 보관 기간: 5년
  - 폐기 시 결재권자 승인 필수

Module: SAP DMS (Document Management System)
```

**4. 거래 증명** (완전성)
```
T코드: FBL3N (개별 거래 내역)
  - 모든 거래: 문서번호 + 포증빙 필수
  - 공란 금지: 자동 부여 번호 (예: GJ000001)

T코드: FB03 (증빙 검색)
```

**5. 정보 시스템 통제** (IT General Controls)
```
- 변경 관리: 모든 코드 변경은 CAB 승인 필수
- 백업: 일일 백업 + 월간 복구 테스트
- 접근 제한: 직무 분리 (개발/운영/테스트)
- 로깅: 모든 로그인/거래 기록
```

**SAP Note**: **2142308** (SAP & K-SOX Compliance)

## 6. 망분리 & 보안

### 6.1 망분리 의무 대상
- **금융기관**: 은행, 증권사, 보험사 (필수)
- **공공기관**: 정부부처, 지자체, 공기업 (필수)
- **방산업체**: 국방부 납품 업체 (필수)

### 6.2 망분리 구성

```
┌─────────────────────────────────────────┐
│        SAP 내부망 (업무용)              │
│    (ERP 시스템, 업무 관리)              │
│    - IP: 172.16.0.0/12                  │
│    - DB: On-premise PostgreSQL 또는 Oracle
│    - 접근: VPN + 공인 IP 차단           │
└─────────────────────────────────────────┘
                   ↓
        (방화벽 + 게이트웨이)
                   ↓
┌─────────────────────────────────────────┐
│      인터넷망 (외부용)                   │
│    (홈택스, 은행 시스템)                │
│    - IP: 공인IP (DHCP/Static)           │
│    - VPN: 서명된 인증서                 │
│    - 데이터: 암호화 (AES-256)           │
└─────────────────────────────────────────┘
```

### 6.3 SAP on 망분리 환경

**금지 사항**:
```
❌ SAP → 홈택스 직접 통신 불가
❌ SAP → 은행 시스템 직접 API 불가
❌ RFC 외부 호출 불가
```

**허용 워크어라운드**:
```
✅ Batch Job (배치)로 자료 생성
   → 승인자가 수동으로 홈택스 접속
   → 파일 업로드

✅ 게이트웨이 서버 (DMZ)
   - SAP (내부망) ↔ 게이트웨이 (암호화된 API)
   - 게이트웨이 ↔ 홈택스/은행 (공개망)

✅ MFT (Managed File Transfer)
   - SAP → 파일 생성
   - MFT → 서명 검증 + 암호화
   - 홈택스 연동 시스템 → 파일 수신
```

**SAP 구성 (망분리 고려)**:
```
T코드: SM04 (시스템 간 연결 제한)
  - RFC Destination: 내부 사용만
  - Job Schedule: 특정 시간대만 (예: 오전 9시)

T코드: STMS (Transport System)
  - 내부 개발/테스트/운영 환경 분리
```

**SAP Note**: **2900571** (SAP & Air-Gapped Deployment)

```
T코드: F110 (자동 지급 운영)
```

| 포맷 | 용도 | 상태 |

```
T코드: FBZP (파일 변형)

  - 송금인: 회사 통장 성명 (한글 또는 영문)
```

## 8. 손익계산서 & 재무제표

### 8.1 한국 표준 재무제표 구조

**손익계산서** (분기/연간):

## 11. 자주 묻는 질문 (FAQ)

**Q1. 연중에 회계연도를 변경할 수 있나?**
A: 국세청 승인 필수 (매우 드문 사례). 통상적으로 새 회계연도 시작 시점에만 변경 가능. 변경 시 감사 복잡성 가중.

**Q2. 중소기업인데 K-SOX를 따라야 하나?**
A: 내부회계관리제도는 **상장사 및 감시대상회사**만 의무. 비상장 중소기업은 권장사항이지만, 대출이나 투자 시 신용도 향상에 도움.

**Q3. 부가세를 매월 신고할 수 없나?**
A: 불가. 한국은 **분기별(3개월 단위)** 신고 의무. 다만, 수출업체는 월별 환급 신청 가능.

**Q4. SAP Cloud PE도 한국 CVI를 지원하나?**
A: 네. S/4HANA Cloud PE는 모든 CVI를 지원하며, 자동 업데이트로 최신 법제 반영.

**Q5. 망분리 환경에서 SAP와 홈택스를 연동할 수 없나?**
A: 직접 연동은 보안상 불가. 대신 MFT(관리형 파일 전송) 또는 DMZ 게이트웨이 방식 권장.
