Symptom You convert your system to SAP S/4HANA and then migrate from SAP S/4HANA Grouping, Pegging, Distribution (GPD) to SAP S/4HANA Project Manufacturing Management and Optimization (PMMO).

A side-by-side approach of SAP S/4HANA GPD and PMMO is not possible. You must migrate your data from GPD to PMMO before using PMMO (unless you use a greenfield approach).

## Reason and Prerequisites

GPD is a deprecated component in SAP S/4HANA. This means that it is still supported but will be replaced by the PMMO solution in the long term. The date is not set yet.

This note is relevant if you use GPD in an SAP ECC release or SAP S/4HANA release.

## Solution

## Description

Project Manufacturing Management and Optimization (PMMO) with the application component LO-PMM is planned to replace the solution Grouping, Pegging, Distribution (GPD) with the application component IS-ADEC-GPD.

PMMO is a standard function in the project-based production environment and is a core application component in SAP S/4HANA Enterprise Management. It contains functions that allocate the individual cost types from grouped stock to the WBS elements that drive demand requirements.

## Business Value

The following items in PMMO list the change of existing functions.

## With SAP S/4HANA 2020 release:

PMMO offers multiple improvements compared with GPD, including:

PMMO uses parallel processing in the pegging and distribution programs and code pushdown in the pegging program. Calibration reports are available to aid in determining optimal parameters for the parallel processing.

Many improvements have been made in PMMO to the application log of pegging and distribution programs with error, information and warning messages.

The purchase order item schedule lines are tracked for open orders in PMMO in sync with MRP.

There are material document checks during goods movement for error handling in place of termination messages.

The goods receipt system date is used instead of the document date or posting date on the stock table.

For Stock Transport Order (STO), GPD Pegging could only support a maximum of two plants in case of Type-2 grouping, whereas in PMMO Pegging STO parts are not constrained.

Lower-level granular locking during pegging (the option to not lock during pegging) is also available.

The excess handling logic was improved by assigning excess to the WBS elements in the group that caused the excess.

The purchase order number, item number, material number and plant from the pegging assignment is used to fill the corresponding CO posting record (no customer modification is necessary as in GPD).

The CO posting document number is displayed with a navigation link in the distribution results when executing online.

Redundant data and aggregates in the distribution of actual costs are eliminated.

The following new functionality were added in PMMO:

The pegging table has a new data structure that provides component to top-level product information.

The pegging assignment table includes the 'Stock Ownership Group WBS' field to be consistent with the other pegging tables.

The original goods receipt date is used for the goods issue reversal to maintain FirstIn First-Out (FIFO).

Additional modernization efforts and flexibility options in PMMO include:

Classes and methods are provided to enable customers to incorporate custom code in place of BAdIs in GPD.

There is MRP area support at plant level that is also consistent with SAP S/4HANA.

Support for additional goods movement type processes

## With SAP S/4HANA 2021 release:

PMMO offers multiple improvements compared with GPD, including:

Support for collective order(or direct production) chain in the pegging table where several planned or production orders form a chain (linked) from top leading order on down including multiple branches

Support for blocked and restricted stock depending on the MRP configuration setting

Support for serial number consumption on the component side of the order selecting the replenishment off the stock table that will be consumed on the next higher.

Support for rounding in pegging based on unit of measure.

Materials are tracked at the subcontract vendor level like inventory adjustments and scrapping

During conversion of planned orders to production order, planned order to purchase requisitions, and purchase requisition to purchase orders the pegging records are updated in the pegging table.

Support for vendor down payments by distributing the down payments cleared to received (in stock) quantities and the remaining to the open (in procurement) quantities.

Support of costed option of non-valuated stock gains by creating a production order without a header material number.

PMMO to support MRP configuration include/exclude vendor confirmation and Inbound delivery doc.

The following new functionality are available:

Production order split to move components from parent order to child order.

Additional modernization efforts and flexibility options in PMMO include:

A Fiori report that will provide pegging assignments information with associated distributed costs.

## With SAP S/4HANA 2022 release:

PMMO offers multiple improvements compared with GPD, including:

Configurable option to record pegging assignment history to track any changes during pegging update run and pegging manual changes.

Manually adjust pegging assignment entries.

Prevent distribution of actual costs posting to closed WBS via a configurable option.

Record pegging and distribution update run header data to PMMO_PEG_HEADER and PMMO_DIS_HEADER tables.

The following new functionality is available:

Production order split functionality that enables actual cost distribution for split orders between parent and child in product cost controlling.

Additional modernization efforts and flexibility options in PMMO include:

Current Pegging Assignments Fiori App to analyze the pegging assignment results.

Pegging Assignments History Fiori App to analyze the pegging assignment history as of a certain date and time.

Order Actual Cost Rollup Fiori App to get the true actual cost (direct costs only) for PMMO relevant orders.

Enhancements by using SAP ABAP class events to enhance the standard coding.

## With SAP S/4HANA 2023 release:

PMMO offers multiple improvements compared with GPD, including:

A new configuration will enable customers the ability to create costed stock gain for selective movement type for products that have to be cost relevant or not.

Stock consistency can be checked using PMMO_STOCK_CHECK transaction to compare the PMMO_STOCK table with core stock tables including stock at vendor, and display the differences in ALV format.

In Manual Change pegging, an option to add 'Change Reason' text is available.

The following new functionality are available:

Breakpoint upload through Excel to upload the breakpoints information through an excel instead of creating entries manually through the customizing.

Pegging Assignments History analysis by Time Range report is available to Display the Assignment History between two date ranges.

## Additional modernization efforts in PMMO include:

Cost Distribution Fiori App to analyze the cost posting results for any MRP Element or WBS element or Material.

## Business Process related information

There are three major features within PMMO:

Grouping (provided by Project Systems)

Pegging Assignment

Cost Distribution

The PMMO application is designed for multiple industries and costing methods. One or more of the three features may benefit the customers' business models. The first release is planned for an Engineer-To-Order (ETO) business model that requires all three.

## Pegging

PMMO Pegging data is tightly aligned with MRP by capturing the 'Next Higher Assembly (NHA) Order Number' data and 'Top Level Demand' data on each order assignment record.  When Project Systems is involved, their WBS element numbers will be captured. The pegging records are the cost mapping information for cost distribution.

## Distribution

The assignment records are the mapping information for the cost distribution program to push costs to the appropriate destination.  This is how costs get expensed to the project which is a Non-Valuated (True Actual) costing method.

Costing scenarios that may see PMMO distribute costs:

PMMO-relevant purchase order items from the goods receipt postings and from vendor invoices being posted

PMMO-relevant production order costs, such as labor, overheads, and plant stock material costs (non-valuated, such as material master amounts).  Also, any costs from purchase order items attached to such production orders.

A report-based migration tool is provided as part of PMMO included in SAP S/4HANA. Prerequisite: GPD customers (on ECC / Suite on HANA) planning to deploy PMMO would first need to move the data to SAP S/4HANA GPD tables purely for technical purposes only.

The report can be used to migrate SAP S/4HANA GPD data to the new SAP S/4HANA PMMO pegging and distribution tables.

Customers would be responsible for enhancing the delivered report to migrate custom data fields and tables.

## More information can be found here

After GPD to PMMO Migration (and before any new goods movements transactions are posted), pegging results between GPD and PMMO may differ and could result in cost distribution postings differences. These differences can be anticipated due to the improvements in PMMO pegging compared to GPD pegging.

## 48.3.1 Required and Recommended Action(s)

New functionality and enhancements are planned for the PMMO solution according to the SAP roadmap. GPD in SAP S/4HANA will be in maintenance (until a date yet to be decided).

PMMO features shipped in the SAP S/4HANA releases and features planned for any future release can be found in the Roadmap Explorer. Based on this information, the customer may decide on the relevant SAP S/4HANA release to move from GPD to PMMO, considering their specific business scope.

A Migration report will be provided for moving data from SAP S/4HANA GPD to PMMO.

The functionality provided in PMMO in SAP S/4HANA 2020 release is recommended for blueprinting/template development activities in implementation projects. Additional functionality and enhancements (including comparable scope as GPD) is available with PMMO in SAP S/4HANA 2021 release and onwards, which can be implemented as baseline for productive use depending on customer business requirements.

It is recommended that new SAP customer (greenfield) implementations start with PMMO.
