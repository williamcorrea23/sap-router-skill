## Symptom

You are performing a system conversion to SAP S/4HANA.

## Reason and Prerequisites

This SAP Note is relevant to you if your source release for the system conversion is an ERP release.

It is not relevant if your source release is SAP S/4HANA Finance or SAP S/4HANA.

## Solution

## Description

Packages with development objects that were already obsolete in SAP Reinsurance Management 7.0 are deleted in SAP Reinsurance Management for SAP S/4HANA and in SAP S/4HANA Insurance for reinsurance management.

## Information related to the business process

The impact on business processes depends on the use of the obsolete objects.

## Required actions

Check if the obsolete objects in the object list are used in user-specific source code. If this is the case, you must adjust the user-specific source code.

Some of the deleted programs are replaced by new programs. To do this, you can perform the following steps:

Replace the obsolete program /MSG/VRP_BA_DEL_DTA with the new program: /MSG/VRP_BA_DT_DELETE

Replace the obsolete program /MSG/VRP_BA_DEL_DTA_PP with the new program: /MSG/VRP_BA_DT_DELETE_PP

Replace the obsolete program /MSG/VRP_BA_MOVEMT_PPROC with the new program: /MSG/VRP_BA_MOV_PROC_PP

Replace the obsolete program /MSG/VRP_BA_MOVEMT_PROC with the new program: /MSG/VRP_BA_MOV_PROC

Replace the obsolete program /MSG/VRP_BA_MOVEMT_REWND with the new program: /MSG/VRP_BA_REWIND

Replace the obsolete program /MSG/VRP_BA_CLOSECAP with the new program: /MSG/VRP_BA_CAP_CLOSE

Replace the obsolete program /MSG/VRP_BA_CLOSECAP_PP with the new program: /MSG/VRP_BA_CAP_CLOSE_PP

Replace the obsolete program /MSG/VRP_BA_OPENCAP with the new program: /MSG/VRP_BA_CAP_OPEN

Replace the obsolete program /MSG/VRP_BA_REOPENCAP with the new program: /MSG/VRP_BA_CAP_REOPEN

All other objects in the piece list have been deleted. If you use these in custom code, you must implement your own solution for this.
