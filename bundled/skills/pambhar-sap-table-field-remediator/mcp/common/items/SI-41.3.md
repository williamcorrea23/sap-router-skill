| Business Impact |       3290708 | Impact on TM Location Geo-Coordinate Determination with conversion of SAP ERP or upgrade of S/4HANA any version to S/4HANA OP2023 FPS0 |

## Symptom

The system conversion is planned to S/4HANA 2023 FPS0 or system upgrade is planned from S/4HANA any version to S/4HANA 2023 FPS0.

You use S/4HANA any version to create/update Location.

There is a change in the behaviour for determining the geo-coordinates. As a result, the default geo-coordinate for a location might not be valid after installing the changes for the above-mentioned release.

Reason and Prerequisites Conversion or upgradation to S/4HANA 2023 FPS0.

## Solution

## Description:

Currently, the system picks the strategy from the first entry of the customizing maintained for Geocoding Levels. Please find below the SPRO path for maintaining the Geocoding Levels. With the new behaviour, the system will check if there is a country specific GIS Zone for each Geocoding Level. For a particular GC Level, if the system can find a country specific GIS Zone, then the system determines the corresponding Strategy from the customizing, if not the system will pick the strategy where the GIS Zone is empty. Because of the above change, the locations in customer system need to be checked. If the system finds a location where the current default Geocoding Level & Strategy differ from the default Geocoding Level and Strategy post upgrade, the customer has to fix the location Geo-Coordinates. Alternatively, customer can keep the location Geo-Coordinates as it is.

The customizing for Geocoding Levels is maintained through SPRO->SAP Customizing Implementation Guide-> Transportation Management->Master Data->Transportation Network->Location->Geocoding->Define Geocoding Levels. Alternatively, you can use view /SAPAPO/VLOC_GL to maintain geo coding levels. Please refer table /SAPAPO/LOC_GL for the existing customizing.

The customizing for GIS Zone for a country is maintained through SPRO->SAP Customizing Implementation Guide-> Transportation Management->Master Data>Transportation Network->Transportation Lane->Distance and Duration Determination> Assign Country/Region to GIS Zone. Alternatively, you can use view /SCMB/V_GIS_ZN_A to maintain GIS Zone for a country. Please refer table /SCMB/GIS_ZN_A for the existing customizing.

A simplification item is provided to check this behaviour during the upgrade. The Simplification item will be configured in Software Update Manager (SUM). You run this SUM framework for the simplification item during the upgrade. First decide if it is relevant to check the affected locations and if yes:

A message will be shown with the number of locations that are going to get affected after the upgrade for every system client in case if you choose detailed check.

A generic message will be shown if there are any locations that are going to get affected after the upgrade for every system client in case if you choose non-detailed check.

How to determine check relevancy:

Check customizing in the following path:

SPRO->SAP Customizing Implementation Guide->Transportation Management->Master Data->Transportation Network->Location-> Geocoding->Define Geocoding Levels.

If it has any of the below configuration, it will be relevant for checking the locations with affected geo-coordinates.

If it has single entry and GIS Zone is not initial.

If it has multiple entries.

## Required and Recommended action(s):

You might be getting any of the below messages after the successful run of the simplification item check in SUM.

No action required by you. Continue with the upgrade: In this case, there is no action required by you as mentioned in the message. So, you can go ahead with the upgrade. This can be shown when checking of locations with affected geo-coordinates is not relevant or there is no location that will be affected after upgrade.

Client XXX: Geographical data for locations will be affected. See note 3290708: This is shown when you run the Simplification Item check through SUM framework in non-detailed mode. If you get this message, you have to run the Report  /SAPAPO/UPDATE_GEOCODE_LOC in the particular client mentioned in the message after the upgrade. This report will fix the geo-coordinates of all the affected locations.

Client XXX: Geographical data for XXX locations will be affected. See note 3290708: This is shown when you run the Simplification Item check through SUM framework in detailed mode. It gives you an idea on how many locations will be affected after the upgrade. If you get this message, you have to run the Report /SAPAPO/UPDATE_GEOCODE_LOC in the particular client mentioned in the message after the upgrade. This report will fix the geo-coordinates of all the affected locations.

## How to run Report /SAPAPO/UPDATE_GEOCODE_LOC:

There are two ways to run the report:

In Simulation Mode : In this case, you must check the Simulation Mode checkbox and run the report in background mode. You also need to provide the packet size which denotes how many locations will be processed at a time. The expected packet size is not more than 10000. As the name suggests, the system will run in simulation mode and will not fix the affected locations upon successful execution, a log will be created which you

can check in transaction SLG1 application log by providing the selection parameters along with the object and sub-object mentioned in the output of the report. The SLG1 log will show the list of locations that have been affected after the upgrade and need to be fixed.

In Non-Simulation Mode: In this case, you don't need to check the Simulation Mode checkbox. You can enter the packet size (not more than 10000) and run the report in background mode. It is preferred to schedule a batch job for the report with the variant mentioned above. This will fix all the locations with affected geo-coordinates upon successful execution. A log will be created, and you can check the same in SLG1 application log by providing the selection parameters along with the object and sub-object mentioned in the output of the report. The SLG1 log will show the number of locations that have been successfully updated. It will also show a list of locations that are failed to get updated.
