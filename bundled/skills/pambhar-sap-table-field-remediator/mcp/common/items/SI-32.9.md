## 32.10S4TWL- Deprecation of Project Cost Reports - Overview and Line Items

Application Components:PS-FIO

Related Notes:

| Note Type       |   Note Number | Note Description                                                                                                |
|-----------------|---------------|-----------------------------------------------------------------------------------------------------------------|
| Business Impact |       3331626 | S4TWL - Deprecation of Project Cost Report - Overview (F2513) and Project Cost Report - Line Items (F2538) apps |

## Symptom

You want to use Project Cost Report - Overview (F2513) or Project Cost Report - Line Items (F2538) app.

## Reason and Prerequisites

The Project Cost Report - Overview (F2513) and Project Cost Report - Line Items (F2538) apps will be deprecated in S/4HANA 2023 and will become obsolete at a later point in time.

We recommend you to switch to the successor apps Project Cost Overview (F6991) and Project Cost Line Items (F6992) as soon as possible.

## Solution

## Description

The Project Cost Report - Overview (F2513) and Project Cost Report - Line Items (F2538) apps will be deprecated in S/4HANA 2023 and it is planned to make these apps obsolete in future. However, there is currently no designated timeline for making these apps obsolete. This means, the apps are still available and supported but no new features or enhancements can be expected.

We recommend you to switch to the successor apps Project Cost Overview (F6991) and Project Cost Line Items (F6992) as soon as possible.

## Business process related information

The successor Fiori Apps provide some changed behaviour compared the to previous apps. The following list highlights the most important changes you should consider when switching to the successor apps.

Changes common for Overview and Line Item Report:

## Data Selection:

Single selection for project

Selection of relevant G/L accounts via G/L account hierarchies. No need for semantic tags or Financial Statement Versions for the data selection

Additional accounting objects as filter criteria, e.g. Network, Maintenance Order, Internal Order, Production Order

Support of Virtual Plan Categories in filters for Plan Category and Additional Plan Category

Support of filtering of nonappended order plan costs

Reduced set of filter criteria

## Result Set:

Planned and actual costs of project structure including assigned Networks, Network Activities, header account assigned Maintenance Orders, Internal Orders, Production Orders

Commitments are not included in result set

Values are displayed in global and user-specific currencies only, not in project currency

Additional changes specific to Project Cost Line Items (F6992)

Data Selection:

Single selection for WBS

Additional filter Include WBS Hierarchy controls if only the selected WBS element (i.e. Include WBS Hierarchy = No) or also all child WBS elements (i.e. Include WBS Hierarchy = Yes) are selected

No support of visual filter

Result Set:

Chart view does not visualize cumulated values

## Required and Recommended Actions

Switch to the successor apps.

## How to Determine Relevancy

If you are using Project Cost Report - Overview (F2513) or Project Cost Report - Line Items (F2538) app, this deprecation note is applicable for you.

## Symptom

You are doing a system conversion from SAP ERP to SAP S/4HANA or an upgrade from a lower to a higher SAP S/4HANA release and are using the functionality described in this note. The following SAP S/4HANA Simplification Item is applicable in this case.

## Solution

## Description

Production Resources and Tools functions for projects is part of the SAP S/4HANA compatibility scope, which comes with limited usage rights. For more details on the compatibility scope and it's expiry date and links to further information please refer to SAP note 2269324. In the compatibility matrix attached to SAP note

2269324, Production Resources and Tools functions for projects can be found under the ID 463.

## Business Process related information

Production Resources and Tools (PRT) functions for projects can be used as part of the compatibility scope. This refers only to the assignment of production resources/tools to internal and external activities in networks or standard networks. The usage of PRTs in other scenarios is not affected by this compatibility scope item.

The transactions CN21 and CN22 in the simplification item are only used as an indicator for usage of PRTs, the remaining functionality of these transactions is part of the perpetual scope of SAP S/4HANA.

## Required and Recommended Action(s)

Knowledge Transfer to key- and end user, usage of alternatives to PRTs need to be considered

## 32.13S4TWL - Specific PS archiving transactions

Application Components:PS

Related Notes:

| Note Type       |   Note Number | Note Description                           |
|-----------------|---------------|--------------------------------------------|
| Business Impact |       2270265 | S4TWL - Specific PS archiving transactions |

## Symptom

You are doing a system conversion from SAP ERP to SAP S/4HANA or an upgrade from a lower to a higher SAP S/4HANA release and are using the functionality described in this note. The following SAP S/4HANA Simplification Item is applicable in this case.

## Solution

## Description

Project system specific archiving is part of the SAP S/4HANA compatibility scope, which comes with limited usage rights. For more details on the compatibility scope and it's expiry date and links to further information please refer to SAP note 2269324. In the compatibility matrix attached to SAP note 2269324, Project system specific archiving can be found under the ID 470.

This means that you need to migrate from Project system specific archiving to its designated alternative functionality central archiving (transaction SARA) before expiry of the compatibility pack license. This note shall provide an overview on the scope of Project system specific archiving which is going to expire with the compatibility pack license, how this maps to the designated alternative functionality, if this alternative functionality is already available or when it's planned to be available and how you can migrate to the alternative functionality.

This note will regularly be updated to reflect more details on the alternative functionality and the migration.

## Business Process related information

The following Project system specific archiving transactions are part of the compatibility scope

CN80               Archiving project structures

CN81               PS: Archiving project - preliminary

CN82               PS: Archiving project structures

CN83               PS: Archiving project - Info System

CN84               PS: Archiving project - admin.

CN85               PS: Delete operative structures

CN99               Archiving Standard Networks

KBEA               Trsfr Price Docs: Set   Del.Indicator

KBEB Trsfr Price Docs: Generate Archive

KBEC Trsfr Price Docs: Manage Archive

Going forward it is recommended to use the central archiving transaction SARA instead.

## Required and Recommended Action(s)

As Project system specific archiving can be used until the expiry date of the compatibility pack license, it's up to each customer to decide, when (with which of the coming SAP S/4HANA releases) they want to migrate to the designated alternative functionality. Though the decision, at which point in time this will be done, should be taken early in advance in order to plan accordingly for the migration efforts.

It is recommended to use central archiving transaction SARA instead of project system specific transactions.

Knowledge Transfer to key- and end-user

## Solution

CN04               Edit PS Text Catalog

CN05               Display PS Text Catalog

## Business Process related information

With SAP S/4HANA 2021 FPS1 renewed project texts are available. Renewed PS texts provide a modernized persistency and the Fiori app Project Text for maintenance, while covering similar features like the previous solution.

## Required and Recommended Action(s)

Existing PS texts records can be migrated to the renewed PS texts model by using the report 'CN_PS_TEXT_MIGRATION'. The migration is client-dependent, to activate the renewed PS texts, use IMG node 'Activation of renewed PS Text'.
