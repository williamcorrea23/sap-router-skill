## Symptom

You are performing a system conversion to SAP S/4HANA. In this case, the following entry in the worklist for the transition to SAP S/4HANA applies.

## Reason and Prerequisites

This SAP Note is relevant to you if your source release for the system conversion is an ERP release.

It is not relevant if your source release is SAP S/4HANA Finance or SAP S/4HANA.

## Solution

## Description

In SAP S/4HANA Insurance for reinsurance management, the msg.PM connection is no longer available. It must be replaced by the SAP standard.

## Information related to the business process

The impact on business processes depends on the use of the obsolete objects.

## Required actions

If you were using msg.PM, you must perform the following steps:

You must use the CAIMAN version PM3 or PMQ (see SAP S/4HANA Insurance for reinsurance management -> LRM Configuration Guide, section "msg.PMQ Connection).

To use this version, you must enter the latest version in PM Customizing.

Call transaction /MSG/4_PM_CUSTOMIZE .

Select the application FS-RI-RML-SRA-PM .

Navigate to Configuration -> RFC Destinations -> TCP/IP Connections and select your CAIMAN connection .

Fill the "Program ID" field with the CAIMAN RFC component version (see SAP S/4HANA Insurance for reinsurance management -> LRM Configuration Guide, section "msg.PMQ Connection) .

The customer setting must be made in PM Customizing. This entry is deleted by the conversion.

Call transaction /MSG/4_PM_CUSTOMIZE .

Select the application FS-RI-RML-SRA-PM .

Navigate to Meta Configuration -> Application Customer Setting .

In the "PM" field, enter msg.PM 3.x".

In PM Customizing, check that entries have been made for the request destinations and monitor destinations.

Call transaction /MSG/4_PM_CUSTOMIZE .

Select the application FS-RI-RML-SRA-PM .

Navigate to Configuration -> Request Destinations .

Check the entries:

Application: FS-RI-RML-SRA-PM

RFC Destination: Enter the RFC destination of the PM runtime server.

Choose "Back" and navigate to Configuration -> Monitor Destinations .

Check the entries:

Application: FS-RI-RML-SRA-PM

RFC destination: Enter the RFC destination of the PM runtime server monitor.

In the XML configuration of TomatosX, the pm.rfcnamespace must be changed from /MSG/2 to /MSG/4.

Product data must be re-imported.

Call transaction /MSG/4_PM_CUSTOMIZE .

Select the application FS-RI-RML-SRA-PM .

Navigate to Programs -> Product Library Import .

Enter the following values:

RFC Destination:

Enter your CAIMAN RFC destination

Request/Task: Select a suitable transport request.

Source / File / Export / Workspace / Product Space : Select the product library that you want to import.

Translation : If necessary, select all relevant languages.

Convert Errors to Warnings: If necessary, set the relevant checkmarks.

Logs / Directory : If you want to save a log locally, set the indicator and select a directory.

Choose "Execute Import".

Check the results log and choose "Release" if there are no errors.

If an error occurs during the import of the product data with regard to the positive list, make the following entries in the CAIMAN RFC destination under Logon and Security :

Called function module:        /MSG/4_PM_FETCH_EXPORTINFOBYUP Callback function module:          *

Called function module:        /MSG/4_PM_FETCH_INFORMATION Callback function module:          *
