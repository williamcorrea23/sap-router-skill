## Symptom

You are performing a system conversion to SAP S/4HANA. In this case, the following entry in the worklist for the transition to SAP S/4HANA applies.

## Reason and Prerequisites

This SAP Note is relevant to you if your source release for the system conversion is an ERP release.

It is not relevant if your source release is SAP S/4HANA Finance or SAP S/4HANA.

## Solution

## Description

The "Import of a Data Transmission" background job is no longer available In SAP Reinsurance Management for SAP S/4HANA and in SAP S/4HANA Insurance for reinsurance management. When you navigate to a data transmission, the "LSMW" and "IDocs" tab pages and the import button that are displayed when the data transmission is being processed are also no longer available.

## Information related to the business process

The impact on business processes depends on the use of the obsolete objects.

## Required and recommended action(s)

Check whether you used the obsolete objects. If this is the case, you must implement your own solution for importing a data transmission, if you do not do so already.

54.7 S4TWL - FS-RI - Replacement of a Semantic Object in Fiori App and Transaction

Application Components:FS-RI-LRM, FS-RI

## Related Notes:

| Note Type       |   Note Number | Note Description                                                              |
|-----------------|---------------|-------------------------------------------------------------------------------|
| Business Impact |       2970970 | S4TWL - FS-RI - Replacement of a Semantic Object in Fiori App and Transaction |

## Symptom

You intend to do a release upgrade from SAP S/4HANA 1909 to SAP S/4HANA 2020.

The following SAP S/4HANA 2020 Transition Worklist item is applicable in this case.

## Reason and Prerequisites

This SAP Note is relevant, if the current release is SAP S/4HANA 1909.

## Solution

## Description

In SAP Reinsurance Management for SAP S/4HANA 2020 following objects are not available:

Transaction /MSG/X_V_OPEN_POLICY

Program /MSG/X_V_OPEN_POLICY

semantic object ReinsuranceLifeRiskPolicy

## Business Process related information

The effects on business processes depend on the usage of the obsolete objects.

## Required Action(s)

Check if the obsolete objects, specified in the piece list, are in use. If yes, you must follow these steps:

Replace the obsolete transaction/MSG/X_V_OPEN_POLICY with the new transaction: /MSG/X_V_POLICY_MGMT

Replace the obsolete program /MSG/X_V_OPEN_POLICY with the new program:

/MSG/X_V_POLICY_MGMT

Replace the obsolete semantic object ReinsuranceLifeRiskPolicy with the new semantic object: RMLifePlcyNumber
