## Symptom

You want to convert a system with Add-on OGSD to SAP S/4HANA.

## Reason and Prerequisites

You are using the OGSD Formroutines for Telesales

## Solution

## Description:

Formroutines in Telesales are used to collect data from various sections of the SDprocess and prepare them for display.

## Business related information:

OGSD Formroutines for Telesales allow for customer-own procedures to be integrated into Telesales.

By using Formroutines you may use, read and display data from:

the "Customer History", that is a list of orders and offers relating to a customer, screen inside sales transaction VA01, VA02 etc. meaning: how and where shall data be

Dialogue Settings, these are settings for the screens and subscreens on the OGSDdisplayed?,

Reference Pricing,

Terminal Access Identification Number.

The Formroutines for Telesales are discontinued in SAP S/4HANA, so all coding is lost after a migration to SAP S/4HANA.

## Required and recommended action(s):

If you want to continue using Telesales functionalities currently provided by Formroutines, you need to switch to the Telesales BAdIs functionality.

You need to consider this as a project, as there are no migration tools which may assist you in transfering code of form-routines into BAdI-implementations. All adaptations need to be executed manually. They consist of creating BAdIimplementations and doing some customizing (this is laid out in SAP Note 2458477, see below).

You need to organize knowledge transfer to all users working with Telesales functionalities currently provided by Formroutines as their pattern of work, when it comes to add own functionalities to the Telesales will change.

If you want to migrate the Formroutines you need to check and implement SAP note 2458477 - Elemente zur Ersetzung des Formroutinenwerks im OGSD Telesales  and, as a prerequisite, 2461340 - DDIC-Elemente: Vorausetzung für Hinweis 2458477 .
