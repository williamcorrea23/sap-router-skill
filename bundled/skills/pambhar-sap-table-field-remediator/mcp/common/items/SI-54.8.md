## Symptom

You are performing a system conversion to SAP S/4HANA. In this case, the following entry in the worklist for the transition to SAP S/4HANA applies.

## Reason and Prerequisites

This SAP Note is relevant to you if your source release for the system conversion fulfills one of the following conditions:

ERP system

SAP S/4HANA 1709 (FP 02) to SAP S/4HANA 2020 (Initial Shipment Stack)

## Solution

## Description

Extension Service objects are no longer available in SAP Reinsurance Management for SAP S/4HANA and in SAP S/4HANA Insurance for reinsurance management.

## Information related to the business process

The impact on business processes depends on the use of the obsolete objects.

## Required actions

Check whether you used the obsolete objects. If you did, you must perform the following steps:

During the SPAU and SPDD adjustment, the system may issue warnings stating that an adjustment is required for modified objects. This affects Extension Service structures with the name /MSG/BAPI_ED_* or /MSG/BAPI_EC_*. You can ignore these warnings. You can also reset the modifications. However, this may result in errors and should therefore be avoided.

When the Extension Service is regenerated after the upgrade, these BAPI structures are also regenerated. As a result, the Extension Service is consistent and functioning again.

## Create customer variant

Call transaction /MSG/7FW_START.

Double-click "Workplace" and open the application "10000".

Right-click "10000" and choose "Customer Variant".

Populate the following fields:

Customer Variant

License Key

Storage Package of Classes

Class Prefix

Choose "New".

Save the change and choose a suitable request to do so.

## Choose "Generate".

Call transaction /MSG/7FW_START again and, before choosing "Generate", ensure that all requests that contain the relevant objects have been released.

Call transaction /MSG/2_ES_EXT_SRVC.

Under "Applications", select FS-RI-RML-SRA-PM.

Under "Extendable Tables", select the table /MSG/VDOCLAIM.

Under "Enhancements", select /MSG/ED_OCLM01 and choose "Activate" (Ctrl+F3).

In the dialog box that is displayed, create a request and save it.

Confirm any other dialog boxes that may be displayed. If the system displays a dialog box for the IDocs, choose "No".

If no errors occur, navigate back to transaction /MSG/2_ES_EXT_SRVC and repeat the steps 3 to 6 for the following tables:

/MSG/VDOCOND

/MSG/VDOCONDBASE

/MSG/VDOINSPER

/MSG/VDOLOAD

/MSG/VDOPCSHR

/MSG/VDOPOL

/MSG/VDOPOLCOMP

Navigate back to transaction /MSG/2_ES_EXT_SRVC and select FS-RI-RML-SRA under "Applications".

Repeat the steps 3 to 6 for the following tables:

/MSG/VDOUWASS

/MSG/VDOUWPOLDAT

Note that two enhancements must be generated for /MSG/VDOUWASS.

If errors occur during generation, call the transaction again and generate the enhancement again.

The system may issue the following error messages:

"Unable to load business object element ENTITY"

"Unable to load business object element PERSISTENCE"

"Unable to load business object element CHMODEL"

"Unable to load business object element CTRLMODEL"

## Del-tool for problems with COND

Add the following entry to the table /MSG/0CHECK:

KORVAS-SAP

Call transaction /MSG/2_ES_DEL_TOOL .

Select the entry RCOND01.

Choose "Generate Standard".
