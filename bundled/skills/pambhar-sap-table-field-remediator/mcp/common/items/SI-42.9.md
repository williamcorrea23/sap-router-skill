## Symptom

You are doing a system conversion from SAP ERP to SAP S/4HANA and are using the functionality described in this SAP Note.

To understand if the activity 'Implement the generic DRC Fiori app succeeding the deprecated legal reports' described in Simplification item SI9: GSFIN_SAFT_RO (S4TWL - SAF-T RO: Replacement of ABAP Report Solution with DRC Solution) needs to be performed after the system upgrade, please check if you are using any combination of report and country/region listed in note 2480067

## Solution

## Description

This SAP Note is relevant for you when your starting release for the system conversion is an SAP ERP release or an SAP S/4HANA Finance release, where you have the ERP SAF-T for Romania solution implemented (transactions SAFTN_E & SAFTN_G) in your system.

## Business Process related information

In S/4 Hana, the requirement of generating SAF-T files according to the legal format of Romania is covered by the Document and Reporting Compliance (DRC) Application. For this reason, the SAF-T Romania solution of ERP (transactions SAFTN_E & SAFTN_G) is not to be used anymore and shall be replaced by the corresponding report defined in the DRC app.

There is no need to migrate data. After the transition to S/4 Hana, it is still possible to generate SAF-T files for previous reporting periods using the DRC application, no matter if files have been generated for the same reporting period before the transition or not

## Required and Recommended Action(s)

To use the DRC report for SAF-T Romania, some general configuration settings have to be made, like for example defining the required reporting entities. Refer to the corresponding help document.

The ERP SAF-T solution for Romania offers a monitoring app (transaction SAFTN_M). This transaction is not supported anymore in SAP S/4HANA. If you need to keep the details about extracted data and generated files, we recommend to download the monitoring details before the transition. In addition, we recommend to clean up the extraction data using transaction SAFTN_D before the transition to save database storage space as these entries are not used anymore after the transition to SAP S/4HANA

Please note that with the transition from SAP ERPSAF-T solution for Romania to SAP DRC, some ABAP objects ofpackage /CEECV/RO_SAFT have become obsolete. Therefore, run the ABAP Test Cockpit (ATC) using the latest simplification item DB content and remove all detected usages of the correspondingobjects from your custom code.

In case you have implemented any extension to the selection logic according to note 3237660, check the need for a corresponding extension after the transition. In case you still require an extension to the standard logic, use the available DRC extension techniques to implement them. Find more information on how to do this in the corresponding help document.
