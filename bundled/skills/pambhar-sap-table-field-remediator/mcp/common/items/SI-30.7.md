## Symptom

You are doing a system conversion to SAP S/4HANA. The following SAP S/4HANA Transition Worklist item is applicable in this case.

## Solution

## Context

Standard value calculation with "Computer Aided Process Planning" CAPP is intended to compute standard values and processing durations for production operations based on work center or product properties. CAPP allows to define formulas, methods, and procedures to this end.

In general standard values can be determined in several ways:

Estimating

Polling, sampling, and statistical procedures

Time and motion study (REFA in Germany)

Methods-time measurement MTM

Period time sheets

Standard value calculation with CAPP can be used to compute standard values depending on variables, which can be material attributes, routing attributes, or manually entered variables.

CAPP has been used to adjust standard values to the variant configuration of the manufactured material.

CAPP has been used to add work center or standard text specific surcharges.

CAPP has been used to multiply standard values for duration with a season factor.

CAPP has been used to multiply standard values with product dimensions.

The example given in various documents is the following:

MachineTime = NumberOfDrillings * DrillDepth / DrillSpeed.

Obviously parameters like NumberOfDrillings or DrillDepth are neither material nor routing attributes. Therefore, all parameters would have to be entered manually during production order creation.

## Description

In SAP S/4HANA CAPP is available, but defined as "Functionality Deprecated". Functional equivalent is not available.

## Required and Recommended Action

You can continue to use CAPP after migrating to SAP S/4HANA. It is recommended to consider alternatives (See below).

## Future Direction

CAPP formulas can contain variables, which must by specified by the user. The classic production order UI prompts the user to specify such variables. In future releases SAP intends to provide more user-friendly Fiori UIs for production orders. When using FioriUIs the backend cannot prompt the user to enter data. CAPP cannot be supported in combination with FIORI production order UIs.

S/4HANA processes focus on automatic conversion of planned orders into production orders. CAPP variables cannot be entered during automatic planned order conversion.

In S/4HANA Manufacturing for Production Engineering and Operations PEO, routing operations are detailled by operation activities explaining to production operators the work that has to be done in more detail. All operation activities of one operation are produced without interruption by other operations in the same work center and are subject to the same wage group, business process, and incentive wage policy. Therefore scheduling and costing will still be on operation level but standard value calculation could refer to operation activity data rather than CAPP data. The total duration of an operation for example is the sum of the durations of its operation activities.

SAP advocates open standards. Open standards for estimating standard values are time studies, MTM, REFA, and others. To calculate standard times using one of the established methods reach out to one of the following SAP partners:

https://www.dmc-group.com/dmc-ortim/produkte/arbeitsplanung/capp-knowledge

https://mtm.org/software/mtm-software https://refa-consulting.de/refa-software
