| Business Impact |       3014556 | S4TWL - Deprecation of transaction EA22_TOOL |

## Symptom

You are doing a system conversion or an upgrade to SAP S/4HANA 2020 or higher. You are using functionality of Utilities Meter Reading, Utilities Billing and Utilities Invoicing. The following SAP S/4HANA Transition Worklist item is applicable in this case.

## Solution

## Description

SAP S/4HANA Utilities provides a rich set of transactions and processes for Utilities Meter Reading, Utilities Billing and Utilities Invoicing exposed in the SAP Menu for the Utilities industry.

Beside this functionality, earlier releases of SAP Business suite and SAP S/4HANA provide the transaction EA22_TOOL ("Billingtool") that exposes a set of functionality in the functional utilities areas mentioned above. Due to these overlapping functionalities, transaction EA22_TOOL is not needed and therefore deprecated in SAP S/4HANA.

## Business Process related information

For the overview of all meter reading / billing / print documents and orders, you can use the screen "Billing periods" in transaction ES32 "Display Installation".

| Business Impact |       3326146 | S4TWL - IS-U refund claim (energy price caps in Germany) |

## Symptom

This note is relevant for you under folowing conditions:

You are using SAP for Utilities (IS-U).

As a German Utility in the market role supplier, you are using the SAP standard solution for the German laws for energy price caps according to note 3279875.

## Reason and Prerequisites

The solution for energy price caps in Germany according to the German laws EWPBG and StromPBG is available for SAP S/4HANA 2022 and higher releases.

The length of the fields EPB_REFWERT and EPB_BETRGEW in the database table EPB_REFUND_POS has been changed after the initial delivery. This modification has been delivered with note 3322223. If this note has not been applied to your system, there would be negative impacts during system upgrades that should be avoided.

## Solution

## Description

You should prepare your system for the planned upgrade as described below. After upgrade, there are no more specific actions required in this context.

## Required and recommended actions

If you want to upgrade your system (upgrade to higher support package or enhancement package within SAP ERP 6.0 or upgrade to any SAP S/4HANA release greater or equal 2022), you should consider the following:

Make sure that the solution for energy price caps is supported by the target release you want to upgrade to. Information are provided in note 3279875.

Make sure that note 3322223 is implemented or, precisely expressed, data element EPB_REFWERT has a length of 12. If not, please implement note 3322223 before performing any upgrade.

## Business Process related information

The increased length of data elements EPB_REFWERT and EPB_BETRGEW became necessary to process values larger than initially considered.

The SOA services UtilitiesInteractionCreateRequest_Out and UtilitiesScoreBulkReplicateRequest_Out and the determination of the utilities customer score (BRFplus application ISU_MKT_FINANCIAL_HEALTH_SCORE), available since release SAP S/4HANA 2020, are deprecated and shall not be used anymore on SAP S/4HANA 2023 FPS2 or higher releases.

You are doing an upgrade to SAP S/4HANA 2023 FPS2 or higher. You are using the SOA services  UtilitiesScoreBulkUtilitiesInteractionCreateRequest_Out and ReplicateRequest_Out, the later to create and transfer the utilities customer score. ## Description

The SOA service UtilitiesScoreBulkReplicateRequest_Out contains a special feature. Upon service execution, a utilities customer score for a business partner is determined and sent by this SOA service. This is a utilities specific approach that is only very loosely integrated with FI-CA information and processes. The concept of FI-CA creditworthiness provides higher capabilities and value to the users of SAP S/4HANA Utilities.

The SOA service UtilitiesScoreBulkReplicateRequest_Out and the determination of the utilities customer score (BRFplus application ISU_MKT_FINANCIAL_HEALTH_SCORE) are deprecated. The SOA service UtilitiesInteractionCreateRequest_Out has been specifically designed for the integration of SAP S/4HANA Utilities and SAP Marketing Cloud. It will be not considered for the integration of SAP S/4HANA Utilities with SAP Emarsys or SAP Customer Data Platform. If you are using UtilitiesInteractionCreateRequest_Out for the integration of SAP S/4HANA Utilities with another third party application, you must exchange UtilitiesInteractionCreateRequest_Out either by your own custom specific API or request for an API via SAP Customer Influence.

## 62.14S4TWL - Machine Learning Scenario for Utilities Billing

Application Components:IS-U-BI

Related Notes:

| Note Type       |   Note Number | Note Description                                                             |
|-----------------|---------------|------------------------------------------------------------------------------|
| Business Impact |       3480987 | S4TWL - Performance in IS-U Machine Learning for outsorted billing documents |

## Symptom

The training of the machine learning model for outsorted billing documents (UTI_BI_OUTSORTED) terminates due to problems with the memory requirement or due to runtime reasons.

In the application log for the object ISLM (transaction SLG1), there is an unusually high memory requirement.

This SAP Note changes the machine learning solution for outsorted billing documents to the new model UTI_BI_OUTSRTD_DOC. The previous model is no longer used.

The previous scenario UTI_BI_OUTSORTED and the associated model UTI_BI_OUTSORTED are deleted with release SAP S/4HANA 2025 FPS00.

(SAP Note 3345830 "Machine Learning for IS-U Billing" already mentions possible problems when using the previous model.)

## Reason and Prerequisites

In the previous solution, values from the interim billing were not saved. Determining the statistics is very extensive and leads to a high memory requirement and a long runtime. This SAP Note significantly improves the memory requirement and the runtime behavior. The solution works in a similar way to machine learning for implausible meter readings.

## Solution

## For SAP releases SAP S/4HANA 2023 or lower:

Install APL version APL 4.203 SP 2405. For the availability of the APL version, check the related SAP Note 2631182.

Important: The SAP HANA database must be restarted to register the corresponding C++ libraries.

## Now import the relevant support package.

To use the new machine learning model UTI_BI_OUTSRTD_DOC for outsorted billing documents, proceed as follows:

Set the period for the machine learning application in the maintenance view (transaction SM30) TEML_OUTBILPARAM in the same way as before.

Start the report REML_FILL_OUTBILCHAR ("Determine ML Characteristics for outsorted Billing Documents") to precalculate the characteristic values. Then execute the calculation for the documents that have already been resolved. To do this, select the "Historical Results" checkbox.

The precalculated characteristic values are now stored in the database table TEML_OUTBILCHAR.

Start the Fiori Launchpad. The role SAP_BR_ANALYTICS_SPECIALIST is assigned to you.

Under Analytics, open the Fiori app "Intelligent Scenario Management" with the scenario "UTI_BI_OUTSRTD_DOC".

Train and activate the model.

A corresponding active model version is visible. This completes the machine learning preparation.

Now apply machine learning to outsorted billing documents that have not yet been resolved.

To do this, start the report REML_FILL_OUTBILCHAR with the "Current Results" checkbox set.

The report now calculates the characteristic values and stores them in the database table TEML_OUTBILCHAR. The number of records processed is displayed.

Start the report REML_BILLING_OUTSRTD_RELPRED to calculate the release confidence values in the same way as before.

Note: The report directly uses the new model UTI_BI_OUTSRTD_DOC with the new version. The old model (UTI_BI_OUTSORTED) is not intended to be used.

In the Fiori Launchpad, start the "Resolve Outsorted Billing Documents" app to display the result of the release confidence calculation.

## Other Terms

Outsorted billing, machine learning, IS-U, training
