## Symptom

In SAP S/4HANA, some of the existing legal notes have been replaced by SAP S/4HANA for advanced compliance reporting (ACR) based reports. However, the replacement of reports does not apply to all the countries/regions and reports. Some of the existing country/region - specific reports are still relevant.

To understand if the activity 'Implement the generic ACR Fiori app succeeding the deprecated legal reports' described in Simplification item SI5: GSFIN_LEG_REP (S4TWL - Replacement of legal reports with SAP S/4HANA ACR) needs to be performed after the system upgrade, please check if you are using any combination of report and country/region listed in note 2480067.

## Solution

Advanced Compliance Reporting is a unique and competitive legal reporting solution that allows you to manage your regulatory requirements across the globe. The solution offers features that ensure your complex and diverse legal reporting needs are addressed in uniform, easy to use, and efficient manner. As an all-encompassing global reporting solution, advanced compliance reporting provides a holistic and real time view of your compliance status across countries and empowers you to draw meaningful inferences. One of biggest strengths of advanced compliance reporting is that it offers an extensibility platform to SAP partners and customers to create new legal reports or enhance the existing reports. For more details about advanced compliance reporting and country-specific report features, see the documentation note 2465179 (for OP 1610) and the SAP help portal. In case of any generic questions/feedback, raise an incident in component FI-LOC-SRF-RUN.

## Business Process related information

Scope of SAP ERP, add-on for Polish SAF-T regulatory requirements

"SAP ERP, add-on for Polish SAF-T regulatory requirements 1.0" is a stand-alone solution. It is not related to the SAP standard solution "SAP Starter Solution". For further information on the "SAP Starter Solution" see SAP Note 2304519 SAFT-PL: SAP solution strategy and release dates guidance.

"SAP ERP, add-on for Polish SAF-T regulatory requirements 1.0" is comprised of two parts:

ERP add-on "PLVAT": This solution is responsible for the generation of the SAF-T XML files.

Desktop Application "eTransfer" (also named "PLVAT_DESKTOP"): This application is responsible for the transmission of SAF-T XML files to the Polish tax authorities.

The solution comprises data extraction only from the following SAP ERP and SAP S/4HANA modules:

SAP FI

SAP MM

SAP SD

For more information about this solution, see SAP Note 2351241 - SAF-T Poland: Release Strategy for "SAP Add-on for Polish SAF-T Regulatory Requirements (Private Cloud Edition/On-Premise)".

## SAF-T Strategy: 'SAP Document and Reporting Compliance' future direction

SAP Document and Reporting Compliance is SAP's strategic solution for e-invoicing and statutory reporting. Please read more about SAP's SAF-T Strategy in the SAP Note 3526880.

For Poland, SAP Document and Reporting Compliance already covers PEF/PEPPOL and KSeF. For SAP S/4HANA Cloud Public Edition customers, it also includes SAF-T reporting in Poland (see SAP Note 2926588).

SAP plans to extend SAF-T Poland coverage to SAP S/4HANA Cloud Private Edition and SAP S/4HANA (this is the current state of planning and can be changed by SAP at any point in time without further notice). Currently, SAF-T Poland functionality for these systems is provided via the software component PLVAT with the SAP Add-on for Polish SAF-T Regulatory Requirements.

It is strongly recommended to plan the implementation leveraging SAP Document and Reporting Compliance capabilities, as PLVAT with all software versions ('SAP ERP, add-on f Polish SAF-T rgltr req', 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition', 'SAP add-on for Polish SAF-T reg req,pto', 'Cloud SAP ERP, add-on Polish SAF-T rgltr') will be out of maintenance after Dec. 31, 2027. A list of the already supported compliance tasks, including reports for Poland in SAP Document and Reporting Compliance for SAP S/4HANA, can be found on the SAP Help Portal.

The future functionality planned for Q1 and Q2 2025 for SAP Document and Reporting Compliance and SAF-T reporting in Poland can be found on the SAP Road Map Explorer.

Information on SAF-T Poland: Changes in accounting file JPK_KR (extended JPK_KR_PD file and JPK_ST_KR file with a new structure for fixed assets) as of 2025 can be found in the regulatory change manager.

The scope of the alternative functionality 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition'

The solution 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition' has the same functionality as the current solution including the desktop application 'eTransfer'.

Customers migrating to SAP S/4HANA Cloud Private Edition need to convert their SAFT add-on license to 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition'. This license is off the pricelist and will be closed after Q4 2024.

PLVAT with all software versions ('SAP ERP, add-on f Polish SAF-T rgltr req', 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition', 'SAP add-on for Polish SAF-T reg req,pto', 'Cloud SAP ERP, add-on Polish SAF-T rgltr') will be out of maintenance after Dec. 31, 2027. Until that date, SAP plans to provide legal changes and maintenance for the 'SAP Add-on for Polish SAF-T Regulatory Requirements' for SAP ERP and PCE solutions (this is current state of planning and can be changed by SAP at any point in time without further notice).

Please read SAP Note 3526880.

The scope of the alternative functionality ' SAP Starter Solution'

The 'SAP Starter Solution' comprises SAP ERP Finance and Logistics data.

It covers two stages of the end-to-end process

Settings for Data Extraction (definition of relevant parameters that control the data extraction)

Data extraction (collection of relevant transactional data needed for file creation)

Note that the file creation, and file submission step is not covered in the 'SAP Starter Solution'. For this reason, the logic required for creating the XML file will be within customer's responsibility.

For guidelines about using 'SAP Starter Solution' , please refer to SAP Note 2304519 SAFT-PL: SAP solution strategy and release dates guidance. This note will also be regularly updated regarding any future roadmap decisions.

## Required and Recommended Action(s)

As SAP ERP, add-on for Polish SAF-T regulatory requirements can be used until the expiry date of the compatibility pack license (read more in SAP note 2269324), it is up to each customer to decide, with which of the coming SAP S/4HANA releases they want to switch to the designated alternative functionality of another SAP solution or to switch to an alternative partner solution.

Depending on your future landscape, you have the following options concerning SAP solutions:

S/4HANA on-premise: "SAP Document and Reporting Compliance", 'SAP Starter Solution'

S/4HANA Cloud Private Edition: "SAP Document and Reporting Compliance, 'SAP Starter Solution', 'SAP add-on for Polish SAF-T regulatory requirements, private cloud edition' (deprecated, end of maintenance after Dec. 31, 2027)

S/4HANA Cloud Public Edition: 'SAP Document and Reporting Compliance'

Sufficient switching time should be considered.

For guideline about uninstalling SAP ERP, add-on for Polish SAF-T regulatory requirements, please refer to SAP Note 2869393 - Uninstalling ABAP Add-On PLVAT.
