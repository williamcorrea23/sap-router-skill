## Symptom

You use Convergent Invoicing (FI-CA-INV) and are upgrading to SAP S/4HANA release 2023 or a later release from SAP S/4HANA release 2022 or earlier.

During this upgrade, the data types of the billing document number and invoicing document number have been changed from data type NUMC length 12 to data type CHAR length 12.

This change is a technical prerequisite for a potential future use of alphanumeric document numbers in Convergent Invoicing.

## Reason and Prerequisites

You use FI-CA-INV and are upgrading to SAP S/4HANA release 2023 or a later release from SAP S/4HANA release 2022 or earlier.

Task : Check that Convergent Invoicing is active in your system.

Procedure: Go to the implementation guide using transaction FICAIMG --> Convergent Invoicing --> Invoicing --> Define Invoicing Process. If there is a invoicing process configured, then this SAP Note is relevant for you.

## Changes to Customer Coding Required

The domain change has several technical consequences. This note contains a detailed list of the different technical consequences of the type change, and how you must adapt your coding. The two following aspects illustrate the necessary code changes:

## Checks that a Billing/Invoicing Document Number is Assigned - IS INITIAL Checks

The ABAP statements "IS INITIAL" and "IS NOT INITIAL" can no longer be used to check if fields of type BILLDOCNO_KK or INVDOCNO_KK are empty.

The previous initial value of billing/invoicing document number was '000000000000', while the new initial value will be 'space'. Since the existing database entries containing the old initial value are not updated, either the old or the new initial value can occur and must be checked.

## Solution

## Business-Process Related Information

The change of the data type of the billing and invoicing document number has no business-related impact.

## 44.1.1 Required and Recommended Actions

To avoid any errors in the system behaviour due to the change of the data types, you need to adapt your customer coding for this change. The adaption of the initial value check needs to be carried out.

## What to Do Before the Upgrade

Please ensure that you update the check logic to the most recent version by implementing all relevant SAP Notes from SAP Note 2436688,  otherwise you might get wrong or missing check results.

Whether you are using the initial check or select statement, please investigate if you have set BILLDOCNO or INVDOCNO to 'empty' in your customer coding. The change of the underlying technical domain in the ABAP dictionary could lead to changed behavior when moving billing/invoicing document numbers into different types. The billing and invoicing document header store references to subsequent documents in fields such as REVERSEDDOC, REVERSALDOC, ADJUSTMENTDOC, ADJUSTEDDOC.

Besides BILLDOCNO and INVDOCNO, you must investigate the use of these fields in your custom coding as well.

As described before, the initial value of the fields BILLDOCNO and INVDOCNO will be 'space' rather than '000000000000' for new documents. However, old values on the database will not be updated during the upgrade process. This means that the code checking whether the fields BILLDOCNO and INVDOCNO are empty or not, needs to check for both values.
