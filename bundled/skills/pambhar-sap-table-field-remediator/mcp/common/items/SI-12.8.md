| Business Impact |       3572033 | S4TWL - Deprecation of classic profit center accounting (EC-PCA) |

## Symptom

You are doing a system conversion from SAP ERP to SAP S/4HANA or an upgrade from a lower to a higher SAP S/4HANA release and are using the functionality described in this note. The following SAP S/4HANA Simplification Item is applicable in this case.

## Solution

## Description

Actual postings for classical Profit Center Accounting (PCA), using GLPCA and GLPCT, have been adopted to S/4HANA product standards and, therefore, are available as part of perpetual scope in SAP S/4HANA and SAP S/4HANA Cloud, private edition. This means that PCA actual postings are no longer part of the compatibility scope. Therefore, you can fully use actual postings for classical Profit Center Accounting in SAP S/4HANA without restrictions.

However, classical PCA for actuals has been set to 'deprecated' because this concept is outdated and not supporting our SAP S/4HANA Finance strategy. In the deprecated phase, customers can still use actual postings in PCA and will get support, but PCA will not receive any further developments or updates and will also not be integrated with new Finance solutions such as Universal Parallel Accounting or AIdriven innovations.

## Business Process related information

Classic PCA in SAP S/4HANA is active if the active indicator is set in transaction 0KE5 (field PCRCH for the corresponding controlling area in the table TKA00 is set to '2') and the online posting of data records in the tables (GLPCA = actual line items table, GLPCT = totals table, GLPCP = plan line items table) has been activated using transaction 1KEF.

Going forward with SAP S/4HANA, SAP follows a separate product strategy for actual postings and for planning in EC-PCA.

Profit Center Planning based on table GLPCP is in compatibility scope and must not be used beyond they expiry date of the compatibility package usage rights. Please refer to SAP note 2993220.

## Required and Recommended Action(s)

SAP strongly recommends transitioning to Profit Center Accounting on Universal Journal after completion of your next fiscal year.

In SAP S/4HANA, PCA is mapped in the universal journal by default. This means, if you still use classical PCA, Profit Center Accounting in the universal journal is also always active in parallel. To stop using classical PCA you need to deactivate it as described in note 702854.

Refer to SAP note 2425255 - Profit Center Accounting in the universal journal in SAP S/4HANA, on-premise edition for more information. In this note, you also find a general description of

how PCA is mapped in the context of SAP S/4HANA and

to what extent classical PCA can be used in parallel with SAP S/4HANA.

what changes if classical PCA is deactivated in SAP S/4HANA
