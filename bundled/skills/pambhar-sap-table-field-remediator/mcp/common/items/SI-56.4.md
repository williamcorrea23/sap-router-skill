## Symptom

You are doing a system conversion of ACM (Agricultural Contract Management) from SAP ECC(ACCGO 400) to SAP S/4HANA on-premise edition 2020 FPS01 or a higher release.

## Reason and Prerequisites

Application document process changed with respect to batch managed material

Going forward from the S4HANA release 2020 FPS01 the Application document will not be split on the basis of the batches mentioned on the Load Data Capture(LDC) if Feature switch (MERGE_APP)( Logistics - General under Global Trade Management -> Agricultural Contract Management -> Basic Settings -> Activate Feature Switches .) is checked.

## Solution

Recommended action:Its recommended to check all the active application documents which have been split due to more than one batch on the LDC and complete those transactions before performing the converison to S4HANA 2020 FPS01 or higher release.

If already converted to Higher release Activate the feature switch Feature switch (MERGE_APP)( Logistics - General under Global Trade Management -> Agricultural Contract Management -> Basic Settings -> Activate Feature Switches .) then use the transaction code /ACCGO/ACM_BSPL_CORR to correct the Application document by performing the Correction of the Same LDC Document and form the New sets of the  logistics document.
