| Business Impact |       2964772 | S4TWL - FS-RI deletion of obsolete functionality for FI-AR |

## Symptom

You are doing a system conversion to SAP S/4HANA. The following SAP S/4HANA Transition Worklist item is applicable in this case.

## Reason and Prerequisites

This SAP Note is relevant for you when your start release for the system conversion is an ERP release.

It is not relevant if your start release is S/4HANA Finance or S/4HANA.

Solution

Description In SAP Reinsurance Management for SAP S/4HANA and in SAP S/4HANA Insurance for reinsurance management, obsolete FS-RI functionality objects for FI-AR are deleted.

## Business Process related information

Following business processes are no longer available:

Goto Accounts Receivable (Transaction /MSG/RFDMN)

Transfer Posting (Transaction /MSG/R_AU)

Cash Loss/Noted Item (Transaction /MSG/R_AS and /MSG/R_AS1)

Broker View of Line Items (Transaction /MSG/R_KM)

For Goto Accounts Receivable use Transaction RFDMN instead of /MSG/RFDMN. For Transfer Posting use Transaction /MSG/R_AU1 instead of /MSG/R_AU.

Effects on customer business processes depends on the usage of obsolete objects in custom code.

## Required and Recommended Action(s)

Check if the obsolete objects in the piece list are used in custom code. If yes, you must adapt the custom code.

## Symptom

You are doing a system conversion from SAP ERP to SAP S/4HANA or an upgrade from a lower to a higher SAP S/4HANA release and are using the functionality described in this Note.

The following SAP S/4HANA Simplification Item SI8: GSFIN_Insurance_TR is applicable in this case.

## Reason and Prerequisites

Prerequisite: Insurance Localization FS-CM for Turkey installed.

Reason for deprecation:

TRAMER functionality could be used to export automotive insurance related information in a specific format as XML in order to submit it to authorities in Turkey.

the functionality is not usable anymore without customer modification, as the export format defined by the Turkish government has changed since initial delivery of the functionality, but it was never adapted to the new format.

hence today customers already use other means to export this data from the system in the required format (eg. export to external text file and perform mapping to a suitable XML format with XSLT or via 3 rd party, own code…).

## Solution

## Description

Package /GSINS/TR and it contents becomes obsolete because of no customer usage. Objects stored in the package and corresponding subpackages are removed including their references from other objects.

TRAMER functionality has already been put on the blocklist and hence will not be usable anymore without intervention in S/4HANA OP2022 release.

## Business Process related information

Insurance localization for Turkey stored in the package /GSINS/TR and it's subpackages is not being used and has been deprecated in S/4HANA OP2022 and became obsolete in all the supported lower versions.

Regarding S/4HANA OP2022 release, for transition period of 2 years the package and it's subpackages has been moved to SW component 'S4DEPREC' if the restoration of the functionality is needed.
