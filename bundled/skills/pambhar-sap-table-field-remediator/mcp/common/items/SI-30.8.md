## Symptom

You are doing a system conversion to SAP S/4HANA, on-premise edition. The following SAP S/4HANA Transition Worklist item is applicable in this case.

## Solution

## Description

Starting with SAP R/3 4.6 C, the previously used ABAP-list-based process instruction sheets have been replaced by browser-based PI sheets (see SAP Note 397504).

The functionality of the ABAP-list-based PI sheets has been partly deactivated and underlying code is only partially available in SAP S/4HANA on-premise.

ABAP-list-based process instruction sheets will be carved out fully in a future release.

With SAP S/4HANA 2023, SAP delivered browser-based Execution Sheets (see SAP user assistance).

They support an easier system conversion to SAP S/4HANA because you can reuse your investments in SAP PP-PI process management application content.

Like XStep-based process instructions and Standard XSteps (SXS).

Execution Sheets can be displayed and maintained using SAP GUI for Windows and SAP GUI for HTML. Details are described in the following SAP Notes:

3413777 Support SAPGUI for HTML (Web GUI) for PMA Application - Display & Maintenance (Execution Sheets)

3417131 Execution Sheets - Supported Functionality (SAP GUI for HTML)

Therefore, Execution Sheets are a good starting point to convert your manufacturing processes of SAP S/4HANA to web-based solutions like SAP Digital Manufacturing (mid-to long-term).

In addition, the direct generation of Execution Sheets from manufacturing orders simplifies your previous business processes.

As it is no longer required to create and send control recipes (PP-PI)/control instructions (PP-SFC). For details, see SAP Note 3413719.

For lower SAP S/4HANA releases, you can use browser-based PI sheets in SAP S/4HANA, on-premise edition for the time being.

Preferably use XStep-based process instructions instead of the characteristic-based process instruction for defining the PI Sheet content:

XStep-based process instructions provide reusable and configurable building blocks (Standard XSteps [SXS]).

They are organized in a Standard XStep repository.

They can be used across different plants which allows a better standardization of manufacturing business processes.

Costs for maintenance of the process instruction content are also significantly lower when the reusable SXS building blocks are applied.

For similar capabilities offered by SAP Digital Manufacturing, check the corresponding user assistance and roadmap.

## Business Process related information

Customers that used ABAP-list-based PI sheets temporarily have to switch to Execution Sheets (SAP S/4HANA 2023+) or browser-based PI sheets.

The corresponding control recipe destinations of type 1 (ABAP-list-based PI sheets) have to be replaced with:

Destinations of type X (Execution Sheets [SAP S/4HANA 2023+]).

These destinations need to be maintained using the corresponding IMG activity (transaction CMX_XSH_DEST_SETUP ).

Control recipe destinations of type 4 (browser-based PI sheets), if SAP S/4HANA release is lower than SAP S/4HANA 2023.

This replacement mostly affects the used master recipes and process orders.

Execution Sheets and  browser-based PI sheets offer a better support of the business processes.

No effects on business processes are expected.

Transactions CO56 and CO58 have been deactivated in SAP S/4HANA (on-premise and Private Cloud Edition [PCE]).

## Required and Recommended Action(s)

If you use Execution Sheets:

Exchange your control recipe destinations of type 1 (ABAP-list-based PI sheets) with destinations of type X (Execution Sheets).

Preferably activate direct generation of Execution Sheets (see SAP Note 3413719). Create test Execution Sheets first.

If you want to continue using control recipes, then create and send test control recipes for the exchanged control recipe destinations.

Check the generated test Execution Sheets.

If you use browser-based PI Sheets:

Exchange your control recipe destinations of type 1 (ABAP-list-based PI sheets) with destinations of type 4 (browser-based PI Sheets).

Create and send test control recipes for the exchanged control recipe destinations.

Check the generated browser-based PI Sheet.

Recommended:

Use XStep-based process instructions instead of characteristic-based process instructions

Replace generation scopes and filters that are defined in customizing (of the control recipe destination) with XStep-based generation scopes and valuation symbols

For mid- to long-term planning consider migrating to worker UIs offered by SAP Digital Manufacturing.
