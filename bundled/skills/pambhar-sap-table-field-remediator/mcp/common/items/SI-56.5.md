## Symptom

You are doing a system conversion to SAP S/4HANA, on-premise edition. The following SAP S/4HANA Transition Worklist item is applicable in this case.

A system upgrade/conversion is about to be done and the source system is one of the following releases: -

ACM 4.0

ACM on SAP S/4HANA 1709 - All releases

ACM on SAP S/4HANA 1809 - FPS00

ACM on SAP S/4HANA 1809 - FPS01

ACM on SAP S/4HANA 1809 - FPS02

ACM on SAP S/4HANA 1909 - FPS00

ACM on SAP S/4HANA 1909 - FPS01

ACM on SAP S/4HANA 1909 - FPS02

ACM on SAP S/4HANA 2020 - FPS00

The target system is 'ACM on SAP S/4HANA 2020 FPS01' or a higher release.

Post-conversion/upgrade, dumps occurs because of the existence of multiple active BAdI implementations of the BAdI definition BADI_CPE_FE_RED_DATE_ROUTINE for the same filter value '9999999'. This prevents execution of various ACM transactions and prevents successful execution of the SDM /ACCGO/CL_SDM_ACM_V_SDMNPEL.

## Important Note : -

The manual instruction in this note must only be implemented after all the productive activities in the system have been halted in preparation for the conversion/upgrade. The changes need to be performed before any workbench locks are made as part of the upgrade activities.

The transport-request in which the implementation of this note is captured must be released to all follow-on systems to prevent occurence of this issue in those systems.

## Reason and Prerequisites

This step is needed to prevent the occurrence of dumps because of the existence of multiple BAdI implementations of the BAdI definition BADI_CPE_FE_REF_DATE_ROUTINE for the same filter value '9999999'.

## Solution

## Required and Recommended Action(s)

Run the steps mentioned in SAP Note - 3094714- S4TC ACM Pre-Checks for Data Migration of /ACCGO/T_NPELOTS table in the source system

The manual instruction in this note must only be implemented after all the productive activities in the system have been halted in preparation for the conversion/upgrade.

The transport-request in which the implementation of this note is captured must be released to all follow-on systems to prevent occurence of this issue in those systems.

Please implement the manual correction instructions attached to this note.

## Other Terms

Pre-upgrade/conversion step for upgrade/conversion to ACM on S/4HANA 2021 FPS01 or higher releases
