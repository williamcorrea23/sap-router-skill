## Symptom

You plan a system conversion from SAP ERP to SAP S/4HANA or an SAP S/4HANA upgrade to SAP S/4HANA 2023 FPS02 or higher and you use the functionality described in this note.

## Solution

Non Ferrous Metals is part of the SAP S/4HANA compatibility scope, however, with SAP S/4HANA 2023 FPS02, NFM solution has been modernized as described below. With this modernization, NFM can be used as part of perpetual scope as of this release also beyond 2025. For an upgrade or conversion to release SAP S/4HANA 2023 FPS02 or higher please consider the functional changes and required activities described below.

## Business Process-related information

The functional scope of Non-Ferrous Metal in SAP S/4HANA remains unchanged.

Two new CDS Views are provided for NFM SD transactional reporting (equal to reporting transaction: /NFM/POSLS) with S/4HANA 2023 FPS02:

## Sales :

Consumption View: C_NonFerrousMatSlsDocItmTP UI ODATA V2 Service Binding: UI_NONFERROUSMATSALES_DISP OData V2 - Web API Binding: API_NONFERROUSMATSALES

## Billing :

Consumption View: C_NonFerrousMatBillgDocItmTP UI ODATA V2 Service Binding: UI_NONFERROUSMATBILLG_DISP OData V2 - Web API Binding: API_NONFERROUSMATBILLING

The CDS views includes same fields and nearly equal logic as the report.

Columns which are calculated:

Charge Value in Company Code Currency Company

Code Currency Rate in Company Code Currency

Base Value in Company Code Currency

Purchasing Value in Company Code Currency

Struct. Value in Company Code Currency

Column 'Charge weight'

Charge weight is displayed with value = 0 KG, if SD order item field 'ATP confirmed quantity' is zero.

Not included: Delivery date field /NFM/POSLS (=VBEP-EDATU) is dropped. (In old report a specific logic shows the delivery date from LAST item of the SD order schedule line.)

New: Column "Total charge weight"

## Required and Recommended Actions

If you do a conversion from SAP ERP to SAP S/4HANA, the following activities are required before using NFM in S/4HANA.

NFM field 'Exchange key' in transaction 'Business Partner':

Since SAP S/4HANA, on-premise edition 1511, Customer and Vendor master transactions are combined into new Business Partner transaction "BP". The Non Ferrous Metals (NFM) field 'Exchange key' is not shown in transaction "BP" by default. In DDIC tables LFM1 [Vendor master record purchasing organization data] and KNVV [Customer Master Sales Data], the 'NFM Exchange Key' field /NFM/BOESL is still available. Re-activation by project development is possible. You can add field 'Exchange key' into Transaction 'BP' by BadI development. SAP note 2815395 - Enhancement of transaction BP with Non Ferrous Metals (NFM) "exchange key" field

## Metal price quotations:

If you import metal price quotations from stock exchanges by data feed with transaction TBD4 please read SAP note 2918965 - NFM (Non Ferrous Metal) Datafeed via TBD4 in S4/HANA

NFM fields in Material master:

In case the NFM fields in material master are not displayed, please read SAP note 2357870 - SAP S/4HANA: Settings to enable Non Ferrous Metals data in material master
