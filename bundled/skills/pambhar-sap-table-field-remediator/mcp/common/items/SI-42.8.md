## Symptom

You are doing a system conversion or an upgrade to SAP S/4HANA, on-premise 1909 edition. The following SAP S/4HANA Transition Worklist item is applicable in this case.

## Reason and Prerequisites

This note is relevant for you when your start release for the system conversion is an ERP release or SAP S/4HANA on-premise 1809 edition or lower releases.

It is not relevant if your start release is S/4HANA on-premise 1909 or later.

## Solution

## Business Value

New solution to maintain document attributes for Payments to tax authorities before output in Payment Medium.

New Manage Additional Payment Attributes app offers you the following benefits:

Unified and simplified user experience via Fiori interface

Improved Mobility. The switch to Fiori delivers advantages to mobile employees. Fiori's simple interface lets users work with touch screens.

Increases in Productivity - Fiori-designed interfaces are easier, so they enable more productive work.

Integration with a Manage Journal Entry app that is main report to see and check postings.

## Description

For tax payments attributes in S/4HANA on-premise 1909 and future releases Manage Additional Payment Attributes app has to be used:

Find your Payment document in the Manage Journal Entries app and click on the journal entry number.

Select Manage Additional Payment Attributes and enter payment attributes or change existing one.

For more information see Tax Payments help page.

## Business Process related information

Manage Additional Payment Attributes app replaces the solution:

Payment attributes screen during Financial Document posting and Financial Document change (Open FI Functional Modules J_3RF_PDOCV_2218 and J_3RF_PDOC1430)

Note that depending on your SAP S/4HANA release and planned end of support date previous solution will become obsolete. After end of support date these solutions will not be updated by new legal changes or corrections. We highly recommend that you use the Fiori based apps instead of the old previous solution.

## SAP S/4HANA on-premise 1809 and previous releases

No planned end of support date

## SAP S/4HANA on-premise 1909 and future releases

Only Manage Tax Payment Attributes Fiori will be supported. Payment attributes screen during Financial Document posting and Financial Document change (Open FI Functional Modules J_3RF_PDOCV_2218 and J_3RF_PDOC1430) is obsolete within these releases.

## Required and Recommended Actions:

Set up new Fiori app 'Manage Additional Payment Attributes (Refer to SAP Note 2931354).

If you have made customer enhancements to the obsoleted Open FI function modules, check if you need to reimplement them in the new Fiori App. Custom logic can be implemented in the BAdI FIGLO_PMNT_FIELDS.

Train your users.

## How to Determine Relevancy

Check that you have configured the Open FI module J_3RF_PDOCV_2218 for the event 2218 and Open FI module J_3RF_PDOC1430 for the event 1430 or you have entries in Payment Attributes table J_3RF_PLAT.
