## Symptom

How to define the Integration of Sales Order Scheduling in S/4HANA TM 2023 FPS 01?

You are planning to use the Integration of Sales Order Scheduling in SAP S/4HANA Transportation Management (TM) version 2023 FPS 01.

According to the listed prerequisites, you have to define:

transportation relevance of sales documents, containing

a control key for document integration a logistics integration profile one or more transportation duration determination profiles one or more control keys for document integration one or more logistics integration profile, containing a transportation duration determination profiles In 2023 FPS 01 , this set of customizing activies offers that in the transportation relevance of sales documents the same control key but different logistics integration profiles can be used for different sales documents. As in different logistics integration profiles different transportation duration determination profiles can be used, in the end, different transportation duration determination profiles can be used for the same control key .

With 2023 FPS 02 onwards , new features will require a simplification to "one control key will have at maximum exactly one transportation duration determination profile". Accordingly, a simplification item check (SI04: TM_CUSTOMIZING_4) will be executed during upgrade from 2023 FPS01 to 2023 FPS02 or higher which checks for this cardinality, and in case of finding more than one transportation duration determination profiles for the same control key aborts SUM (software update manager).

Alternative symptom: TRS consistency check of simplification item SI04: TM_CUSTOMIZING_4 fails (red traffic lights).

## Reason and Prerequisites

Planned simplification item for S/4HANA Transportation Management (TM) 2023 FPS2 and higher.

## Solution

To enable a smooth upgrade from 2023 FPS 01 to 2023 FPS 02 or higher, ensure that one control key at maximum is connected to exactly one transportation duration determination profile. This means that in case for different sales document the same control key, but different logistics integration profiles are configured, all logistics integration profiles that are used in conjunction with the same control key have to have the same transportation duration determination profile.
