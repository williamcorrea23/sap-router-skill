## Symptom

You are using "new functionality" in SAP S/4HANA for product compliance and are performing a system upgrade to SAP S/4HANA 2022 or higher. The following SAP S/4HANA transition worklist item SI15: Logistics_PSS is applicable in this case. To understand the differences between Product Compliance "classic functionality" and "new functionality" and the strategy please refer to SAP note 3015382.

## Reason and Prerequisites

You have activated the event type linkages in database view SWFDVEVTYP for the receiver type PCEP_V1.0 and are using a customer-specific receiver class implementation and not the standard class CL_EHFND_PCEP_EVT_HANDLER. See also Customizing for Activate Event Type Linkage under Product Compliance -> Foundation for Product Compliance -> Product Compliance Background Processing -> Activate Event Type Linkage .

If you are using standard event receiver settings or perform an initial installation of SAP S/4HANA 2022 or higher (greenfield installation), this note is not relevant.

## Solution

## Description / Business Value

In SAP S/4HANA for product compliance, the event type linkages have been introduced for the following main reasons:

Decoupling of a transcation, such as the release of a composition, from the asynchronous processing of this event to ensure proper follow-on processing of this event Providing a possibility to trigger customer-specific actions, such as sending an e-mail or notifying external systems that a certain event has been raised and needs follow-on processing Due to technical restrictions, it is not allowed to use local update tasks in the context of a logical unit of work of RAP and BOBF business objects. The update task module SWF_EVT_RAISE_IN_UPDATE_TASK to raise events and carry them out in a separate update task violates the contract, which may lead to short dumps.

In order solve these technical restrictions, the overall processing logic has been changed in the following way with SAP S/4HANA 2022. The event type linkages are replaced with a central determination that has been added to all affected business objects.

The determination is carried out during save and is supposed to check if events have been created in the same session.

The business object saves the data (for example, as an analytical composition), creates the corresponding new event(s) (for example, Analytical Composition Released ) in the table EHFNDD_PCE, and raises the system event SAP_EHFND_PCEP_EVT_CRTED to process these events in a follow-on activity.

The system event triggers the periodically scheduled background job to process all new and failed product compliance events and tasks.

## Business Process-Related Information

If you are using standard event type linkages, there are no changes to the business process. If you are using a customer-specific event type linkage in SWFDVEVTYP for the receiver type PCEP_V1.0 and not the standard class CL_EHFND_PCEP_EVT_HANDLER, you have to take action.

## Required and Recommended Action(s)

You have to replace your customer implementation of the receiver class to reflect the architectural change.
