| Business Impact |       2400737 | Ariba Cloud Integration Solutions for SAP: Supported SAP Product Versions |

## Symptom

You are planning to use SAP and Ariba Solutions integrated with Integration Adapters provided by the SAP Ariba Cloud Integration program, or you are already live with it and are considering an upgrade of one of the solutions. To plan your project, you need to know the supported versions and constraints with respect to upgrades.

During upgrade or conversion check, an error is issued in check class CLS4SIC_ARIBA_CI_CHECK (RC = 7): Go to Solution -> Information for upgrade and conversion

## Reason and Prerequisites

You are using or you are planning to use the Ariba Cloud Integration Solution (CI), that you can download from Ariba Connect. Or you are planning to use the Ariba Cloud Integration Gateway (CIG) Add-On for SAP ERP or S/4HANA.

This note is not relevant for integration solutions provided by the 'Ariba Integration Suite' (Add-On) that you can download from the SAP Service Marketplace.

## Information for upgrade and conversion

You can only upgrade to a SAP Product Version that is supported by the Ariba Cloud Integration program. Some integration scenarios will be broken if you upgrade to a non-supported SAP Product Version. Support is not granted by SAP Ariba for any inquiries related to such incidents.

SAP Ariba delivered the Cloud Integration programs (CI-9) through SAP transport files within an exclusive namespace registered with SAP. The solution is supported by SAP Ariba as per the Ariba Support policy (see section below 'Supported Versions for CI'). However, due to the nature of transport objects, the programs are considered as customer objects during upgrade with Software Update Manager (SUM). During the upgrade, you have to ensure that Ariba Objects including table contents are retained. Due to their origin from transport files, Ariba Objects can get deleted during an upgrade. To avoid this, make sure that note 2402191 is applied in your system, if you want to continue using this solution.

During upgrades and conversions, a detailed check is executed by Software Update Manager (SUM). In case that CI objects can be found in the system, that have been imported through transport files (for example CI-9), the check returns error code RC = 7. In this case, you have to confirm that you have either migrated to CIG already or that you have run the report attached to note 2402191. You can confirm this by adding an exemption to the check result. This procedure is described in detail here: https://blogs.sap.com/2018/03/26/sap-s4hanasimplification-item-check-how-to-do-it-right./

After you have migrated to CIG, you do not longer need to run run the report attached to note 2402191 prior to upgrades. CI objects, that have been imported through transport files (for example CI-9) will be deleted during the next upgrade.

With SAP's standard integration platform released with SAP Ariba Cloud Integration Gateway (CIG) to integrate SAP Ariba solutions with SAP ERP systems, previous integration tools such as  Cloud Integration (CI) and Business Suite Add-On are no longer available for new installations and no further enhancements will be made. At this time, only CI-9 version is the only supported version for break fixes while previous CI versions are deprecated and no longer supported.

## * CIG Support Pack - Support Policy:

To adhere with the standard SAP support model, SAP Ariba will only support the current (version N) and the version preceding the current version (version N-1) for the SAP Ariba Cloud Integration Gateway. CIG has 3 components ARBCI1, ARBCI2 & ARBCIGR and each of these components could be at different SP level. Current version is determined from the component with highest Support Pack level.

For example, with the support package SP 0008 release, ARBCI1 is SP008 while ARBCI2 is SP006 and ARBCIGR is SP0001 and SAP Ariba supports the following versions:

Current released version (version n) - for example, ARBCI1: SP 0008

Preceding released version (version n-1)- for example, ARBCI1: SP 0007

SAP Ariba recommends that customers must always upgrade to the latest SP release that is available at GA (General Availability) / RTC (Released To customer). The new releases always include additional enhancements and new features and customers can benefits from these functionalities.
