# /docs

Generated evidence documents. These are outputs of gate commands, not
hand-written prose — regenerate them rather than editing them by hand.

## Contents

- `parse-failures.md` — the parser proof for the abapGit workspace: total
  `.abap` files, parsed OK, failed (with per-file reason) and the success rate.
  Regenerated by `npm run parse-proof`.

Parse failures never block a seed run: a failed object is stored with
`parse_status='failed'` plus its raw source, and the seeder prints a
parse-success-rate summary.

The narrative documentation for this project lives at the repository root:
`README.md` (product spec and core principles) and `CLAUDE.md` (conventions).

## Commands

```bash
npm run parse-proof     # regenerates docs/parse-failures.md
```
