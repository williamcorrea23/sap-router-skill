场景知识后端 — 每个场景的 references/
🧠
判断框架.md — Decision Logic
场景核心判断逻辑。以 YAML/表格形式沉淀财务判断规则（如内控缺陷分级标准、KPI 选择逻辑、投资可行性判断树）。供 Atomic Skill 读取执行，不直接暴露给用户。
🔌
数据源清单.md — Connector Mapping
场景所需数据 → 对应 Connector → 系统/表/字段 映射。包括系统连接状态表（Operational / Configured-unreachable / Not configured）和关键阈值配置。Agent 读取后决定工具调用路径。
🗺️
查询路径.md — SOP
场景的标准操作流程（SOP）。当没有 Connector 时，Agent 按 SOP 引导用户手动完成数据收集和决策。每一步标注信息来源和预期输出。

Cold-Start 首次配置向导
P0
连接器检测
轻量 API 调用验证 SAP / Power BI / BlackLine / Concur 等是否真正可达，标记 Operational / Configured-unreachable / Not configured 三种状态。
P1
公司配置
收集 ERP 类型 / 审计周期 / 财务职能结构 / 公司名称 / 所属行业，写入 ~/.config/finance-skills/CLAUDE.md。
P2
场景选择
用户选择相关场景（可多选），安装对应 references/ 和 skills/，激活 Connector 覆盖。
P3
关键阈值设置
设置场景级阈值（内控缺陷分级标准、KPI 基准值、预算容许偏差%等），写入场景级 CLAUDE.md。
P4
写入配置
将所有配置持久化到 ~/.config/finance-skills/，生成操作日志，完成冷启动。
触发方式
/finance:cold-start
或说"开始配置"、"初始化财务技能"、
"财务助手设置"、"cold start"、"setup finance"
配置检查
/finance:cold-start --check-connections
仅检测连接器状态，不触发完整向导

├──CLAUDE.md ← 公司级配置
├──credentials/ ← API 密钥（Agent 不可见）
└──projects/ ← 所有财务项目
└──2026-annual-audit/
├──project.md ← 项目核心上下文
├──scope.md ← 审计范围
├──findings/ ← 缺陷评估
├──control-testing/ ← 测试底稿
└──documents/ ← 相关文档

project_type: "annual-audit"
