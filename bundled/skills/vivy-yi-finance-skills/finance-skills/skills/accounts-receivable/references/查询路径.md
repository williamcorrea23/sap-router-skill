# 查询路径 — 应收账款

## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | ERP AR 模块（[ERP]）| 客户应收余额/账龄/收款记录 | 最优先 |
| 2 | CRM 系统（[CRM]）| 客户信息/信用额度/合同 | 客户背景 |
| 3 | 合同/订单系统（[SO]）| 销售订单/收入确认 | 确认依据 |
| 4 | 发票系统（[INV]）| 开票记录/发票状态 | 收款依据 |
| 5 | 资金系统（[CASH]）| 到账确认/银行流水 | 收款确认 |

---

## 常用查询场景

### 场景 1：账龄分析

**查询入口：**
```
ERP SAP：
- FD10N：客户余额（应收账款视图）
- FBL5N：客户行项目（显示所有未清项）
- S_ALR_00001231：应收账款账龄分析表

用友/金蝶：
- 应收账款 → 客户余额表
- 应收账款 → 账龄分析表
```

**账龄分析维度：**
```
按账龄区间：
  - 1-30 天：[X] 万 / [X] 笔
  - 31-60 天：[X] 万 / [X] 笔
  - 61-90 天：[X] 万 / [X] 笔
  - 91-180 天：[X] 万 / [X] 笔
  - > 180 天：[X] 万 / [X] 笔

按客户：
  - TOP 5 大客户应收
  - TOP 5 逾期客户
  - 逾期客户数量占比

按客户信用等级：
  - A 级客户应收分布
  - B 级客户应收分布
  - C/D 级客户应收分布
```

### 场景 2：逾期客户跟进

**查询入口：**

- 催款函发送记录

**跟进记录查询：**
```
□ 已发送催款函：[X] 封

□ 客户承诺付款日期：[YYYY-MM-DD]
□ 客户承诺付款金额：[XXX万]
□ 实际到账情况：[已到账/未到账]

```

### 场景 3：客户信用评估

**查询入口：**
```
CRM 系统（[CRM]）：
- 客户主数据（信用额度/账期/信用等级）
- 客户财务信息（年报/信用报告）
- 客户联系人及沟通记录

外部数据：
- 天眼查/企查查：客户工商信息/法律诉讼/经营状态
- 中国执行信息公开网：客户是否被被执行
- 裁判文书网：客户涉及的重大诉讼
```

**信用评估维度：**
```
□ 客户工商状态是否正常（存续/在业）
□ 客户是否有失信/被执行记录
□ 客户涉及诉讼金额是否过大
□ 客户财务状况（年报/审计报告）
□ 客户付款历史（是否按时付款）
□ 客户行业地位（是否龙头企业）
□ 客户授信额度（银行授信情况）
```

### 场景 4：合同条款查询

**查询入口：**
```
合同管理系统：
- 销售合同/框架协议
- 付款条款（账期/付款方式）
- 信用额度约定
- 违约条款

ERP（[SO]）：
- 销售订单（已确认订单/已发货订单）
- 收入确认记录
```

**合同关键条款：**
```
□ 付款账期：[月结30/45/60天]
□ 信用额度：[XXX万]
□ 付款方式：[银行转账/票据/供应链金融]
□ 违约条款：[逾期付款违约责任]
□ 争议解决条款：[仲裁/诉讼]
```

---

### 场景 5：到账确认

**查询入口：**
```
资金系统（[CASH]）：
- 银行流水（到账记录）
- 到账认领（AR 收款认领）

ERP（[ERP]）：
- 收款录入
- 应收核销
- 预收账款
```

**到账认领流程：**
```
1. 银行到账 → 资金系统显示
2. AR 认领到账 → 匹配客户/发票
3. 核销应收 → ERP 中标记为已收款
4. 如有多付 → 记入预收账款
```

## 查询结果处理

### 数据完整性判断

```
三方数据一致（[ERP] + [CRM] + [INV]）：
→ 高置信度，可直接生成账龄报告

两方一致，有差异：
→ 中置信度，核查差异原因后再分析

数据差异较大：
→ 低置信度，暂停账龄分析，核查后处理
```
