# 判断框架 — KPI 管理异常处理

## 核心原则

**KPI 管理的目标是"目标清晰、追踪及时、偏差可纠"。**
遇到异常时，优先判断是"目标设定问题"还是"执行问题"，确保 KPI 真正驱动业务改进。

---

## 异常类型与处理方式

### 类型一：KPI 连续未达标

**典型场景：**
- 同一 KPI 连续 [X] 个月低于目标
- KPI 跌破预警线
- 行业对标显著落后

**判断流程：**

```
第一步：确认未达标情况

□ KPI 名称：[名称]
□ 目标值：[X]
□ 实际值：[X]
□ 达成率：[X%]

□ 未达标期间：
□ [月份1] — 目标 [X] — 实际 [X] — 达成 [X%]
□ [月份2] — 目标 [X] — 实际 [X] — 达成 [X%]
□ [月份3] — 目标 [X] — 实际 [X] — 达成 [X%]

□ 预警状态：[✅ 黄灯 / 🔴 红灯]
□ 连续未达标月数：[X] 个月

第二步：分析原因

□ 外部因素：
□ 市场环境变化：[是/否] — [描述]
□ 政策变化：[是/否] — [描述]
□ 竞争对手变化：[是/否] — [描述]

□ 内部因素：
□ 资源不足（人员/资金）：[是/否] — [描述]
□ 执行偏差：[是/否] — [描述]
□ 目标设定过高：[是/否] — [描述]
□ 跨部门协调问题：[是/否] — [描述]

□ 原因分类：
□ 可控原因：[描述]
□ 不可控原因：[描述]

第三步：制定改进计划

□ 短期措施（1-3 个月）：
□ 措施1：[描述] — 责任人 [X] — 完成 [YYYY-MM-DD]
□ 措施2：[描述] — 责任人 [X] — 完成 [YYYY-MM-DD]

□ 中期措施（3-6 个月）：
□ 措施1：[描述] — 责任人 [X] — 完成 [YYYY-MM-DD]

□ 目标调整（如需要）：

□ 资源申请（如需要）：
□ 资源需求：[描述]
□ 资源类型：[人员/资金/系统]

```

### 类型二：KPI 数据不一致

**典型场景：**
- 不同系统 KPI 数据不一致
- 财务口径与业务口径不一致
- 计算逻辑错误

**判断流程：**

```
第一步：确认数据差异

□ KPI 名称：[名称]
□ 系统A 数据：[X]
□ 系统B 数据：[X]
□ 差异：[±X]

□ 数据来源：
□ [ERP] — 数据 [X]
□ [BI] — 数据 [X]
□ [CRM] — 数据 [X]

□ 计算逻辑核查：
□ [系统A] 计算公式：[描述] — 正确 [✅/⚠️]
□ [系统B] 计算公式：[描述] — 正确 [✅/⚠️]

□ 差异原因：
□ 定义不一致：[是/否]
□ 数据同步延迟：[是/否]
□ 计算错误：[是/否]

第二步：确定正确数据

□ 正确数据源：[ERP/BI/CRM]
□ 正确计算口径：[描述]
□ 正确值：[X]

□ 历史数据修正：
□ 是否须修正历史：[是/否]
□ 修正影响期间：[YYYY-MM 至 YYYY-MM]
□ 对考核的影响：[描述]

第三步：整改措施

□ 数据源整改：
□ 系统打通：[ERP ↔ BI ↔ CRM]
□ 数据同步机制：[实时/每日/每周]

□ 定义统一：
□ 统一 KPI 定义文档：[✅ 已更新 / ⚠️ 待更新]
□ 口径说明：[描述]
□ 沟通下发：[✅ 已通知 / ⚠️ 待通知]
```

### 类型三：KPI 预警触发

**典型场景：**
- KPI 实时监控跌破预警线
- 异常波动触发自动预警
- 竞争对手数据变化触发对标预警

**判断流程：**

```
第一步：确认预警内容

□ 预警 KPI：[名称]
□ 预警类型：[跌破预警/异常波动/对标落后]
□ 当前值：[X]
□ 预警阈值：[X]
□ 触发时间：[YYYY-MM-DD HH:MM]

□ 预警来源：
□ 仪表板自动触发
□ 人工识别
□ 竞争对手对标触发

第二步：初步评估

□ 影响评估：
□ 对业务影响：[✅ 低 / ⚠️ 中 / 🔴 高]
□ 对财务影响：[✅ 低 / ⚠️ 中 / 🔴 高]
□ 对客户影响：[✅ 低 / ⚠️ 中 / 🔴 高]

□ 紧迫性：
□ 是否须立即行动：[✅ 是 / ⚠️ 否]
□ 响应时间要求：[X] 小时内

第三步：启动响应

□ 即时响应：
□ 通知责任人：[X]
□ 初步分析完成：[YYYY-MM-DD HH:MM]

□ 根因分析：
□ 分析负责人：[X]
□ 完成时间：[YYYY-MM-DD]

□ 应对措施：
□ [措施1] — 启动时间 [YYYY-MM-DD] — 预期效果 [描述]
□ [措施2] — 启动时间 [YYYY-MM-DD] — 预期效果 [描述]
```

## 置信度评估

| 置信度 | 条件 | 操作 |
|---|---|---|---|
| 高 | 多系统数据一致 + 计算逻辑确认 | ✅ 正常追踪 |
| 中 | 单系统数据 + 计算逻辑待确认 | ⚠️ 交叉核对 |
| 低 | 多系统数据不一致 | 🔴 立即核查 |

---

## 升级触发条件

- 关键 KPI 连续 3 个月未达标须 CFO 关注
- KPI 跌破红线须立即升级业务负责人
- 须调整年度 KPI 目标须 CEO 审批
