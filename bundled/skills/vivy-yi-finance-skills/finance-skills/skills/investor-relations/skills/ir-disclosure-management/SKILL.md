# /ir-disclosure-management — IR 信息披露管理


## Examples

→ 示例：用户说"我们有重大收购在谈，IR 什么时候可以开始和投资者沟通"，系统应调用本技能，制定重大事项的信息披露时间窗口建议。

→ 示例：用户说"帮我们审一下这份业绩预告，有没有选择性披露的风险"，系统应调用本技能，审阅业绩预告的合规性。

→ 示例：用户说"社交媒体上有人泄露了未经公告的财务数据，怎么办"，系统应调用本技能，评估泄露事件的 IR 影响并制定应对方案。
## 第一步：披露类型确认

**披露类型判断：**
```
□ 披露类型：
  → [定期报告]：年报/半年报/季报
  → [临时公告]：重大事项/业绩预告/股价异常波动
  → [IR 材料]：路演材料/投资者PPT/新闻稿
  → [其他]：自愿性披露

□ 是否为必须披露：[✅ 是（监管要求）/ ⚠️ 自愿性披露 / 🔴 禁止披露]
□ 披露依据：[具体法规条款]
```

**信息披露标准核对：**
```
□ 信息重要性标准（是否构成"重大信息"）：
  → 对股价可能产生重大影响：[是/否]
  → 对投资者决策可能产生重大影响：[是/否]
  → 信息确定性程度：[高/中/低]

□ 保密性检查：
  → 信息是否已处于公开状态：[是/否]
  → 是否须申请豁免保密：[是/否 — 依据]
```

## 第二步：披露合规审核

**合规审核清单：**
```
□ 披露内容审核：
  → 信息准确性：[✅ 已核实 / ⚠️ 待核实]
  → 信息完整性：[✅ 完整 / ⚠️ 缺失部分]
  → 表述中立性：[✅ 中立 / ⚠️ 存在倾向性表述]
  → 风险披露充分性：[✅ 充分 / ⚠️ 不足]

□ 披露格式审核：
  → 符合监管格式要求：[✅ 是 / ⚠️ 需调整]
  → 必要声明/脚注：[✅ 已包含 / ⚠️ 缺失]
  → 数据单位/口径：[✅ 统一 / ⚠️ 不一致]

□ 禁止事项检查：
  → 禁止虚假陈述：[✅ 合规]
  → 禁止重大遗漏：[✅ 合规]
```

**审核问题记录：**

## 升级触发条件

- 发现须披露的重大信息但尚未披露（须立即启动披露程序）
- 披露内容存在重大错误或遗漏（须立即更正）
- 内幕信息管理存在漏洞（信息泄露风险）
- 披露时机选择与监管要求冲突
- 信息披露引发市场异常波动须启动临时停牌
- 媒体或分析师提前获知未披露信息（合规事件）
