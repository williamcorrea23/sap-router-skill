# /esg-materiality-assessment — ESG 重大性评估


## Examples

→ 示例：用户说"帮我们做一次 ESG 重大性评估，确定哪些议题最关键"，系统应调用本技能，执行 ESG 重大性评估。

→ 示例：用户说"去年 materiality assessment 的结果今年还用吗，需要更新吗"，系统应调用本技能，评估是否需要更新重大性评估。

→ 示例：用户说"投资者问我们为什么不披露某个议题，需要解释决策依据"，系统应调用本技能，提供重大性评估决策依据说明。

## 升级触发条件

- 重大性评估结果与管理层判断存在重大分歧
- 利益相关方调研样本严重不足（< [X]%）影响可靠性
- 议题落入"高影响 + 低重要性"象限（需解释为何优先处理）
- 行业政策或标准变化导致议题边界变化
- 重大业务变化（M&A/剥离/新业务）须重新评估
- ESG 评级机构对重大性评估提出质疑
