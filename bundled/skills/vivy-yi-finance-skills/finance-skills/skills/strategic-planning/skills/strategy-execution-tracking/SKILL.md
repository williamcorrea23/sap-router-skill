# /strategy-execution-tracking — 战略执行追踪


## Examples

→ 示例：用户说"战略举措执行了大半年了，帮我看看对业绩指标的实际影响"，系统应调用本技能，执行战略举措与业绩指标关联分析。

→ 示例：用户说"有几个战略举措落后于计划，帮我识别瓶颈和调整建议"，系统应调用本技能，执行战略执行差距分析和改进建议。

→ 示例：用户说"半年度战略回顾，帮我准备一份战略执行进度全景报告"，系统应调用本技能，生成半年度战略执行报告。
## 第一步：战略目标达成评估

**财务目标追踪：**
```
□ 战略规划周期：[YYYY - YYYY]
□ 本期追踪期间：[YYYY-QX]
□ 规划执行进度：[X]%（[X]/[X] 年）

□ 财务目标达成：
| 指标 | [YYYY]年目标 | 本期实际 | 达成率 | 趋势 |
|------|------------|---------|--------|------|
| 收入 | [X]亿 | [X]亿 | [X]% | [↑↓—] |
| EBITDA | [X]亿 | [X]亿 | [X]% | [↑↓—] |

□ 整体达成评估：
  → 综合达成率：[X]%（[X] 项达标 / [X] 项）
  → 达成评价：[✅ 超额/符合/低于]
```

**战略目标达成：**
```
□ 战略目标追踪：
| 战略目标 | 成功标志 | 当前状态 | 达成评估 |
|---------|---------|---------|---------|
| [目标1] | [标志] | [状态] | [✅/⚠️/🔴] |
| [目标2] | [标志] | [状态] | [✅/⚠️/🔴] |

□ 整体战略目标达成：[X]/[X] 项达标（[X]%）
```

## 第二步：里程碑达成评估

```
□ 本期应达成里程碑：[X] 项

□ 达成率：[X]%

□ 里程碑明细：
| 里程碑 | 原计划日期 | 实际日期 | 状态 | 偏差说明 |
|--------|-----------|---------|------|---------|
| [里程碑1] | [日期] | [日期/—] | [✅/⚠️/🔴] | [说明] |
| [里程碑2] | [日期] | [日期/—] | [✅/⚠️/🔴] | [说明] |
```

**延期里程碑分析：**
```
□ 延期里程碑：[X] 项
□ 延期原因分类：
  → 内部执行问题：[X] 项 — [描述]
  → 外部因素：[X] 项 — [描述]
  → 资源不足：[X] 项 — [描述]
  → 计划过于激进：[X] 项 — [描述]
```

## 第三步：执行偏差分析

**偏差识别：**
```
□ 重大偏差项（> [X]% 偏离目标）：
  → [指标/里程碑] — 目标 [X]，实际 [X]，偏差 [±X]%
  → 原因：[描述]
  → 影响：[对战略目标的影响]

□ 中等偏差项（[X-X]% 偏离）：
  → [指标/里程碑] — 偏差 [±X]%
  → 原因：[描述]
```

**偏差根因分析：**
```
□ 针对 [重大偏差项] 的根因：
  → 直接原因：[描述]
  → 深层原因：[描述]
  → 根因分类：[执行问题/资源问题/外部环境/计划问题]

□ 执行问题 vs 环境变化：
  → 执行问题导致：[X]% 的偏差
  → 环境变化导致：[X]% 的偏差
```

**战略健康度维度：**
```

□ 执行健康度：[✅ 良好 / ⚠️ 承压 / 🔴 危险]

  → 战略举措推进：[描述]

□ 组织健康度：[✅ 良好 / ⚠️ 承压 / 🔴 危险]
  → 关键人才到位：[X]%
  → 组织能力：[描述]

□ 整体战略健康度：[✅ 绿色 / 🟡 黄色 / 🔴 红色]
```

**战略风险信号：**
```
□ 风险信号：
  → 信号 1：[描述] — 影响 [目标/里程碑]
  → 信号 2：[描述]

  → 机会 1：[描述] — 潜在收益 [描述]
```

## 升级触发条件

- 财务目标达成率 < [X]%（如 80%），且无改善趋势
- 里程碑连续 [X] 期延期
- 战略健康度为 🔴 红色
- 须修正战略目标（重大战略调整）
- 执行偏差根因为外部环境重大变化（行业下行/政策逆转）
- 资源需求大幅超出原计划（> [X]%）
