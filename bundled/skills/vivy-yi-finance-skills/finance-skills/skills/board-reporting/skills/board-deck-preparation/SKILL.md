# /board-deck-preparation — Board Deck 编制


## Examples

→ 示例：用户说"下周三董事会，需要准备 Q2 的 Board Deck，帮我看看框架有没有问题"，系统应调用本技能，按董事会模板生成 Deck 框架和内容要点。

→ 示例：用户说"Board Deck 里需要加一页 ESG 进展，评级刚提升了"，系统应调用本技能，补充 ESG 进展页面。

→ 示例：用户说"这次 Board Deck 要特别说明一下收购标的的整合进展"，系统应调用本技能，生成整合进展专项页面。
## 第一步：确认会议类型和内容框架

**会议类型对应的标准内容：**
```
□ 董事会（Board Meeting）：
  → 执行摘要（1 页）
  → 财务业绩（2-3 页）：收入/利润/现金流/关键财务比率
  → 战略进展（2-3 页）：战略目标达成/市场变化/竞争态势
  → 重大事项（1-2 页）：M&A/融资/组织调整/诉讼
  → 风险分析（1 页）：主要风险及应对
  → 下阶段计划（1 页）：重点工作和里程碑

□ 审计委员会（Audit Committee）：
  → 审计发现跟踪（1-2 页）
  → 内控状态（1-2 页）：SOX 合规/缺陷整改
  → 财务报告质量（1 页）：关键会计判断
  → 合规状态（1 页）：监管合规

□ 战略委员会（Strategy Committee）：
  → 战略执行进度（2-3 页）
  → 市场与竞争分析（1-2 页）
  → 战略调整建议（1-2 页）
  → 新战略规划（如适用）（3-5 页）
```

**补充内容（如有特殊事项）：**
```
□ 须单独披露的重大事项：[列表]
□ 保密信息处理：[脱敏/删除/单独附件]
```

## 第二步：收集和核实数据

**数据提取：**
```
□ 财务数据（从 [BI/ERP/财务系统]）：
  → 期间：[YYYY-MM]
  → 收入：[X] 亿 — YoY [±X]%
  → EBITDA：[X] 亿 — YoY [±X]%
  → 净利润：[X] 亿 — YoY [±X]%
  → 经营现金流：[X] 亿
  → 关键比率：毛利率 [X]%、净利率 [X]%、ROE [X]%

□ 业务数据（从 [CRM/运营系统]）：
  → 客户数量：[X] — QoQ [±X]%
  → 订单量：[X] — QoQ [±X]%
  → 重点项目进度：[X] 项 — 其中里程碑达成 [X] 项

□ 数据来源确认：
  → 财务数据：[✅ 已从审计口径确认 / ⚠️ 待确认]
  → 业务数据：[✅ 已核实 / ⚠️ 待核实]
  → 数据截止时间：[YYYY-MM-DD HH:MM]
```

**数据一致性检查：**
```
□ Deck 数据 vs 审计数据：[✅ 一致 / ⚠️ 差异 [X]，原因：[描述]]
□ 同比/环比数据计算口径：[✅ 一致 / ⚠️ 口径差异]
□ 四舍五入差异：[✅ 在合理范围内 / ⚠️ 需调整]
```

## 第三步：设计 Deck 结构

**页面结构（参考，可按实际情况调整）：**
```
总页数：[X] 页

1. 执行摘要（1 页）
   → 核心亮点 [3-5 条，每条一句话]
   → 本期 vs 上期 vs 全年目标
   → 风险预警（如有）

2. 财务业绩（2-3 页）
   → 收入/利润/现金流趋势图（3 年）
   → 部门/业务线贡献分解
   → 关键财务比率

3. 战略进展（2-3 页）
   → 战略目标达成追踪
   → 市场与竞争态势变化
   → 重点战略项目进度

4. 重大事项（1-2 页）
   → M&A/融资进展
   → 组织调整
   → 重大诉讼/监管事项

5. 风险分析（1 页）
   → Top 3 风险及应对
   → 风险趋势变化

6. 下阶段计划（1 页）
   → 重点工作计划
   → 下次会议前里程碑
```

**Board 风格规范：**
```
□ 配色：[深蓝/企业蓝] — 与公司品牌一致
□ 字体：[无衬线字体，如 Helvetica/Arial]
□ 图表风格：[简洁/数据优先/减少装饰]
□ 数据呈现：[同比 YoY / 环比 QoQ / 完成率]
□ 每页建议字数：≤ [X] 字（Board 层注意力有限）
```

## 第四步：撰写内容

**执行摘要（1 页模板）：**
```
□ 核心信息（3-5 条）：
  1. 收入 [X] 亿，YoY [+X]%，完成全年目标的 [X]%
  2. EBITDA [X] 亿，YoY [+X]%，利润率 [X]%
  3. 重点战略项目 [X] 项进展符合预期，[X] 项偏差
  4. 主要风险：[风险1]（影响可控/须关注）
  5. 下阶段重点：[一句话]

□ 数据佐证（如须详细数据）：
  → 见财务业绩页 [X]/战略进展页 [X]
```

**各 section 要点：**
```
□ 财务业绩：
  → 每页一个核心信息（收入/利润/现金流）
  → 图表优先，减少文字
  → 标注关键变化和原因

□ 战略进展：
  → 用里程碑图展示重点项目进度
  → 标注偏差项和原因
  → 量化成果（市场占有率/客户数等）

□ 重大事项：
  → 标题说明事项本质（不是详细描述）
  → 附页展示详情（如须保密，单独提供）
```

## 升级触发条件

- 财务数据与审计数据存在重大差异（> [X]%）
- 涉及须披露的重大事项（股价敏感信息）
- Deck 内容涉及 M&A/融资等敏感事项
- 审计委员会对内控报告提出质疑
- 数据无法在会议前 [X] 天获取，导致 Deck 无法按时完成
- 存在须向董事会特别说明的重大风险事件
