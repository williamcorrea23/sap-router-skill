# 判断框架 — 董事会汇报异常处理

## 核心原则

**董事会汇报的目标是"信息真实、披露合规、决策有用"。**
遇到异常时，优先判断是"数据问题"还是"合规问题"，确保向董事会提供可靠、完整的决策信息。

---

## 异常类型与处理方式

### 类型一：汇报数据与审计数据不一致

**典型场景：**
- Board Deck 数据与审计报告不一致
- 月度数据与年度数据须调整
- 发现前期数据有误

**判断流程：**

```
第一步：确认数据差异

□ 报告名称：[Board Deck 名称]
□ 涉及期间：[YYYY-MM]
□ Board Deck 数据：[XXX万]
□ 审计/确认数据：[XXX万]
□ 差异：[±XXX万]

□ 差异原因：
□ 审计调整未更新 Board Deck
□ 数据提取口径不一致
□ 录入错误
□ 披露规则差异

第二步：评估影响

□ 对董事会决策的影响：
□ 是否影响重大决策：[是/否]
□ 影响决策类型：[投资/融资/人事/战略]

□ 合规影响：
□ 是否违反信息披露规则：[是/否]
□ 是否须更正公告：[是/否]

第三步：更正处理

□ 立即更正 Board Deck：
□ 更正版 Deck — 完成时间 [YYYY-MM-DD]
□ 更正说明：[描述]

□ 内部追责：
□ 错误责任人：[姓名]
□ 错误原因：[描述]
□ 整改措施：[描述]
```

---

### 类型二：须临时增加重大议题

**典型场景：**
- 突发重大事项须在原定议程外增加
- 重大交易须紧急上会
- 监管问询须向董事会汇报

**判断流程：**

```
第一步：确认事项

□ 事项名称：[描述]
□ 紧急程度：[✅ 紧急（24小时内）/ ⚠️ 一般（3天内）]
□ 信息保密级别：[公开/内部/最高机密]

□ 上会理由：
□ [重大交易须董事会审批]
□ [监管要求须董事会知悉]
□ [紧急风险须董事会决策]

□ 材料准备状态：
□ 已有材料：[✅ 是 / ⚠️ 否]
□ 材料完整度：[X%]

第二步：通知与议程调整

□ 通知范围：
□ 董事长：[✅ 已通知 / ⚠️ 待通知]
□ 独立董事：[✅ 已通知 / ⚠️ 待通知]
□ 审计委员会：[✅ 已通知 / ⚠️ 待通知]

□ 议程调整：
□ 原有议题顺延：[是/否]
□ 新增议题增加：[是/否]

□ 会议形式：
□ 现场会议
□ 通讯表决（仅紧急事项）
□ 视频会议

第三步：补充材料与汇报

□ 材料补充：
□ 补充材料截止：[YYYY-MM-DD]
□ 材料审核人：[CFO/CEO]

□ 汇报准备：
□ 汇报人：[姓名]
□ 汇报时间：[X] 分钟
□ Q&A 准备：[✅ 是 / ⚠️ 否]
```

---

### 类型三：审计委员会关注事项

**典型场景：**
- 审计委员会对汇报内容有疑问
- 要求补充数据或解释
- 提出调整建议

**判断流程：**

```
第一步：确认关注事项

□ 关注事项：[描述]
□ 提问人：[审计委员会成员]
□ 收到时间：[YYYY-MM-DD]
□ 须回复时间：[YYYY-MM-DD]
□ 剩余时间：[X] 天

□ 问题类型：
□ 数据核实（须补充数据）
□ 合规问询（须解释政策）
□ 风险质疑（须说明应对）
□ 审计调整（须评估影响）

第二步：准备回复

□ 回复材料：
□ 数据补充：[描述] — 完成 [YYYY-MM-DD]
□ 合规解释：[描述] — 完成 [YYYY-MM-DD]
□ 风险应对：[描述] — 完成 [YYYY-MM-DD]

□ 内部协调：
□ 须协调部门：[X]
□ 协调完成：[✅ 是 / ⚠️ 否]

□ 回复提交：
□ 提交形式：[书面/口头/会议]
□ 提交时间：[YYYY-MM-DD]
```

## 置信度评估

| 置信度 | 条件 | 操作 |
|---|---|---|---|
| 高 | 数据经审计确认 + 多源交叉验证 | ✅ 可直接使用 |
| 中 | 数据待最终确认 | ⚠️ 标注待确认 |
| 低 | 数据存在差异须核实 | 🔴 先核实后汇报 |

---

## 升级触发条件

- 须披露重大信息须 CEO/董事会秘书确认
- 涉及关联交易须独立董事审批
- 业绩重大偏差须立即汇报董事长
