# 判断框架 — 月末结账异常处理

## 核心原则

**月末结账的目标不是"不出错"，而是"发现错误后知道如何处理"。**
遇到异常时，优先原则是：宁可停止结账，不可带着不平衡的报表进入下期。

---

## 异常类型与处理方式

### 类型一：试算表不平衡

**典型场景：**
- 借贷方合计差异正好等于某个科目的金额（疑似做反方向）
- 借贷方合计差异为小数（精度问题，如 0.01 元）
- 借贷方合计差异为大数（>1000 元，疑似漏做或重做分录）

**处理方式：**

| 差异金额 | 处理方式 |
|---|---|
| ≤ 0.01 元 | ⚠️ 标注"精度误差，可忽略"，记录差异原因 |
| 0.01-100 元 | ⚠️ 定位差异最可能的来源，通知总账会计排查 |
| 100-1000 元 | 🔴 停止结账，要求总账会计逐笔排查 |
| > 1000 元 | 🔴 停止结账，财务经理介入 |

**排查顺序：**
```
第一步：从 ERP 导出当月所有凭证，按科目汇总
第二步：对比各科目余额与上月期末余额 + 本月发生额
第三步：检查是否有凭证借贷方向反向
第四步：检查是否有期末汇率重述导致的差异（外币业务）
```

---

### 类型二：跨期凭证

**典型场景：**
- 凭证日期在上月末之前（漏做上月凭证）
- 凭证日期在下月初（提前确认收入/费用）
- 跨期调整申请已批准，但调整分录未录入

**处理方式：**
```
判断：凭证日期距今 > 1 个月
  → 确认为跨期凭证

处理路径：
  - 月末最后 3 个工作日内发现：要求当日补录或调整
  - 月末最后 1 个工作日发现：评估金额影响，小额可先结账下月调整
  - 金额 > [填空：如 5 万元]：必须本月调整，不允许跨期结转

判断：跨期调整申请已批准但未录入
  → 立即录入调整分录
  → 录入后重新执行试算平衡检查
```

---

### 类型三：银行未达账项

**典型场景：**
- 银行已付但企业未付（银行扣费但未记账）
- 企业已付但银行未付（跨行转账未到账）
- 银行已收但企业未收（客户汇款已到账）
- 企业已收但银行未收（支票已收但未进账）

**处理方式：**

| 情形 | 处理 |
|---|---|
| 金额 ≤ 100 元，且长期存在 | ⚠️ 记录银行调节表，暂不处理 |
| 金额 100-5000 元 | 🔴 查明原因，补录或通知银行调整 |
| 金额 > 5000 元 | 🔴 必须查明原因，不允许长期挂账 |
| 同一银行账户连续 3 个月存在同一方向未达 | 🔴 停止结账，财务经理介入排查 |

**银行调节表格式：**
```
企业账面余额：XXXXX.XX
加：银行已收企业未收：XXXXX.XX（含 X 笔）
减：银行已付企业未付：XXXXX.XX（含 X 笔）
调整后余额：XXXXX.XX
银行对账单余额：XXXXX.XX
差异：XXXXX.XX（差异应为 0）
```

### 类型四：往来长期未清

**典型场景：**
- 供应商应付账款账龄 > 90 天
- 客户应收账款账龄 > 180 天
- 其他应收款中有员工借款挂账 > 12 个月

**处理方式：**

| 账龄 | 处理 |
|---|---|
| 90-180 天 | ⚠️ 标注"长期未清"，通知责任会计跟进 |
| 180-360 天 | 🔴 发送催办函，要求说明预计清理时间 |
| > 360 天 | 🔴 升级至财务经理，确认是否需做坏账准备 |
| 员工借款 > 12 个月 | 🔴 停止该员工的新借款审批，直至清账 |

---

### 类型五：报表钩稽不一致

**典型场景：**
- 资产负债表"未分配利润"≠利润表"净利润"累计本期金额
- 现金流量表"现金及等价物期末余额"≠资产负债表"货币资金"
- 所有者权益变动表与资产负债表不匹配

**处理方式：**

| 钩稽关系 | 异常原因 | 处理 |
|---|---|---|
| 未分配利润勾稽 | 期初未分配利润 + 本期净利润 - 本期分红 ≠ 期末未分配利润 | 排查是否有分红/利润分配凭证漏做 |
| 现金勾稽 | 现金流量表净额 + 期初现金 ≠ 期末现金 | 排查现金流量表分类是否正确 |
| 权益勾稽 | 期末权益变动未体现在资产负债表中 | 排查权益变动表与凭证对应关系 |

**排查路径：**
```
第一步：从 ERP 导出三表（资产负债表/利润表/现金流量表）
第二步：执行标准钩稽公式（见下）
第三步：如不一致，定位差异金额最接近的科目
第四步：检查该科目的凭证是否完整
```

**标准钩稽公式：**
```
资产负债表：
  资产总计 = 负债总计 + 所有者权益总计

利润表 + 资产负债表：
  期末未分配利润 = 期初未分配利润 + 本期净利润 - 本期分红 - 本期计提盈余公积

现金流量表 + 资产负债表：
  现金及等价物期末余额 = 资产负债表货币资金期末余额

所有者权益变动：
  期末所有者权益 = 期初所有者权益 + 本期增碱变动
```

### 类型六：税务申报数据与账面不一致

**典型场景：**
- 增值税申报销售额 > 账面收入（差异可能合理）
- 企业所得税申报的利润总额与账面利润不一致
- 个税代扣代缴金额与工资系统扣款不一致

**处理方式：**
```
差异 ≤ 1000 元：
  → 记录差异说明，允许结账
  → 下月申报时调整

差异 1000-50000 元：
  → ⚠️ 标注差异
  → 税务专员确认是否影响申报
  → 如不影响，提供书面说明

差异 > 50000 元：
  → 🔴 停止结账
  → 税务专员排查差异原因
  → 不允许在差异查明前完成结账
```

## 置信度评估表

| 置信度 | 条件 | 操作 |
|---|---|---|
| 高 | 试算平衡 ✅ + 银行已对账 ✅ + 往来已核销 ✅ + 报表钩稽 ✅ | ✅ 可提交结账 |
| 中 | 试算平衡，但有少量银行未达账项或往来未清项 | ⚠️ 标注异常，可有限结账 |
| 低 | 试算不平衡，或报表钩稽不一致 | 🔴 停止结账，须排查 |
| 无 | 存在重大金额差异（> 10 万元）或跨期问题 | ❌ 不允许结账 |

---

## 升级触发器

满足以下任一条件，自动停止结账，升级至财务经理：

- 试算表借贷差异 > 100 元
- 银行未达账项涉及金额 > 5000 元且无法查明原因
- 同一供应商/客户往来账龄 > 180 天
- 报表钩稽差异 > [填空：如 1 万元]
- 税务申报数据与账面差异 > 5 万元
- 结账进度落后于规定时间节点 2 个工作日
- ERP 系统无法在规定时间内导出报表
