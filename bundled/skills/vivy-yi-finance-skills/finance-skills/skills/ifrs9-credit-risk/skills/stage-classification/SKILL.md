# /stage-classification — IFRS 9 三阶段分类

## 第一步：收集敞口信用信息

```
□ 对每个客户/交易收集：
□ 客户名称：[...]
□ 产品类型：[对公贷款/零售贷款/信用卡/债券投资/表外]
□ 账面余额：[X] 万元
□ 当前评级：[AAA/AA/A/BBB/BB/B/C/D]
□ 到期日：[YYYY-MM-DD]
□ 逾期天数：[X] 天（如有）
□ 抵押品：[无 / 有 — 类型/评估价值/折价率]
□ 初始确认日：[YYYY-MM-DD]
□ Watch List 状态：[是/否]

□ 数据完整性检查：
□ 评级缺失客户：[X] 个 — [✅ 已补全 / ⚠️ 须处理]
□ 抵押品信息缺失：[X] 个 — [✅ 已补全 / ⚠️ 须处理]
```

---

## 第二步：Stage 3 判断（先排除法）

```
□ Stage 3 客观减值证据（任一触发即 Stage 3）：

□ 🔴 逾期 90 天以上
□ 债务人重大财务困难（审计/评级下调至 D 级）
□ 债务人获得贷款优惠（债务重组条款变更）
□ 债权人已处置抵押品
□ 债务人进入破产/清算程序
□ 其他客观证据：濒临违约的合同条款变更

□ Stage 3 判断结果：
□ Stage 3 敞口：[X] 万元（[X] 个）
□ 主要触发原因：[...]
□ 是否已逾期但尚未识别：[✅ 是 / ⚠️ 否]
```

---

## 第三步：Stage 2 SICR 判断（排除 Stage 3 后）

```
□ SICR 触发条件（来自 CLAUDE.md）：

□ 🔴 逾期 30-89 天
□ ⚠️ PD 评级下降 [X] 个等级（vs 初始确认日评级）
□ ⚠️ 进入 Watch List / Early Alert
□ ⚠️ 负面公开信息（法律诉讼/监管调查/舆情负面）
□ ⚠️ 宏观高风险行业暴露（参考 CLAUDE.md 行业清单）

□ PD 下降计算：
□ 初始评级：[X]
□ 当前评级：[X]
□ 评级下降：[X] 级 — [✅ 触发 SICR / ⚠️ 未触发]

□ Stage 2 判断结果：
□ Stage 2 敞口：[X] 万元（[X] 个）
□ 主要触发原因：[...]
```

---

## 第四步：Stage 1 判断（其余敞口）

```
□ Stage 1 条件：
□ 无 Stage 3 条件
□ 无 Stage 2 SICR 条件
□ 初始确认即 Stage 1（除非后续触发 Stage 2/3）

□ 低信用风险豁免（来自 CLAUDE.md）：
□ 适用条件：外部评级 [A-及以上] + 到期日 < [X] 个月
□ 是否适用豁免：[是/否]
□ 豁免依据：[评级文件]

□ Stage 1 判断结果：
□ Stage 1 敞口：[X] 万元（[X] 个）
□ 豁免低信用风险：[X] 个（[X] 万元）
```

---

## 第五步：Stage 转移记录与转回判断

```
□ 转移记录：
□ 期初 Stage 1 → 期末仍 Stage 1：[X] 万元
□ 期初 Stage 1 → 期末 Stage 2：[X] 万元（[X] 个）
□ 期初 Stage 2 → 期末 Stage 3：[X] 万元（[X] 个）
□ 期初 Stage 2 → 期末转回 Stage 1：[X] 万元（[X] 个）

□ 转回判断（Stage 2 → Stage 1）：
□ 转回条件：连续 [X] 期正常还款 + 当前评级处于投资级
□ 本期可转回客户：[X] 个 — [✅ 已转回 / ⚠️ 待验证]

□ 转移原因摘要：
□ [原因 1]：[X] 个客户 / [X] 万元
□ [原因 2]：[X] 个客户 / [X] 万元
```
