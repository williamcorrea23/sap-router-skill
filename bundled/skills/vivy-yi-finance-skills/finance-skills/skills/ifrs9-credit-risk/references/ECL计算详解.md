# IFRS 9 ECL 计算详解

本文件为 ECL 计量场景提供详细的算法说明、计算公式、行业基准参数和真实案例。配合 `ecl-calculation` / `stage-classification` / `macro-forward-adjustment` 三个 skill 使用。

---

## 一、ECL 基本公式

```
ECL = EAD × PD × LGD × DF
```

其中：
- **EAD**（Exposure at Default）= 违约时的敞口（含表外承诺 × CCF）
- **PD**（Probability of Default）= 违约概率
- **LGD**（Loss Given Default）= 违约损失率（1 - 回收率）
- **DF**（Discount Factor）= 折现因子 `1 / (1 + EIR)^t`，t 是违约时点

按 Stage 区分：

| Stage | PD 范围 | LGD | 折现 | 公式 |
|-------|--------|-----|------|------|
| Stage 1 | 12 个月 PD | 12 个月 LGD | 12 个月 | `ECL_1 = EAD × PD_12m × LGD_12m / (1 + EIR)^0.5` |
| Stage 2 | 存续期累计 PD | 存续期 LGD | 整个存续期 | `ECL_2 = Σ[t=1..T] (EAD_t × marginal_PD_t × LGD_t / (1 + EIR)^t)` |
| Stage 3 | PD=100% | 已减值 LGD（净现值） | 实际损失日 | `ECL_3 = EAD - PV(回收额)` |

**简化实操**：

实务中常用"现期敞口法"（current exposure approach），不对每期敞口重估：

```
ECL_Stage2 = EAD_current × cumulative_PD_remaining × LGD × avg_DF_remaining
```

## 二、PD 估算方法

### 2.1 内部评级法（IRB）

```
PD_internal = f(内部评级, 产品类型, 期限, 宏观因子)
```

**典型内部评级 PD 映射（巴塞尔 IRB 校准参考）**：

| 内部评级 | 12 个月 PD 区间 | 平均 PD | 适用客户 |
|---------|----------------|---------|---------|
| AAA | 0.01% - 0.05% | 0.03% | 大型央企/政策性银行 |
| AA | 0.05% - 0.10% | 0.07% | 大型国企/上市银行 |
| A | 0.10% - 0.30% | 0.20% | 优质上市公司 |
| BBB | 0.30% - 1.00% | 0.60% | 投资级 |
| BB | 1.00% - 3.00% | 2.00% | 投机级（关注类） |
| B | 3.00% - 8.00% | 5.50% | 高风险 |
| CCC | 8.00% - 20.00% | 14.00% | 高度投机 |
| CC | 20.00% - 50.00% | 35.00% | 重大风险 |
| C | 50.00% - 99.99% | 75.00% | 濒临违约 |
| D | 100% | 100% | 违约 |

**注意**：上表为参考值。实际机构应基于自身历史违约数据校准。

### 2.2 外部评级映射法

小型机构或新业务无足够内部数据时，用外部评级映射：

```
PD_internal = f(外部评级, 行业, 期限)
```

**标普 / 穆迪 PD 区间（年化）**：

| 外部评级（S&P / Moody's） | 1 年 PD | 5 年累计 PD |
|--------------------------|---------|------------|
| AAA / Aaa | 0.01% | 0.10% |
| AA / Aa | 0.05% | 0.40% |
| A / A | 0.10% | 0.80% |
| BBB / Baa | 0.30% | 2.50% |
| BB / Ba | 1.50% | 12.00% |
| B / B | 5.00% | 35.00% |
| CCC / Caa | 25.00% | 75.00% |
| D / C | 100% | 100% |

### 2.3 零售业务 PD 估算

零售业务（信用卡/个人贷款）通常**不用内部评级**，改用**行为评分 + 期限**：

```
PD_零售 = f(行为评分, 账龄, 逾期历史, 行业指数)
```

**典型零售 PD 区间**：
- 优质客户（行为评分 750+）：12 个月 PD 0.3% - 0.8%
- 正常客户（评分 650-750）：12 个月 PD 0.8% - 3.0%
- 关注客户（评分 550-650）：12 个月 PD 3.0% - 10.0%
- 高风险（评分 < 550）：12 个月 PD 10.0% - 30.0%

## 三、LGD 估算方法

### 3.1 有抵押 LGD

```
LGD = 1 - (抵押品价值 × 折价率 + 净现金回收) / EAD
```

**典型抵押品折价率**（Basel III 监管值参考）：

| 抵押品类型 | 折价率（haircut） | 适用情形 |
|----------|------------------|---------|
| 现金 / 银行存款 | 0% | 完全对等 |
| 国债 / 央行票据 | 5% - 10% | 高流动性债券 |
| 政策性金融债 | 10% - 15% | 准主权级 |
| 优质公司债（AA+及以上） | 15% - 25% | 上市债券 |
| 普通公司债（A - BBB） | 25% - 40% | 上市债券 |
| 上市股权（主板蓝筹） | 30% - 40% | 大盘股 |
| 上市股权（中小板/创业板） | 40% - 60% | 高波动 |
| 房地产（住宅） | 30% - 40% | 重点城市 |
| 房地产（商业） | 40% - 50% | 一线商业 |
| 房地产（工业） | 50% - 60% | 标准厂房 |
| 机器设备 | 60% - 80% | 专用设备 |
| 应收账款 | 50% - 70% | 优质客户应收 |
| 存货 | 60% - 80% | 标准化存货 |

### 3.2 无抵押 LGD

无抵押 / 抵押不足时使用**经验值**：

| 客户类型 | 无抵押 LGD 区间 | 典型值 |
|---------|----------------|--------|
| 大型对公（上市公司） | 40% - 60% | 50% |
| 中型对公 | 50% - 70% | 60% |
| 小型对公 | 60% - 80% | 70% |
| 个人住房贷款（LTV ≤ 80%） | 5% - 15% | 10% |
| 个人住房贷款（LTV > 80%） | 15% - 30% | 20% |
| 信用卡无抵押 | 70% - 90% | 80% |
| 小微企业贷款 | 60% - 80% | 70% |

### 3.3 LGD 时间结构

回收需要时间，ECL 计量中应反映 LGD 的"时间分布"：

```
LGD_存续期 = Σ[t=1..T] (回收_t × DF_t)
```

**典型回收周期**：
- 抵押品处置：12 - 36 个月
- 担保代偿：6 - 18 个月
- 现金回收：3 - 12 个月
- 法律诉讼：18 - 48 个月

**简化**：用平均回收周期对应的 DF 调整 LGD。

## 四、EAD 与 CCF

### 4.1 表内敞口

```
EAD_表内 = 账面余额 - 已减值部分
```

### 4.2 表外敞口

表外承诺（未提取信贷额度 / 担保 / 信用证）：

```
EAD_表外 = 已承诺金额 × CCF
```

**CCF 监管参考值**（Basel 框架）：

| 表外项目 | CCF | 备注 |
|---------|-----|------|
| 不可撤销贷款承诺（原到期 ≤ 1 年） | 20% | 短期周转额度 |
| 不可撤销贷款承诺（原到期 > 1 年） | 50% | 中长期授信 |
| 担保 / 信用证 | 50% | 履约担保 |
| 跟单信用证 | 20% | 短期贸易融资 |
| 票据发行便利（NIF） | 50% | 备用融资 |
| 循环授信未提取部分 | 50% - 75% | 信用卡/循环贷 |

**重要**：表外敞口对 Stage 1 / Stage 2 同样要计提 ECL。

## 五、SICR 判断（Stage 转移核心）

**信用风险显著上升（Significant Increase in Credit Risk）** 是 Stage 1 → Stage 2 的触发条件。

### 5.1 SICR 量化标准

**主要触发条件**（任一触发即认定 SICR）：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 逾期天数 | ≥ 30 天 | **绝对底线**（30 DPD） |
| 评级下降 | ≥ 2 个等级 | 内部评级迁移 |
| PD 相对变化 | PD_当前 / PD_初始 > 某个阈值 | 例如 2.0x 或 1.5x |
| PD 绝对变化 | PD_当前 - PD_初始 > 0.5% | 经济意义 |
| 评级映射外部评级 | 投资级 → 投机级 | 重要节点 |
| Watch List | 进入预警名单 | 主动监控 |
| 监管评级 | 监管评级下调 | 间接信号 |
| 其他定性指标 | 重大不利变化 | 重大诉讼/重组/收购 |

### 5.2 常用 SICR 规则设计

**规则 1：评级迁移法**（推荐）
```
SICR = (内部评级下降 ≥ 2 个等级) OR (外部评级投资级 → 投机级)
```

**规则 2：PD 倍数法**
```
SICR = (PD_12m_当前 / PD_12m_初始) > [2.0] OR (PD_当前 - PD_初始) > 0.5%
```

**规则 3：组合规则**（最稳健）
```
SICR = (
    (评级下降 ≥ 1 等级 AND 评级从投资级降至投机级)
    OR (PD 相对变化 > 2.0x)
    OR (逾期 ≥ 30 天)
    OR (进入 Watch List)
    OR (其他重大不利变化)
)
```

### 5.3 Stage 转回规则

| 来源 Stage | 目标 Stage | 条件 |
|----------|----------|------|
| Stage 2 | Stage 1 | SICR 因素消失 + 连续 [N] 期正常还款（典型 3 期/6 个月） |
| Stage 3 | Stage 2 | 减值因素消失 + 信贷委员会审批 |
| Stage 3 | Stage 1 | 极少发生，通常直接到 Stage 2 |

**重要**：Stage 2 → Stage 1 的"观察期"是关键 — 太短会过早转回，太长会过度拨备。

## 六、宏观前瞻调整详细方法

### 6.1 三情景法（IFRS 9 推荐）

```
ECL_前瞻 = Σ[i=1..3] weight_i × ECL_scenario_i
```

**典型权重**：
- 乐观：20% - 30%
- 基准：40% - 60%
- 悲观：20% - 30%

**注意**：权重之和必须 = 100%。权重反映管理层对未来经济走势的判断，需文档化。

### 6.2 宏观因子选择

**核心三因子**（推荐）：

| 因子 | 数据来源 | 频率 | 转换 |
|------|---------|------|------|
| GDP 实际增速 | 国家统计局 | 季度 | 直接使用 |
| CPI 同比 | 国家统计局 | 月度 | 通胀传导 |
| 失业率 | 国家统计局 | 季度 | 失业率敏感度 |

**行业因子**（按需）：

| 行业 | 因子 |
|------|------|
| 房地产 | 国房景气指数 / 70城房价 |
| 制造业 | PMI / PPI |
| 批发零售 | 社零增速 |
| 出口企业 | 人民币汇率 / 海外 PMI |
| 基建 | 固定资产投资增速 |

### 6.3 PD 调整系数建模

**方法 A：简单敏感度**

```
PD_调整 = PD_基础 × (1 + β × ΔX)
```

其中：
- ΔX = 宏观因子偏离基准的百分比
- β = 弹性系数（历史回归得到）

**典型 β 值**（参考）：

| 客户类型 | GDP β | CPI β | 失业率 β |
|---------|-------|-------|---------|
| 大型对公 | -0.3 | +0.5 | +0.4 |
| 中型对公 | -0.5 | +0.8 | +0.6 |
| 小微对公 | -0.8 | +1.2 | +0.9 |
| 个人住房 | -0.2 | +0.3 | +0.5 |
| 信用卡 | -0.6 | +1.0 | +0.8 |

**方法 B：宏观回归模型**（更严谨）

建立 PD 与宏观因子的回归方程：
```
logit(PD) = α + β₁ × GDP + β₂ × CPI + β₃ × UNEMP + ε
```

**方法 C：违约率时序模型**

直接建模行业违约率与宏观因子的关系：
```
ln(行业违约率) = α + β × GDP_gap + ε
```

其中 GDP_gap = 实际 GDP - 潜在 GDP。

### 6.4 完整计算案例

**场景**：
- 客户 A（中型制造业，BBB 评级）
- EAD = 1000 万元
- 12 个月 PD_基础 = 0.60%
- LGD = 60%（无抵押）
- 3 情景宏观假设（评估日 2026-06-30）：

| 情景 | 权重 | GDP 2026E | 调整后 PD |
|------|------|----------|----------|
| 乐观 | 25% | +5.5%（基准 5% +0.5） | 0.45% |
| 基准 | 50% | +5.0% | 0.60% |
| 悲观 | 25% | +3.5%（基准 -1.5） | 0.90% |

**计算**：
```
ECL_乐观 = 1000 × 0.45% × 60% × 1/(1+5%)^0.5 = 2.66 万
ECL_基准 = 1000 × 0.60% × 60% × 1/(1+5%)^0.5 = 3.55 万
ECL_悲观 = 1000 × 0.90% × 60% × 1/(1+5%)^0.5 = 5.32 万

ECL_加权 = 2.66 × 25% + 3.55 × 50% + 5.32 × 25% = 3.77 万
```

**Stage 分类**：
- 当前评级 BBB，无逾期，无 Watch List → **Stage 1**
- 选用 12 个月 PD → `ECL = 3.77 万元`
- 覆盖率 = 3.77 / 1000 = **0.38%**

## 七、Stage 2 完整例子

**场景**：
- 客户 B（中型制造业，A 评级 → BB 评级，下降 2 等级）
- EAD = 5000 万元，剩余期限 3 年
- 当前评级 BB，存续期累计 PD = 12%
- LGD = 60%
- 3 情景 PD 调整：8% / 12% / 18%

**计算**：
```
ECL_乐观 = 5000 × 8% × 60% × avg_DF(3年) = 240 × 0.91 = 218.4 万
ECL_基准 = 5000 × 12% × 60% × 0.91 = 327.6 万
ECL_悲观 = 5000 × 18% × 60% × 0.91 = 491.4 万

ECL_加权 = 218.4 × 25% + 327.6 × 50% + 491.4 × 25% = 341.25 万
```

**Stage 分类**：
- 评级下降 2 等级，触发 SICR → **Stage 2**
- 选用存续期 PD → `ECL = 341.25 万元`
- 覆盖率 = 341.25 / 5000 = **6.83%**

## 八、Stage 3 完整例子

**场景**：
- 客户 C（小型零售，逾期 95 天，评级 D）
- EAD = 200 万元
- 抵押品：住宅评估值 180 万，折价 40%
- 抵押品净回收现值 = 180 × 60% × 0.95 (时间折扣) = 102.6 万

**计算**：
```
LGD = 1 - 102.6 / 200 = 48.7%
ECL = 200 × 100% × 48.7% = 97.4 万
```

**Stage 分类**：
- 逾期 95 天 > 90 天 → **Stage 3**
- `ECL = 97.4 万元`
- 覆盖率 = 97.4 / 200 = **48.7%**

## 九、ECL 覆盖率合理性验证

完成 ECL 计量后，对比覆盖率与行业基准：

| 客户类型 | Stage 1 覆盖率 | Stage 2 覆盖率 | Stage 3 覆盖率 |
|---------|----------------|----------------|----------------|
| 大型对公 | 0.1% - 0.5% | 1% - 5% | 30% - 60% |
| 中型对公 | 0.3% - 1.0% | 3% - 10% | 40% - 70% |
| 小型对公 | 0.5% - 2.0% | 5% - 15% | 50% - 80% |
| 个人住房 | 0.05% - 0.30% | 0.5% - 2% | 5% - 30% |
| 信用卡 | 0.5% - 1.5% | 5% - 20% | 60% - 90% |

**异常情况警示**：
- Stage 1 覆盖率 > 2%：可能 SICR 判断过松（应进 Stage 2）
- Stage 2 覆盖率 < 1%：可能 LGD 估计过低
- Stage 3 覆盖率 < 30%：可能抵押品过度乐观

## 十、ECL 披露要点（IFRS 7）

### 10.1 必披露项

```
- Stage 1/2/3 敞口余额（按产品/行业/地区拆分）
- Stage 1/2/3 ECL 余额 + 当期变动（迁徙表）
- 前瞻性信息：宏观假设、情景权重、调整后 PD/LGD
- 信用风险显著上升（SICR）判断方法
- 评级体系说明 + 评级映射
- 抵押品类型与折价率
- 最大信用风险敞口（MR4 reconciliation）
```

### 10.2 推荐披露

```
- ECL 敏感性分析（宏观因子 ±1%）
- Stage 转移矩阵（本期迁徙 vs 上期）
- ECL 拨备覆盖率（按产品/行业）
- 集中度风险（单一客户/行业占比）
- 模型验证（backtesting）结果
```

详见 [ecl-disclosure skill](../skills/ecl-disclosure/SKILL.md)。

## 十一、模型验证与 Backtesting

**季度验证**：
```
- 实际 PD vs 预测 PD（中心趋势检验）
- 实际 LGD vs 预测 LGD
- Stage 转移准确性（SICR 误判率）
- 宏观调整合理性（调整方向 vs 实际违约方向）
```

**年度验证**：
```
- 模型整体重检
- 评级体系回顾
- PD/LGD/EAD 重新校准
- 关键假设压力测试
```

**Backtesting 关键指标**：

| 指标 | 公式 | 阈值 |
|------|------|------|
| 二项检验（Binomial） | 实际违约 vs 预测 PD | p-value > 0.05 |
| Hosmer-Lemeshow | 分组实际 vs 预测 | p-value > 0.05 |
| PSI（Population Stability Index） | Σ(实际% × ln(实际%/预测%)) | < 0.1 稳定 |
| CSI（Characteristic Stability Index） | 各评级实际 PD 稳定性 | < 0.1 稳定 |
