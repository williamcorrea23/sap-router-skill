# 判断框架 — IFRS 9 信用风险

## 核心原则

IFRS 9 信用风险的核心是"用前瞻性量化模型估计预期信用损失"。遇到判断时，优先区分"是否触发 SICR"、"处于哪个 Stage"、"宏观调整是否合理"。

---

## Stage 判断

```
□ 初始确认即 Stage 1，除非满足 Stage 2 或 Stage 3 条件。

□ Stage 转移判断：

□ Stage 1 → Stage 2：
  触发条件（来自 CLAUDE.md）：
  → 逾期 30 天以上
  → PD 评级下降 [X] 个等级
  → 进入 Watch List / Early Alert
  → 负面公开信息（法律诉讼/监管处罚/舆情）
  
□ Stage 2 → Stage 1 转回条件：
  → 连续 [X] 期正常还款
  → 当前 PD 评级处于投资级
  → 已移出 Watch List
  
□ Stage 3 触发（客观减值证据）：
  → 逾期 90 天以上
  → 债务人重大财务困难
  → 债务人重组条款（grant relief）
  → 可变现抵押品价值不足以覆盖敞口
  → 债权人放弃要求立即偿付的权利

□ Stage 3 → Stage 2/1：
  → 仅在特殊情况下由信贷委员会审批转回
  → 通常须重新评估信用质量
```

```
□ PD 合理性检查：
□ PD 与历史实际违约率的对比：[X]% vs 实际 [X]% — [✅ 一致 / ⚠️ 偏差较大]

□ 迁移矩阵是否与历史数据一致：[✅ 一致 / ⚠️ 偏差较大]

□ LGD 合理性检查：
□ LGD vs 历史回收率的对比：[X]% vs 实际 [X]% — [✅ 一致 / ⚠️ 偏差较大]
□ 抵押品价值确认：[✅ 已核实 / ⚠️ 须核实]
□ 回收周期假设：[X] 个月 vs 历史 [X] 个月

□ EAD 合理性检查：
□ CCF 系数是否与历史使用率一致：[✅ 一致 / ⚠️ 须调整]
□ 表外敞口计量是否完整：[✅ 是 / ⚠️ 遗漏项：...]
```

## 前瞻调整（Forward-Looking）判断

```
□ 宏观因子的选择：
□ GDP 增速：基准 [X]%，乐观 [+X]%，悲观 [-X]%
□ 房价指数（如适用）：基准 [X]%，乐观 [+X]%，悲观 [-X]%
□ 失业率：基准 [X]%，乐观 [-X]%，悲观 [+X]%

□ 宏观调整的合理性：
□ 基准宏观假设 vs 市场一致预期：[✅ 一致 / ⚠️ 偏离]
□ 乐观/悲观情景权重分配：[X]% / [X]% / [X]%
□ 是否参考央行/IMF/WB 的宏观预测：[✅ 是 / ⚠️ 否]

□ 避免的前瞻调整错误：
□ 🔴 零前瞻调整：IFRS 9 明确要求，前瞻是核心
□ 🔴 机械套用历史数据：须反映当前和预期经济状况
□ 🔴 过度依赖单一宏观因子：建议至少 [X] 个因子

```

```
□ ECL 覆盖率是否充足：[✅ 充足 / ⚠️ 偏低 / 🔴 不足]
□ 是否低于同业均值：[X]% vs 同业均值 [X]% — [✅ 在合理范围 / ⚠️ 须关注]

□ 迁徙分析（NCR）：
□ 期初 → 期末 Stage 迁徙矩阵
□ 各 Stage 净转出（转好）vs 净转入（转差）
□ 迁徙是否符合预期：[✅ 是 / ⚠️ 异常]
```

## 升级触发条件

- Stage 3 敞口占比 > [X]%
- ECL 覆盖率低于监管最低要求
- 前瞻调整与市场共识严重偏离
- 模型验证发现 PD/LGD 显著高估或低估
- 外部审计对 ECL 模型提出保留意见
