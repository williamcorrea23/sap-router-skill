# /confirmation-management — 函证管理


## Examples

→ 示例：用户说"年度审计的银行询证函发出去了，有几家没回，需要再发一次吗"，系统应调用本技能，跟踪银行询证函回函情况。

→ 示例：用户说"有个客户是上市公司，对账函的格式要求特别严格，帮我调整一下"，系统应调用本技能，按客户要求格式准备对账函。

→ 示例：用户说"审计师要求我们对所有余额超过 100 万的客户发函证，帮我安排一下"，系统应调用本技能，按余额阈值制定询证函发送计划。
## 第一步：确认函证范围

**确认须发函的银行和往来单位：**

```
□ 银行函证范围：
□ 须发函银行：[X] 家
□ 须函证内容：[存款/借款/担保/票据/其他]

□ 往来函证范围：
□ 须发函客户：[X] 家
□ 须发函供应商：[X] 家
□ 须发函其他往来：[X] 家

□ 发函截止：[YYYY-MM-DD]
□ 回函截止：[YYYY-MM-DD]
```

```

□ 处理方案：

□ 银行冻结/抵押：[X] 家
```
