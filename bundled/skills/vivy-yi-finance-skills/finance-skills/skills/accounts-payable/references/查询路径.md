# 查询路径 — 应付账款

## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | ERP AP 模块（[ERP]）| 供应商应付余额/到期日/付款记录 | 最优先 |
| 2 | 合同/采购订单系统（[PO]）| 采购订单/框架合同 | 核对依据 |
| 3 | 收货系统（[RCV]）| 采购收货/服务验收 | 确认应付 |
| 4 | 发票系统（[INV]）| 供应商发票登记 | 付款依据 |
| 5 | 资金系统（[CASH]）| 可用资金/银行余额 | 付款排期 |

---

## 常用查询场景

### 场景 1：应付账款到期分析

**查询入口：**
```
ERP SAP：
- FK01N：显示供应商（应付账款视图）
- FBL1N：供应商行项目（显示所有未清项）
- MIRA：发票校验记录

用友/金蝶：
- 应付账款 → 供应商余额表
- 应付账款 → 到期分析表
```

**到期分析维度：**
```
按供应商：
  - 各供应商应付余额
  - 到期日分布
  - 逾期天数

按金额：
  - 单笔金额分布（小额/中额/大额）
  - TOP 10 大额应付

按到期日：
  - 已到期应付
  - 未来 7 天到期应付
  - 未来 30 天到期应付
```

---

### 场景 2：三单匹配查询

**查询入口：**
```
ERP SAP：
- ME21N：创建采购订单
- MIGO：收货确认
- MIRO：发票校验

用友/金蝶：
- 采购管理 → 采购订单
- 库存管理 → 采购入库
- 应付管理 → 供应商发票
```

**匹配状态：**
```
完全匹配（GR/IR）：
  → 采购订单 - 收货单 - 发票三者一致
  → 可安排付款

差异状态：
  → 金额差异：发票 vs 订单
  → 数量差异：发票 vs 收货
  → 价格差异：发票 vs 合同单价
```

---

### 场景 3：供应商付款历史

**查询入口：**
```
ERP SAP：
- FBL1N：供应商行项目（历史付款记录）
- FB03：显示付款凭证

用友/金蝶：
- 应付账款 → 付款明细
```

**付款历史分析：**
```
付款习惯：
  - 正常付款周期（是否按合同执行）
  - 是否有过逾期（逾期次数/天数）
  - 是否有付款条款变更请求

供应商绩效：
  - 发票准确率（发票与订单匹配率）
  - 交付及时率（是否按时交货）
  - 质量问题频率
```

---

### 场景 4：资金安排分析

**查询入口：**
```
资金系统（[CASH]）：
- 现金管理系统 → 可用资金查询
- 银行余额查询 → 各账户余额

ERP：
- SAP FCH5：显示付款计划
- 资金管理系统 → 现金流预测
```

**资金排期：**
```
可用资金 vs 应付到期：
  - 可用资金足够：按计划付款
  - 可用资金不足：按优先级支付

付款优先级：
  1. A 级战略供应商
  2. 已逾期应付
  3. 到期日最早
  4. 其他应付
```

---

### 场景 5：供应商争议查询

**查询入口：**
```
ERP：
- 供应商行项目中的争议项标记
- 历史争议记录

OA 审批系统（[OA]）：
- 采购投诉/质量问题的审批记录
```

**争议处理跟踪：**
```
争议状态：
  - 新争议（待确认）
  - 沟通中（与供应商协商）
  - 已解决（部分付款/全额付款/扣款）
  - 已升级（升级至管理层/法务）

争议金额分布：
  - 按争议类型（质量/数量/价格/交付）
  - 按供应商（哪个供应商争议最多）
```

## 查询结果处理

### 数据完整性判断

```
三方数据一致（[ERP] + [PO] + [INV]）：
→ 高置信度，可直接安排付款

两方一致，有差异：
→ 中置信度，核查差异原因后再付款

数据差异较大：
→ 低置信度，暂停付款，升级处理
```
