# /regulatory-reporting — 监管报告编制与报送


## Examples

→ 示例：用户说"银保监的监管报表这个月截止日期提前了 5 天，帮我调整一下时间表"，系统应调用本技能，更新监管报告截止日期提醒。

→ 示例：用户说"有个监管指标计算口径变了，帮我重新计算一下历史数据"，系统应调用本技能，执行新口径下的监管指标重述。

→ 示例：用户说"监管检查要来，需要提前准备哪些材料"，系统应调用本技能，准备监管检查材料清单。
## 第一步：确认报告要求

**根据报告类型确认报送要求：**

```
□ 报告类型：[月度财务数据/季度报告/年度报告]
□ 报告期间：[YYYY年MM月/年度]
□ 报送对象：[国资委/统计局/财政厅等]
□ 报送截止：[YYYY-MM-DD]
□ 剩余天数：[X] 天

□ 报告内容清单：
□ 主要财务指标
□ 资产负债表
□ 利润表
□ 其他：[具体要求]
```

## 第三步：核查数据准确性

**核查数据是否与 ERP 一致：**

```
□ 数据核对：
□ 营业收入：ERP [XXX万] — 报告填列 [XXX万] — 一致 [✅/⚠️]
□ 利润总额：ERP [XXX万] — 报告填列 [XXX万] — 一致 [✅/⚠️]
□ 资产总额：ERP [XXX万] — 报告填列 [XXX万] — 一致 [✅/⚠️]
□ 负债总额：ERP [XXX万] — 报告填列 [XXX万] — 一致 [✅/⚠️]
□ 所有者权益：ERP [XXX万] — 报告填列 [XXX万] — 一致 [✅/⚠️]

□ 数据准确性：[✅ 准确 / ⚠️ 存在差异]
□ 差异项：[...]
□ 差异原因：[...]
□ 更正状态：[✅ 已更正 / ⚠️ 待更正]
```
