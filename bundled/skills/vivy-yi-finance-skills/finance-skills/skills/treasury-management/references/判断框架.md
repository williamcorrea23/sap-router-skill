# 判断框架 — 资金管理异常处理

## 核心原则

**资金管理的目标是"让公司永远有钱花，同时不让钱闲置"。**
遇到异常时，优先判断是"短期波动"还是"长期结构问题"，对资金缺口风险保持高度警惕。

---

## 异常类型与处理方式

### 类型一：账户余额不足

**典型场景：**
- 单个账户余额 < 预警值 [X] 万
- 付款时账户余额不足
- 账户出现透支

**判断流程：**

```
第一步：确认数据准确性

□ [CASH] 与 [BANK] 数据是否一致？
□ 是否有未达账项（银行已付但系统未记账）？
□ 是否有定时扣款未到账（水电费/税费）？

→ 如数据不一致：立即核对银行对账单

第二步：判断不足原因

短期流动性不足（暂时性）：
□ 收入未及时到账（大额客户晚付款）
□ 付款集中（月底供应商付款高峰）
□ 税款/工资等刚性支出集中扣款
→ 短期借款/调整付款时间

结构性资金不足（长期性）：
□ 收入持续低于支出（亏损经营）
□ 应收账款回收困难（DSO 持续上升）
□ 库存积压（资金占用在存货）
→ 须系统性解决（融资/资产处置/业务改善）

第三步：紧急应对

□ 立即措施：
  → 检查其他账户是否有足够资金调拨
  → 延迟非紧急付款（联系供应商协商）
  → 紧急申请银行授信

□ 短期措施（[X] 天内）：
  → 催收应收账款（优先催收大额/逾期）
  → 与银行沟通增加授信
  → 减少非必要支出

第四步：预防措施

□ 资金预测改进：
  → 提高预测准确性
  → 提前 [X] 天预警资金缺口
  → 设置多级预警阈值

□ 账户管理改进：
  → 保持主账户余额 > [X] 万
  → 设立备用金账户
  → 优化收付款账户结构
```

### 类型二：资金缺口预警（未来有缺口）

**典型场景：**
- 资金预测显示 [X] 天内可能出现缺口
- 大额支出即将发生（贷款到期/资本支出）
- 收入预测大幅下调

**判断流程：**

```
第一步：确认缺口规模

□ 预测资金收入：[XXX万]
□ 预测资金支出：[XXX万]
□ 预测净缺口：[XXX万]

□ 缺口出现时间：[YYYY-MM-DD]
□ 缺口持续时间：[X] 天
□ 最大缺口金额：[XXX万]

第二步：分析缺口原因

收入端减少：
□ 客户回款延迟（具体客户/金额）
□ 销售预期过高（实际订单减少）
□ 投资收益减少（理财产品到期）

支出端增加：
□ 大额贷款/债券到期
□ 资本支出集中（项目付款）
□ 税款/社保等刚性支出

外部因素：
□ 银行收紧信贷（抽贷/压降）
□ 市场资金面紧张

第三步：评估缺口风险

缺口风险等级：[⚠️中/🔴高/🔴极高]

□ 短期小额缺口（< [X] 万，< [X] 天）：
  → 催收应收账款
  → 延迟非紧急付款
  → 风险等级：⚠️ 中

□ 中期较大缺口（[X]-[X] 万，[X]-[X] 天）：
  → 申请银行短期借款
  → 处置短期金融资产
  → 风险等级：🔴 高

□ 长期大额缺口（> [X] 万，> [X] 天）：
  → 引入战略投资
  → 资产处置
  → 业务重组
  → 风险等级：🔴 极高

第四步：制定应对方案

□ 立即措施（[X] 天内）：
  → 催收到期应收账款
  → 延迟非紧急支出
  → 申请银行短期授信

□ 中期措施（[X]-[X] 天）：
  → 发行短期融资券
  → 供应链金融（应收账款融资）
  → 出售低效资产

□ 长期措施（> [X] 天）：
  → 股权融资
  → 资产证券化
  → 业务模式调整
```

### 类型三：大额异常付款

**典型场景：**
- 单笔付款 > [X] 万（超出权限）
- 新供应商首单付款
- 付款对象与合同不符
- 付款频率异常（同一供应商频繁付款）

**判断流程：**

```
第一步：确认付款信息

□ 付款申请单号：[编号]
□ 供应商名称：[名称]
□ 付款金额：[XXX万]
□ 付款用途：[描述]
□ 合同/订单号：[编号]

□ 是否在预算内？[是/否]
□ 是否有对应合同？[是/否]
□ 付款对象是否与合同一致？[是/否]

第二步：核实业务背景

□ 付款是否与业务需求匹配？
□ 供应商是否为公司认可的供应商？
□ 价格/数量是否与合同一致？
□ 是否有特殊审批？

□ 正常付款：
  → 进入正常审批流程
  → 优先安排付款

□ 可疑付款：
  → 暂停付款
  → 联系业务部门确认
  → 联系供应商确认
  → 如确认诈骗：立即报警

第三步：处理决策

□ 确认为正常付款：
  → 按权限完成审批
  → 安排付款
  → 记录备注

□ 确认为可疑付款：
  → 拒绝付款
  → 通知资金总监/CFO
  → 调查原因
  → 如确认诈骗：立即报警+冻结账户

□ 确认为业务变更导致的合理付款：
  → 补充审批
  → 事后补充合同/说明
  → 完成付款

第四步：事后改进

□ 付款异常原因分析：
  → 业务部门未及时沟通
  → 供应商信息变更未更新
  → 合同执行偏差

□ 内控改进措施：
  → 新供应商首单须额外审批
  → 大额付款须电话核实
  → 定期供应商名单审核
```

### 类型四：银行授信过度使用

**典型场景：**
- 授信使用率 > [X]%
- 单一银行授信即将用尽
- 授信即将到期但续期困难

**判断流程：**

```
第一步：确认授信状况

□ 各银行授信总额：[XXX万]
□ 已使用授信：[XXX万]
□ 可用授信：[XXX万]

□ 授信使用率：[X%]

□ 主要使用银行：
  - [银行A]：[XXX万] 已用 / [XXX万] 总授信（[X%]）
  - [银行B]：[XXX万] 已用 / [XXX万] 总授信（[X%]）

□ 授信到期日：
  - [银行A]：[YYYY-MM-DD]
  - [银行B]：[YYYY-MM-DD]

第二步：判断使用率高的原因

□ 业务扩张（合理）：
  → 收入增长带动授信需求增加
  → 主动增加借款支持业务
  → 须评估还款能力

□ 流动性紧张（风险）：
  → 回款困难导致依赖授信
  → 亏损导致自有资金不足
  → 须立即改善流动性

□ 授信结构不合理：
  → 短期授信占比过高
  → 缺少长期授信
  → 须优化授信结构

第三步：评估授信到期风险

□ 即将到期授信：[XXX万]（[银行]，[日期]）
□ 续期难度：[低/中/高]

□ 低难度：银行关系良好，财务指标正常
□ 中难度：需补充材料，财务指标略有波动
□ 高难度：银行收紧，行业风险，或公司财务恶化

第四步：制定应对方案

□ 增加授信：
  → 目标银行：[描述]
  → 目标额度：[XXX万]
  → 时间：[YYYY-MM-DD]
  → 责任人：[X]

□ 优化授信结构：
  → 增加长期授信比例
  → 减少短期授信依赖
  → 分散银行集中度

□ 减少授信使用：
  → 加速应收账款回收
  → 延长付款账期（与供应商协商）
  → 处置低效资产

□ 续授信准备：
  → 提前 [X] 个月启动续期谈判
  → 准备审计报告/财务报表
  → 维护银行关系
```

### 类型五：外汇风险

**典型场景：**
- 外币账户汇率大幅波动
- 外汇敞口超过限额
- 境外账户资金汇回困难

**判断流程：**

```
第一步：确认外汇敞口

□ 外币资产：[XXX万]（[币种]）
□ 外币负债：[XXX万]（[币种]）
□ 净敞口：[XXX万]（[币种]）

□ 汇率变动对利润影响：[±XXX万]
□ 是否触发止损线？[是/否]

第二步：判断汇率走势

□ 本币贬值趋势：
  → 对出口有利（收入增加）
  → 对进口不利（成本增加）
  → 对美元负债有利（债务减少）

□ 本币升值趋势：
  → 对出口不利（收入减少）
  → 对进口有利（成本减少）
  → 对美元负债不利（债务增加）

第三步：评估风险敞口

外汇风险等级：[低/中/高]

□ 低风险（敞口 < [X] 万美元）：
  → 无需对冲
  → 定期监控

□ 中风险（敞口 [X]-[X] 万美元）：
  → 考虑远期合约锁汇
  → 部分对冲敞口

□ 高风险（敞口 > [X] 万美元）：
  → 必须对冲
  → 使用远期/期权合约
  → 评估是否减少外币敞口

第四步：制定应对措施

□ 低风险应对：
  → 定期监控汇率变动
  → 设定止损/目标汇率

□ 中风险应对：
  → 签订远期外汇合约
  → 锁定期望汇率
  → 减少部分外币敞口

□ 高风险应对：
  → 全面对冲外币敞口
  → 与银行合作套期保值
  → 评估提前结汇/购汇
  → 考虑自然对冲（调整业务结构）
```

## 置信度评估

| 置信度 | 条件 | 操作 |
|---|---|---|---|
| 高 | [CASH] 与 [BANK] 一致，无未达账项 | ✅ 正常分析 |
| 中 | [CASH] 与 [BANK] 略有差异，少量未达 | ⚠️ 标注未达项 |
| 低 | 大量未达账项，或存在未确认付款 | 🔴 暂停，先核对银行对账单 |
