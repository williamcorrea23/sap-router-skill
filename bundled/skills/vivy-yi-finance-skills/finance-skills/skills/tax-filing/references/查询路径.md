# 查询路径 — 税务申报

## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | 电子税务局（[TAX]）| 申报记录/缴纳记录/完税证明 | 最优先 |
| 2 | 开票系统（[INV]）| 销项发票明细 | 销项数据 |
| 3 | 增值税发票管理系统（[TAX-VAT]）| 进项发票认证/勾选记录 | 进项数据 |
| 4 | ERP 总账（[ERP]）| 税金计提/应交税费科目余额 | 账务数据 |
| 5 | 薪酬系统（[HR]）| 工资薪金/社保公积金 | 个税数据 |
| 6 | 金税系统（[TAX-GOLD]）| 发票电子底账/协查信息 | 风险监控 |

## 常用查询场景

### 场景 1：增值税申报准备

**查询入口：**
```
电子税务局（[TAX]）：
- 增值税及附加税费申报（一般纳税人）/（小规模纳税人）
- 查看已填报的申报表
- 查看待补税/待退税金额

开票系统（[INV]）：
- 销项发票统计 → 按税率/按期间/按客户筛选
- 导出当月所有销项发票明细（发票代码/号码/开票日期/金额/税率/税额）

增值税发票管理系统（[TAX-VAT]）：
- 发票勾选 → 查看已勾选进项发票
- 统计当月进项税额
```

**申报前核对清单：**
```
□销项：开票系统当月开具的销项发票合计 = 申报表销项合计
□进项：当月勾选认证的进项发票合计 = 申报表进项合计
□进项转出：当月须进项转出的金额（如有异常凭证/用于非应税项目）
□销项税额 = 销售额 × 税率，逐行核对
□应纳税额 = 销项税额 - 进项税额 + 进项转出
□附加税 = 增值税实缴金额 × 附加税税率
```

### 场景 2：企业所得税预缴申报

**查询入口：**
```
电子税务局（[TAX]）：
- 企业所得税月（季）度申报
- 查看看已填报的申报表
- 查看弥补以前年度亏损数据

ERP 总账（[ERP]）：
- 利润表（累计）→ 营业收入/营业成本/利润总额
- 期间费用明细 → 管理费用/销售费用/财务费用
```

**预缴数据核对：**
```
营业收入：
  → ERP 利润表"营业收入"累计数 vs 申报表

营业成本：
  → ERP 利润表"营业成本"累计数 vs 申报表

利润总额：
  → ERP 利润表"利润总额"累计数 vs 申报表

注意：季度预缴按"实际利润额"计算，非累计
实际利润额 = 利润总额 - 不征税收入 - 免税收入 ± 税收优惠
```

### 场景 3：个人所得税代扣代缴

**查询入口：**
```
自然人电子税务局（[TAX]）：
- 分类所得申报（股息红利/稿酬/特许权使用费）
- 综合所得申报（工资薪金）— 通常由 [HR] 薪酬系统推送

薪酬系统（[HR]）：
- 工资明细 → 税前工资/社保公积金/专项附加扣除
- 查看当月个税扣缴金额
```

**个税核对清单：**
```
□工资薪金：月度工资表税前合计 vs 申报系统
□社保公积金：月度社保公积金个人部分合计 vs 申报系统
□专项附加扣除：当月采集的专项附加扣除信息 vs 申报系统
□实发金额 = 税前工资 - 社保公积金 - 个税
□检查员工专项附加扣除是否有效采集（否则按0扣除）
```

### 场景 4：发票验真

**查询入口：**
```
国家税务总局全国增值税发票查验平台：
- https://inv-veri.chinatax.gov.cn
- 支持增值税专用发票/普通发票/电子发票/卷式发票查验

增值税发票管理系统（[TAX-VAT]）：
- 勾选平台 → 查看发票状态（正常/作废/红冲/失控）
```

**验真操作：**
```
增值税专用发票/普通发票：
  1. 登录发票查验平台
  2. 输入发票代码/号码/日期/金额/验证码
  3. 查看发票状态和明细

异常发票状态：
  - 作废：发票已作废，不能抵扣
  - 红冲：发票已冲红，不能抵扣
  - 失控：开票方已走逃，不能抵扣，须进项转出
  - 异常：须进一步核实
```

### 场景 5：税收优惠备案查询

**查询入口：**
```
电子税务局（[TAX]）：
- 税收优惠备案 → 查看已备案的优惠项目
- 优惠事项：[高新技术企业/研发费用加计扣除/小型微利企业/...]
- 备案有效期：查看各优惠的到期时间

研发费用加计扣除：
  - 须有科技部门出具的研发项目立项文件
  - 须在企业所得税汇算清缴时填报《研发费用加计扣除优惠明细表》
```

**优惠资格核查：**
```
高新技术企业：
  → 证书编号 → 在"高新技术企业认定管理工作网"核查有效期
  → 证书过期则不得享受 15% 税率

研发费用加计扣除：
  → 核查研发项目是否在科技部门备案
  → 核查研发费用是否单独核算（未与生产费用混同）
  → 核查研发人员是否真实参与（考勤/工时记录）
```

### 场景 6：税务风险自查

**查询入口：**
```
金税系统（[TAX-GOLD]）：
- 增值税风险监控 → 税负率预警/进销项异常/发票风险
- 企业所得税风险监控 → 收入成本异常/期间费用异常

电子税务局（[TAX]）：
- 我的提醒 → 查看系统推送的各类风险提示
- 纳税信用评级 → 查看当年信用等级
```

**风险核查步骤：**
```
步骤 1：查看系统预警
  → 登录 [TAX-GOLD] 金税系统
  → 查看所有橙色/红色预警项

步骤 2：逐项核实
  → 对每项预警，判断是否为误报（业务原因）
  → 如是误报，准备书面说明
  → 如是真实风险，制定整改计划

步骤 3：主动纠错
  → 如发现申报错误，在申报期内更正申报
  → 如超过申报期，须提交更正申报申请
```

## 查询结果处理

### 数据完整性判断

```
数据完整（所有系统可用）：
→ 高置信度，可直接进行申报

数据部分缺失：
→ 中置信度，用其他系统数据补充
→ 标注数据来源

数据不可用：
→ 低置信度，暂停申报准备
→ 使用上次申报数据作为参考
→ 尽快联系系统管理员恢复
```
