# 判断框架 — 税务申报异常处理

## 核心原则

**税务合规的目标是"该交的税一分不少交，不该交的税一分不多交"。**
遇到异常情形时，优先判断是"合理商业安排"还是"恶意避税/偷逃税"。

---

## 异常类型与处理方式

### 类型一：税负率异常

**典型场景：**
- 增值税税负率长期低于行业参考值
- 企业所得税税负率异常偏低
- 税负率突然大幅下降（无明显业务原因）

**判断标准：**

```
第一步：确认数据准确性

□ 申报数据与 [TAX] 电子税务局一致
□ 销项税额与 [INV] 开票系统一致
□ 进项税额与 [TAX-VAT] 勾选认证记录一致
□ 计提税金与 [ERP] 总账一致

→ 如数据不一致：先调整数据，再判断是否异常

第二步：判断异常原因

合理原因（可解释）：
  - 业务季节性波动（淡旺季）
  - 存货结构变化（期初库存大/期末库存小）
  - 采购时点变化（集中采购导致进项多）
  - 产品结构变化（低税率产品占比增加）
  - 出口占比增加（增值税零税率）

不合理原因（须关注）：
  - 隐匿销售收入
  - 虚增进项税额
  - 关联交易转移利润
  - 阴阳合同/两套账

第三步：评估风险等级

税负率偏离参考值 ≤ 30%：
  → ✅ 正常范围内，持续关注

税负率偏离参考值 30-50%：
  → ⚠️ 中风险，须提供书面说明

税负率偏离参考值 > 50%：
  → 🔴 高风险，须立即核查
  → 须提交详细原因分析至税务经理
```

### 类型二：进项税额异常

**典型场景：**
- 进项税额持续大于销项税额
- 进项发票集中于少数供应商
- 农产品收购发票金额异常
- 旅客运输发票抵扣不合规

**判断标准：**

```
第一步：进项 vs 销项结构分析

连续 3 个月进项 > 销项：
  → ⚠️ 须关注
  → 分析业务原因（存货增加/业务淡季/产品转型）

连续 6 个月进项 > 销项：
  → 🔴 须税务经理介入
  → 核查是否存在滞留票/失控票

第二步：进项来源分析

进项发票集中于少数供应商（TOP 3 占比 > 60%）：
  → ⚠️ 须提供合理性说明
  → 核查供应商真实性（合同/物流/资金流）

农产品收购发票金额异常增长：
  → 🔴 高风险
  → 须核查农产品收购发票的真实性
  → 是否有实际货物入库

旅客运输发票抵扣不合规：
  → 常见不合规情形：
     - 日期不在在职员工出差期间
     - 抬头为个人而非公司
     - 金额明显异常（头等舱/高铁一等座超标准）
  → 须在认证截止日前做进项转出
```

### 类型三：申报与财务账差异

**典型场景：**
- 增值税申报收入与财务报表收入不一致
- 企业所得税申报收入与财务报表利润不一致
- 附加税计税依据与增值税不一致

**差异类型：**

```
暂时性差异（正常）：
  - 增值税：确认收入时间与开票时间不同
  - 企业所得税：确认收入时间与会计收入时间不同

永久性差异（须关注）：
  - 不征税收入（财政补贴/减免税款）
  - 免税收入（国债利息/居民企业股息）
  - 不可扣除项目（行政罚款/超标准业务招待费）

异常差异（须核查）：
  - 申报收入 < 财务账收入（可能隐匿收入）
  - 申报收入 > 财务账收入（可能确认收入规则错误）
  - 差异金额 > 申报收入的 10%（须查明原因）
```

### 类型四：发票异常

**典型场景：**
- 收到失控发票（开票方已走逃）
- 收到异常凭证（增值税扣税凭证异常）
- 发票与实际业务不符
- 发票日期/金额/内容异常

**失控发票处理：**

```
第一步：确认是否已抵扣

已抵扣：
  → 🔴 须做进项税额转出
  → 须向主管税务机关申请核实
  → 在未核实前不得再次抵扣

未抵扣：
  → ⚠️ 暂不抵扣，保留相关证据
  → 向主管税务机关报告

第二步：追查上游

→ 联系供应商，核实发票来源
→ 核查合同/物流单据/入库单
→ 如确认为虚假发票，向税务机关举报
```

**异常凭证处理：**

```
异常凭证（已抵扣）的处理时限：
  - 须在收到税务机关通知之日起 60 日内申请核实
  - 如核实后仍异常，须做进项转出

补救措施：
  - 进项转出后，相应成本须调增企业所得税
  - 如能重新取得合规发票，可申请核实抵扣
```

### 类型五：税收优惠资格存疑

**典型场景：**
- 高新技术企业证书到期但仍享受 15% 税率
- 研发费用加计扣除归集不合规
- 小型微利企业条件模糊

**研发费用加计扣除判断：**

```
不合规情形：
  - 研发人员比例不符合要求
  - 研发费用与生产费用混同（未分开核算）
  - 研发项目未通过科技部门立项/备案
  - 仅有设备折旧，无其他研发支出

合规要求：
  - 研发费用须单独设置明细账
  - 研发人员须是直接参与研发的人员
  - 须有项目立项文件/研发计划
  - 须保存相关凭证（记录研发过程）
```

## 置信度评估

### 申报数据置信度

| 置信度 | 条件 | 操作 |
|---|---|---|
| 高 | 三方数据（[TAX]+[INV]+[ERP]）完全一致 | ✅ 可直接申报 |
| 中 | 两方一致，有合理差异原因 | ⚠️ 标注差异原因后申报 |
| 低 | 三方均有差异或存在异常凭证 | 🔴 暂停申报，核查后申报 |

## 升级触发条件

满足以下任一条件，升级至税务经理：

- 税负率偏离参考值 > 50%
- 连续 6 个月进项 > 销项
- 收到失控发票/异常凭证（已抵扣）
- 申报与财务账差异 > [填空：如 5 万元]
- 须补税金额 > [填空：如 10 万元]
- 享受税收优惠资格存疑（高新技术/研发费用加计扣除）
- 触发税务机关评估/稽查通知
- 发票认证（勾选）逾期（360 天内未认证）
