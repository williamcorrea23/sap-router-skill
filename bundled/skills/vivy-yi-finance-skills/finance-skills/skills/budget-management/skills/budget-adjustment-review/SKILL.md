# /budget-adjustment-review — 预算调整审核


## Examples

→ 示例：用户说"销售部门申请追加 200 万市场费用，说是为了冲 Q4 业绩"，系统应调用本技能，评估追加预算的合理性并给出批准建议。

→ 示例：用户说"有几个部门科目间需要调剂，帮我按现有政策判断哪些能批"，系统应调用本技能，按政策执行预算调剂审核。

→ 示例：用户说"预算调整申请说是因为汇率损失，帮我核实一下这个理由是否充分"，系统应调用本技能，评估预算调整理由的真实性。
## 调整类型分类

| 调整类型 | 说明 | 审批层级 |
|---|---|---|
| **追加预算** | 增加某科目/部门的预算 | 见下方审批矩阵 |
| **削减预算** | 减少某科目/部门的预算 | 部门负责人即可 |
| **科目间调剂** | 同一部门内科目间调整 | 部门负责人审批 |
| **跨部门调剂** | 不同部门间预算调整 | 财务总监审批 |
| **跨期动用** | 用下期预算弥补本期 | CEO + 财务总监 |

## 第一步：接收申请信息

**必须获取的信息：**
```
基本信息：
  - 申请部门：[部门名称]
  - 调整科目：[科目名称]
  - 调整金额：[X万元]
  - 调整类型：[追加/削减/调剂]
  - 申请日期：[YYYY-MM-DD]
  - 申请人：[姓名]

调整原因（必须填写）：
  - 原因类别：[业务扩大/价格调整/突发需求/预算编制错误/其他]
  - 详细说明：[一段话描述]

业务回报（追加预算必须填写）：
  - 预期回报：[具体描述]
  - 回报时间：[YYYY-MM]
  - 回报衡量指标：[KPIs]
```

---

## 第二步：判断调整合理性

### 合理性检查清单

```
[ ] 调整原因是否清晰具体（不是泛泛的"业务需要"）
[ ] 调整金额是否与业务变化匹配（不是随意报大数）
[ ] 是否有可替代方案（不调整也能解决问题？）
[ ] 调整后是否影响其他部门/科目
[ ] 时间节点是否合理（不是季度末才想起来调整）
```

### 不合理调整的典型特征

```
特征 1：Q1/Q2 刚过就申请全年预算调整
  → 暗示预算编制质量有问题
  → 质疑：为什么不一开始就用正确数据

特征 2：调整幅度 > 30%
  → 暗示原预算假设有根本性错误
  → 质疑：原预算是否还有参考价值

特征 3：频繁申请调整（> 2次/年）
  → 暗示预算管控流程失效
  → 质疑：该部门的预算管理能力

特征 4：追加理由为"行业普遍超支"
  → 不能接受（别人的问题是别人的问题）
  → 质疑：本公司的具体情况是什么
```

## 第三步：判断资金来源

### 追加预算的资金来源

| 资金来源 | 可接受性 | 说明 |
|---|---|---|
| 从本部门其他科目削减 | ✅ 可接受 | 须提供削减科目明细 |
| 从其他部门调剂 | ⚠️ 需对方部门同意 | 须 OA 审批记录 |
| 从预备费/应急费 | ⚠️ 须说明动用理由 | 预备费有既定用途 |
| 新增预算（公司整体增加）| 🔴 须 CEO 审批 | 公司利润影响 |

### 削减预算的接受性

```
削减金额 ≤ 10%：
  → ⚠️ 评估对业务的影响
  → 如影响可控，可批准

削减金额 > 10%：
  → 🔴 须部门负责人说明业务影响
  → 如影响业务正常运转，不批准

削减科目为战略性投入：
  → 🔴 不批准
  → 战略性投入削减须经财务总监+CEO审批
```

---

## 第四步：确定审批层级

### 审批矩阵

| 调整金额 | 追加预算 | 削减预算 | 跨部门调剂 |
|---|---|---|---|---|
| ≤ 5 万 | 部门负责人 | 部门负责人 | 部门负责人 | 财务总监 |
| 5-20 万 | 财务经理 | 部门负责人 | 财务经理 | 财务总监 |
| 20-50 万 | 财务总监 | 财务经理 | 财务总监 | CEO |
| > 50 万 | CEO + 财务总监 | 财务总监 | CEO | CEO |

### 特殊规则

```
资本支出预算调整：须 CEO + 财务总监双重审批
跨期动用预算：须 CEO + 财务总监双重审批
同一科目连续 3 次申请调整：须财务总监介入审查预算流程
```
