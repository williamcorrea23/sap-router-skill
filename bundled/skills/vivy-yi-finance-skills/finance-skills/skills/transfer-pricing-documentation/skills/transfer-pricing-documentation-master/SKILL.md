# /transfer-pricing-documentation-master — 转让定价文档管理主流程

## 完整文档管理流程

```
Step 1：关联方交易数据收集（截止日后 X 周内）
  → 收集全期关联交易数据
  → 确认须准备的文档类型

Step 2：同期资料准备
  → 执行 master-document-preparation（主体文档）
  → 执行 comparable-analysis-tp（可比性分析）
  → 执行 beps-compliance（BEPS 合规）

Step 3：APA 管理（如适用）
  → 执行 apa-management

Step 4：文档汇总与提交
  → 核对完整性
  → 生成文档包
  → 提交关联申报表
```

---

## 第一步：关联交易数据收集

```
□ 关联交易类型确认：
□ 有形资产销售/采购
□ 服务提供
□ 无形资产许可
□ 金融交易（借款/担保）
□ 费用分摊

□ 须准备文档类型确认：
□ 主体文档（Master File）：[✅ 须准备 / ☐ 不须]
□ 本地文档（Local File）：[✅ 须准备 / ☐ 不须]
□ 国别报告（CbCR）：[✅ 须准备 / ☐ 不须]

□ 关联交易金额阈值检查：
□ 年度关联交易总额 > [X] 万元 → 须准备同期资料
□ 海外关联方存在 → 须准备主体文档
□ 集团合并收入 > [X] 亿元 → 须准备 CbCR
```

## 第二步：同期资料准备

**执行 Skill：** `master-document-preparation`

```
□ 执行状态：[成功/失败/部分成功]

□ 主体文档完整性：
□ 组织架构描述：[✅ 有 / ☐ 缺失]
□ 价值链分析：[✅ 有 / ☐ 缺失]
□ 关联交易概况：[✅ 有 / ☐ 缺失]
□ 功能/风险/资产分布：[✅ 有 / ☐ 缺失]
□ 可比性分析：[✅ 有 / ☐ 缺失]
□ TP 方法说明：[✅ 有 / ☐ 缺失]
```

**执行 Skill：** `comparable-analysis-tp`

```
□ 执行状态：[成功/失败/部分成功]

□ 可比性分析质量：
□ 可比公司数量：≥ [X] 家
□ 行业筛选：准确
□ 财务指标区间：符合警戒线
□ 差异调整：有充分依据

□ 利润水平指标：
□ 毛利率：[X]% — 风险：[低/中/高]
□ 营业利润率：[X]% — 风险：[低/中/高]
□ 完全成本加成率：[X]% — 风险：[低/中/高]
```

**执行 Skill：** `beps-compliance`

```
□ 执行状态：[成功/失败/部分成功]

□ BEPS 行动 13 合规：
□ 主体文档格式符合要求
□ 本地文档格式符合要求
□ CbCR 数据完整且准确
□ 各管辖区纳税情况已披露
```
