# 查询路径 — 现金管理

## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | 资金系统/司库（[CASH]）| 账户余额/调拨记录 | 最优先 |
| 2 | 网上银行（[BANK]）| 银行流水/实时余额 | 实时核对 |
| 3 | ERP 总账（[ERP]）| 凭证/科目余额 | 账务核对 |
| 4 | OA 审批系统（[OA]）| 付款申请/审批记录 | 付款依据 |
| 5 | 预算管理系统（[BK]）| 资金计划/预算 | 计划核对 |

---

## 常用查询场景

### 场景 1：资金余额查询

**查询入口：**
```
资金系统/司库：
- 账户余额总览 → 查看所有账户余额
- 实时余额 → 实时查看各账户
- 历史余额 → 查看历史某日余额

网上银行：
- 企业网银 → 账户余额查询
- 账户明细查询 → 流水和余额

ERP：
- 现金日记账 → 账面余额
- 银行存款明细账 → 各账户余额
```

**资金余额核查：**
```
□ 运营账户余额：[XXX万]
□ 工资账户余额：[XXX万]
□ 纳税账户余额：[XXX万]
□ 其他账户余额：[XXX万]
□ 合计可用余额：[XXX万]

三方核对：
□ 资金系统余额：[XXX万]
□ 网上银行余额：[XXX万]
□ ERP 账面余额：[XXX万]
→ 如三方一致：高置信度
→ 如有差异：须核查原因
```

### 场景 2：资金缺口分析

**查询入口：**
```
ERP：
- 到期应付账款 → 未来 30 天须付款清单
- 到期银行贷款 → 到期本息清单

OA 审批系统：
- 待支付申请 → 已审批待支付清单
- 付款申请流程 → 审批中申请

预算管理系统：
- 资金计划 → 未来 30 天收支计划
```

**资金缺口分析：**
```
未来 30 天资金需求：
□ 到期应付账款：[XXX万]
□ 到期银行贷款：[XXX万]
□ 本月工资/社保：[XXX万]
□ 本月税款：[XXX万]
□ 其他固定支出：[XXX万]
□ 合计：[XXX万]

未来 30 天资金来源：
□ 预期客户回款：[XXX万]
□ 银行融资到账：[XXX万]
□ 其他来源：[XXX万]
□ 合计：[XXX万]

净缺口：[XXX万]（需求 - 来源）
→ 如为正：须安排融资
→ 如为负：资金富余
```

### 场景 3：付款优先级确认

**查询入口：**
```
OA 审批系统：
- 待支付申请列表 → 按金额/到期日排序
- 紧急付款申请 → 标注优先处理

ERP：
- 到期应付清单 → 按到期日排序
- 供应商付款历史 → 付款优先级参考

预算管理系统：
- 资金计划 → 确认已批准支出
```

**付款优先级排序：**
```
P0（立即支付）：
□ 员工工资：[XXX万] — 到期日 [日期]
□ 税款：[XXX万] — 到期日 [日期]
□ 到期贷款：[XXX万] — 到期日 [日期]

P1（优先支付）：
□ 紧急供应商 [XXX万] — 到期日 [日期]
□ [供应商名称]：[XXX万] — 到期日 [日期]

P2（正常支付）：
□ 常规供应商 [XXX万] — 到期日 [日期]
□ [供应商名称]：[XXX万] — 到期日 [日期]
```

### 场景 4：银行流水核对

**查询入口：**
```
网上银行：
- 历史流水查询 → 按日期范围筛选
- 到账通知 → 实时到账提醒

ERP：
- 银行到账认领 → 匹配客户/发票
- 收款凭证 → 已认领记录

资金系统：
- 到账流水 → 银行到账记录
```

**银行流水核对：**
```
□ 昨日银行流水（[日期]）：
  - 收入：[X] 笔 / [XXX万]
  - 支出：[X] 笔 / [XXX万]
  - 净流量：[XXX万]

□ 待认领到账：
  - 待认领：[X] 笔 / [XXX万]
  - 已匹配：[X] 笔 / [XXX万]
  - 无法匹配：[X] 笔 / [XXX万] — 须跟进

□ 异常流水：
  - 大额转出：[X] 笔 / [XXX万] — 是否审批？
  - 不明来源：[X] 笔 / [XXX万] — 须核实
```

## 数据源组合使用

### 资金日报
```
[CASH] 实时余额 → [BANK] 流水核对 → [ERP] 账面余额 → 生成日报
```

### 资金缺口预警
```
[ERP] 到期应付 → [OA] 待支付申请 → [BK] 资金计划 → 预测缺口
```

### 付款安排
```
[OA] 审批通过清单 → [BANK] 余额 → [CASH] 优先级排序 → 生成付款计划
```
