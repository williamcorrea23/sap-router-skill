# 判断框架 — 固定资产管理异常处理

## 核心原则

**固定资产管理的目标是"账实相符、资产完好、折旧准确、处置合规"。**
遇到异常时，优先判断是"账务问题"还是"实物问题"，确保资产安全完整。

---

## 异常类型与处理方式

### 类型一：资产盘盈

**典型场景：**
- 盘点时发现账上无记录但实际存在的资产
- 历史遗留的无主资产
- 合并报表中新发现的资产

**判断流程：**

```
第一步：确认盘盈事实

□ 盘点差异表：[编号]
□ 资产名称：[名称]
□ 资产类别：[类别]
□ 估计原值：[XXX万]
□ 估计已使用年限：[X] 年

□ 是否有历史采购记录？[ERP 中查不到/查到 XX 年 XX 万]
□ 是否有发票/合同？[有/无/不完整]

第二步：分析盘盈原因

□ 账外资产（历史遗留）：
→ 接受捐赠未入账
→ 自行建造未转固
→ 历史差错（漏记账）

□ 盘点差错：
→ 标签贴错导致串号
→ 存放地点变更未更新
→ 资产已处置但账未销

□ 其他原因：
→ 合并报表新增
→ 分立/改制发现

第三步：确定入账价值

□ 如能查到原值/发票：
□ 入账原值：[XXX万]
□ 入账累计折旧：[XXX万]（按使用年限估算）
□ 入账账面价值：[XXX万]

□ 如无法查到原值：
□ 评估入账：[XXX万]（须评估报告或参考市场价）
□ 入账累计折旧：[XXX万]
□ 入账账面价值：[XXX万]

□ 净盘盈：[+XXX万] → 计入"以前年度损益调整"

第四步：审批与入账

□ 审批层级：
□ < [X] 万：[财务经理] 审批
□ [X]-[X] 万：[CFO] 审批
□ > [X] 万：[CEO] 审批

□ 入账分录：
借：固定资产 [原值]
  贷：累计折旧 [已提折旧]
    以前年度损益调整 [净值]

□ 同时更新资产标签和管理台账
```

### 类型二：资产盘亏

**典型场景：**
- 盘点时发现账上存在但实物不存在的资产
- 被盗/丢失的资产
- 损毁无法修复须报废的资产

**判断流程：**

```
第一步：确认盘亏事实

□ 盘点差异表：[编号]
□ 资产编号：[编号]
□ 资产名称：[名称]
□ 账面原值：[XXX万]
□ 累计折旧：[XXX万]
□ 账面净值：[XXX万]

□ 盘亏发现时间：[YYYY-MM-DD]
□ 最后见到实物时间：[YYYY-MM-DD]

第二步：分析盘亏原因

□ 实物损失：
→ 被盗（须报警）
→ 丢失（须说明原因）
→ 损坏报废（须评估）

□ 账务差错：
→ 重复记账
→ 分类错误（已在另一卡片）
→ 存放地点记录错误

□ 其他：
→ 已处置但账未销
→ 合并范围变更

第三步：处理决策

□ 如为被盗：
→ 立即报警，取得报警回执
→ 通知保险公司（是否投保）
→ 确认为非责任事故 → 报公安
→ 确认为责任事故 → 追究赔偿

□ 如为丢失：
→ 查明原因（保管不当/管理缺失）
→ 责任人赔偿协商
→ 书面报告说明

□ 如为损坏报废：
→ 评估残值（变卖收入）
→ 确认是否为正常使用损坏
→ 责任人赔偿协商（如适用）

第四步：审批与账务处理

□ 审批层级：
□ < [X] 万：[CFO] 审批
□ [X]-[X] 万：[CEO] 审批
□ > [X] 万：[董事会] 审批

□ 账务处理：
借：营业外支出 [净损失]
  累计折旧 [已提折旧]
  其他应收款 [责任人赔款/保险赔款]（如有）
  贷：固定资产 [原值]

□ 涉及增值税进项税额转出（如为非正常损失）
```

### 类型三：资产提前报废

**典型场景：**
- 技术进步导致设备过时
- 损坏无法修复
- 环保政策关停
- 产能调整淘汰

**判断流程：**

```
第一步：确认报废事实

□ 资产编号：[编号]
□ 资产名称：[名称]
□ 账面原值：[XXX万]
□ 累计折旧：[XXX万]
□ 账面净值：[XXX万]
□ 预计残值：[XXX万]

□ 报废原因：[技术进步/损坏/政策/调整]

□ 鉴定报告：[有/须补充]
□ 残值评估：[XXX万]

□ 预计处置损失：[XXX万]（净值 - 残值）

第二步：判断是否属于正常报废

□ 正常报废（正常使用到期）：
→ 折旧已足额计提
→ 技术进步但仍可使用
→ 使用寿命已到

□ 非正常报废：
→ 意外损坏（火灾/水灾/被盗）
→ 政策强制淘汰
→ 严重技术落后

□ 是否为责任事故：[是/否]
□ 责任人是否赔偿：[是/否]

第三步：审批流程

□ 正常报废：
□ < [X] 万：[CFO] 审批
□ [X]-[X] 万：[CEO] 审批
□ > [X] 万：[董事会] 审批

□ 非正常报废：
□ 须增加 [CFO/CEO] 额外审批
□ 须查明原因和责任

第四步：残值处置

□ 残值变卖：[XXX万]
□ 是否需要评估：[是/否]（> [X] 万须评估）

□ 变卖收入入账：
借：银行存款 [收到金额]
  营业外支出 [损失，如收入<净值]
  贷：固定资产清理 [净值]
    应交税费-增值税（简易计税）[如有]

□ 如收入 > 净值，计入"营业外收入"
```

### 类型四：资产减值

**典型场景：**
- 资产市价大幅下跌
- 资产闲置（停用 > [X] 个月）
- 技术落后（市场价低于账面价值）
- 损坏/陈旧导致可收回金额下降

**判断流程：**

```
第一步：确认减值迹象

□ 外部迹象：
→ 市价在当期大幅下跌（跌幅 > [X]%）
→ 利率/汇率提高导致折现率上升
→ 资产所在行业市场萎缩

□ 内部迹象：
→ 资产闲置（停用 > [X] 个月）
→ 资产陈旧/损坏
→ 经营亏损（现金流恶化）

□ 存在减值迹象：[是/否]

第二步：计算可收回金额

□ 可收回金额 = max(公允价值 - 处置费用 , 未来现金流量的现值)

□ 公允价值 - 处置费用：
→ 有市场报价：[市场价 - 处置费用]
→ 无市场报价：参考类似资产市场价

□ 未来现金流量现值：
□ 预计未来各年现金流：[XXX万]/年
□ 折现率：[X%]
□ 折现期：[X] 年
□ 现值计算：[具体数字]

□ 可收回金额：[XXX万]

第三步：计算减值损失

□ 账面价值：[XXX万]
□ 可收回金额：[XXX万]

□ 减值损失 = 账面价值 - 可收回金额 = [XXX万]

□ 减值损失 > 0 → 须计提减值
□ 减值损失 <= 0 → 无需计提

第四步：审批与账务处理

□ 审批层级：
□ < [X] 万：[CFO] 审批
□ [X]-[X] 万：[CEO] 审批
□ > [X] 万：[董事会] 审批

□ 账务处理：
借：资产减值损失 [减值金额]
  贷：固定资产减值准备 [减值金额]

□ 减值损失确认后，后续折旧须按扣除减值后的账面价值重新计算
```

### 类型五：折旧政策变更

**典型场景：**
- 会计准则/税法调整
- 公司经营方针变化
- 资产实际使用情况与原估计不符
- 会计估计变更（折旧年限/残值率）

**判断流程：**

```
第一步：确认变更类型

□ 会计政策变更：
→ 折旧方法改变（直线法 → 加速法）
→ 须采用追溯调整法
→ 须披露变更原因和影响

□ 会计估计变更：
→ 折旧年限调整
→ 残值率调整
→ 采用未来适用法，不追溯调整

□ 错误更正：
→ 历史差错导致折旧错误
→ 须追溯重述

第二步：分析变更原因

□ 外部原因：
→ 会计准则修订
→ 税法规定变化
→ 行业监管要求

□ 内部原因：
→ 资产实际磨损情况与原估计不符
→ 资产使用方式变化
→ 经营策略调整

第三步：计算影响金额

□ 会计政策变更（追溯调整）：
□ 变更前各年累计折旧：[XXX万]
□ 变更后各年累计折旧：[XXX万]
□ 累计影响数：[±XXX万]

□ 会计估计变更（未来适用）：
□ 变更时账面价值：[XXX万]
□ 剩余折旧年限：[X] 年（变更后估计）
□ 残值率：[X]%（变更后）
□ 变更后月折旧额：[XXX万]
□ 变更影响：[±XXX万]/月

第四步：审批与披露

□ 审批层级：
□ 会计政策变更：[CFO + 审计委员会] 审批
□ 会计估计变更：[CFO] 审批
□ 重大会计政策变更须 [董事会] 审批

□ 披露要求：
□ 须在报表附注中披露
□ 说明变更原因
□ 列明变更对当期及可比期间的影响
```
