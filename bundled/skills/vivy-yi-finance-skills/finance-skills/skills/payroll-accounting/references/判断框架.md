# 判断框架 — 薪资核算异常处理

## 核心原则

**薪资核算的目标是"发对、发准、发及时"。**
遇到异常时，优先判断是"计算错误"还是"数据来源问题"，确保员工权益不受损、公司支出不浪费。

---

## 异常类型与处理方式

### 类型一：员工投诉薪资错误

**典型场景：**
- 员工反馈实发金额与预期不符
- 社保/公积金扣款与政策不符
- 个税计算错误
- 绩效工资与实际不符

**判断流程：**

```
□ 员工反馈项目：
□ 应发工资：[描述]
□ 社保公积金扣款：[描述]
□ 个税扣款：[描述]
□ 其他扣款：[描述]
□ 实发工资：[描述]
```

```
第二步：核查薪资计算

□ 应发工资核查：
□ 基本工资：[XXX元] — 与合同一致 [✅/⚠️]
□ 岗位工资：[XXX元] — 与岗位一致 [✅/⚠️]
□ 绩效工资：[XXX元] — 与绩效评定一致 [✅/⚠️]
□ 加班费：[XXX元] — 与考勤一致 [✅/⚠️]

□ 社保公积金核查：
□ 缴费基数：[XXX元] — 与申报基数一致 [✅/⚠️]
□ 养老保险扣款：[XXX元] — 应扣 [XXX元] — 差异 [XXX元]
□ 医疗保险扣款：[XXX元] — 应扣 [XXX元] — 差异 [XXX元]
□ 失业保险扣款：[XXX元] — 应扣 [XXX元] — 差异 [XXX元]

□ 如发现扣款错误：
□ 多扣须退还：[XXX元]
□ 少扣须补扣：[XXX元]（须员工同意/政策允许）

□ 计税收入：[XXX元]

□ 如发现个税错误：
□ 多扣须退还：[XXX元]

```

```
第三步：确定处理方案

□ 公司原因导致少发：
□ 须补发：[XXX元]
□ 补发时间：[次月薪资/当月另行发放]
□ 须重新计算补发月的社保/个税

□ 员工原因导致多发：
□ 须退还：[XXX元]
□ 退还方式：[次月扣回/转账退还]
```

```
第四步：审批与执行

□ 补发/退还审批：
□ < [X] 元：[HR 经理] 审批
```

### 类型二：社保公积金欠缴

**典型场景：**
- 员工入职未及时参保
- 社保公积金基数调整漏报
- 员工离职当月漏停
- 银行代扣失败

**判断流程：**

```
第一步：确认欠缴情况

□ 欠缴员工：[姓名]
□ 工号：[编号]
□ 欠缴期间：[YYYY年MM月至YYYY年MM月]
□ 欠缴项目：[社保/公积金/两者]

□ 欠缴金额：
□ 养老保险：[XXX元]
□ 医疗保险：[XXX元]
□ 失业保险：[XXX元]
□ 住房公积金：[XXX元]
□ 合计个人部分：[XXX元]
□ 合计公司部分：[XXX元]

□ 欠缴原因：[入职未及时/调基漏报/离职漏停/代扣失败]
```

```
第二步：分析欠缴原因

□ 入职未及时参保：
□ 入职日期：[YYYY-MM-DD]
□ 首次参保日期：[YYYY-MM-DD]
□ 漏缴月数：[X] 月
□ 是否公司原因：[是/否]

□ 基数调整漏报：
□ 原申报基数：[XXX元]
□ 应申报基数：[XXX元]
□ 漏报月数：[X] 月
□ 是否公司原因：[是/否]

□ 离职当月漏停：
□ 离职日期：[YYYY-MM-DD]
□ 社保公积金截止月：[YYYY年MM月]
□ 多缴月数：[X] 月（可退/不可退）

□ 代扣失败：
□ 代扣账户余额不足
□ 银行系统问题
□ 须立即充值重缴
```

```
第三步：制定补缴方案

□ 补缴可行性：
□ 欠缴在 [X] 个月内可补缴（各地政策不同）
□ 超过 [X] 个月一般不可补缴，须折算补偿

□ 补缴金额计算：
□ 补缴本金：[XXX元]（个人+公司部分）
□ 滞纳金：[XXX元]（按日万分之五）
□ 利息：[XXX元]（按补缴时利率）

□ 责任划分：
□ 公司原因 → 公司承担全部费用
□ 员工原因 → 员工须承担个人部分

□ 补缴审批：
□ < [X] 万：[HR 总监] 审批
□ [X]-[X] 万：[CFO] 审批
□ > [X] 万：[CEO] 审批
```

```
第四步：执行补缴

□ 补缴流程：
□ 向社保公积金中心提交补缴申请
□ 获批后缴纳本金+滞纳金+利息
□ 员工签字确认（如须员工承担部分）
□ 入账处理

□ 入账分录（公司原因）：
借：管理费用-社保 [公司部分]
  其他应收款-员工 [个人部分]（如须员工承担）
  营业外支出-滞纳金 [滞纳金]
  贷：银行存款 [合计]

□ 后续改进：
□ 入职流程增加 [X] 天内参保提醒
□ 离职流程增加当月参保截止核查
□ 调基流程增加复核环节
```

### 类型三：个税申报错误

**典型场景：**
- 专项附加扣除信息未填报
- 收入类型混淆（工资 vs 劳务）
- 年终奖计税方式选错
- 重复扣除免税项目

**判断流程：**

```
第一步：确认申报错误类型

□ 申报期间：[YYYY年MM月/年度]
□ 涉及员工：[X] 人

□ 错误类型：
□ 专项附加扣除未填报（员工已提交但未录入）
□ 收入类型错误（工资/劳务混淆）
□ 年终奖计税方式选错
□ 其他：[描述]
```

```
第二步：评估影响

□ 对员工影响：
□ 多缴个税：[XXX元]（须退还员工）
□ 少缴个税：[XXX元]（须补缴+滞纳金）

□ 对公司影响：
□ 未履行代扣代缴义务
□ 须补申报 + 滞纳金（日万分之五）
□ 可能面临 [X] 元罚款

□ 错误发现时间：
□ 当期发现 → 可更正申报，无滞纳金（特殊情况下）
□ 年度汇算清缴发现 → 须更正申报，可能有滞纳金
□ 超过 [X] 年 → 一般不可更正
```

```
第三步：更正申报

□ 更正流程：
□ 收集正确专项附加扣除信息
□ 在个税申报系统提交更正申报
□ 补缴少缴税款（如有）
□ 滞纳金处理（公司原因/员工原因）

□ 年终奖计税方式更正（如过渡期政策允许）：
□ 方式一：单独计税
□ 方式二：并入综合所得
□ 重新计算应纳税额
□ 差额多退少补
```

```
第四步：预防措施

□ 专项附加扣除管理：
□ 每月 [X] 日提醒员工更新扣除信息
□ 新入职员工 [X] 天内完成信息填报
□ 年度末检查信息有效性

□ 申报复核：
□ 申报前 [X] 天内完成数据复核
□ 复核人员与申报人员分离
□ 保留复核签字记录
```

### 类型四：薪资延迟发放

**典型场景：**
- 银行代发失败
- 资金账户余额不足
- 审批流程延误
- 系统故障

**判断流程：**

```
第一步：确认延迟情况

□ 应发薪日：[YYYY-MM-DD]
□ 实际发放日：[YYYY-MM-DD]
□ 延迟天数：[X] 天

□ 影响员工：[X] 人
□ 应发总额：[XXX元]
□ 已发总额：[XXX元]
□ 待发总额：[XXX元]
```

```
第二步：分析延迟原因

□ 银行代发失败：
□ 失败原因：[账号错误/银行系统维护/超限额]
□ 成功补发日：[YYYY-MM-DD]

□ 资金不足：
□ 缺口金额：[XXX元]
□ 缺口原因：[收入未到账/付款优先级错误]
□ 解决时间：[YYYY-MM-DD]

□ 审批延误：
□ 延误环节：[描述]
□ 责任人：[X]
□ 延误原因：[描述]
```

```
第三步：紧急应对

□ 立即措施：
□ 确认延迟原因
□ 通知工会/员工代表（如适用）
□ 确定补发时间

□ 补发安排：
□ < [X] 天延迟：次日补发
□ [X]-[X] 天延迟：当日补发 + 书面说明
□ > [X] 天延迟：补发 + 补偿 [X] 天利息

□ 内部问责：
□ 查明原因
□ 责任人处理
□ 改进措施
```

```
第四步：后续改进

□ 资金管理改进：
□ 发放日前 [X] 天确认账户余额
□ 建立备用金账户 [XXX元]
□ 与银行建立快速响应通道

□ 流程改进：
□ 薪资发放审批前置 [X] 天
□ 建立节假日提前发放机制
□ 系统自动校验账号
```


```
第二步：核查离职薪资

□ 离职当月工资：[XXX元]
□ 计入工资的绩效/提成：[XXX元]
□ 加班费（如未调休）：[XXX元]
□ 未休年假折算：[X] 天 × 日工资 = [XXX元]
□ 其他应发：[XXX元]

□ 扣除项：
□ 社保公积金个人部分：[XXX元]
□ 个税：[XXX元]
□ 其他扣款：[XXX元]

□ 实发工资：[XXX元]

□ 合规性核查：
□ 离职当月工资是否足额发放：[✅ 是 / ⚠️ 否]
□ 未休年假是否按规定折算：[✅ 是 / ⚠️ 否]
□ 经济补偿金是否正确：[✅ 是 / ⚠️ 不涉及]
```

```
第三步：处理争议

□ 如员工主张有误：

□ 如公司计算有误：
□ 确认错误金额：[XXX元]
□ 须补发：[XXX元]
□ 补发时间：离职当日/次月

□ 经济补偿金（如适用）：

```
