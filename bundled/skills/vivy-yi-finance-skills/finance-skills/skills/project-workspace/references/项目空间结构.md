# 项目空间结构 — Finance Skills Layer 2

财务工作不是单次任务，而是有明确周期和范围的**项目**。本文件定义项目目录结构和 project.md 字段规范。

---

## 项目根目录

```

├── 2026-audit-ic/                  ← 年度内部控制审计

## project.md 必填字段

| 字段 | 必填 | 说明 |
|------|------|------|
| `code` | ✅ | 项目代码，全小写字母/数字/- |
| `type` | ✅ | annual-audit / budget-review / tax-filing / m-and-a / financing / internal-control / other |

2026-06-11 09:20:12 [internal-audit] findings/C-15-重要.md created
```

用于审计可追溯性，符合 SOX/合规要求。

## 与 CLAUDE.md 场景配置的关系

**场景 CLAUDE.md**（如 `skills/internal-audit/CLAUDE.md`）是**全局场景配置**，影响所有项目。

**project.md** 是**单项目配置**，覆盖或补充场景级配置：

```
场景 CLAUDE.md（全局）
  ├── 重大缺陷门槛: 500 万
  ├── 重要缺陷门槛: 100 万
  └── SOX 控制: 72 个

project.md（项目级）
  ├── 重大缺陷门槛: 800 万  ← 项目特殊调高
  ├── 重要缺陷门槛: 150 万
  └── 范围: 仅华东区域
```

**优先规则**：project.md 的阈值覆盖场景 CLAUDE.md。

## 与 finance-skills/cold-start 的关系

`/finance:project-new` 是 cold-start 的**第二阶段**：
1. `/finance:cold-start` → 全局配置（公司、ERP、连接器）
2. `/finance:project-new` → 创建第一个项目
3. `/finance:project-switch` → 激活项目上下文

首次用户应按顺序跑这三个命令。
