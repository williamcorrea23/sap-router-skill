# 查询路径 — 费用报销审核

## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | 企业报销制度（PDF/文档）| 最优先，标准以制度为准 |
| 2 | ERP 系统（SAP/用友/金蝶）| 历史数据/员工职级/供应商信息 |
| 3 | OA 审批记录（钉钉/飞书/企微）| 申请历史/审批节点记录 |
| 4 | 费控平台（分贝通/胜意）| 预算余额/标准校验/申请记录 |
| 5 | 国家税务总局发票验真 | 发票真伪和状态 |
| 6 | 天眼查/启信宝 | 供应商资质/工商信息核查 |

## 常用查询场景

### 场景 1：发票真伪验真

**查询入口：**
```
国家税务总局增值税发票查验平台
https://inv-veri.chinatax.gov.cn

查验范围：
- 增值税专用发票（最核心）
- 增值税普通发票（卷式/电子）
- 机动车销售统一发票
- 通行费发票（有限查验）

注意：发票开具后 5 日内数据同步，约 1 天延迟
```

**查询判断标准：**

| 查验结果 | 含义 | 操作 |
|---|---|---|
| 一致 | 发票信息与系统记录完全匹配 | ✅ 通过 |
| 不一致 | 金额/税号/品名与系统不符 | ❌ 驳回，要求核实 |
| 不存在 | 发票号码/代码无法查到 | ❌ 驳回，疑似假票 |
| 作废 | 发票已被开票方作废 | ❌ 驳回，不可使用 |
| 红冲 | 发票已被冲销 | ❌ 驳回，不可使用 |
| 已认证（失控）| 专票被异地认证，税局已标记失控 | ❌ 驳回 + 升级 |

### 场景 2：员工职级确认

**查询入口：**
```
优先级1：HR 系统（钉钉/飞书/Workday）
  → 员工花名册/职级/部门

优先级2：OA 审批系统
  → 审批时会读取申请人的职级信息

优先级3：ERP 系统
  → 用户主数据（User Master）中的职级字段
```

**判断逻辑：**
```
差旅住宿标准 = 城市基础标准 × 职级系数

高管（系数 2.0）→ 城市 A 类 500 × 2.0 = 1000 元/天上限
中层（系数 1.5）→ 城市 A 类 500 × 1.5 = 750 元/天上限
基层（系数 1.0）→ 城市 A 类 500 × 1.0 = 500 元/天上限
```

### 场景 3：预算余额查询

**查询入口：**
```
优先级1：费控平台（分贝通/胜意）
  → 实时显示科目预算已用/可用余额

优先级2：ERP 系统（SAP CO / 用友 U8）
  → 预算模块（FM）查询

优先级3：Excel 预算台账（无系统时）
  → 要求财务人员手动查询
```

**判断标准：**
```
预算余额 ≥ 报销金额 → ✅ 预算充足
预算余额 < 报销金额，但 ≥ 0 → ⚠️ 超支，需要调拨或特批
预算余额 < 0 → ❌ 驳回，须先申请预算调整
```

### 场景 4：三单匹配（采购类费用）

**三单定义：**
- 发票（Invoice）：供应商开具
- 合同/订单（PO）：采购协议
- 收货单（GR）：仓库/需求部门确认收货

**匹配检查点：**
```
检查1：合同金额 vs 发票金额 vs 付款金额
  → 三者差异在 [填空：如 5%] 以内 → ✅ 正常
  → 超过 [填空：如 5%] → ⚠️ 要求说明差异原因

检查2：收货日期逻辑
  → 收货日期早于合同签订日期 → ❌ 逻辑矛盾，要求核实
  → 发票日期早于收货日期 → ⚠️ 要求说明（允许合理间隔）

检查3：供应商一致性
  → 合同签约方 vs 发票开具方 vs 收款账户方须一致
  → 不一致 → ⚠️ 要求提供授权文件或代理协议
```

### 场景 5：供应商工商信息核查

**查询入口：**
```
天眼查：https://www.tianyancha.com
启信宝：https://www.qixin.com
国家企业信用信息公示系统：http://www.gsxt.gov.cn
```

**核查要点：**
```
□ 供应商是否处于经营异常名录
□ 营业执照是否有效（注册资本/经营范围）
□ 是否存在严重违法失信名单
□ 法定代表人是否被限制高消费
□ 与公司是否存在关联关系（关联交易披露）
```

### 场景 6：差旅行程核实

**查询入口：**
```
携程/飞猪：机票/酒店订单截图
铁路 12306：火车票订单
企业差旅系统（如有）：完整行程记录
```

**核实要点：**
```
□ 机票/火车票日期与出差申请日期一致
□ 目的地与报销目的地一致
□ 多人同行时，是否存在虚假同行人
□ 里程数与交通费是否匹配（过路过桥费/租车费）
□ 住宿日期与出差天数匹配（通常出差天数 = 住宿天数 + 1）
```

## 查询结果处理

### 情形一：所有数据源结论一致
→ 高置信度，直接给出审核结论

### 情形二：部分数据源有结论，部分无数据
→ 中置信度，给出结论并标注"建议人工复核"

### 情形三：数据源结论矛盾
→ 低置信度，不自行判断，升级至财务专员

### 情形四：所有数据源均无数据
→ 无法审核，要求员工提供补充材料，不设置自动通过路径
