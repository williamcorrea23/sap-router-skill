# 判断框架 — 融资结构异常处理

## 核心原则

**融资结构的目标是"成本最优、风险可控、灵活性足"。**
遇到异常时，优先判断是"市场变化"还是"信用变化"，确保融资结构与公司战略匹配。

---

## 异常类型与处理方式

### 类型一：融资成本大幅上升

**典型场景：**
- 市场利率上升导致新融资成本超出预算
- 信用评级下调导致融资成本上升
- 银行收紧信贷导致额度不足

**判断流程：**

```
第一步：确认成本变化

□ 融资工具：[银行贷款/债券/其他]
□ 原预期成本：[X%]（利率 + 费用）
□ 当前市场成本：[X%]
□ 成本上升：[+X bp]

□ 变化原因：
□ 宏观利率上行：[是/否]
□ 公司信用评级变化：[是/否]
□ 银行信贷政策收紧：[是/否]
□ 融资市场流动性变化：[是/否]

□ 对公司影响：
□ 年度利息支出增加：[XXX万]
□ 财务费用率变化：[+X%]
□ 对利润影响：[XXX万]

第二步：评估影响

□ 对项目可行性的影响：
□ 项目 IRR：[X%] — 原融资成本 [X%] — 现融资成本 [X%]
□ 项目是否仍可行：[✅ 可行 / ⚠️ 须重新评估 / 🔴 不可行]

□ 对资本结构的影响：
□ D/E 比率变化：[从 X 到 X]
□ 资产负债率变化：[从 X% 到 X%]
□ 是否仍符合目标结构：[✅ 符合 / ⚠️ 偏离]

□ 融资策略调整必要性：
□ 必须调整：[✅ 是 / ⚠️ 否]

第三步：应对方案

□ 方案A：调整融资结构
□ 减少债务融资比例
□ 增加股权融资比例
□ 成本影响：[描述]

□ 方案B：选择替代融资工具
□ 替代工具：[融资租赁/保理/可转债]
□ 成本对比：[X%] vs [X%]

□ 方案C：推迟融资时机
□ 等待市场利率回落
□ 成本节约：[XXX万]
□ 业务影响：[描述]

□ 方案D：接受成本上升并执行
□ 原因：[描述]
□ CFO 审批：[✅ 已审批 / ⚠️ 待审批]
```

### 类型二：信用评级下调

**典型场景：**
- 三大评级机构之一下调评级
- 评级展望从稳定转为负面
- 评级下调导致融资条件恶化

**判断流程：**

```
第一步：确认评级变化

□ 评级机构：[机构名称]
□ 原评级：[AAA/AA/A/BBB]
□ 新评级：[AAA/AA/A/BBB]
□ 评级展望：[稳定/负面/正面]

□ 下调原因：
□ 财务指标恶化：[是/否] — [描述]
□ 经营风险增加：[是/否] — [描述]
□ 行业环境恶化：[是/否] — [描述]
□ 重大事件影响：[是/否] — [描述]

□ 影响评估：
□ 债券融资成本上升：[+X bp]
□ 银行贷款条件收紧：[是/否]
□ 评级触发条款：[是/否]

第二步：制定应对措施

□ 即时影响管控：
□ 是否有评级触发条款：[✅ 是 / ⚠️ 否]
□ 是否触发提前还款：[✅ 是 / ⚠️ 否]
□ 应急措施：[描述]

□ 融资渠道维护：
□ 银行关系维护：[描述]
□ 投资者沟通：[描述]
□ 替代融资渠道：[描述]

□ 评级恢复计划：
□ 关键指标改善目标：[描述]
□ 计划时间表：[YYYY-MM-DD]
□ 评级机构沟通：[YYYY-MM-DD]
```
