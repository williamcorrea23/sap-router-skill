# /debt-restructuring — 债务重组方案设计


## Examples

→ 示例：用户说"有几笔贷款快到期了，继续用还是做债务置换，帮我分析一下"，系统应调用本技能，执行债务到期分析和置换方案评估。

→ 示例：用户说"穆迪下调评级后，我们的债券价格跌了不少，需要评估要不要回购"，系统应调用本技能，评估债券回购机会和成本。

→ 示例：用户说"子公司有高息贷款，想通过集团统一做债务置换降低成本"，系统应调用本技能，制定债务置换方案和成本测算。

## 第三步：重组方案设计

**重组工具选择：**
```
□ 展期（Extension）：
  → 适用情况：短期流动性紧张但长期偿付能力可
  → 方式：延长贷款到期日 [X] 年
  → 优势：操作简单，债权人接受度高
  → 劣势：仅延迟问题，未根本解决

□ 置换（Refinancing）：
  → 适用情况：现有债务利率偏高或条件不佳
  → 方式：以新债务替换旧债务
  → 优势：可改善债务条件
  → 劣势：需有新资金方愿意承接

□ 债务减记（Debt Write-off）：
  → 适用情况：资不抵债或濒临破产
  → 方式：债权人豁免部分债务
  → 优势：减轻债务负担
  → 劣势：影响债权人利益，谈判难度大

□ 债转股（Debt-to-Equity）：
  → 适用情况：债权人愿意转为股东
  → 方式：债务转为股权
  → 优势：无需现金偿还
  → 劣势：稀释原股东股权
```

**推荐方案：**
```
□ 推荐重组方案：[方案组合]
□ 重组后债务结构：
| 指标 | 重组前 | 重组后 | 改善 |
|------|--------|--------|------|
| 总债务 | [X]万 | [X]万 | — |

| 月还款压力 | [X]万 | [X]万 | [-X]万 |
```
