# 判断框架 — 内部审计异常处理

## 核心原则

内部审计的核心是"独立、客观地评价并改善组织运作"。遇到异常时，优先判断是否构成"审计发现"以及严重程度。发现按三类处理：控制缺陷、改进机会、合规问题。

---

## 审计发现分类

### 控制缺陷（Process/Control Deficiency）

```
□ 设计缺陷（Design Deficiency）：
  → 控制目标明确但现有控制无法实现该目标
  → 例：审批权限设置错误，任何人可批准大额付款

□ 运行缺陷（Operating Deficiency）：
  → 控制设计有效但未被正确执行
  → 例：审批权限正确但无人实际审核

□ 影响评估：
  → 可能导致重大错报：[是/否]
  → 可能导致资产损失：[是/否]
  → 影响范围：[局部/广泛]
```

### 改进机会（Improvement Opportunity）

```
□ 不构成控制缺陷但存在更优做法
□ 例：流程效率可以提升、数据分析可以深化
□ 处理方式：纳入管理建议书，不记入正式审计报告
```

### 合规问题（Compliance Issue）

```
□ 违反法规/内部制度
□ 须评估：主动披露 vs 发现后整改
□ 升级：重大合规问题须立即升级至审计委员会
```

## 审计发现严重程度判断

### 风险矩阵

```
影响程度      | 高可能性 | 中可能性 | 低可能性
--------------|----------|----------|----------|
高财务影响    | 🔴 高    | 🔴 高    | ⚠️ 中
中财务影响    | 🔴 高    | ⚠️ 中    | ✅ 低
低财务影响    | ⚠️ 中    | ✅ 低    | ✅ 低
```

### 缺陷等级定义

```
🔴 Major（重大缺陷）：
□ 直接导致重大错报或资产损失
□ 或多个中风险缺陷组合暴露系统性漏洞
□ 影响管理层舞弊风险
□ 须立即升级至审计委员会

⚠️ Moderate（中度缺陷）：
□ 可能导致中等错报或效率损失
□ 控制存在但执行不足
□ 须在 [X] 天内整改

✅ Low（轻度）：
□ 改进机会，不构成正式缺陷
□ 纳入管理建议
```

## 舞弊调查判断流程

### 启动条件

```
□ 举报线索（经初步评估有价值）
□ 异常数据模式（数据分析发现）
□ 管理层要求
□ 外部监管要求

□ 初步评估（48 小时内完成）：
□ 涉及人员：[姓名/岗位]
□ 涉及金额估算：[X 万元]
□ 舞弊类型：[挪用/贪污/虚假报表/...]
□ 证据可获取性：[高/中/低]
```

### 调查流程

```
第一步：证据保护
□ 冻结相关系统访问权限（如必要）
□ 保存相关文件/邮件/数据
□ 限制知情范围

第二步：事实核查
□ 访谈相关人员
□ 调取系统日志/审批记录
□ 核对财务数据

第三步：影响评估
□ 损失金额：[X 万元]
□ 对报表影响：[是/否]
□ 对业务影响：[描述]

第四步：报告与整改
□ 报告对象：[审计委员会/CFO/法务/CEO]
□ 是否报警/诉讼：[是/否]
□ 整改措施：[...]
```

---

## 审计发现整改跟踪

### 超期未整改处理

```
□ 整改超期 [X] 天：
□ 发送提醒至被审计单位负责人

□ 整改超期 [X] 天：
□ 升级通知至 CFO

□ 整改超期 [X] 天且无合理解释：
□ 升级至审计委员会
□ 纳入绩效考核
```

---

## 升级触发条件

- 发现 Major 缺陷
- 涉及管理层舞弊
- 财务影响 > [X] 万元
- 被审计单位拒绝配合审计
- 外部监管调查启动
