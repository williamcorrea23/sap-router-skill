# 判断框架 — 高级司库异常处理

## 核心原则

**高级司库的目标是"资金安全、收益合理、流动性充裕"。**
遇到异常时，优先判断是"流动性风险"还是"收益风险"，确保资金管理在安全边界内运作。

---

## 异常类型与处理方式

### 类型一：LCR 跌破监管红线

**典型场景：**
- 流动性覆盖率（LCR）低于监管要求 [X]%
- 合格流动性资产（HQLA）不足
- 30 天净现金流出超出预期

**判断流程：**

```
第一步：确认 LCR 状况

□ 当前 LCR：[X%]（监管要求 [X%]）
□ HQLA 余额：[X] 万

□ 问题来源：

□ 净流出增加：[是/否] — 原因 [描述]
□ 两者皆有：[是/否]

□ 持续时间：
□ 短暂（< 3 天）
□ 中期（3-30 天）
□ 长期（> 30 天）

□ 监管报告状态：
□ 是否已向监管报告：[✅ 是 / ⚠️ 否]

第二步：评估影响

□ 合规影响：
□ 违反监管要求：[✅ 是]
□ 监管处罚风险：[高/中/低]
□ 须向监管解释：[✅ 是 / ⚠️ 否]

□ 业务影响：
□ 对付款能力影响：[✅ 高 / ⚠️ 中 / 🔴 低]
□ 对评级影响：[描述]

第三步：紧急补充流动性

□ 方案A：增加 HQLA
□ 措施：购买国债/央行票据
□ 资金来源：[描述]
□ 预计完成：[YYYY-MM-DD]
□ 效果：LCR 提升至 [X]%

□ 方案B：减少净流出
□ 措施：延后非必要付款/加速收款
□ 金额：[X] 万
□ 执行时间：[YYYY-MM-DD]

□ 方案C：银行授信启用
□ 启用授信额度：[X] 万
□ 银行：[名称]
□ 预计到账：[YYYY-MM-DD]

□ CFO 审批：[✅ 已审批 / ⚠️ 待审批]
□ CEO 知悉：[✅ 是 / ⚠️ 否]
□ 董事会知悉（如须）：[✅ 是 / ⚠️ 否]
```

### 类型二：银行授信突然收紧

**典型场景：**
- 主结算银行突然降低授信额度
- 银行要求提前还款
- 银行拒绝新授信申请

**判断流程：**

```
第一步：确认收紧情况

□ 银行：[名称]
□ 原授信额度：[X] 万
□ 现有额度：[X] 万
□ 减少：[X] 万

□ 收紧原因（银行口径）：
□ [原因描述]

□ 银行真实原因（推测）：
□ 银行内部信贷政策收紧：[是/否]
□ 对公司信用评估下调：[是/否]
□ 行业整体风险：[是/否]

□ 对公司影响：
□ 流动性缺口：[X] 万
□ 到期债务覆盖：[X] 万

第二步：应对措施

□ 即时措施：
□ 紧急联系其他银行：[✅ 是 / ⚠️ 否]
□ 激活其他银行授信：[✅ 是 / ⚠️ 否]

□ 替代银行：
□ 候选银行：[X 家]
□ 预计新授信：[X] 万
□ 到位时间：[X] 天

□ 资产抵押：
□ 可抵押资产：[描述]
□ 可获授信：[X] 万

□ 与收紧银行沟通：
□ 沟通层级：[客户经理 → 分行行长 → 总行]
□ 恢复额度可能性：[高/中/低]
□ 条件：[描述]

□ CFO 审批：[✅ 已审批 / ⚠️ 待审批]
□ CEO 知悉：[✅ 是 / ⚠️ 否]
```

### 类型三：闲置资金投资亏损

**典型场景：**
- 购买的银行理财出现浮亏
- 货币基金净值下跌
- 结构性存款收益低于预期

**判断流程：**

```
第一步：确认亏损

□ 投资产品：[产品名称]
□ 投资金额：[X] 万
□ 买入成本：[X] 万
□ 当前市值：[X] 万
□ 浮亏：[X] 万
□ 亏损率：[X%]

□ 亏损原因：
□ 市场利率上升（债券价格下跌）
□ 信用风险暴露
□ 流动性紧张

□ 是否超过公司风险限额：
□ 产品风险评级：[RPR X]
□ 公司限额：[RPR X]
□ 是否超限：[✅ 是 / ⚠️ 否]

□ 产品合规性：
□ 是否在批准投资清单内：[✅ 是 / ⚠️ 否]
□ 是否违反投资政策：[是/否]

第二步：评估与决策

□ 继续持有分析：
□ 到期是否可收回本金：[✅ 是 / ⚠️ 不确定 / 🔴 否]
□ 继续持有期间是否有进一步下跌风险：[⚠️ 是 / 🔴 否]
□ 到期预计回收：[X] 万

□ 止损分析：
□ 止损后实际亏损：[X] 万
□ 剩余资金再做何用：[描述]

□ 决策选项：
□ 选项A：继续持有至到期 — 理由 [描述]
□ 选项B：止损卖出 — 理由 [描述]
□ 选项C：寻找买家转让 — 理由 [描述]

□ CFO 审批：[✅ 已审批 / ⚠️ 待审批]
□ CEO 审批（重大亏损）：[✅ 已审批 / ⚠️ 待审批]
```

### 类型四：跨境现金池资金调动受限

**典型场景：**
- 外汇管制政策突然收紧
- 境外资金无法调回
- 跨境支付延迟或被拒

**判断流程：**

```
第一步：确认限制情况

□ 涉及地区：[地区]
□ 涉及金额：[X] 万
□ 限制类型：
□ 外汇管制：[是/否]
□ 银行政策：[是/否]
□ 监管要求：[是/否]

□ 限制持续时间：
□ 临时（< 1 周）
□ 中期（1-4 周）
□ 长期（> 1 个月）

□ 对运营影响：
□ 境内付款影响：[✅ 高 / ⚠️ 中 / 🔴 低]
□ 境外付款影响：[✅ 高 / ⚠️ 中 / 🔴 低]

第二步：评估与应对

□ 即时措施：
□ 启用境内银行授信补充流动性：[✅ 是 / ⚠️ 否]
□ 安排替代跨境路径：[✅ 是 / ⚠️ 否]

□ 合规应对：
□ 与监管机构沟通：[✅ 是 / ⚠️ 否]
□ 合规顾问咨询：[✅ 是 / ⚠️ 否]

□ 业务调整：
□ 延迟非紧急跨境付款：[✅ 是 / ⚠️ 否]
□ 改用境内供应商替代：[✅ 是 / ⚠️ 否]

□ 结构优化（如长期限制）：
□ 考虑利润留存境外用于再投资
□ 调整跨境资金池结构
□ 评估 tax impact
```

## 升级触发条件

- LCR < 监管要求须立即升级 CFO/CEO
- 银行授信突然收紧须立即升级 CFO
- 投资亏损 > [X] 万须 CFO + CEO 审批
- 跨境资金调动受限须 CFO + CEO 知悉
