# /investment-compliance-check — 投资合规检查


## Examples

→ 示例：用户说"有一笔理财要投，帮我确认一下是否在授权投资范围内"，系统应调用本技能，执行投资合规性检查。

→ 示例：用户说"帮我们审一下这个理财产品的风险等级，是否符合我们的风险偏好"，系统应调用本技能，评估理财产品合规性。

→ 示例：用户说"有一笔投资要到期了，想续做，但授权额度超了"，系统应调用本技能，评估超授权投资的审批路径。
## 第一步：投资信息确认

**投资基本信息：**
```
□ 投资项目：[名称]
□ 投资类型：[股权投资/债权投资/项目投资/基金投资]
□ 投资金额：[X] 万
□ 投资主体：[公司名称]
□ 投资目的：[战略投资/财务投资/业务协同]

□ 投资标的：[标的名称]
□ 标的估值：[X] 万（如适用）
□ 持股比例：[X]%（如适用）
```

## 第二步：投资政策核对

**投资政策合规检查：**
```
□ 投资类型合规：
  → 投资类型是否在政策允许范围内：[✅ 是 / 🔴 否 — 超出范围]
  → 政策规定：[允许类型列表]

□ 投资金额合规：
  → 单笔投资限额：[X] 万（授权矩阵规定）
  → 本次投资金额：[X] 万
  → 是否超过限额：[✅ 否 / 🔴 是 — 须特别审批]

□ 投资禁止项检查：
  → 禁止投机性投资：[✅ 合规]
  → 禁止投资高风险项目：[✅ 合规]
  → 禁止违规投资：[✅ 合规]

□ 合规结论：[✅ 符合政策 / 🔴 存在违规点：列表]
```

**授权检查：**
```
□ 授权层级要求：
  → 根据金额 [X] 万，须审批层级：[CXO/CEO/董事会/股东大会]
  → 当前已获审批：[已获 [层级1/层级2] 审批]

□ 审批完整性：
  → [层级1] 审批：[✅ 已完成（[日期]）/ 🔴 未完成]
  → [层级2] 审批：[✅ 已完成（[日期]）/ 🔴 未完成]

□ 审批完整性结论：[✅ 完整 / 🔴 缺失：列表]
```

## 第三步：监管合规检查

**证券监管合规（如涉及上市公司）：**
```
□ 信息披露合规：
  → 是否须披露：[✅ 是 / 否]
  → 披露类型：[临时公告/定期报告]
  → 披露时间：[日期前]
  → 当前披露状态：[✅ 已披露 / ⏳ 待披露 / 🔴 未披露]

□ 内幕信息管理：
  → 是否涉及内幕信息：[是/否]
  → 知情人登记：[✅ 已完成 / ⏳ 待完成]
  → 禁止交易期提醒：[✅ 已发出]
```

**反垄断合规（如适用）：**
```
□ 经营者集中申报：
  → 是否达到申报标准：[是/否 — 标准：营业额 [X] 亿]
  → 须申报机关：[商务部/市场监管总局]
  → 当前申报状态：[✅ 无需申报/⏳ 申报中/🔴 未申报]

□ 是否可能构成垄断：[是/否 — 分析]
```

**外汇/跨境投资合规（如适用）：**
```
□ 外汇管理：
  → 是否涉及跨境资金流动：[是/否]
  → 是否须外汇管理局备案/审批：[✅ 无需/⏳ 备案中/🔴 需审批]

□ 境外投资合规：
  → 是否须发改委备案/核准：[✅ 无需/⏳ 备案中/🔴 需核准]
```

## 升级触发条件

- 投资政策不合规（超出授权或违反禁止项）
- 存在未完成的必备审批
- 须申报未申报（反垄断/外汇）—— 存在法律风险
- 未进行信息披露（上市公司）—— 监管处罚风险
- 合规整改项未在执行前完成
- 风险等级超过公司风险偏好
