# /cash-pool-configuration — 现金池配置


## Examples

→ 示例：用户说"帮我们设计一个现金池方案，把境内子公司的闲置资金归集上来"，系统应调用本技能，设计现金池归集方案。

→ 示例：用户说"现金池上线后，帮我们检查一下各成员账户的计息是否正确"，系统应调用本技能，验证现金池计息准确性。

→ 示例：用户说"有个子公司在外地，加入现金池需要考虑哪些税务问题"，系统应调用本技能，评估现金池跨地区税务影响。

## 第二步：现金池结构设计

**现金池类型选择：**
```
□ 境内实体现金池（Physical Cash Pooling）：
  → 适用：同一税收居民实体间
  → 优势：简单，无税务复杂问题
  → 劣势：无实际法律隔离

□ 虚拟现金池（Notional Cash Pooling）：
  → 适用：跨法律实体
  → 优势：无需实际划转资金
  → 劣势：部分银行不支持

□ 跨境资金池：
  → 适用：境内外资金联动
  → 须考虑：外汇管理、预提税、转让定价
```

**推荐现金池方案：**

```
□ 归集规则：
  → 归集频率：[每日/每周]
  → 归集方式：[全额归集/限额保留]
  → 保留金额：[X] 万（每个子账户）

□ 下拨规则（如需）：

```

## 升级触发条件

- 跨境现金池涉及外汇管理审批（须外管局沟通）
- 涉及大额预提所得税（须税务专项评估）
- 主办银行无法满足技术对接要求
- 转让定价风险评估为中高风险
- 实施涉及 [X] 家以上子公司协调（复杂度高）
- 银行费用谈判无法达成目标
