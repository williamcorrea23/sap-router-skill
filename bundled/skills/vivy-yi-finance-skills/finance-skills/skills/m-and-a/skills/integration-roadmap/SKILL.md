# /integration-roadmap — 并购整合路线图


## Examples

→ 示例：用户说"交割后 100 天了，整合进展怎么样，帮我做一次回顾"，系统应调用本技能，执行 100 天整合进展回顾。

→ 示例：用户说"帮我们制定一个财务系统整合的路线图，分几步走"，系统应调用本技能，制定财务系统整合路线图。

→ 示例：用户说"整合中遇到文化冲突，帮我从财务管控角度设计过渡方案"，系统应调用本技能，设计财务管控过渡方案。

```

□ Phase 2：协同期（[X]-[X] 个月）
  → 目标：实现成本协同，推进业务整合
```

## 升级触发条件

- 关键人才流失超预期（保留率 < [X]%）
- 协同效应实现率低于计划 [X]% 以上
- 整合进度滞后超过 [X] 周
- 重大客户流失（单客户收入 > [X] 万）
- IT 系统整合出现重大技术障碍
- 文化冲突导致整合无法推进
- 整合成本超出预算 [X]%
