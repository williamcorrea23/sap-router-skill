Привіт! Потестив підключення на тому JSON, що скидав. Токен валідний,
scope повний, але `/calm-projects/v1/projects` повертає порожній
список.

Підозрюю, що справа в тому, що цей `clientid`
(`sb-CALMExtensionAPI!b570849|sapcloudalm-ext-01!b16907`) не прив'язаний
до жодного CALM-проєкту і не має жодних ролей у Project & Engagement
Management — тому CALM не показує нічого, незалежно від scope. Доказів
напряму нема, це гіпотеза.

Чесно — не впевнений, де саме це налаштовується: чи в BTP-cockpit на
рівні subaccount (role collections / scope grants для XSUAA-клієнта),
чи всередині самого CALM (Project Team / Members). У CALM мені права
видані, але я постійно потрапляю в документацію замість адмінки —
очевидно, дивлюсь не туди.

Можеш, будь ласка:

- і, якщо можливо, додати цьому `clientid` ролі / прив'язку до хоча б
